# 第9章 定义水文领域的 Skills（工作流）

## 9.1 什么是 Skill？将业务经验封装为可执行流

在引入了基于大模型的 Agent 之后，如果每次遇到复杂水务问题，Agent 都要“从头开始思考”如何调用底层模型和 API，不仅效率极低，而且极易出错。
Skill（工作流技能）的出现就是为了解决这一问题。它不是一段死板的代码，也不是一句模糊的 Prompt。在 HydroDesktop 的定义里，Skill 是一套“标准化操作程序（SOP）”的代码化表达，它规定了当触发某个任务时，Agent 必须按何种顺序读取数据、调用哪些模型、应用哪些规则进行判断。
这意味着老专家的“直觉”被固化成了系统里可重复执行的流水线。

## 9.2 典型 Skill 范例：降雨预报复核、异经常归因分析、合规报告生成

在 `core/hydrology/skills/review_forecast.py` 中，我们实现了一个“预报异常复核”技能。当系统接收到极值暴雨预报时，该 Skill 会强制要求提取多模式（如雷达外推、数值天气预报）的数据进行相互比对。
而在 `core/hydrology/skills/compliance_report.py` 中，我们把月末的合规性数据盘点做成了自动化流水线，释放了大量人力。
`tests/hydrology/test_ch09_skill_workflow.py` 确保了这些封装好的 Skill 能够在输入正确时，得出确定性的业务结论，而无需依赖每次大模型的随机“发挥”。

## 9.3 Skill 的质量控制与测试验证

一旦大模型被赋予了调用底层水网的能力，Skill 就是它的“驾照”。没有经过严苛测试的 Skill 绝不能上线。
通过引入测试驱动开发（TDD），每一个新编写的水务调度 Skill 都必须配备完整的单元测试和场景回归测试。我们通过持续集成（CI）来保障水网“操作手册”的绝对安全。

## 本章小结与证据锚点

- **核心结论**：Skill 是连接大模型认知能力与底层水文物理模型的高速公路。
- **代码锚点**：`core/hydrology/skills/review_forecast.py`, `core/hydrology/skills/compliance_report.py`
- **测试证据**：`tests/hydrology/test_ch09_skill_workflow.py`
