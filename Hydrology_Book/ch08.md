# 第8章 MCP：让大模型读懂水文世界

## 8.1 MCP (Model Context Protocol) 核心机制：Tools, Resources, Prompts

在 HydroDesktop 等智能化水网工作台的背后，大语言模型（LLM）需要频繁与上述（第7章）复杂的水文 API 进行交互。
但大模型并不能“凭空猜出”如何传参，这就需要 Model Context Protocol (MCP) 来提供标准化连接。
MCP 的三大支柱：
1. **Tools**：将底层的预报和调度计算暴露为大模型可执行的工具。
2. **Resources**：为大模型提供背景上下文（如该城市的降雨极值表）。
3. **Prompts**：为大模型限定思考框架与输出规范。

## 8.2 封装水文能力：将 `run_forecast` 包装为 FastMCP Server

利用 FastMCP 框架，我们将极其复杂的洪水演进预报模型包裹在一层极薄的描述代码中。
以至于大模型在遇到“明天暴雨，水库该怎么调度？”这种模糊询问时，能准确找到对应的水文仿真 Server 进行请求分发。

## 8.3 标准化 Schema：如何确保大模型稳定输出控制指令

防止大模型“产生幻觉（Hallucination）”的关键，在于为其施加强类型的 Schema 约束。
在 `core/hydrology/mcp/schema.py` 和 `prompt_contract.py` 中，我们明确了输入参数的合法类型（比如 `rain` 必须是 `number`）。
同时，System Prompt 在每次对话前都会强制规定：“You are a hydrology assistant.” 这种刚性的人设与接口契约，在 `tests/hydrology/test_ch08_mcp_schema_stability.py` 中得到了自动化捍卫，确保 AI 不管怎么聊，都不会给出违背物理法则和水利规范的灾难性指令。

## 本章小结与证据锚点

- **核心结论**：MCP 协议及其严格的 Schema，是构建新一代专业认知水务大模型最坚固的安全护栏。
- **代码锚点**：`core/hydrology/mcp/schema.py`, `core/hydrology/mcp/prompt_contract.py`
- **测试证据**：`tests/hydrology/test_ch08_mcp_schema_stability.py`
