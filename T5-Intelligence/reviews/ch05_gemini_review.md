### P0 致命问题

**P0-01【第5.5.1节Lagrangian公式渲染双行重复且缺少$$开头】**
第5.5.1节中Lagrangian函数公式出现**完全相同的两行**：
```
\mathcal{L}(\pi, \boldsymbol{\lambda}) = J_R(\pi) - \sum_{i=1}^{m} \lambda_i \left(J_{C_i}(\pi) - d_i\right)
\mathcal{L}(\pi, \boldsymbol{\lambda}) = J_R(\pi) - \sum_{i=1}^{m} \lambda_i \left(J_{C_i}(\pi) - d_i\right)
$$
```
该公式既缺少开头的`$$`，又重复出现两次，且闭合符号位置错误（$$在末尾）。Lagrange乘数更新公式同样存在开头`$$`缺失的问题。这是必须修复的渲染错误。

**P0-02【第5.2.2节状态空间正弦编码公式格式错误】**
```
$$\phi_{\text{season}}(t) = \left[\sin!\left(\frac{2\pi t}{365}\right), \cos!\left(\frac{2\pi t}{365}\right)\right]$$
```
`\sin!` 和 `\cos!` 中的感叹号是LaTeX排版的间距控制符，不应出现在教材正文公式中（会渲染为多余字符），应改为标准的 `\sin\left(` 形式。此问题在第5章多处出现（奖励函数中 `\min!`、`\max!` 等）。

### P1 重要问题

**P1-01【知识依赖框中章节引用不一致】**
知识依赖框中写：
- "HIL（Hardware-in-the-Loop）仿真框架 → 对应第4章"
- "HydroOS L2内核的水文-水动力耦合机制 → 对应第3章"
- "模型预测控制（MPC）基础 → 对应第2章"

但实际第3章（时序异常检测）和第2章（水文预报）的内容并不包含HIL框架和MPC基础，这些映射关系存在错误。应根据全书实际内容修正依赖章节。

**P1-02【SAC温度参数损失函数公式中目标熵H0定义存在争议】**
第5.3.2节 $H_0 = -\dim(\mathcal{A})$，这是SAC原论文（Haarnoja et al., 2018）针对连续动作空间的经验设置（负的动作维数），但对离散动作空间并不适用。闸门开关等离散控制场景需要不同的目标熵计算方式（通常设为 $-\log(1/|\mathcal{A}|)$）。建议补充说明："连续动作空间推荐 $H_0=-\dim(\mathcal{A})$；离散动作空间建议 $H_0=-\log(1/|\mathcal{A}|)$，其中 $|\mathcal{A}|$ 为动作数量。"

**P1-03【WNAL等级表格只有L0-L3，实际水利场景需要说明L4的存在】**
表格列出WNAL L0-L3，但SAE自动驾驶等级共有L0-L5，对应水利场景是否存在L4（全自主无人监管）应明确说明，即使当前水利系统不推荐使用L4，也应在脚注中说明"L4（全自主）因公共安全监管要求，当前阶段不建议在关键基础设施中部署"，避免读者误以为L3是最高等级。

**P1-04【"三库联合调度状态空间100^8"计算示例有误】**
第5.1.1节写："若每个水库库容离散化为100个等级，时段数为12个月，则状态空间规模达 $100^8$"。三个水库各有100等级则状态空间为 $100^3$，加上时段12个月，状态空间最多为 $12 \times 100^3 = 12,000,000$，而非 $100^8$。$100^8$ 的计算方式未解释清楚（可能是将时段数也纳入离散化，或考虑了其他状态变量）。建议重新计算并给出清晰的推导过程，否则该数字会被读者直接引用。

### P2 改进建议

**P2-01【奖励函数权重优先级原则需补充定量说明】**
正文写"权重设置应遵循 $w_5 \gg w_1 \geq w_2 > w_3 \geq w_4$ 的优先级原则"，但未给出典型取值示例。建议补充一组工程实践中的参考权重（如 $w_5=10, w_1=3, w_2=2, w_3=1, w_4=0.5$），使读者有直观参考。

**P2-02【域随机化表格中"渗漏系数k"单位可更规范】**
表格中"渗漏系数k: 1.5e-6 m/s"建议改写为 $1.5 \times 10^{-6}$ m/s，避免科学计数法的工程符号"1.5e-6"出现在正式教材表格中。

**P2-03【CQL损失函数来源应补充说明】**
第5.4.4节引用CQL（Kumar et al., 2020），但该方法原本是为离线RL（Offline RL）设计的，用于防止对数据集外动作的Q值高估。在水利调度中如何与在线训练结合需要额外说明，否则读者可能误以为CQL是标准在线RL的增强版本。

**P2-04【开篇故事"18%防洪效益"的指标定义未说明】**
开篇故事写"DQN在1998年特大洪水场景下的防洪效益相比传统调度规则高出18%"，但"防洪效益"未定义（是最大水位降低?还是溃坝概率减小?还是下游受灾面积减小?）。建议明确该指标的定义，增强科学严谨性。

**P2-05【本章末尾是否有习题需要确认】**
从读取内容判断，第5章内容包含多节技术内容，但未见本章小结和习题板块。需确认这两个必备体例是否已存在于草稿末尾。

**P2-06【"Zhao et al., 2023"引用真实性待核实】**
第5.3.4节引用"Zhao et al., 2023"验证MAPPO在水资源调度中的应用，该文献无法通过常规学术数据库核实（可能是未公开发表的内部研究或预印本）。建议提供更完整的引用信息或改引已发表的MAPPO水利应用文献。
