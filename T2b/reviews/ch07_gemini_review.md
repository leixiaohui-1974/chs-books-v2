## P1 重要问题

**P1-1【D3公式正确性】线性共识算法收敛条件描述不够精确**
§7.3.2公式(7-5)文中表述"当通信图连通且步长选取合适时，所有Agent状态收敛到初始值的平均"，这对无向连通图成立，但对有向图（文中引理7-1使用有向生成树条件）需要更严格的步长上界：`ε < 1/Δ_max`（其中Δ_max为最大度数）。当前表述可能使读者在有向稀疏图中选择过大步长导致发散。建议明确区分有向/无向图的收敛条件，或增加步长选取的具体指导。

**P1-2【D1技术准确性】ADMM推导被移至T2a但正文逻辑存在缺口**
§7.3.4说明"ADMM的完整数学推导已在T2a第七章§7.7中详细给出"，但随后直接进入"工程含义"讨论，使未读T2a的读者无法理解"增广Lagrange函数"和"对偶变量"等术语。建议在§7.3.4保留增广Lagrange函数的基本形式（3行公式），并明确三步迭代公式，以确保本章的自包含性。

**P1-3【D5编号连续性】表格编号系统不一致**
ch07中部分表格有正式编号（"表7-1""表7-2"），部分表格（如§7.2.4中的耦合约束说明、§7.4.3的代码注释）无编号。建议对全部表格统一编号，并确保正文有相应"如表7-X所示"的引用。

**P1-4【D2图表完整性】缺少通信拓扑结构对比图**
§7.2.3描述了完全图、链式图、星形图、分层图四种通信拓扑，但无对应示意图。对于研究生读者，可视化拓扑对理解Laplacian矩阵特征值意义至关重要。建议增加图7-1：四种通信拓扑对比图。

## P2 改进建议

**P2-1【D1技术准确性】AgentCard示例代码中List[str]需更新为Python 3.10+语法**
```python
capabilities: List[str]
```
根据全书Python规范（优先用Python 3.10+语法），应更新为 `capabilities: list[str]`，并添加 `from __future__ import annotations` 或直接使用内置类型注解。

**P2-2【D6教材体例】习题设计偏少**
ch07仅7道习题（5基础+2思考），与ch05（15道）、ch08（15道）相比明显偏少。建议将§7.5-§7.6相关的消息总线和DAG编排内容各增加1-2道应用题，达到10道以上，与全书体例一致。

**P2-3【D7代码示例】AgentContext代码示例可增加错误处理**
§7.2.2的`AgentContext`字典示例无错误处理。考虑到水系统运行场景，建议增加说明：当`water_balance_result`键不存在时的回退行为，以展示工程健壮性的设计原则。

**P2-4【D4参考文献】参考文献27条，体量合理，但缺少中文水利期刊引用**
ch07的27条参考文献中，中文文献仅2条（均为Lei 2025a/b），缺少国内多智能体水系统控制领域的代表性工作（如中国水科院、华北水利水电大学等相关研究）。建议补充2-3条国内中文期刊文献。

**P2-5【D3公式正确性】全局目标函数(7-3)的量纲说明需要增强**
公式(7-3)的加权聚合式中，正文已提到"通常对各项进行归一化处理"，这是正确的工程实践，但建议将量纲一致性要求（$J_i$和$J_\text{coupling}$均应为无量纲量）直接写入公式说明，而非仅在括号内附注，以更突出对研究生读者的指导意义。
