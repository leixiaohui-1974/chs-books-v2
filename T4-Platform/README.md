**英文书名：** Hydro Network Operating System: From Protocols to Platforms in System Engineering

## 本书定位

传统水利自动化系统（SCADA）在实时性、互操作性、智能化和安全性方面已难以满足现代水网精细化管理的需求。现有通用操作系统缺乏对水利领域特有物理过程、控制逻辑和异构设备环境的深度支持，而传统控制系统又过于封闭和碎片化。本书旨在填补这一空白，系统性地构建一套面向水网的专用操作系统（HydroOS）理论与实践体系。

本书的核心价值在于将系统工程理念贯穿于水网操作系统的设计与实现全过程，强调从底层通信协议到上层应用平台的整体性、开放性与可扩展性。它不仅关注软件架构设计，更深入探讨了水利物理过程与控制算法的紧密耦合，以及如何通过标准化接口和分层架构实现对复杂水网的统一管理和智能控制。

与通用操作系统教材不同，本书聚焦于水利领域的独特挑战：大尺度分布式部署、强实时性要求、多物理场耦合、异构设备协同、长生命周期运维以及对功能安全与网络安全的极高要求。本书引入"水网自治等级"（Water Network Autonomy Level，WNAL）框架，将平台能力与业务场景需求对齐，使读者能够在工程实践中做出有据可查的技术决策。

本书特别阐述了"四预双层架构"——Skill级四预（事件驱动、异步松耦合）与步长级四预（同步紧耦合、Δt内共推进）——这是HydroOS区别于通用信息化平台的核心架构关键，也是工程实践中被广泛忽视的设计盲区。

## 目标读者画像

### L1 管理层

关注战略规划、投资回报、系统整体效益与风险控制。

**建议阅读路径：** 第1章（平台演进逻辑）→ 第2章（架构总览）→ 第6章（安全子系统）→ 第7章（平台集成）→ 第8章（工程案例）。各章均设有"[管理层速览]"框，可快速获取核心决策信息，无需阅读技术细节。

### L2 水利工程师

具备水利工程背景，负责系统操作、配置、调度与日常维护。

**建议阅读路径：** 第1章 → 第2章 → 第3章（Skill系统）→ 第4章（步长级耦合）→ 第5章（通信协议栈）→ 第8章（工程案例）。

### L3 系统架构师

负责系统整体设计、模块开发与平台集成，需深入掌握每一层的技术原理与接口约定。

**建议阅读路径：** 全书所有章节，尤其重点关注第2、3、4、5、6章的技术细节与设计权衡。

## 与其他五册的依赖关系

```
T1《水动力学控制论基础》          T2《水网调度数学》
   物理模型 / 数值求解                图论 / 优化 / 博弈论
        │                                  │
        └──────────────┬───────────────────┘
                       ↓
T3《传感器与边缘计算》 ──→  T4《水网操作系统》（本书）──→ T5《水网智能决策》
   设备接入 / IoT 协议       HydroOS 架构 / WNAL 框架       AI 算法 / CV / 安全
                                       │
                                       ↓
                             T6《水利CPS》
                             数字孪生 / 自主调度
```

**依赖说明：**

- **T1 → T4：** 步长级仿控耦合（第4章）的实时内核需要T1提供的Saint-Venant方程数值求解基础，才能实现精确物理过程仿真。
- **T2 → T4：** Skill系统（第3章）是T2中图论优化与博弈论调度算法的运行平台，二者需要接口对齐。
- **T3 → T4：** HydroOS物理接入层直接依赖T3的传感器技术与IoT协议，T3是T4连接物理世界的基础。
- **T4 → T5：** T4平台服务层为T5的AI算法、视觉模块与安全策略提供运行环境和数据接口。
- **T4 → T6：** HydroOS是水利CPS的核心软件基础设施，T6的数字孪生与自主调度以T4为操作系统平台。
