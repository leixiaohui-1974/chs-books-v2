## P0问题（必须立即修复）

- [ ] **P0-1 全章零公式，不符合控制论教材标准。** 讨论MPC（§14.1.4）时必须给出代价函数 $J_i = \sum_{k=0}^{N_p-1}[\|y_i(k)-r_i(k)\|^2_{Q_i} + \|\Delta u_i(k)\|^2_{R_i}]$；讨论EKF（§14.1.3）时应给出预测-更新两步公式；讨论IDZ降阶（§14.1.2）时应复述Family α传递函数 $G(s)=(1+\tau_m s)e^{-\tau_d s}/(A_s \cdot s)$。这是物理AI引擎的"核心算法"章节，没有公式就像乐谱没有音符。

- [ ] **P0-2 原理四名称误引。** §14.1.4末段写"第七章原理四'安全内生'"，但ch07_final.md §7.5标题为"原理四：安全包络"。"安全内生"不是八原理的正式名称，必须改为"安全包络"。

- [ ] **P0-3 图编号非标。** 图14-2A、图14-4A使用字母后缀，全书其他章节（ch01—ch13）均无此惯例。应重新编号为连续整数（如图14-3、图14-5），后续图号顺延。

- [ ] **P0-4 图14-4悬浮无引用。** `![图14-4 策略网关与安全裁剪机制](./H/fig_14_04_strategy_gateway.png)` 出现在§14.6结尾与§14.7开头之间，正文中无任何段落引用该图。必须在相关正文中添加引用，或将该图移至§14.1.4安全包络约束器处。

- [ ] **P0-5 §14.4与§14.2.2内容大面积重复。** §14.2.2已详述知识图谱四步构建流程（本体设计→知识抽取→知识融合→持续更新），§14.4又以几乎相同结构重述（步骤一至步骤五）。必须合并或大幅差异化——建议§14.2.2保留原理层叙述，§14.4改为纯工程实操指南（工具选型、数据标注量估算、质量指标等）。

- [ ] **P1-2 大渡河案例（§14.8）缺乏定量数据。** §14.8.3"实施效果"全部使用"显著提升/显著降低/显著缩短"，与§14.7.3胶东案例的具体数据（±5cm、降低83%、90秒等）形成鲜明反差。如果没有真实数据，宁可删除该案例或标注为"预期目标"，不应用模糊定性描述冒充实施结果。

- [ ] **P1-3 安全包络三区缺乏数学定义。** §14.1.4定义了绿/黄/红三区，但仅用文字描述。应给出形式化定义，例如：绿区 $h_{\min}+\delta \leq h \leq h_{\max}-\delta$，黄区 $h_{\max}-\delta < h \leq h_{\max}$，红区 $h > h_{\max}$，其中 $\delta$ 为安全裕度参数。

- [ ] **P1-4 表14-5A与表14-5存在信息冗余。** 两表均对比DAL/PAI/CAI的响应时间、安全性、故障影响等维度，建议合并为一张综合对比表。

- [ ] **P1-5 §14.5（大语言模型应用）与§14.2存在边界模糊。** §14.2已讨论CAI的RAG架构（§14.2.3）、LLM角色（§14.2.4末段），§14.5又单独展开LLM选型、微调、六类场景。建议将§14.5的内容整合入§14.2，或明确§14.2侧重"CAI架构设计"、§14.5侧重"LLM工程落地"的区分。

- [ ] **P1-6 Chi-Square检验缺乏公式。** §14.1.6描述了Chi-Square检验的原理但未给出检验统计量的计算公式。应至少给出 $\chi^2 = \sum_{i=1}^{N} \left(\frac{y_i - \hat{y}_i}{\sigma_i}\right)^2$ 及其判定条件。

- [ ] **P1-7 "断连可活"术语未在本章定义。** §14.2.7使用"断连可活"设计原则但未给出出处或定义，读者需翻阅其他章节理解。应添加括号注释或脚注标明来源章节。

## P0问题（必须立即修复）
- [ ] **§14.1.4 安全包络“后置裁剪”存在稳定性与可行性风险**：MPC算完再裁剪会破坏原优化可行域，可能引发边界振荡/约束反复触发。改为“安全滤波QP/参考治理器（Reference Governor）+不变集约束（Tube/Robust MPC）”，并补充切换滞回与驻留时间。
- [ ] **关键算法缺可实施数学定义**：至少补齐 `J(u)`、状态方程、约束方程、`J_k=r_k^T S_k^{-1} r_k`、告警规则 `J_k > χ²_{α,m}`、N窗口与误报率设计值（如 `P_FA<=10^-3`），否则工程团队无法复现。
- [ ] **§14-5A“CAI覆盖范围：全部工况（含未知）”不安全**：应改为“未知工况仅给建议，必须经OOD检测+PAI验证+人工确认”，避免误导为可直接自治。
- [ ] **安全机制缺SCADA必需控制**：未明确命令签名、防重放、时钟同步、最小权限、双人复核、失联降级与只读隔离区（IEC 62443分区分域）。需补“控制链安全最小闭环”。

## 亮点
- **绝妙的跨界类比**：将 PAI 与 CAI 的协同比喻为"小脑"（运动控制）与"大脑皮层"（认知解释），以及"只开药的医生" vs "解释病理的医生"的类比，极大地降低了交叉学科的认知门槛，堪称教科书级别的表达。

- **极致的工程落地指南**：不仅在理论层面构建架构，还在 14.6.5 和 14.7.6 节中详细拆解了软硬件成本估算、开源平替方案、所需人力周期，甚至投资回报率(ROI)。这种"从理论到算账"的连贯性，对于需要实际推动项目落地的工程决策者来说具有不可估量的价值。
