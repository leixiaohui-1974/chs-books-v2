## P0问题（必须立即修复）
- [ ] **式(4-6)的模型类型写错。** 现在写成 `P_IDZ(s)=e^{-τ_d s}/(A_s(s+σ))`，这不是 IDZ，而是“带泄漏的一阶/积分延迟”形式，没有 `τ_m` 零点；同时又与式(4-3)、式(4-8)冲突。建议拆开写：低频近似先得到 `ID/ID+leak`，IDZ 单独给出 `P(s)=(1+τ_m s)e^{-τ_d s}/(A_s s)`，并说明零点来自边界条件/回水效应。 |
- [ ] **`A_s` 的定义、量纲和算例数值不一致。** 文中定义 `A_s` 单位为 `m²`，但又写 `A_s=A_0·L`，这在量纲上是 `m³`；案例 4-2 的 `15 km` 渠池给出 `A_s=23,800 m²` 也明显不合常识。建议统一为“回水区水面面积/等效蓄量系数”一种定义，重算案例 4-2 和表 4-2 全部数据。 |
- [ ] **`τ_m`、`τ_d` 的解析公式当前不可实现。** `ε`、`T_p` 未定义，`τ_m` 公式量纲不闭合，工程人员无法据此编程或校核。建议删去这组未经闭合的闭式公式，改为“机理初值 + 阶跃/PRBS 辨识 + 多工况校准”的可执行流程。 |
- [ ] **图表不符合本书出版规范。** 图 4-1 至图 4-6 都未使用 `./H/...` 本地路径，文件编号与图号多处错位，且 alt 文本含 `</b></div>` 残片。这个问题会直接影响交付版排版与后续维护。 |

## P1问题（重要但非阻塞）
- [ ] **“IDZ 三参数完全由几何尺寸决定、无需反复整定”的表述过满。** 工程上 `A_s、τ_d、τ_m` 至少受运行点、下游边界、糙率季节漂移和建筑物局部损失影响。建议改成“可由机理计算获得初值，但需现场辨识或在线重标定”。 |
- [ ] **MPC 落地描述偏理想化。** 文中把 `N` 个渠池近似成 `N` 维系统，但一旦用 Pade 或移位寄存器处理时滞，状态维度会明显增加。建议补 `N_p / N_c / Δt / solver / CPU` 的示例配置，否则“0.5s 在线求解”没有工程依据。 |
- [ ] **安全机制还停留在概念层。** 目前只写“模型精度下降则降级/MRC”，没有触发条件。建议明确：残差阈值、持续时间、通信中断判据、执行器跟踪误差、watchdog 超时、输出仲裁优先级、fail-safe 动作、人工接管优先。 |
- [ ] **[4-4] 参考文献状态不可接受。** 正文已带“web search 未检索到该文，请确认”的内联注释，说明该条文献尚未完成终审。出版前要么补 DOI/官网页面，要么替换为可核验来源。 |
- [ ] **CHS 映射不够明确。** 建议在章末加一个小表：状态空间/传递函数对应 P1，状态选择与估计对应 P2，降阶与分解对应 P3，ODD 与降级接口支撑 WNAL L3 的 P1-P5 最小集。 |

## P2问题（建议改进）
- [ ] **“自调节型对象如水库水位”需要加边界条件。** 只有在固定出流-水位关系成立时才可近似为自调节对象；带闸门主动调节的库水位不能笼统这样写。 |
- [ ] **§4.7 既说“说明性设定”，又给出很具体的精度提升数字，容易被误读为实测结果。** 建议统一标成“仿真/试点推演结果”。 |
- [ ] **练习题缺少安全闭环。** 建议加一道“模型残差越界后如何触发降级/MRC”的题，把形式化、控制和安全连起来。 |
- [ ] **ODD 表中的数值阈值应明确为示例，不宜让读者误认为行业通用标准。** |
