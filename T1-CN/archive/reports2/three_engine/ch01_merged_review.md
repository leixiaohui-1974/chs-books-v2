## P0问题（必须立即修复）

- [ ] **P0-1：WNAL vs WSAL 术语严重不一致。** 全章使用"WNAL"（Water Network Autonomy Levels），但CHS体系（P1a WRR §7.4、P2A、CLAUDE.md术语表）统一使用"WSAL"（Water Systems Autonomy Level）。"WNAL"在CHS体系中不存在。必须全文替换为"WSAL"。涉及位置：§1.3.2标题及正文、§1.3.5对比表、§1.5.2、§1.5.5贡献二、§1.6全书结构多处、本章术语表、练习题等——几乎所有出现该缩写的地方。

- [ ] **P0-2：八原理名称与P1a WRR不一致。** §1.5.2中的八原理名称与P1a §5定义的"Eight Governing Principles"存在多处偏差：

  | 本章名称 | P1a WRR名称 | 问题 |
  |---------|------------|------|
  | 原理一：传递函数化 | P1 Feedback（反馈） | 完全不同的概念 |
  | 原理二：可控可观性 | P2 Decoupling（解耦） | 完全不同的概念 |
  | 原理三：分层分布式 | P3 Order Reduction（降阶） | 不同 |
  | 原理四：安全包络 | P4 Emergence（涌现） | 不同 |
  | 原理五：在环验证 | P5 Robustness（鲁棒性） | 不同 |
  | 原理六：认知增强 | P6 Coordination（协调） | 不同 |
  | 原理七：人机共融 | P7 Adaptation（自适应） | 不同 |
  | 原理八：自主演进 | P8 Hierarchy（层级性） | 不同 |

  P1a的八原理分为Operational Tetrad（P1-P4）和Resilience Tetrad（P5-P8），本章的八原理看起来是一套**完全不同的分类体系**。这是教材自建的八原理体系还是与P1a的八原理是同一套？如果是同一套，则名称必须统一；如果是教材扩展版本，必须明确说明与P1a的对应关系，否则读者阅读P1a论文时会产生严重混淆。**这是全书体系一致性的根基问题。**

- [ ] **P0-3：HDC层级编号与WSAL等级编号冲突。** §1.2.4中HDC架构使用L0/L1/L2编号（L0本地控制层、L1渠段控制层、L2全系统调度层），而§1.3.2中WSAL也使用L0-L5编号。两套"L编号"在同一章中共存，极易混淆。P1a WRR中HDC层级使用"Layer 0/1/2/3"，WSAL使用"Level 0-5"，有清晰区分。建议：HDC层级统一用"第0层/第1层/第2层"或"Layer 0/1/2"，WSAL用"L0-L5等级"。

## P1问题（重要但非阻塞）

- [ ] **P1-1：图片路径不符合规范。** 所有图片使用GitHub raw URL（`https://raw.githubusercontent.com/...`），应使用本地相对路径 `./H/fig_01_XX.png`。共5处：图1-1、图1-2、图1-3、图1-4、图1-5。部分图片使用`<img>`标签，部分使用`![]()`，格式不统一。

- [ ] **P1-2：§1.5.2八原理层次结构描述与P1a不一致。** 本章将八原理分为"基础层→架构层→验证层→智能层→演进层"五层，而P1a将八原理分为"Operational Tetrad（运行四元组）"和"Resilience Tetrad（韧性四元组）"双四元组结构，并定义了四对tradeoff。本章的五层结构是否是教材的教学性重组？如果是，应明确说明"本书为教学目的将八原理重新组织为五层结构，与P1a论文中的双四元组分类互为补充"。

- [ ] **P1-3：§1.3.4中"原理一'传递函数化'"与"原理四'安全包络'"等引用，依赖P0-2的修正。** 如果八原理名称修正后，这些交叉引用也需同步更新。当前原理名称与编号的对应关系需全章检查。

- [ ] **P1-4：§1.2.4中HDC三层架构缺少P1a中的Layer 3。** P1a WRR Fig3定义了四层控制层级：Layer 0 Safety（PLC, ms）、Layer 1 Regulation（local MPC, s-min）、Layer 2 Coordination（DMPC, min-h）、Layer 3 Planning（nested opt, day-yr）。本章仅列出三层（L0/L1/L2），遗漏了Layer 3 Planning层。对于绪论章的简化介绍，应至少提及四层结构并注明"详见第七章"。

- [ ] **P1-5：§1.5.4 HydroOS三层架构名称需检查一致性。** 本章SIL定义为"调度与智能层（Scheduling & Intelligence Layer）"，而P1c中SIL层的定义可能不同。需确认与第十三章的术语完全一致。同时，"物理AI引擎"和"认知AI引擎"的位置描述需与P1a/P1c保持一致——P1a中MAS = HDC + ODD + Cognitive Intelligence，本章的PAI/CAI分法是否与此对应需明确。

- [ ] **P1-6：§1.4节"设计锁定效应"的MBD框架描述为"四层一闭环"，与P1a/CKG-1中MBD"七层框架"不一致。** CLAUDE.md中CKG-1论文核心内容为"MBD七层框架"。本章的"四层一闭环"是否为七层框架的简化版？如果是，应注明"此处为简化概述，完整的MBD七层框架见第十二章"。

- [ ] **P1-7：自引率略高。** 22篇参考文献中，Lei/雷晓辉/王浩团队文献约7篇（[1-13][1-14][1-17][1-19][1-20][1-21][1-22]），占~32%，超过25%上限。建议：补充2-3篇第三方重要文献（如Bequette过程控制教材、Åström反馈系统教材、Mayne MPC综述等），或将[1-19][1-20]中选择性移除一篇。

- [ ] **P1-8：§1.3.2 WSAL表格中L0定义为"人工运行"，但概念速览框和正文中有时将L0描述为"无驾照/完全不会开车"。** P1a WRR中WSAL定义L0为"No Automation"还是"Manual"需确认。当前L0和L1的区分不够清晰——L0是"完全人工"，L1也是"完全人工但有SCADA"，两者的差异主要在信息获取方式而非决策方式，这一点应更明确表述。

## 亮点

- **引导案例出色**：开篇的暴雨调度场景生动具体，直接切入核心矛盾，能有效抓住读者注意力。

- **概念速览框设计精良**：WSAL和ODD的驾驶类比直观易懂，对非控制论背景的水利工程师特别友好。"阅读指引"框帮助不同背景读者快速定位。

- **三个工程案例的渐进对比**（§1.3.5）是全章最有价值的部分之一：L1→L2→L3的具体数据对比使抽象的自主等级概念落地，对比表格设计清晰。

- **§1.4设计锁定效应**是一个有原创性的贡献：从物理层/信息层/社会层三维度分析锁定效应，并用水锤计算类比引入MBD，逻辑链条完整，对工程背景读者特别有说服力。

- **多路径阅读指南**（§1.6.2）设计周到：五类读者、快速分流表、具体章节推荐，体现了教材的用户导向设计理念。

- **五代演进框架**虽非完全原创（类似分代法在其他领域有先例），但对水利行业的适配和叙述节奏把握很好，从"看得见够得着"到"重感知轻控制"的批评尤为精准。

## 总体评价

本章作为绪论在**教学可读性**和**问题驱动的叙事**方面表现优秀，能有效激发读者对CHS的兴趣。但存在一个**系统性的体系一致性风险**：八原理名称和编号与P1a WRR论文不匹配（P0-2），WNAL/WSAL术语混用（P0-1），HDC层级与WSAL等级编号冲突（P0-3）。这三个P0问题如果不修复，将在后续章节产生连锁矛盾，影响全书的学术可信度。建议**优先修复P0问题后再进入下一轮评审**。
