## P0问题（必须立即修复）

- [ ] **P0-1：WNAL与WSAL命名冲突。** 全章使用"WNAL (Water Network Autonomy Levels)"，但CHS统一术语表（CLAUDE.md §H）明确定义为"WSAL (Water Systems Autonomy Level, 水系统自治等级)"。P1a WRR §7.4、P2A、Nature Water Fig3均使用WSAL。若本书有意改名（"水系统"→"水网"，"自治"→"自主"），必须在首次出现处给出显式说明并与其他章节/跨书（T2b ch08, M8 ch01）统一；否则应全部改回WSAL。当前的沉默式改名会导致读者困惑并与论文体系脱节。

- [ ] **P0-2：八原理编号与P1a WRR完全不对应。** P1a WRR §5定义的八原理为：
  - 运行四元组：P1 Feedback, P2 Decoupling, P3 Order Reduction, P4 Emergence
  - 韧性四元组：P5 Robustness, P6 Coordination, P7 Adaptation, P8 Hierarchy
  
  但本章使用的八原理为：P1传递函数化, P2可控可观性, P3分层分布式, P4安全包络, P5在环验证, P6认知增强, P7人机共融, P8自主演进。这是**两套完全不同的原理体系**，共享P1-P8编号但含义截然不同。若第七章已做了教学重构，则需要：(a) 在本章明确声明"本书八原理"与"P1a八原理"的映射关系；(b) 在CLAUDE.md术语表中增补本书版本。当前状态下，任何熟悉P1a的读者会产生严重困惑。

- [ ] **P0-3：L0-L5 vs L1-L5层级数量不一致。** CLAUDE.md §H术语表写"WSAL L1-L5"（五级），WRR-Fig5提示词写"L1→L5"（五级阶梯），但本章定义L0-L5（六级）。若本书有意增加L0，需确认WRR论文、P2A论文和其他章节的一致性——P1a §7.4描述的是"Five-Level WSAL"还是"Six-Level"？

## P1问题（重要但非阻塞）

- [ ] **P1-1：§10.3.2 原理一的归属存疑。** 原文："L1仅涉及原理一（传递函数化）的最简形式——用PID控制器对单个控制回路进行闭环控制"。但紧接着说"PID控制器的参数整定可以通过经验试凑法完成，不需要建立传递函数模型"。这自相矛盾：若L1不需要传递函数模型，为何说它"涉及原理一（传递函数化）"？建议改为"L1涉及原理一的**萌芽形式**——存在闭环控制回路，但控制器设计不依赖显式传递函数模型"或将L1与原理一的关系改为"—"。

- [ ] **P1-2：表10-2"最小原理集"列与正文不一致。** 表中L2写"P1+P2+P4初步"，但§10.3.3正文写"L2需要原理一…原理二…和原理四的初步形式"，两者一致。但L3表中写"P1+P2+P3+P4+P5"，§10.3.4正文写"L3不要求原理六/七/八的完整实现，但需要原理七（人机共融）的基本形式"——说明L3也需要P7初步，表中应补充。

- [ ] **P1-3：缺乏对四态机的形式化定义。** §10.3.4描述四态机（正常→受限→降级→接管）及其迁移逻辑，但未给出状态迁移的形式化表示（如状态迁移矩阵或guard条件的数学表达）。作为控制论教材，建议至少给出一个简化的状态迁移图或形式化定义，例如：$\sigma \in \{N, R, D, T\}$，迁移函数 $\delta(\sigma, \text{ODD\_margin})$。

- [ ] **P1-4：§10.4.1 ODD五元组被引用但未在本章定义。** 原文说"第二章§2.2.4已经介绍了ODD的基本概念和五元组定义"，ODD示例中给出了H, Q, E, S, G五个维度，但未明确声明这就是五元组的内容。建议在示例前增加一行："ODD五元组 $\Omega = (H, Q, E, S, G)$ 的具体规格如下："以建立与第二章的显式关联。

- [ ] **P1-5：§10.6.1 表10-4中场景覆盖率标准表述模糊。** 第5行写"场景覆盖满足标准要求"，第6行写"每条边界有测试"，但未给出L3的具体数值要求。§10.3.4提到"场景覆盖率满足标准要求"，§10.5.1提到"L3≥95%，L4≥99%"。表10-4应将"满足标准要求"替换为具体数值"≥95%"以保持一致。

- [ ] **P1-6：§10.8.1 [10-24]引用内容与文献标题不匹配。** [10-24]标注为"雷晓辉, 王浩, 等. 梯级水电站群多目标优化调度[J]. 水利学报, 2018"，但正文用它来支撑"大渡河瀑深枕梯级还系统引入了ODD及对应的降级机制"的论断。一篇2018年的多目标优化调度论文是否包含ODD和降级机制的内容？请核实引文与论述的对应关系。

- [ ] **P1-7：投资估算缺乏来源。** §10.8.3给出了详细的投资估算（如传递函数建模200-300万元），但未说明数据来源——是基于已有工程的实际成本，还是专家估计？建议标注数据来源或注明"基于CHS课题组对XX工程的成本分析"。

## 亮点

- **引导案例设计精良。** "三层缺口"的层次递进（感知→决策→复盘）直接对应WNAL的核心设计动机，且"综合提炼"的声明规避了数据真实性风险。

- **SAE对比分析深度恰当。** 五个关键差异的提炼（失效后果社会属性、时间尺度跨度、运行连续性、治理结构、验证环境）精准抓住了水系统的本质特殊性，未落入简单类比的陷阱。

- **L3四态机降级逻辑的工程叙述。** §10.3.4中闸门卡死场景的L2 vs L3对比，以及通信中断场景中"MRC策略不能依赖触发MRC的失效资源"这一设计原则，展示了深厚的工程直觉。

- **表10-4 L3准入检查清单。** 12项检查条目结构完整，每项关联到对应门槛和八原理，且附有可操作的测量方法——这是全章最具工程指导价值的产出。

- **§10.8.3投资估算。** 在学术教材中给出具体的投资金额范围（1250-2000万元/工程）虽属罕见，但极大提升了决策者的参考价值，体现了工程教材应有的实用导向。

- **L5的审慎定位。** "理论方向标"而非工程目标的定位，避免了学术界常见的技术乐观主义陷阱，三个不可达原因（非平稳性、伦理约束、利益博弈）的论证逻辑严密。
