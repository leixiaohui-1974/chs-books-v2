## P0问题（必须立即修复）

- [ ] **P0-1：全章"WNAL"应为"WSAL"**。CHS体系标准术语为 **WSAL**（Water Systems Autonomy Level，水系统自治等级），见CLAUDE.md §H术语表及P1a §7.4、P2A。本章全文（含标题、正文、表格、术语表、练习题）使用"WNAL"（Water Network Autonomy Levels），与CHS体系严重不一致。须全局替换，术语表中的英文全称也须改为"Water Systems Autonomy Level"。涉及位置：§6.1.3、§6.2.2、§6.2.4、§6.4.2、§6.4.3、§6.8、§6.9、§6.13、§6.14、小结、练习题、术语表。

- [ ] **P0-2：八原理编号/名称与P1a WRR不一致**。P1a定义的八原理为——运行四元组：P1反馈(Feedback)、P2解耦(Decoupling)、P3降阶(Order Reduction)、P4涌现(Emergence)；韧性四元组：P5鲁棒性(Robustness)、P6协调(Coordination)、P7自适应(Adaptation)、P8层级(Hierarchy)。本章§6.13映射表和阅读指引中使用的原理名称（"原理二：可控可观性""原理三：分层分布式""原理四：安全包络""原理六：认知增强""原理八：自主演进"）与P1a完全不对应。须与第七章核实确认后统一，若第七章已重编号则需在本章注明对照关系。

- [ ] **P0-3：阅读指引中"第七章原理二：可控可观性"引用不确**。开篇阅读指引声称本章是"第七章八原理中'原理二：可控可观性'的工程实施指南"，但P1a中P2为Decoupling（解耦）。可控可观性在P1a中并非独立原理编号，而是系统可控可观性作为控制设计前提在多个原理中体现。须修正引用，或确认第七章确实将可控可观性列为独立原理并标注与P1a的对应关系。

## P1问题（重要但非阻塞）

- [ ] **P1-1：§6.1.3/§6.2.4/§6.8三张WSAL等级表内容高度重复**。§6.1.3给出可控性vs WSAL表，§6.2.4给出可观性vs WSAL表，§6.8又给出可控+可观vs WSAL合并表，且三表之间L3/L4/L5的描述不完全一致（§6.1.3中L3="冗余可控"，§6.8中L3="自适应可控"；§6.1.3中L4="自适应可控"，§6.8中L4="全场景可控"）。建议删除§6.1.3和§6.2.4中的表格，各用一句话引用§6.8的合并表，并统一各等级描述。

- [ ] **P1-2：图片路径应使用相对路径**。全部7张图使用`https://raw.githubusercontent.com/...`绝对URL，应改为相对路径`./H/fig_06_XX_name.png`，以确保离线阅读和本地构建正常。

- [ ] **P1-3：§6.5.2可观性矩阵实例的教学效果可加强**。当前实例中$CA = [-0.1, 0]$与$C = [1, 0]$线性相关导致不可观，结论正确但物理直觉不够清晰。建议增加一句解释：**"物理含义：仅测量第一渠池水位时，由于$A$矩阵中$a_{12}=0$（第二渠池对第一渠池无反向影响），第二渠池状态在输出中完全不可见。"**这能帮助读者理解矩阵秩亏与物理拓扑的对应。

- [ ] **P1-4：§6.10.2中SB04QD子程序归属错误**。文中称"LAPACK中的结构化Sylvester求解器（如SB04QD子程序）"，但SB04QD属于**SLICOT**库（Subroutine Library in Control Theory），非LAPACK。须修正为"SLICOT库中的SB04QD子程序"。

- [ ] **P1-5：§6.5.4的E最优/D最优设计缺少定义**。文中直接使用"E最优设计"和"D最优设计"术语，但未给出定义。建议增加括号注释：E最优（maximizing $\lambda_{\min}(W_o)$）、D最优（maximizing $\det(W_o)$或等价地$\ln\det(W_o)$）。

- [ ] **P1-6：缺少与P1a统一传递函数族的显式关联**。本章讨论IDZ模型用于可观性分析，但未将其与P1a Theorem 2的Family α传递函数$G(s) = (1+\tau_m s)e^{-\tau_d s}/(A_s \cdot s)$显式对应。建议在§6.6.2"建立渠道状态空间模型"处增加一句：**"IDZ传递函数即P1a Theorem 2中Family α的Level ii近似"**，强化与CHS体系的关联。

## P2问题（建议改进）

- [ ] **P2-1：引导案例中"可观性矩阵在末端区域严重秩亏"表述需斟酌**。严格来说，可观性矩阵是系统级属性，不是区域级属性。更精确的表述是"系统可观性矩阵的某些行（对应末端区域状态）几乎线性相关，导致这些状态的可观性Gramian特征值极小"。

- [ ] **P2-2：§6.3.3中"合理密度与管网规模的平方根大致成正比"缺少理论依据或文献支撑**。这是一个有用的经验规律，但目前作为断言给出。建议补充来源（文献或工程统计依据）或标注为"经验观察"。

- [ ] **P2-3：§6.9.1中大渡河梯级2012年AGC停用事件**。作为具体工程事件引用，建议确认事实准确性并考虑是否需要文献支撑或注明信息来源。

- [ ] **P2-4：公式(6-1)的目标函数可更完整**。当前仅为成本最小化，但§6.3.1的约束条件（可观性约束、冗余约束等）未用数学形式表达。作为教材，建议至少将可观性约束写为$\lambda_{\min}(W_o(z)) \geq \lambda^*$的形式，使优化问题定义完整。

- [ ] **P2-5：练习题9引用"案例5-6"应为"案例6-3"或§6.7**。题目说"从案例5-6到设计规范"，但第六章不存在"案例5-6"编号，应核实是否指§6.7的城市管网案例。

- [ ] **P2-6：§6.4节位于§6.3之后、§6.5之前，结构略显突兀**。可验证性（MIL/SIL/HIL）在逻辑上更接近§6.8-6.9的运行验证主题，插在传感器布局优化和数学判据之间打断了"工程判据→数学工具→工程应用"的主线。建议考虑移至§6.9之后或合并为独立一节。

- [ ] **P2-7：术语表中"WNAL"定义与CHS标准不符**（与P0-1关联）。修正术语表后，建议同时标注英文缩写由来：WSAL = Water Systems Autonomy Level（参见P1a §7.4，P2A）。

## 亮点

- **引导案例质量高**：开篇城市供水管网案例以"仪表全绿但漏损18%"切入，直觉震撼力强，"有传感器≠可观测"的核心论点在第一段即建立。
- **双通道观测原则**具有很强的工程指导性，将理论可观性要求转化为"直接测量+模型估计"的可操作方案，且与WSAL等级跃迁门槛绑定，形成闭环。
- **§6.6-6.7两个工程实例**质量高，包含完整的"问题→方法→结果→启示"闭环，数据量级合理，且明确标注为"说明性设定"，避免了伪精确。
- **§6.14六类典型误区**非常实用，特别是误区一（数据覆盖率≠可观性）和误区六（只优化可观性忽视可控性），直接针对工程界常见偏见。
- **§6.10大规模分解方法**提供了四种工程可行的方法（子系统分解、稀疏矩阵、PCA降维），并给出了清晰的适用范围对比表，实用价值高。
- **数学部分（§6.5）严谨正确**，从Kalman判据到Gramian到Luenberger观测器到Lyapunov方程，推导链完整无误，且配有物理直觉解释（如Gramian椭球轴长度的含义）。
