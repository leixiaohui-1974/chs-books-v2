## P0问题（必须立即修复）
- [ ] **8.3.1 水击模型变量定义混乱且公式量纲错误**：`q` 同时像“流量”又像“流速”在用；连续方程和动量方程都缺少与截面积 `A` 相关的项。建议统一用 `Q-H` 口径重写，否则后续状态方程、能量函数、控制器全都不稳。
- [ ] **8.2.2-8.2.3 能量函数与稳定性证明不成立**：`V_h` 第二项应是 `1/2·ρLQ^2/A`（若 `Q` 为流量），不是 `1/2·ρLAq^2`；`\dot V` 推导中也不能推出恒非正。建议把这部分降级为“能量视角直觉分析”，正式稳定性证明改成线性化模型或严格的 Hamiltonian/port-Hamiltonian 形式。
- [ ] **8.3.2 机械功率/力矩公式错误**：应为 `P_m = ρgQHη_t`、`T_m = P_m/ω`；当前式子把 `9.81` 和 `g` 重复乘了，工程上不能用。
- [ ] **8.1.2 引导案例参数不闭合**：文中写“管道长度 200m、波速 1000m/s、T_w=2.5s”，这三者不匹配；“PID 到 5s 才开始响应”也不符合调速器实际。建议明确 `T_w` 是全引水系统综合惯性时间而非 200m 压力钢管的传播延迟。
- [ ] **8.7.2 控制架构不可直接落地**：AGC 不应直接下沉为“10ms 功率 PID”。工程上应是“上层给 MW 偏置/功率参考，底层仍由调速器速度下垂/功率模式 + 导叶伺服回路执行”，并加开度、开度速率、压力变化率硬限制。
- [ ] **8.4.4 导叶卡涩后的处置策略错误**：卡涩不是“增大控制增益”能补偿的问题，增益提上去只会把伺服顶死、加剧振荡。应改成“故障隔离 + 冻结 AGC/MPC + 限负荷/保守下垂 + 超阈值进入 MRC/人工接管”。
- [ ] **图8-1/图8-2/图8-3 路径违规且文件缺失**：这是出版级阻塞项，不修正文档无法验收。

## P1问题（重要但非阻塞）
- [ ] **WNAL 映射过度简化**：`L2` 不是“只有 Physical+Cyber”，`L3` 也不是“仅多一个 Social 层”；真正分水岭是 ODD、MRC、xIL 和接管责任闭环。
- [ ] **滑模/反馈线性化写法偏理论化**：缺执行器饱和、死区、回差、导叶限速、压力波动约束。建议至少补 `rate limiter + anti-windup + bumpless transfer`。
- [ ] **MPC 稳定性结论写得太满**：文中是非线性对象，却直接套线性 Riccati 和终端集保证。应注明“以下结论基于工作点线性化/局部模型”。
- [ ] **水轮机单位参数公式需核对**：`n_{11}` 写法缺少常见的转轮直径项，和工程特性曲线口径不一致，后续查表/插值会出问题。
- [ ] **安全机制不充分**：缺独立安全 PLC/硬联锁、通信中断后的本地自治策略、优化器失败后的降级链、人工接管时间窗和演练要求。
- [ ] **案例经济数据缺复算口径**：罚款 8 万、磨损 2 万、成本下降 18%/21% 都没有把计算式和评价窗口讲清楚，可信度不够。
- [ ] **符号体系需收口**：前文 CHS 已有 `A_s, τ_d, τ_m` 固定含义，本章又用 `A_s` 表示调压室截面积。建议改成 `A_st` 或 `A_surge`，并在章首放符号表。
- [ ] **PSS 的层级位置不准**：8.7.1 把 PSS 放到 Cyber 层不合适。PSS 属于励磁侧本地快速控制，不应和 AGC/状态估计并列。

## P2问题（建议改进）
- [ ] **经济优化目标建议补工程约束**：水价值曲线、振动区/空化区惩罚、生态流量、启停成本、检修约束。
- [ ] **观测器示例不够贴近现场**：调压室水位通常可测，更难的是引水系统分布式压力、内部水力状态和有效水头。
- [ ] **8.6.2 在线增益自适应要加护栏**：没有投影、速率限制、影子模式、回滚条件，不应写成可直接在线投运。
- [ ] **习题参数不全**：8.3、8.6 未给出对象矩阵或采样周期，学生无法真正算“最大允许时滞”和 MPC 参数。

## 亮点
- 引导案例选得好，抓住了水电站控制最真实的矛盾：频率、水压、爬坡率、安全边界同时存在。
- 尝试把 CHS 八原理、CPSS 三层和水电站控制问题放到一个框架里，教材组织性强。
- 8.7 给了硬件、软件、协议三个层次，至少开始从“理论框架”走向“系统架构”。
- 参考文献组合合理，经典水电模型、非线性控制、MPC、电力系统稳定性都有覆盖。
