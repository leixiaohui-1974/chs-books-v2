## P0问题（必须立即修复）
- [ ] `§7.2.1.1` 的简化 ID 模型必须回到 CHS 标准口径。若保留简化式，应明确写成 `G(s)=e^{-τ_d s}/(A_s s)`，或至少说明 `k=1/A_s` 的物理意义、量纲和适用边界；同时核对“详见第五章式5-1”的交叉引用。
- [ ] `§7.5` 与 `§7.8.3` 的安全机制需要重写为“硬约束/软约束”两层。结构安全、漫顶、机组保护、闸门机械极限这类硬约束不能写成“授权后可突破”；人工超控只能作用于运行性约束，并需双授权、原因码、时间窗和审计链。
- [ ] `§7.6.3` 的 MIL/SIL/HIL 门槛值需与第十章、第十一章统一。当前本章写 `L3≥90%`，而后文章节已使用 `L3≥95%` 口径；且“覆盖率”分母未定义，工程上无法验收。
- [ ] 图7-1至图7-5必须改成本地相对路径，并同步本地文件名。现在既不符合出版规范，也会导致离线构建和后续维护直接失败。

## P1问题（重要但非阻塞）
- [ ] 正文中的 `Lei 2025a/Lei 2025b` 属悬空引用，必须并入编号文献体系，否则参考文献质量会被质疑。
- [ ] `§7.5.2` 和实例 4 中“黄区降级为 PID”过于粗暴。对长时滞、多约束渠道系统，PID 并不天然比 MPC 安全；建议改为“切换到经 HIL 验证的保守约束控制/规则曲线/MRC 策略”。
- [ ] 实例 1 的“上游闸门开度阶跃增加 10%”不适合作为大系统在运辨识示范。更可落地的写法是“小扰动 PRBS/夜间小窗口试验/历史运行数据辨识”。
- [ ] 实例 3 中“闸站毫秒级”不符合真实 SCADA/PLC/水力过程节拍。应区分 PLC 扫描周期和水工控制周期，建议写成“本地联锁 100 ms-1 s，设备闭环 1-30 s，区域协调分钟级”。
- [ ] `§7.5.2` 的“黄区一般取安全上限 80%-90%”、`§7.9.3` 的“30 秒回滚”、`≥5 年保留`、实例 8 的“参数校正幅度 ±20%”等，都缺工程依据。建议统一标注为“示例值，定稿前由 T3/工程规范确认”。
- [ ] `§7.5.5` 提到“安全包络覆盖率”“黄区自恢复成功率”为 L3 必检项，但这些指标在第十章口径中未见明确定义，需补定义或统一术语。

## P2问题（建议改进）
- [ ] 在 `§7.2-§7.6` 增加一个贯穿算例，用同一渠段把 `A_s、τ_d、τ_m`、三区阈值、MIL/SIL/HIL 场景串起来，工程读者会更容易落地。
- [ ] 给 `§7.5` 补一个最简形式化表达，例如硬约束集合、黄区权重切换、约束传播关系，不然“安全包络”更像理念，不像控制器接口。
- [ ] P6 参考文献建议补更贴近“工业解释、RAG、知识图谱、因果诊断”的直接文献；目前 `[7-14][7-15]` 对本章“认知增强”支撑偏弱。
- [ ] `§7.6-§7.9` 可增加一个“哪些是原则，哪些是示例门槛”的提示框，避免教材读者把示例参数直接抄进招标技术规范。

## 亮点
- 引导案例非常好，准确打中了“单点智能不等于系统自治”的工程痛点。
- 五层依赖结构和 `P4→P8` 约束回边设计得好，既有理论美感，也符合工程安全逻辑。
- `§7.10.4` 的最小实施清单很实用，已经接近项目立项/验收检查表。
- 对认知 AI 的边界把握是对的，明确了“认知增强不能直接替代物理安全闭环”。
