## P0问题（必须立即修复）

- [ ] **P0-1: WNAL与WSAL术语冲突。** 全章使用"WNAL"（Water Network Autonomy Level），但CHS体系（P1a WRR §7.4、P2A、CLAUDE.md术语表）统一定义为**WSAL**（Water Systems Autonomy Level）。术语表中也写作WNAL。这是全书核心术语的不一致，涉及§2.2.4、§2.3表、§2.5.2、§2.5.4、§2.9、§2.10、术语表等至少7处，**必须全部替换为WSAL**。

- [ ] **P0-2: 八原理命名/编号体系与P1a WRR完全不一致。** P1a定义的八原理为——运行四元组：P1 Feedback(反馈)、P2 Decoupling(解耦)、P3 Order Reduction(降阶)、P4 Emergence(涌现)；韧性四元组：P5 Robustness(鲁棒)、P6 Coordination(协调)、P7 Adaptation(自适应)、P8 Hierarchy(层次)。本章使用的命名为"原理一(传递函数化)"、"原理二(可控可观性)"、"原理三(分层分布式)"、"原理四(安全包络)"、"原理五(在环验证)"、"原理六(认知增强)"、"原理七(人机共融)"、"原理八(自主演进)"——与P1a无任何对应关系。§2.3概览表、§2.10映射表均受影响。需要明确：若本书有意采用不同于P1a的教学化重编体系，必须在§2.10或第三章开篇**显式声明映射关系**（例如"本书原理一'传递函数化'综合了P1a中P1 Feedback和P3 Order Reduction的内容"），否则读者对照论文时将产生严重混淆。

- [ ] **P0-3: 图片路径格式不规范且不统一。** 评审规范要求`./H/fig_XX_YY_name.png`，但实际使用了两种GitHub绝对URL：图2-1至2-4使用`https://raw.githubusercontent.com/.../T1-CN/H/fig_02_01.png`，图2-5至2-8使用`https://github.com/.../T1-CN/figure-prompts/generated/fig_02_08a_topology.png`。此外图2-5至2-8的文件名为`fig_02_08a/b/c/d`，与实际图号2-5/2-6/2-7/2-8不匹配，说明重编图号后文件未同步更新。应统一为相对路径`./H/fig_02_05_topology.png`等。

## P1问题（重要但非阻塞）

- [ ] **P1-1: 式(2-3) IDZ传递函数分子因子顺序与P1a不同。** 本章写作$G(s) = \frac{e^{-\tau_d s}(1 + \tau_m s)}{A_s \cdot s}$，P1a写作$G(s) = \frac{(1+\tau_m s) e^{-\tau_d s}}{A_s \cdot s}$。数学上等价，但为与P1a形式完全一致（全书符号一致性原则），建议调换为$(1+\tau_m s)e^{-\tau_d s}$在前的写法。

- [ ] **P1-2: 跨章引用"第十四章物理AI与认知AI引擎"存疑。** CLAUDE.md章节速查表显示第14章为（未明确命名），§2.2.2和§2.6.4多次引用"第十四章物理AI与认知AI引擎"。需确认第14章实际标题是否匹配，避免引用悬空。

- [ ] **P1-3: §2.3.1稳定性判据表述可更严谨。** 文中称"闭环稳定的充分必要条件是控制增益$K < \frac{\pi A_s}{2\tau_d}$"。经Nyquist分析验证数学正确（相穿频率$\omega_{pc} = \pi/(2\tau_d)$处增益条件），但此处省略了IDZ模型中零点$(1+\tau_m s)$的影响——完整IDZ模型$G(s)=\frac{(1+\tau_m s)e^{-\tau_d s}}{A_s s}$的稳定性判据与纯积分延迟$\frac{e^{-\tau_d s}}{A_s s}$不同。建议明确标注"此处以简化纯积分延迟模型为例"。

- [ ] **P1-4: §2.2.4 ODD五元组引用SAE J3016 [2-9]后直接跳到形式化定义，但ODD五元组$\{\mathcal{H},\mathcal{I},\mathcal{E},\mathcal{R},\mathcal{C}\}$是CHS自定义的，非SAE原文。** 应明确注释"CHS将SAE J3016的ODD概念扩展为水系统五元组"，避免读者误以为此五元组出自SAE标准。

- [ ] **P1-5: §2.4.2 MPC公式(2-5a)中求和上限与预测时域/控制时域的关系未说明。** 标准MPC公式通常区分预测时域$N_p$和控制时域$N_c$（$N_c \leq N_p$），此处统一用$N$模糊了二者。对于教材级严谨性，建议至少脚注说明。

- [ ] **P1-6: §2.8前馈增益整定公式表述不够清晰。** 文中说"IDZ传递函数含积分环节$1/s$，不存在有限静态增益"，然后给出$\Delta u_{ff} = -\Delta Q_d / \alpha$。但$\alpha$（执行器增益）尚未在本章定义，而是引用"第四章式(4-5)"。建议在此处至少简要说明$\alpha$的物理含义（单位开度变化引起的流量变化），避免前向引用过度。

## 亮点

- **引导案例质量极高。** "看得见风险、来不及控制"的场景从开篇贯穿到§2.8算例和§2.10小结，形成完整的"提出问题→分析原因→给出方案→验证效果"闭环。三类问题（超调、振荡、设备疲劳）与算例策略A的定量对比令人信服。

- **§2.3.1稳定性判据$K < \pi A_s/(2\tau_d)$的推导简洁且正确**，将Nyquist判据与工程直觉（"长渠道既不能太猛也不能太慢"）对接，是控制论教材的优秀示范。

- **概念速览框设计精巧。** IDZ三参数的"石头入水"类比、四态机的"电梯vs自动扶梯"类比、ODD的"新手司机"类比，降低了跨学科读者的认知门槛。

- **§2.7四态机工程实例**（梯级泵站调水系统）完整展示了正常→受限→降级→接管→逐级恢复的全过程，每步均有具体触发条件和数值，可操作性强。

- **§2.8算例的"合规声明"** 明确标注参数为说明性设定而非实测数据，规避了虚假引用风险，符合学术诚信要求。

- **§2.6.4前沿展望节制得当**，PINN/LLM的讨论不过度膨胀，明确"增强而非替代"的定位，与CHS"机理保底"哲学一致。
