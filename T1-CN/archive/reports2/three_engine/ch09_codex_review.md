## P0问题（必须立即修复）
- [ ] **9.2.1 弹性水击模型的低频近似和频域解释有误。** 对 `G_w(s) = -(1/Z_c) tanh(T_e s)`，在频域应代入 `s = jω`，得到 `tanh(jωT_e) = j tan(ωT_e)`；文中“高频时趋于1”“退化为 `-T_w s`”都不成立，且量纲也不一致。建议重写这一段，明确区分拉普拉斯域与频域，并给出含阻尼近似或模态截断近似，避免误导控制设计。
- [ ] **9.2.2 Thoma条件的“简化推导”不成立。** 现有二阶特征方程若系数均正，本身已稳定，无法直接推出文中的Thoma不等式；当前推导链条在数学上断裂。建议删除该“推导”，改为标准文献引用版，并明确适用前提、参数定义和量纲。
- [ ] **9.7.2 长距离输水案例的性能指标与物理时滞矛盾。** 若纯时滞为 `τ = 600 s` 且输出在末端测量，则系统输出不可能在 `65 s` 或 `180 s` 内完成调节。这个案例目前属于“理论图上可画、现场绝不可能”。建议改成“两层控制”：本地压力/液位内环 + 慢速流量外环/MPC，或重新定义受控量。
- [ ] **图文件路径不合书稿规范。** 现为 `./figures/ch09_xxx.png`，应统一改为 `./H/fig_09_xx_name.png`，否则后续排版和资源校验会出问题。

## P1问题（重要但非阻塞）
- [ ] **引水系统/水轮机模型前后不一致。** 9.2.1 用 `k_q/(T_w s+1)`，9.2.3 又引入线性化矩阵，9.4.1 则切到 IEEE 常用 ` (1 - T_w s)/(1 + 0.5 T_w s)`。建议选一个主模型作为正文主线，其他模型作为“简化层级/适用范围”说明。
- [ ] **9.2.3 水轮机矩阵维度不完整。** 力矩方程含 `Δn`，但矩阵只写了 `Y(s)` 和 `H(s)` 两个输入；这在建模上是不闭合的。建议改成3输入模型，或把 `Δn` 明确列为扰动通道。
- [ ] **9.5.1 的相频公式默认零点都在左半平面，不适用于本章强调的非最小相位零点。** 对 `1 - T_w s`，相位贡献应是额外滞后，不是普通零点的相位超前。建议补充“RHP零点修正项”。
- [ ] **调速器工程实现信息不足。** 当前只有理想PID，缺少永久差率、死区、抗积分饱和、导叶行程/速率限制、无扰切换、故障回退等内容。对SCADA/PLC落地来说，这些不是细节，是必需项。
- [ ] **发电机-电网模型缺少工况边界。** 9.3.2/9.4.1 省略功角 `Δδ` 后，只适用于非常特定的小扰动场景；并网、孤网、甩负荷三类工况不能混写。建议明确适用工况。
- [ ] **案例1的调速设计偏“仿真可用”，不够“机组可投运”。** 300 MW机组仅凭ZN整定和二阶辨识模型，不足以作为工程交付依据；至少要补充压力升高、导叶速率、超速峰值、调压室水位等约束校核。
- [ ] **案例3的“S区快速穿越 10%/s”缺少工程依据。** 这个速率是否超过伺服能力、是否触发水击、是否与导叶/桨叶协调冲突，都没交代。建议给出试验依据和硬保护逻辑，而不是只给一个经验数值。

## 亮点
- 章节结构顺，读者能看出作者想建立“从水力到机电再到控制器”的完整链条。
- 9.4.1 抓住了水电调速最关键的非最小相位问题，这个方向是对的。
- 9.7.2 已经意识到Smith预估器的模型失配问题，没有把它写成“万能补偿器”，这点比很多教材更实在。
- 9.7.3 关注抽蓄S区避开，选题有工程价值，不是停留在一般控制教材层面。
- 参考文献的自引率控制在合理区间，没有明显“文献表为自家论文服务”的问题。
