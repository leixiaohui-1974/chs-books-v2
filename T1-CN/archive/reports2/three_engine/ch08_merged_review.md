## P0问题（必须立即修复）

- [ ] **P0-1：WNAL应为WSAL**。全章使用"WNAL (Water Network Autonomy Level)"，CHS体系统一术语为"WSAL (Water Systems Autonomy Level)"（见CLAUDE.md §H术语表）。涉及8.1.3(c)节表格、正文多处，需全文替换。

- [ ] **P0-2：八原理P1-P8名称和映射全部错误**。8.1.3(a)节的映射表中：
  - P1应为"反馈(Feedback)"，非"传递函数化"
  - P2应为"解耦(Decoupling)"，非"可控可观性"
  - P3应为"降阶(Order Reduction)"，非"分层分布式"
  - P4应为"涌现(Emergence)"，非"安全包络"
  - P5应为"鲁棒性(Robustness)"，非"在环验证"
  - P6应为"协调(Coordination)"，非"认知增强"
  - P7应为"自适应(Adaptation)"，非"人机共融"
  - P8应为"层级(Hierarchy)"，非"全生命周期演进"
  
  必须按P1a WRR §5双四元组（运行四元组+韧性四元组）重新建立CPSS与八原理的映射关系。

- [ ] **P0-3：水力能量函数$V_h$缺失密度$\rho$**。8.2.2节公式：
  
  **错误**：$V_h = \frac{1}{2} A_s g z_s^2 + \frac{1}{2} \rho L A q^2$
  
  **正确**：$V_h = \frac{1}{2} \rho g A_s z_s^2 + \frac{1}{2} \rho L A q^2$
  
  推导过程本身写了$\rho A_s z_s \cdot g \cdot \frac{z_s}{2}$，结果公式却丢了$\rho$，量纲不一致（第一项为$\text{m}^2 \cdot \text{m/s}^2 \cdot \text{m}^2$，缺少$\text{kg/m}^3$）。

- [ ] **P0-4：CHS六元组与CPSS三层的概念冲突**。CHS定义$\Sigma = (P, A, S, D, C, O)$中的P是"Plant（被控对象）"，C是"Controller（控制器）"。但CPSS的Physical层包含了Plant，Cyber层包含了Controller和Sensor，两套框架的"P"和"C"含义冲突。必须在8.1.3节明确说明二者的对应关系，避免读者混淆。建议增加一个映射表：Σ六元素如何分布在CPSS三层中。

## P1问题（重要但非阻塞）

- [ ] **P1-1：Lyapunov证明（8.2.3节）跳步严重**。最终结论$\dot{V} \approx A_s g z_s (q_t - q) \leq 0$未证明为何$\leq 0$。调压室质量守恒$A_s \dot{z}_s = q_t - q$意味着$z_s$上升时$q_t > q$，所以$z_s(q_t-q) > 0$，即$\dot{V} > 0$！这与结论矛盾。需要重新推导，可能需要考虑调压室阻尼项或重新选择Lyapunov函数。

- [ ] **P1-2：能量耗散定理公式无推导**。8.2.2节直接给出$\dot{V}(\mathbf{x}) = -\mathbf{x}^T \mathbf{D} \mathbf{x} + \mathbf{u}^T \mathbf{B}^T \mathbf{x}$，但前面的能量函数$V = V_h + V_m + V_e$是非线性的，不可能直接得到这种二次型形式。需要说明这是线性化后的结果，或给出完整推导。

- [ ] **P1-3：反馈线性化（8.4.2节）推导不完整**。仅给出"步骤1"和"步骤2"的框架，但$L_f^2 y$的具体表达式和相对阶$r$的值均未给出。对于水电系统这个具体问题，应至少完成到控制律的显式表达式。提到"非最小相位特性"但未解释为什么水轮机系统是非最小相位的（水击效应导致初始响应方向与最终方向相反）。

- [ ] **P1-4：符号$A_s$含义冲突**。CHS P1a中$A_s$定义为"水面面积/储能系数"（渠道/水库的储能参数），本章中$A_s$用于"调压室截面积"。虽然物理量纲相同（m²），但在CHS体系内同一符号表示不同物理实体会造成混淆。建议调压室截面积使用$A_{sr}$或$F_s$以区分。

- [ ] **P1-5：水击方程中$q$的定义不清**。8.3.1节连续性方程$\frac{\partial h}{\partial t} + \frac{a^2}{g} \frac{\partial q}{\partial x} = 0$——若$q$是体积流量（m³/s），则方程量纲不平衡（左$\text{m/s}$，右$\text{m}^2/\text{s}^2 \cdot \text{m}^3/(\text{s} \cdot \text{m})$）。标准水击方程中通常$q$代表流速$v$（m/s），或者方程应含$1/A$因子。需明确并修正。

- [ ] **P1-6：定理8.5 Tikhonov定理表述过于简略**。仅一句话"若快慢子系统分别稳定,则复合系统在$\epsilon$充分小时稳定"缺少条件：需要边界层系统的平衡点指数稳定，降阶系统的平衡点渐近稳定。建议补充完整条件或引用参考文献。

- [ ] **P1-7：图片路径格式不符合规范**。当前使用`./figures/ch08_xxx.png`，应统一为`./H/fig_08_XX_name.png`格式（H为chapter assets目录）。

## P2问题（建议改进）

- [ ] **P2-1：CPSS术语来源需要更早说明**。8.1.3(d)节才定义CPSS三层含义，但8.1.3(a)已大量使用。建议在8.1.2节末或8.1.3节开头即给出CPSS定义。

- [ ] **P2-2：[8-12]李德毅文献为僵尸引用**。正文中未见引用[8-12]，参考文献列表中却列出。要么在正文中自然引用，要么删除。

- [ ] **P2-3：CHS传递函数族（Family α/β）未在本章出现**。CHS的核心数学成果是统一传递函数族$G(s)$和$H(s)$，但本章水轮机建模完全采用状态空间方法，未与CHS传递函数族建立联系。建议在8.3.1节增加一个备注，说明水电站压力管道的传递函数属于Family α的特殊形式。

- [ ] **P2-4：MPC稳定性保证部分的Lyapunov方程形式有误**。8.4.2节写"$\mathbf{A}^T P + P \mathbf{A} - P \mathbf{B} R^{-1} \mathbf{B}^T P + Q = 0$"——这是连续时间代数Riccati方程(CARE)，但MPC是离散时间的，应使用离散Riccati方程(DARE)。

- [ ] **P2-5：经济调度目标函数中$C_{fuel}(P)$不适用**。8.5.1节写"燃料成本(对于水电站为水资源机会成本)"，但水电站无燃料成本。建议直接使用$C_{water}(P)$或$C_{opp}(P)$表示水资源机会成本，避免概念混淆。

- [ ] **P2-6：案例数据对比表缺少置信区间**。8.7.4节的对比表给出了点估计值（如"频率偏差±0.15Hz"），但既然进行了100次蒙特卡洛仿真，应给出均值±标准差或置信区间。

- [ ] **P2-7：习题难度分布不均**。8题中6题偏重计算/设计，仅8.7和8.8涉及概念理解和工程实践。建议增加1-2道概念辨析题（如"CPSS三层与CHS六元组的关系"）。

- [ ] **P2-8：8.5.2节一致性协议公式与CHS DMPC代价函数不一致**。CHS体系中多Agent协调使用DMPC（分布式MPC），但本节使用一致性协议（consensus protocol），二者是不同的协调机制。应说明为何此处选择一致性协议而非DMPC，或将二者联系起来。

## 亮点

- **引导案例设计优秀**（8.1.2节）：300MW水电站负荷突增的时间线叙述生动，问题分析切中要害，经济损失估算有"两个细则"依据，是教材写作的范例。
- **数据来源标注诚实**：所有案例数据均明确标注"来自仿真实验"并给出仿真参数，避免了工程教材中常见的"某电站"模糊引用问题。
- **物理推导附有直觉解释**：每个公式后都有"物理意义"段落，有助于非控制论背景的水利工程读者理解。
- **三层融合架构的软件架构图**（8.7.1节ASCII图）清晰实用，贴近工程实际。
- **习题8.3**（Lyapunov-Krasovskii泛函）和**习题8.6**（MPC设计）的设定具有研究前沿性，适合研究生教材。

## P0问题（必须立即修复）
- [ ] **8.3.1 水击模型变量定义混乱且公式量纲错误**：`q` 同时像“流量”又像“流速”在用；连续方程和动量方程都缺少与截面积 `A` 相关的项。建议统一用 `Q-H` 口径重写，否则后续状态方程、能量函数、控制器全都不稳。
- [ ] **8.2.2-8.2.3 能量函数与稳定性证明不成立**：`V_h` 第二项应是 `1/2·ρLQ^2/A`（若 `Q` 为流量），不是 `1/2·ρLAq^2`；`\dot V` 推导中也不能推出恒非正。建议把这部分降级为“能量视角直觉分析”，正式稳定性证明改成线性化模型或严格的 Hamiltonian/port-Hamiltonian 形式。
- [ ] **8.3.2 机械功率/力矩公式错误**：应为 `P_m = ρgQHη_t`、`T_m = P_m/ω`；当前式子把 `9.81` 和 `g` 重复乘了，工程上不能用。
- [ ] **8.1.2 引导案例参数不闭合**：文中写“管道长度 200m、波速 1000m/s、T_w=2.5s”，这三者不匹配；“PID 到 5s 才开始响应”也不符合调速器实际。建议明确 `T_w` 是全引水系统综合惯性时间而非 200m 压力钢管的传播延迟。
- [ ] **8.7.2 控制架构不可直接落地**：AGC 不应直接下沉为“10ms 功率 PID”。工程上应是“上层给 MW 偏置/功率参考，底层仍由调速器速度下垂/功率模式 + 导叶伺服回路执行”，并加开度、开度速率、压力变化率硬限制。
- [ ] **8.4.4 导叶卡涩后的处置策略错误**：卡涩不是“增大控制增益”能补偿的问题，增益提上去只会把伺服顶死、加剧振荡。应改成“故障隔离 + 冻结 AGC/MPC + 限负荷/保守下垂 + 超阈值进入 MRC/人工接管”。
- [ ] **图8-1/图8-2/图8-3 路径违规且文件缺失**：这是出版级阻塞项，不修正文档无法验收。

## P1问题（重要但非阻塞）
- [ ] **WNAL 映射过度简化**：`L2` 不是“只有 Physical+Cyber”，`L3` 也不是“仅多一个 Social 层”；真正分水岭是 ODD、MRC、xIL 和接管责任闭环。

- [ ] **MPC 稳定性结论写得太满**：文中是非线性对象，却直接套线性 Riccati 和终端集保证。应注明“以下结论基于工作点线性化/局部模型”。
