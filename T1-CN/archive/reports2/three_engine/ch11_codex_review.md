## P0问题（必须立即修复）
- [ ] **图表资源与编号不合规。** `./H/fig_11_01_safety_envelope.png`、`./H/fig_11_02_xil_wnal.png`、`./H/fig_11_03_safety_wnal_evolution.png`、`./H/fig_11_04_p4_p5_synergy.png`、`./H/fig_11_05_l2_l3_verification_path.png` 与现有资源名不一致；`./figures/ch11_cbf_safe_set.png`、`./figures/ch11_cbf_qp_architecture.png` 看起来也不存在；§11.16 又把图号重置为“图11-1/11-2/11-3”。建议统一为 `./H/fig_11_XX_name.png`，并全章唯一编号。
- [ ] **WNAL/xIL 准入口径前后冲突。** §11.3 写成 “L2建议，L3必须”，§11.5.4 又写 “L2 必须 MIL/SIL/HIL 关键回路”，§11.10 又变成 “L2 仅 MIL/SIL”；这与第七章“L2 基本 MIL、SIL 推荐、HIL 不要求；L3 必须 SIL+关键 HIL”的口径不一致。必须统一，否则 CHS 体系会断裂。
- [ ] **§11.16 的 CBF 工程化写法当前不可发布。** 水位变量和屏障函数都写成 `h`，符号冲突；泵站例子中 `C_h = 100 m·s²` 单位错误，后续 `Q_pump` 约束式维度也不闭合；“绿区软约束、红区硬约束”破坏了 CBF 的安全不变性前提。建议把 CBF 改成“理论补充附录”，或重写为严格的、可落地的状态约束示例。
- [ ] **供水反例的事故机理错误。** §11.2.6 把“出口压力降到 0.25 MPa”直接推到“泵启动气蚀”，这不成立。气蚀看的是吸入口压力/NPSHA 与 NPSHR，不是单看出压。建议改成“低吸水池水位 + 启停过快 + 未校核 NPSH + 瞬态水锤”组合机理，并把红区动作改成“分级启泵、斜坡升速、校核最小启停间隔”，不能写“强制全开”。

## P1问题（重要但非阻塞）
- [ ] **三区阈值没有真正接入 CHS 参数体系。** 当前只给 `δ=kσ` 或 `δ≥v_maxT_response`，没有把测量时延 `τ_m`、通信/执行时延 `τ_d`、执行器动作时间、模型误差预算纳入。建议改为类似 `δ ≥ v_max(τ_m+τ_d+τ_exec)+ε_model`，明渠再叠加传递函数峰值响应。
- [ ] **§11.2.2 的“80%-90% / 95%-100%”经验阈值过粗，且与前文 δ 法冲突。** 对渠道、水库、泵站、管网，这种按百分比切阈值都不稳妥。建议改成“按工程类型分表给法”，至少分明渠、压力管网、泵机设备三类。
- [ ] **黄区/红区控制策略写得太理想化。** “黄区缩短预测时域”“一律降级为 PID”“红区 MRC+人工接管”都不是通用工程答案。长时滞渠道缩短预测时域可能更危险；很多场景红区应由 PLC/联锁直接进保护，不是上位机再切 MRC。建议改成“收紧约束、限制控制增量、冻结上层优化、必要时切本地规则控制/联锁”。
- [ ] **SIL/HIL 工具表有工程事实问题。** `dSPACE ControlDesk`、`NI VeriStand` 更像测试编排/HIL 环境，不宜当成典型 SIL 核心工具；`Raspberry Pi + Simulink Real-Time` 这条基本不成立。建议按“虚拟 PLC/目标代码主机执行/实时仿真平台/网络仿真器”重写。
- [ ] **价格、周期、覆盖率等数字缺依据。** 如“MATLAB 50 万/年”“HIL 12-18 个月”“覆盖率≥95%”“求解 1.2 ms”“72 小时”“验证周期缩短到 3 天”等都需要注明来源、年份、适用规模；否则像投标宣介，不像教材。

## P2问题（建议改进）
- [ ] **正文重复偏多。** §11.3-11.7、§11.9-11.10、§11.13-11.14 有明显重复，压缩后可把篇幅腾给一个真正可算、可实施的完整案例。
- [ ] **术语缩写存在歧义。** 本章同时出现 Software-in-the-Loop 的 `SIL` 和 IEC 61508 的 Safety Integrity Level `SIL`，建议至少首次出现时强制全称区分。
- [ ] **参考文献有僵尸引用。** [11-17] 至 [11-23] 未在正文实引；若保留，应补正文落点，否则删除。
- [ ] **案例数据需要“来源标签”。** 现在不少案例像综合编写，建议明确“真实脱敏 / 综合重构 / 示意案例”，避免读者误判其证据等级。

## 亮点
- 引导案例切入好，能把“安全包络不是报警，而是控制边界”讲明白。
- 把安全包络放回 P4、把在环验证放回 P5，这条主线是对的。
- 自引率控制合理，4/23=17.4%，没有明显“文献堆砌式自引”。
- 强调黄区自恢复、红区确定性保护、上线前逐级验证，这些工程价值判断是正确的。
- 如果把 CBF 部分改严谨，这章会形成“工程规则 + 数学补充”的很强结构。
