## P0问题（必须立即修复）
- [ ] **锁定度指标需重构为可实施口径**：`N_total` 不可枚举，建议改为“场景库概率加权自动化覆盖率”口径，并给出离散化规则与最小样本量。  
- [ ] **安全机制缺“网络安全+恶意工况”闭环**：当前只覆盖故障，不覆盖欺骗、重放、越权下发；需补 IEC 62443 分区分级、白名单、双通道校验、SIL/HIL 的网络攻击注入测试。  
- [ ] **MRC触发时延不可接受**：写成“≤2个控制步长”在5-15min步长下可达10-30min；应增加事件驱动硬保护链（PLC联锁/ESD）并给出秒级上限。  
- [ ] **时延阈值体系冲突**：同章同时出现5-15min采样与500ms/1000ms阈值，工程语义不一致；需分层定义“过程控制时延、遥测刷新时延、保护动作时延”。  
- [ ] **案例量化口径需复核**：`20.22亿kWh + 节约标煤70万吨 + 减排18.5万吨CO2` 量级不一致，需统一折算系数并给出公式来源。  

## P1问题（重要但非阻塞）
- [ ] **符号体系不统一**：建议统一为CHS符号族（如 $\tau_{comm}, \tau_m, A_s$），避免 `T_*` 与 `\tau_*` 混用。  
- [ ] **OSEM验收指标单一**：仅用“50步收缩至10%”不够，需补 NIS/NEES 一致性检验和异常工况下稳态误差上限。  
- [ ] **HIL通过准则不可审计**：`≤T_comm+T_comp` 建议改为 `p95/p99` 百分位 + 硬截止（hard deadline）。  
- [ ] **WNAL映射需统一**：PBM适用等级在表述中前后不一致，建议给出“按功能而非模型类型”映射表。  
- [ ] **成本效益模型偏乐观**：缺少持续运维成本（模型重标定、人力、网络安全、许可证续费）与停机窗口成本。  
- [ ] **文献规范问题**：如[12-7]期刊名重复“（中英文）”；[12-13]作者信息不完整。
