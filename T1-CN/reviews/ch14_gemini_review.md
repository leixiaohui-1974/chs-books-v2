## P1 重要问题

### P1-1 "检测到传感器故障"段落格式错误（D6）
§14.1.3 中"检测传感器故障。"之后存在明显的段落分割错误（出现孤立的"**"加换行），原文为：
```
**检测传感器故障。

** 当某个传感器读数……
```
这导致该段落标题与正文断开，在排版后将出现破碎的加粗标记。这是明确的格式 Bug。
**建议**：立即修复为 `**检测传感器故障**：当某个传感器读数……`。

### P1-2 EKF 与 UKF 选择未说明工程判据（D1）
§14.1.3 写道"HydroCore 采用的核心算法是扩展卡尔曼滤波器（EKF）或无迹卡尔曼滤波器（UKF）"，但未说明何时选 EKF、何时选 UKF。水利系统的明渠动态通常具有较强非线性（水位-流速关系），UKF 的精度优势应是选择依据之一，但本文未给出。
**建议**：补充选型建议："当系统非线性较弱（如短渠池水位调节）时，EKF 线性化误差可接受，计算量小；当非线性较强（如大流量变化导致的水面形态变化）时，UKF 的Sigma点采样更准确，但计算量约为EKF的3倍"。

### P1-3 安全包络收缩因子 α_s 的工程标定方法缺失（D1）
§14.1.4 给出"控制动作幅度乘以收缩因子 $\alpha_s$（典型值0.5-0.8）"，但未说明如何针对具体工程标定 $\alpha_s$。这是安全关键参数，若选值不当（$\alpha_s$ 过大则黄区保护不足，过小则系统频繁受限），将直接影响安全性。
**建议**：补充"$\alpha_s$ 的工程标定方法：在 MIL 阶段通过参数扫描，找到满足'黄区内最大扰动不超过红区'的最小 $\alpha_s$ 值，再留10%裕度取整定值。典型值0.5-0.8适用于响应时间5-30分钟的渠道系统，对响应更快的管网系统建议取0.3-0.5"。

## P2 改进建议

### P2-1 表14-1 模型验证流水线（D6）
表14-1 的判定标准（如"偏差<5cm"）无括注说明这是哪种标准差或 RMSE 指标。建议注明"5cm 指 RMSE（均方根误差）而非最大误差"，并注明适用渠道规模（典型渠道，水位量程约1-5m）。

### P2-2 Chi-Square 检验的 N 值选择（§14.1.6）（D1）
正文给出"典型值 N=5-10"，但未说明 N 值对检测灵敏度和误报率的影响。小 N 值检测延迟短但误报率高，大 N 值反之。
**建议**：补充"N 的选择取决于两类错误的权衡：N=5（对应95%分位值 $\chi^2_{0.05,5}=11.07$）检测延迟短，适合快速响应场景；N=10 适合允许较长检测窗口的场景。建议在 MIL 阶段通过故障注入实验确定最优 N"。

### P2-3 §14.1.3 中引用 [14-5] 孔令仲、雷晓辉等（D4）
该文献为作者团队自引，引用了"EKF 在南水北调中线工程的研究"——应确认该文献是否已正式发表（期刊/会议/报告），并补充完整引用信息（期刊名、年份、页码）。自引率评估：目前可见 [14-20] 引用 CHS 六元框架，[14-5] 引用团队研究，自引比例需控制在合理范围。

### P2-4 降阶模型精度"±3%"的参照基准（§14.1.2）（D1）
"降阶模型的精度在设计工况下与高保真模型的偏差通常在±3%以内"——3% 是相对误差还是绝对误差？相对于什么基准值（水位量程？设计流量？）？
**建议**：明确"±3%指相对于渠池设计水位量程（通常1-3m）的归一化误差"，或改写为"绝对水位误差<5cm（典型渠道）"。

### P2-5 Tube-MPC 和 Reference Governor 作为替代方案（§14.1.4）（D6）
正文提到"对需要更强稳定性保证的场景，可采用 Tube-MPC 或 Reference Governor 方案"，这是有价值的补充，但对本科读者来说这两种方法未经介绍就作为替代方案列出，理解障碍较大。
**建议**：在后面加一句简短说明："Tube-MPC 通过预先计算不变集保证约束满足，Reference Governor 通过在参考输入前端加滤波器避免约束违反——二者均可提供强理论保证，但计算复杂度较高，适合高安全等级（L4+）场景。"
