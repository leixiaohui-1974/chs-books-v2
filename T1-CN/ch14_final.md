# 第十四章　案例二：大渡河梯级水电站——链状系统的协调自主控制

---

> **引导案例：2012年，一个被迫停用的AGC系统**
>
> 2012年，大渡河深溪沟水电站（装机容量 660 MW）完成了 AGC（自动发电控制）系统的相关试验。按理说，这应该是迈向智能化调度的重要一步。然而，投运不久后，深溪沟 AGC 就因为一个棘手的问题而被迫停用——**它无法协调上游瀑布沟（3600 MW）的出库流量变化对深溪沟水库水位的影响**。
>
> 深溪沟是大渡河梯级的典型"反调节电站"：它的主要功能是平滑上游瀑布沟的出库流量波动，但自身库容极小（只有数百万 $\text{m}^3$ 的有效调节量）。当瀑布沟为了响应电网负荷变化而快速调整出力时，几小时内深溪沟的入库流量可能剧变数百至上千 $\text{m}^3$/s。此时，深溪沟 AGC 系统面临两难困境：**按电网指令运行**（出力固定）则水库迅速被蓄满或拉空；**优先控制水位**（出力跟随来流变化）则违反电网调度纪律，被罚款。
>
> 这一两难局面的根源，不在于深溪沟 AGC 算法不好，而在于**三个独立的 AGC 系统（瀑布沟、深溪沟、枕头坝）各自为政，没有协调机制**。每个电站的控制器都在局部最优化，但三站联合的梯级系统却陷入了"囚徒困境"——各站独立最优但整体最差。
>
> 这就是 CHS 原理三（分层分布式控制）的典型失效场景：缺少全局协调层，导致局部最优无法汇聚为全局最优。2018 年正式投运的**瀑深枕梯级 EDC 系统**，正是对这一失效的系统性修复。

**本章目标**：通过大渡河瀑布沟-深溪沟-枕头坝梯级（瀑深枕梯级）的 EDC 系统案例，系统展示 CHS 理论在**链状系统（Chain）**场景的工程实践。读完本章，读者应能回答：梯级水电站的"水力电力双重耦合"是如何产生的，为何使得各站独立控制失效？梯级 EDC 的负荷分配模型是如何在经济性和安全约束之间取得平衡的？梯级协调控制是如何与沙坪（单站）相互依存的？

## 14.1 工程概况

### 14.1.1 瀑深枕梯级基本参数

**瀑布沟-深溪沟-枕头坝一级梯级**位于四川省大渡河下游汉源-乐山段，是大渡河梯级开发的核心骨干电站群，三站自上游至下游依次串联布置，全程河道距离约60公里。

| **电站** | 装机容量 | 机组型式 | 水库特性 | 正常蓄水位/死水位 | 投运年份 |
|------|---------|---------|---------|-----------------|---------|
| **瀑布沟** | 3600 MW（12×300 MW）| 混流式 | 年调节水库，总库容53亿 $\text{m}^3$，调节库容38.94亿 $\text{m}^3$| 850m/790m | 2010 |
| **深溪沟** | 660 MW（4×165 MW）| 混流式 | 日调节水库，有效库容约200-400万 $\text{m}^3$（**反调节**）| 641m/637m | 2012 |
| **枕头坝一级** | 360 MW（3×120 MW）| 混流式 | 日调节水库，库容小（**二重反调节**）| 622m/617m | 2014 |

三站**总装机容量 4620 MW**，是四川电网最重要的调峰电源之一。从水库调节特性来看，三站形成"强-弱-弱"的不对称组合：瀑布沟具有年调节能力，是流量和负荷的主要调节器；深溪沟和枕头坝仅具备日调节能力，库容极小，是承受瀑布沟调节波动的被动"缓冲器" [14-18]，这一结构性矛盾是梯级控制挑战的根源。

### 14.1.2 水力电力双重耦合特征

瀑深枕梯级呈现独特的"双重耦合"结构：

**水力耦合**：三站依次串联，上游出库即为下游入库，站间无大型支流汇入。根据互相关函数分析（来源：原始研究报告第3.3节），水力传播时间因流量大小而变化：瀑布沟出库到深溪沟入库约100-110分钟（汛期约95分钟，枯水期约110分钟）；深溪沟出库到枕头坝约70-80分钟；枕头坝出库到下游沙坪约70-85分钟（见第十三章）。瀑布沟调整出力的影响将在约1.5-3小时内依次传递至三个下游电站，形成"涟漪效应"。

**电力耦合**：三站均接入四川500kV电网，共同承担调峰调频任务 [14-7]。电力调令频率通常为5分钟一次，调整幅度可达数百MW，而水力响应时滞长达1-2小时，这是EDC控制的本质难点。

以典型汛期工况为例：瀑布沟出库流量可在数小时内变化超过2000 $\text{m}^3$/s（相当于全站满发到半发），而深溪沟有效库容仅约300万 $\text{m}^3$，若不进行主动控制，2000 $\text{m}^3$/s的流量差仅需约25分钟即可使深溪沟水位越限整整1米，已超过深溪沟4米的水位调节范围的四分之一。

### 14.1.3 深溪沟 AGC 停用事件与 EDC 开发背景

理解本章的核心，需要回溯一个标志性事件：**2012年深溪沟AGC被迫停用**。

深溪沟水电站2012年投运后按规划接入集控AGC系统，与瀑布沟共同参与梯级调峰。然而运行实践中，独立运行的深溪沟AGC无法处理来自瀑布沟的大幅变化来流：每当瀑布沟执行大幅度调峰（日内变化幅度常达1000-2000MW），深溪沟入库流量在数小时内剧烈变化，深溪沟AGC按独立控制逻辑维持恒定出力指令，导致水库水位频繁越限、被迫大量弃水。运行人员为保安全，只能手动退出深溪沟AGC，改由人工实时监视调整。此后6年（2012-2018年），深溪沟AGC长期退出，成为梯级调度中一个长期存在的运行欠账。

这一事件清晰地揭示：问题的根源不在于深溪沟AGC算法本身，而在于缺少**全局协调层**来统筹协调瀑布沟的调峰行为与下游反调节电站的承受能力 [14-2]。EDC系统的设计目标，本质上是要在系统架构层面填补这一缺失。

## 14.2 梯级控制的特殊挑战：水力耦合与电力协同

### 14.2.1 "囚徒困境"的系统动力学解析

将三站 AGC 系统各自独立运行的情景用系统动力学模型分析，会发现一个反直觉的现象：**每个单站AGC算法越先进，梯级整体的失调反而可能越严重**。

原因在于：瀑布沟 AGC 收到电网指令后，迅速调整出力（这是正确的，响应快）；深溪沟 AGC 检测到入库流量快速变化（这是正确的，感知准）；但深溪沟 AGC 的决策是"维持电网下达的出力"（这是局部正确的，但全局错误的），导致水位越限。

这种局部最优导致全局次优的现象，在博弈论中称为**纳什均衡陷阱**（Nash Equilibrium Trap）——没有任何一个站有单方面改变策略的动机，但三站集体采取不同策略（例如深溪沟让出部分出力给枕头坝）可以让所有人都受益。梯级水电站AGC优化分配中的这一博弈困境，在李继清等 [14-17] 对梯级AGC分配模型的研究中也有类似发现。

### 14.2.2 站间负荷转移的物理约束

梯级 EDC 允许"负荷在站间转移"（例如，把本来分配给深溪沟的 100 MW 转移给枕头坝），但这种转移受到物理约束，类似于Lund和Guzman [14-14] 分析的串联水库操作规则推导中所面临的耦合约束：

**约束一：站间出力转移速率限制**。为防止电网频率波动，单次负荷转移量不得超过梯级总容量的 10%，且每两次转移之间需间隔至少 3 分钟。

**约束二：机组振动区约束**。三站共有 19 台机组，每台机组都有若干"振动区"出力段（在该区间长期运行会损伤机组）。EDC 分配时必须确保没有机组长时间停留在振动区，必要时采取"快速穿越"策略。

**约束三：水库水位约束**。深溪沟和枕头坝的水位必须维持在安全区间内（黄区/红区触发不同等级响应），且水位变化速率有限制（防止快速水位变化损伤鱼道和水工建筑物）。

## 14.3 EDC 建模与站间负荷分配

### 14.3.1 EDC 系统总体架构

瀑深枕梯级 EDC 系统采用"**集控层统一优化 + 电站层独立执行**"的两级架构，体现了 CHS 原理三（分层分布式控制）的两层架构设计，与第十一章 HydroOS 的"全局协调层"与"本地优化层"划分 [14-4] 相呼应。

```
集控层（大渡河集控中心）
├── 梯级 EDC 优化引擎（负荷分配决策）
├── 水情预报模块（区间入流预测）
└── AGC 接口（接收电网下达的梯级总出力指令，遵循IEC 61970 [14-20] EMS-API规范）
         ↓ 分解后的各站出力指令
电站层（各电站厂控 AGC）
├── 瀑布沟厂控：执行分配出力，管理 12 台机组
├── 深溪沟厂控：执行分配出力，监控水位，触发黄/红区逻辑
└── 枕头坝厂控：执行分配出力，监控水位，与沙坪预调通讯
```

### 14.3.2 站间负荷分配数学模型

EDC 的核心是求解以下优化问题：

**目标函数**（梯级耗水率最小化）：

$$J = \min \sum_{j=1}^{3} \sum_{i \in \mathcal{U}_j} q_{ij}(p_{ij}, h_j)$$

其中：
- $j \in \{1,2,3\}$ 代表瀑布沟、深溪沟、枕头坝
- $\mathcal{U}_j$ 为第 $j$ 站运行机组集合
- $q_{ij}(p_{ij}, h_j)$ 为第 $j$ 站第 $i$ 台机组在当前水头 $h_j$ 下，输出 $p_{ij}$ MW 所需的发电流量（由机组 N-H-Q 特性曲面插值得到）

**约束条件**（完整列表）：

**（C1）功率平衡约束**——梯级总出力必须精确匹配电网指令：

$$\sum_{j=1}^{3} \sum_{i \in \mathcal{U}_j} p_{ij} = P_{cmd}$$

其中 $P_{cmd}$ 为电网下达的梯级总出力指令（MW）。该约束是硬约束，偏差超过电网考核阈值（通常为指令值的 $\pm$2%）将被罚款。

**（C2）水位约束**——反调节电站水库水位必须维持在安全运行区间内：

$$z_{min,j} \leq z_j(t) \leq z_{max,j}, \quad j = 2,3$$

其中深溪沟水位范围 $z_{min,DS}$ = 637 m（死水位）至 $z_{max,DS}$ = 641 m（正常蓄水位），有效调节幅度仅4 m；枕头坝水位范围 $z_{min,ZTB}$ = 617 m（死水位）至 $z_{max,ZTB}$ = 622 m（正常蓄水位），有效调节幅度5 m。在实际控制中，深溪沟和枕头坝均在死水位与正常蓄水位之间设置了"可运行区"（$z_{down}$ ~ $z_{up}$），仅在可运行区内运行属于绿区，超出可运行区但未达极限属于黄区，触及极限为红区。

**（C3）出力约束**——每台机组出力不得超出可用区间，且须避开振动区：

$$P_{min,ij} \leq p_{ij} \leq P_{max,ij}, \quad \text{且} \quad p_{ij} \notin \Omega_{vib,ij}$$

其中 $\Omega_{vib,ij}$ 为第 $j$ 站第 $i$ 台机组的振动区出力集合。振动区约束在EDC中通过引入非高效运行区间惩罚函数实现：当分配出力落入振动区时，目标函数中附加一个大幅惩罚项，迫使优化器将出力推出振动区或采用"快速穿越"策略。

**（C4）流量约束**——各站出库流量须满足上下游水工建筑物和河道安全要求：

$$Q_{min,j} \leq Q_{out,j}(t) \leq Q_{max,j}, \quad j = 1,2,3$$

瀑布沟出库流量上限受下游河道安全泄量限制，深溪沟和枕头坝出库流量下限受鱼道最小生态流量约束。

**（C5）水力耦合约束**——上游出流经传播时滞后成为下游入流：

$$Q_{in,j+1}(t) = Q_{out,j}(t - \tau_j) + Q_{lat,j}(t), \quad j = 1,2$$

其中 $\tau_j$ 为站间水力传播时延（瀑-深约95-110分钟，深-枕约70-80分钟），$Q_{lat,j}$ 为区间旁侧入流。该约束将三站耦合为一条完整的水力链：瀑布沟在 $t_0$ 时刻的出库决策，将在 $t_0 + \tau_1$ 影响深溪沟水位，在 $t_0 + \tau_1 + \tau_2$ 影响枕头坝水位。

**（C6）站间转移约束**——防止负荷大幅转移引起电网频率扰动：

$$|\Delta P_j| \leq 0.1 \cdot P_{total}, \quad j = 1,2,3$$

单次负荷转移量不超过梯级总容量的10%（约462 MW），且相邻两次转移间隔不少于3分钟。

**（C7）水量平衡动态约束**——水库蓄量的离散时间演化：

$$V_j(t+\Delta t) = V_j(t) + \left(Q_{in,j}(t) - Q_{out,j}(t)\right) \cdot \Delta t, \quad j = 1,2,3$$

该约束将水位约束（C2）与流量决策联系起来：出库流量 $Q_{out,j}$ 由发电流量和泄洪流量两部分组成，发电流量又由机组出力分配决定。

**（C8）机组最短开停机时间约束**——防止机组频繁启停损伤设备：

每台机组开机后须持续运行不低于最短开机时间，停机后须持续停机不低于最短停机时间。原始研究报告给出了DLC模式下多台机组顺序开停的具体时间参数（每台机组间隔数分钟），确保启停过程有序可控。

上述八类约束构成了EDC优化问题的完整约束体系，与Yeh [14-12] 对水库管理模型的经典综述中总结的约束分类一致。其中（C1）功率平衡和（C2）水位安全是核心硬约束，（C5）水力耦合是梯级问题区别于单站问题的本质特征，（C3）振动区和（C8）启停时间是水电调度特有的工程约束。

**求解方法**：针对该混合整数非线性规划问题，采用"**改进差分进化算法（DE）+ 动态规划（DP）**"两阶段求解，这一组合策略借鉴了Cheng等 [14-15] 将智能优化算法与传统方法混合求解水电调度问题的思路：
- 第一阶段（DE）：确定各站运行机组组合（整数决策）
- 第二阶段（DP）：在确定机组组合的基础上，优化各机组出力分配（连续决策）

要求求解时间 < 10 秒（满足 AGC 服务的时效性要求）。

### 14.3.3 机组特性参数的在线率定

机组 N-H-Q 特性曲面的准确性直接决定 EDC 分配的经济性。传统做法是直接沿用出厂模型试验曲线，但原型机组安装后的实际特性与模型试验存在偏差，且各台机组因安装工艺和后期大修的差异，特性曲线会呈现出显著的个体差异，若忽略这些差异并在机组间平均分配负荷，将引入可观的经济损耗（来源：原始研究报告第2.1.2节）。

率定方法采用"现场实测+曲线拟合"：在现场实测若干典型水头（如5m、10m、15m等）下的流量、出力数据，剔除明显不合理数据后，按照"质心法"将某一典型水头下的实测数据按平均总趋势修正，再用三次曲线最小二乘拟合成该水头下的机组出力特性曲线 $q_i = a_0 + a_1 p + a_2 p^2 + a_3 p^3$；不同典型水头下的曲线组合，通过"原函数分片近似法"进行水头维度的插值，构成完整的N-H-Q特性曲面。原函数分片近似法的核心思想是：将水头范围分为若干片段，每片段内仅需两条相邻已知水头的单变量曲线通过线性插值即可生成任意水头下的特性曲线，算法简单、计算速度快且占用内存少，满足在线实时调用的需求。

在工程实现中，EDC系统对三站每台机组的特性曲线独立维护：每年汛前汛后各进行一次全站机组真机试验更新曲面；日常运行中通过"等微增率预测值与实测发电流量偏差"的持续监测，识别特性曲线漂移，当偏差超出阈值时触发参数核查，确保曲线始终反映机组的真实运行特性。

### 14.3.4 改进双向逐次逼近（BDPSA）求解算法

站间负荷优化分配是一个混合整数非线性规划问题 [14-8]，决策空间随机组台数和电站数的增加呈指数增长（"维数灾"问题）。研究报告提出了**改进双向逐次逼近（Bidirectional Dynamic Programming Successive Approximation，BDPSA）**算法，是对传统动态规划（DP）的关键改进（来源：原始研究报告第4.2节）。

**传统动态规划的局限**：传统DP [14-10] 将优化过程视为单向多阶段决策，从初始状态（总负荷为$P_{cmd}$）逐步分配到终止状态（各站负荷分配完毕），计算量随阶段数呈几何级增长，无法满足AGC要求的单次求解时间<1秒的实时性要求。

**BDPSA的创新**：利用最优化原理的核心特性——最优路径的任意子路径也必定是最优路径——将原问题分解为正向和逆向两个子问题同时求解，这一思路是对Yakowitz [14-11] 总结的水资源动态规划方法的重要扩展。具体而言，假设全局最优分配路径为 $P^*$，则左半段（从 $P_{cmd}$ 到中间状态 $P_{mid}$）和右半段（从 $P_{mid}$ 到零负荷）必然都是各自的最优路径。因此，EDC系统可以用计算机多线程技术，正向（从上游向下游）和逆向（从下游向上游）同时求解，理论上将计算时间缩短至原来的一半以下。

**BDPSA算法的完整流程**可分为五个步骤：

**步骤一：初始化与变量离散化**。主线程从数据库读取各站水位-库容关系曲线、水位运行限制、出力运行限制、机组振动区参数等基础数据；通过通信系统从各站AGC实时获取当前水库上下游水位、入库流量、区间流量、机组实发出力等运行数据。在此基础上，对状态变量（各站待分配有功负荷）和决策变量（各站分配出力）进行离散化，建立搜索网格。

**步骤二：启动双向并行递推**。主线程启动子线程A（正向递推）和子线程B（逆向递推）。子线程A从初始状态（梯级总待分配负荷 $P_{cmd}$）出发，按电站顺序（瀑布沟→深溪沟→枕头坝）逐阶段搜索最优分配路径；子线程B从终止状态（待分配负荷为零）出发，逆序（枕头坝→深溪沟→瀑布沟）搜索最优路径。两个子线程可采用同步或异步两种并行方式。同步方式下，每轮两个子线程各递推一个阶段后同步等待；异步方式下，两个子线程独立连续递推，互不等待。异步方式在各阶段计算量差异较大时（如瀑布沟12台机组搜索空间远大于枕头坝3台机组）效率优势更为明显。

**步骤三：双向耦合计算**。当子线程A的当前计算阶段 $k_A$ 与子线程B的当前计算阶段 $k_B$ 满足 $k_A \geq k_B$ 时，说明两个子线程已共同覆盖所有阶段，主线程通知两子线程停止递推。选择耦合阶段 $k^*$（同步方式下由阶段总数的奇偶性确定，异步方式下由两子线程实际递推速度决定），在耦合阶段遍历待分配负荷的所有可行取值 $P_{mid}$，计算正向总目标函数 $J^+(P_{mid})$ 与逆向总目标函数 $J^-(P_{mid})$ 之和 $J^+(P_{mid}) + J^-(P_{mid})$，取使总目标函数最小的 $P_{mid}^*$ 为最优耦合点。

**步骤四：拼接最优决策序列**。将 $P_{mid}^*$ 代入正向最优决策子序列和逆向最优决策子序列，拼接为完整的最优分配方案：各站分配出力 $(P_1^*, P_2^*, P_3^*)$，使得梯级总耗水率最小且所有约束均满足。

**步骤五：约束校验与输出**。对拼接后的分配方案进行全约束校验（功率平衡、水位安全、振动区回避、站间转移限制等）。若存在轻微约束冲突（如因机组检修导致可用机组集合变化），执行快速在线修正；若约束冲突严重，退回上一步调整耦合点。校验通过后，输出各站出力指令至电站层AGC执行。

**双向耦合机制**的数学本质：记正向递推得到的耦合阶段目标函数为 $J^+(P_{mid})$，逆向递推为 $J^-(P_{mid})$，则耦合问题转化为一维有约束优化：

$$P_{mid}^* = \arg\min_{P_{mid} \in [P_{min}, P_{max}]} \left[ J^+(P_{mid}) + J^-(P_{mid}) \right]$$

该一维问题可通过在可行域内遍历计算求解，计算量极小。这一机制有效避免了传统DPSA算法中初始轨迹选择不当导致收敛缓慢的问题。

**两层求解架构**：在工程实现中，BDPSA被嵌入"站间负荷分配+站内机组分配"的两层求解架构。上层（站间层）将三个电站概化为单一等效机组，用工程化搜索算法确定各站分配出力——具体做法是：假定深溪沟分配出力 $P_2$、枕头坝分配出力 $P_3$，则瀑布沟分配出力 $P_1 = P_{cmd} - P_2 - P_3$；在满足所有约束条件的前提下，遍历搜索使梯级总耗水量最小的 $(P_1^*, P_2^*, P_3^*)$ 组合。下层（站内层）在确定各站分配出力后，用BDPSA或等微增率法将出力进一步分配到各台机组。这种分层求解大幅降低了计算复杂度：上层仅需搜索三维空间（三站出力），下层在各站独立求解（最大12维，对应瀑布沟12台机组）。

**工程化"经济运行总表"**：在BDPSA基础上，研究报告进一步将不同水头（以0.5m为步长覆盖各站运行水头全范围）和不同总负荷指令（以1MW为步长）的组合预先离线计算，建立"经济运行总表"。实时运行时，EDC系统只需根据当前水头和指令负荷直接查表插值，即可在毫秒级时间内获得接近最优的分配方案，彻底满足实时性要求。对于查表结果与约束有轻微冲突的情形（如机组检修导致可用机组集合变化），再进行快速在线修正。

## 14.4 控制模式与策略库

### 14.4.1 策略库总体设计

EDC 系统设计了**五种主动分配策略模式**（由调度员手动选择或触发条件自动切换）和**三种自动触发应急控制模式**，共计八种控制模式，综合考虑了经济、安全和环境等多重约束之间的平衡 [14-27]。这一"策略库 + 状态机"设计，使 EDC 能够在瀑深枕三站的不同运行状态组合下，始终选用最适合当前水情和电情的分配策略（来源：原始研究报告第3.3节、第5.4节）。

深溪沟和枕头坝各自的水位状态分为三档：可运行区（绿区，$z_{down} \leq z \leq z_{up}$）、高水位区（黄区上，$z_{up}  \lt  z \leq z_{蓄}$）和低水位区（黄区下，$z_{死} \leq z  \lt  z_{down}$）。两站水位状态的不同组合（$3 \times 3 = 9$ 种）构成了策略选择的状态空间。EDC根据当前水位状态组合、是否弃水、负荷变化幅度等条件，按优先级自动选取或推荐策略模式。

### 14.4.2 八种控制模式详表

**表14-1　EDC八种控制模式总览**

| **模式编号** | 名称 | 触发条件 | 控制逻辑 | 优先级 | 适用场景 |
|---------|------|---------|---------|--------|---------|
| **M1** | 水位平稳模式 | 深枕水位均在绿区，调度员选择或默认 | 以深枕水位变化最小为目标，按流量平衡分配站间负荷 | 常规 | 来流平稳、水位正常的日常运行 |
| **M2** | 最大蓄能模式 | 深枕水位均在绿区，调度员选择 | 以瀑布沟水位最高为目标（"以电定水"），调增优先下游、调减优先上游 | 常规 | 丰水期或高价电时段，需最大化蓄能 |
| **M3** | 少调负荷模式 | 深枕水位均在绿区，调度员选择 | 深枕分配负荷变化量最小，调增调减优先瀑布沟承担 | 常规 | 深溪沟AGC投入率低时保护下游设备 |
| **M4** | 深枕负荷平衡模式 | 调度员选择（需人工固定一级电站出力时） | 所有负荷变动分配到瀑布沟；人工改深枕则反向调瀑布沟 | 常规 | 一级电站有特殊运维需求 |
| **M5** | 枕站负荷平衡模式 | 调度员选择（枕头坝需独立维护时） | 负荷变动分配到瀑深两站；人工改枕则反向调瀑深 | 常规 | 枕头坝检修或独立运维需求 |
| **E1** | 水位异常控制模式 | 深溪沟或枕头坝水位进入高水位区或低水位区，且无回归趋势 | 厂间负荷重新分配使异常水位尽快回归可运行区中间值；退出条件：水位回到中间值$\pm$0.8m以内 | **高**（自动触发） | 水位偏离过大的紧急水位调节 |
| **E2** | 弃水控制模式 | 任一电站发生弃水，且深枕水位均在绿区 | 增负荷优先分配给有弃水电站，减负荷优先减无弃水电站；多站弃水时按梯级总弃水最小分配 | **高**（自动触发） | 来水超过发电能力时减少弃水损失 |
| **E3** | 小负荷调节模式 | 相邻两次设定值差值$\leq$50MW | 小负荷差额由单一电站承担，避免多站同时微调引起调节死区 | **中**（自动触发） | 小幅度负荷波动的精细分配 |

**优先级逻辑**：三种应急模式按"E1 > E2 > E3"排列，优先级均高于五种常规模式。当应急条件满足时，EDC自动进入对应应急模式，覆盖当前常规模式选择；当应急条件消失时，自动回退到此前的常规模式。这一"应急模式具有抢占权"的设计，确保安全约束始终优先于经济优化。

### 14.4.3 各模式的决策逻辑详述

**M1 水位平稳模式**是EDC的默认运行模式。其核心思想是"以水定电"——根据各站当前入库流量，按流量平衡原则分配各站发电出力，使深溪沟和枕头坝的水库水位变化最小。具体实现中，EDC首先根据瀑布沟入库流量和区间入流估算各站的"平衡出力"（即使水位不变的出力），然后在平衡出力基础上根据梯级总出力指令进行微调分配。该模式下，水位波动通常控制在$\pm$0.2 m以内，远低于黄区阈值。

**M2 最大蓄能模式**的核心思想是"以电定水"——在满足电网指令的前提下，尽量将水量保留在调节库容最大的瀑布沟。调增负荷时，优先增加深溪沟和枕头坝的出力（消耗已有的下游水量），从而减少瀑布沟出库；调减负荷时，优先减少瀑布沟出力（减少出库、蓄水）。该模式的调增调减策略根据深溪沟和枕头坝各自的水位区（正常/低水位/高水位）动态调整，形成覆盖9种水位状态组合的调度矩阵。例如：深溪沟高水位+枕头坝正常→调增时全额分配给深溪沟；深溪沟低水位+枕头坝高水位→调增时分配给枕头坝，同时减少深溪沟出力让水位回升。

**M3 少调负荷模式**的设计背景是2012-2018年深溪沟AGC长期退出的特殊时期。该模式以深溪沟和枕头坝的分配负荷变化量最小为目标，负荷调增调减优先级为"瀑布沟→深溪沟→枕头坝"。简言之，尽量让装机最大、调节能力最强的瀑布沟多调，使一级电站少调甚至不调，降低一级电站的调节压力和调节次数。在深溪沟AGC投运率低的特殊时期，少调模式可以大幅降低深溪沟的调节次数，减少设备损耗。

**M4/M5 负荷平衡模式**是为人机协同设计的"半自动"模式。M4模式下，所有调令负荷变动全部分配到瀑布沟；若运行人员手动更改深溪沟或枕头坝的分配负荷，EDC程序自动反向调整瀑布沟出力，保持三站总负荷不变。M5模式类似，但将枕头坝"隔离"出来由人工控制，瀑深两站按最大蓄能策略自动分配。这两种模式允许运行人员对特定电站进行手动干预而不破坏EDC框架的完整性，体现了CHS原理七（人机共融）的工程实现。

**E1 水位异常控制模式**是安全优先级最高的应急模式。当深溪沟或枕头坝水位进入高水位区或低水位区且没有返回可运行区的趋势时，EDC自动进入该模式，通过厂间负荷重新分配将水位调节至可运行区中间值。例如：深溪沟进入高水位区→EDC增加深溪沟出力（加速消耗库水），同时减少瀑布沟出力（减少向下游放水）；枕头坝进入低水位区→EDC减少枕头坝出力（减缓出水），同时将差额转移至瀑布沟。退出条件为水位回到可运行区中间值$\pm$0.8 m以内。

**E2 弃水控制模式**在任一电站发生弃水时自动启动。增负荷优先分配给有弃水的电站（加大发电量消纳弃水），减负荷优先在无弃水的电站减少。多站同时弃水时，按梯级总弃水流量最小原则分配，将多余流量存储在上游调节库容最大的瀑布沟以减少总弃水量。

**E3 小负荷调节模式**当相邻两次设定值差值$\leq$50MW时自动启动。该模式将小负荷差额由单一电站承担（通常为瀑布沟），避免多站同时微调引起的调节死区效应，提高小负荷分配精度。在电网负荷平稳、仅有微小波动的夜间低谷时段，该模式可显著减少不必要的跨站调节动作。

### 14.4.4 策略切换的状态机逻辑

八种模式之间的切换遵循"**应急优先、自动切换、无缝衔接**"的原则。EDC内部维护一个有限状态机，每个采样周期（5分钟）检查一次触发条件：

1. 首先检查E1触发条件（水位异常），若满足则进入E1，覆盖一切；
2. E1未触发时，检查E2触发条件（弃水），若满足则进入E2；
3. E1、E2均未触发时，检查E3触发条件（小负荷），若满足则进入E3；
4. 三种应急均未触发时，维持当前常规模式（M1-M5）或按调度员选择切换。

从CHS理论视角看，这一状态机逻辑与HydroOS"四态机"（正常/降级/紧急/维护态）的运行时治理机制是一脉相承的——不同的是，HydroOS四态机管理的是系统整体运行状态，而EDC状态机管理的是策略选择。两者的设计哲学一致：**在事先枚举所有可能的运行状态组合，并为每种组合预先设计最适合的响应策略，通过实时监测自动切换**。这种"策略库+状态机"的设计范式，正是雷晓辉等 [14-1] 所提出的水系统控制论技术框架中"安全包络"与"分层分布式控制"原理在梯级水电站场景的工程映射。

## 14.5 水力传播补偿与时序协调

梯级厂间负荷实时分配中，上游电站调整出力后水流到达下游电站有一定滞时，这使得站间水流滞时引起的能量核算成为实时调度中必须处理的问题（来源：原始研究报告第3.3节）。原始报告指出，实时调度实质上是一个与时间无关的空间优化问题，站间水流滞时引起的能量分配偏差，通过引入水位控制目标（以深溪沟、枕头坝水位变化最小为控制目标的"水位平稳模型"，以及以蓄能最大为目标的"蓄能最大模型"）来隐式补偿：当上游瀑布沟增出力而下游来水尚未到达时，深溪沟水位将短暂下降，EDC通过水位反馈感知这一变化，自动调减深溪沟出力以保持水位平稳，从而在物理层面实现了时序补偿。

**梯级水力时延参数**（基于互相关函数分析，来源：原始研究报告第3.3节）：

| **传播路段** | 枯水期时延（分钟）| 汛期时延（分钟）| 时延变化主因 |
|---------|----------------|--------------|------------|
| **瀑布沟→深溪沟** | ~110 | ~95-100 | 流量增大→波速加快 |
| **深溪沟→枕头坝** | ~80 | ~70-75 | 同上 |
| **枕头坝→沙坪** | ~85 | ~70-75 | 同上（见第十三章）|
| **瀑布沟→沙坪（全程）** | **~195** | **~165-175** | 三段叠加 |

注：上表时延为"最优调控时间"，即上游流量变化量传播到下游50%的时刻，用于指导调控介入时机。

全程时延接近3小时的物理事实，对EDC的决策架构有深刻影响：当瀑布沟在$t_0$时刻调整出力，直到$t_0$+165分钟后沙坪才完整感受到这一变化。这意味着在这2.75小时内，梯级任何一站的控制行为都可能在"不知情"的情况下影响其他站，使得梯级联合控制在时间尺度上具有天然的"信息不对称"特征。EDC的BDPSA和时序补偿策略，本质上是在这一不对称时间结构中为每个站的行动找到合理的时机安排。

### 14.5.1 水力传播时延的预补偿

当电网 AGC 下达新的梯级总出力指令时，EDC 系统需要决定"哪个电站先动，哪个电站后动"——这是梯级协调控制中特有的**时序问题**。

直觉上的"同时调整"策略实际上会引发问题：如果瀑布沟和深溪沟同时大幅增出力，瀑布沟出库流量立即增加，但该流量要 1-2 小时后才到深溪沟，在这 1-2 小时内深溪沟已经增加了出库（增加发电量），但来水还没到，导致水位快速下降。

正确的做法是实施**前馈时序补偿**，其本质是将MPC的预测与前馈机制 [14-6] 应用于梯级站间协调：
1. 瀑布沟先按指令调整出力（$t_0$ 时刻）
2. 预测 $\tau_1 = 70-110$ 分钟后深溪沟入库将增加 $\Delta Q_1$ $\text{m}^3$/s（视当时流量级取时延参数）
3. 深溪沟在 $t_0 + \tau_1$ 时刻再调整出力，分配增量 $\Delta Q_1$ 对应的出力
4. 枕头坝在深溪沟调整后再延迟约75分钟跟随

这种"**上游先动、下游延迟跟随**"的时序策略，本质上是将水力传播延迟显式纳入控制决策，避免了水位的预期性波动。其数学本质是将原本的"空间优化问题"扩展为"时间-空间联合优化问题"：最优决策不仅需要找到好的站间负荷分配，还需要找到正确的执行时序。

**时序误差的容忍度分析**：若时延估计误差为$\pm$15分钟（约时延的$\pm$15%），对深溪沟水位的影响约为$\pm$0.1-0.2 m，在深溪沟4 m调节范围的5%以内，属于可接受误差。这一分析说明，时序补偿对时延参数精度的要求比沙坪单站MPC（要求误差<$\pm$15分钟，误差占预见期的比例~$\pm$18%）更宽松，原因是深溪沟的绝对调节库容比沙坪大得多，对来流波动的缓冲能力更强。

### 14.5.2 来流不确定性的处理

区间降雨产生的区间入流（非来自上游电站）会引入不确定性。EDC 系统引入**鲁棒优化框架**，在目标函数中对区间入流的不确定区间 $[\underline{Q}_{in}, \bar{Q}_{in}]$ 建立最坏情况约束，确保即使区间入流在预测误差范围内波动，水位约束仍然满足。

在实际工程实现中，区间入流不确定性的处理策略与水位区间密切配合：当深溪沟或枕头坝水位处于绿区中间值附近时（有足够的调节裕度），EDC使用点估计（预测区间入流的均值）进行优化，以充分利用调节空间追求经济最优；当水位接近黄区边界时（调节裕度减小），自动切换到最坏情况约束（使用区间入流上界或下界），优先保障水位安全，牺牲部分经济性。这一"变精度"的不确定性处理策略，是EDC约束软化机制（14.6.2节）在不确定性维度的对应实现。

## 14.6 安全约束下的实时优化

### 14.6.1 EDC 投入与退出条件

EDC 系统在原始研究报告第5章中给出了完整的投入/退出条件真值表和47条报警触发规则（来源：原始研究报告第5.2节）。投入条件要求瀑布沟、深溪沟（及后续扩展至枕头坝）三站AGC功能均投入且处于集控控制，同时集控与各电站通信正常、系统频率无异常、无事故告警。任一条件不满足，EDC 即无法投入。退出触发条件包括22条精确量化规则，以下列举典型示例：30秒内全厂有功测值无变化（判断测量故障，立即退出）；系统频率异常且频率异常控制方式未投入（立即退出）；与任一电站AGC通信故障（立即退出）；任一电站上游水位发生跳变（单次变化超过1米，退出并报警）；电站水位过低（水位低于死水位，退出并报警）；电站水位过高（水位高于正常高水位，退出并报警）。此外，EDC还设计了47条其他报警规则，涵盖设定值越限、设定值与实发值偏差过大、频率反调节防止等场景，报警触发后系统自动拒绝执行相关设定指令，防止异常指令进入执行链路。

EDC 系统设计了严格的**安全门禁逻辑**，体现了 CHS 原理四（安全包络）的工程实现：

**EDC 投入条件**（所有条件同时满足）：
1. 梯级总出力指令处于正常范围（不超过 4620 MW 的 95%）
2. 深溪沟水位在正常区间（绿区）
3. 枕头坝水位在正常区间（绿区）
4. 三站通信链路全部正常
5. 瀑布沟 AGC 处于投入状态

**EDC 退出触发条件**（任一条件满足即退出）：
- 任一电站水位进入黄区且恶化趋势未消除
- 电网负荷指令超出梯级安全出力范围
- 任一站通信中断超过 2 分钟
- 任一机组出现故障保护动作

**EDC 退出后的处置**：各站立即切换到本站独立 AGC 模式（深溪沟切换到"带固定负荷"模式），集控中心运行人员接收报警并人工介入。

### 14.6.2 安全约束的软化与优先级

当某约束处于临界状态时，EDC 系统采用**约束软化（Soft Constraint）**机制：将水位越限惩罚从硬约束变为加权惩罚项，使优化器可以在轻微违约（如水位短时略超黄区 0.1 m）与大幅调整出力（可能干扰电网）之间找到平衡。

约束优先级排序（由高到低）：
1. 人身和设备安全（机组保护、坝体安全） → 硬约束
2. 电网调度纪律（出力偏差在考核范围内） → 硬约束
3. 水位安全（不触发红区） → 硬约束
4. 水位优化（维持在绿区经济水位） → 软约束（惩罚项）
5. 经济调度（耗水率最小） → 目标函数

## 14.7 梯级协调与沙坪的依存关系

### 14.7.1 枕头坝出库→沙坪入库的水力传播

枕头坝一级水电站是瀑深枕梯级的最末一级，但大渡河梯级开发并未止步于此——枕头坝的下游约18公里处，就是第十三章详细论述的沙坪二级水电站 [14-22]。枕头坝出库流量经约70-85分钟（枯水期约85分钟，汛期约70-75分钟）的河道传播后到达沙坪水库。这一水力传播时滞意味着：枕头坝在 $t_0$ 时刻的任何出力调整，都将在 $t_0 + 70\sim85$ 分钟后对沙坪的入库流量产生实质性影响。

沙坪的调节库容极小（汛期高水位时仅约38-76万 $\text{m}^3$），对上游来流变化极为敏感——第十三章已详细分析了"2000 $\text{m}^3$/s的流量差仅需约14分钟即可使沙坪水位越限"的物理现实。因此，枕头坝出力的骤变对沙坪的影响远比对枕头坝自身水库的影响更为严峻：枕头坝自身有5米的水位调节幅度来缓冲波动，而沙坪在汛期可能只有不到1米的调节裕度。

### 14.7.2 梯级 EDC 如何向沙坪发送前馈信号

在理想的信息共享架构下，梯级EDC应当在每次调整枕头坝出力时，**同步向沙坪MPC系统发送前馈信号**——告知沙坪"未来80分钟内你的入库流量将增加（或减少）多少"。这一前馈信息对沙坪MPC至关重要：第十三章中沙坪MPC的预见期设计为80分钟，恰好覆盖枕头坝到沙坪的水力传播时滞，如果能在枕头坝出力调整的瞬间就获知这一信息，沙坪MPC可以提前80分钟开始预调闸门和机组出力，而非等到入库流量实际变化后才被动响应。

然而，当前工程实践中，这一信息传递依赖人工通报——枕头坝运行人员在EDC调整出力后，通过电话通知沙坪运行人员 [14-2]。人工通报的问题在于：（1）时间延迟——从EDC下达指令到电话通报再到沙坪人工输入MPC，通常耗时5-10分钟，浪费了宝贵的预见期；（2）信息精度——口头通报难以传达精确的流量变化量和变化时刻；（3）遗漏风险——夜间或高负荷工况下人工通报容易遗漏。

要实现系统级自动对接，需要在梯级EDC和沙坪MPC之间建立数据通道，使EDC在每次修改枕头坝出力指令时自动将以下信息推送至沙坪：预计流量变化量 $\Delta Q$（$\text{m}^3$/s）、预计到达时刻 $t_0 + \tau$、以及预计持续时间。这在技术上并不困难（仅需一条标准通信链路），但涉及管理体制问题——梯级EDC归大渡河集控中心管理，沙坪归另一家运营主体（国投水电），两者之间的数据共享协议和指令权限边界尚不清晰。

### 14.7.3 沙坪作为末级电站的特殊角色

从整个大渡河梯级的视角看，沙坪扮演着"**梯级尾水终端缓冲器**"的特殊角色。瀑布沟的每一次出力调整，经过深溪沟→枕头坝的依次传递和各站自身的调节作用后，最终的残余波动由沙坪承接。一个运行良好的梯级EDC系统，应当在分配枕头坝出力时就考虑到"这个出力变化对沙坪意味着什么"——也就是说，**沙坪的水位安全约束应当成为梯级EDC优化问题的边界条件之一**。

当前EDC的优化目标仅包含瀑深枕三站的约束，沙坪被视为"外部系统"。这一设计的局限在汛期高来流工况下尤为突出：枕头坝按EDC指令大幅增出力时，其出库流量骤增，约80分钟后冲击沙坪水库，若沙坪此时也处于高水位运行，可能触发沙坪的安全预警。如果EDC能够实时感知沙坪水位并在枕头坝出力分配中加入"沙坪水位安全"约束，就可以适当平滑枕头坝的出力变化斜率（给沙坪更多响应时间），在梯级经济性和末级安全性之间取得更好的平衡。

### 14.7.4 从"各自为政"到"信息共享"的转变

深溪沟AGC在2012年的停用，其根源是"各自为政"——三站各有独立的控制器，没有信息共享和协调机制。2018年EDC的投运解决了瀑深枕三站之间的协调问题，但梯级与沙坪之间又出现了新一层的"各自为政"。这一现象反映了水利自主控制升级中的一个规律：**每解决一层协调问题，就会暴露出上一层的协调缺失**。

从CHS理论视角看，完整的大渡河水力链应当涵盖"瀑布沟→深溪沟→枕头坝→沙坪"四站，对应四级串联的链状系统。当前的工程状态是：前三站已建立统一协调层（EDC），第四站（沙坪）仍以独立控制为主，仅通过人工通报实现松散耦合。未来的演进方向是将沙坪纳入梯级EDC统一框架，形成"瀑深枕沙四站联合EDC"。这一扩展在技术上需要解决：（1）沙坪灯泡贯流式机组与瀑深枕混流式机组特性差异的统一建模；（2）四站链条总传播时滞从约175分钟延长至约250分钟后的时序补偿策略调整；（3）跨管理主体的数据共享和指令权限协议。

这一从"各自为政"到"信息共享"再到"统一协调"的渐进路径，正是CHS原理三（分层分布式控制）在工程实践中逐步落地的写照。

## 14.8 实施过程与运行效果

### 14.8.1 分阶段实施路径

**阶段一（2016-2017）：模型开发与 MIL 验证**

本阶段的核心任务是从历史运行数据建立三站水力电力耦合模型，并进行数值仿真验证。这一MIL→SIL→HIL的分阶段验证路径，与雷晓辉等 [14-5] 提出的自主运行智能水网在环测试体系的方法论框架一致。研究团队系统整理了三站的机组效率特性曲线实测数据，采用"质心法+最小二乘拟合"对每台机组独立率定N-H-Q特性曲面（瀑布沟12台、深溪沟4台、枕头坝3台，合计19台机组），并在不同典型水头和总负荷指令组合下构建完整的"经济运行总表"。MIL阶段在数字仿真环境中验证了300+个典型调度场景，覆盖：汛期高来流负荷调增/调减、深溪沟高水位应急退出EDC、枕头坝低水位弃水控制、三站机组组合切换（部分机组检修）、广域通信延迟（100-300ms）对指令响应的影响。仿真阶段发现并修复了3处关键错误：枕头坝机组处于振动区边界时的分配逻辑异常、深枕负荷平衡模式下手动调整枕头坝负荷时瀑布沟反向调整方向错误、小负荷与正常分配模式切换时的竞争条件。

**阶段二（2017-2018）：集成验证与 HIL 测试**

EDC算法部署至大渡河集控中心实际服务器，与三站厂控SCADA系统建立真实通信链路，开展SIL和HIL验证。SIL阶段重点验证EDC与各站AGC的通信协议对接 [14-9]，确认广域网延迟最大300ms条件下控制指令下达和执行回报不影响控制精度。HIL阶段将瀑布沟和深溪沟实际PLC系统接入测试回路，模拟了包括"深溪沟AGC突然故障退出→EDC自动检测→梯级切换到瀑布沟+枕头坝两站分配"的完整故障恢复序列，验证了22条退出触发规则的正确性。枕头坝因地理距离远采用数字孪生代替实物接入。

**阶段三（2018）：渐进式现场投运**

投运遵循"辅助决策→自主控制"两步策略。辅助决策阶段历时3个月（2018年7-9月汛期），EDC每5分钟生成一次分配建议，在调控一体化平台以"待确认指令"形式呈现，调度员确认后执行。这一阶段帮助调度员建立对EDC的信任，同时发现并修复了两处人机交互问题（指令过期后的处理逻辑不合理）。2018年10月起正式进入自主控制模式，调度员仅监督不确认。至2018年底，深溪沟AGC在EDC框架下重新投运，梯级EDC全面正式投运。

| **深溪沟 AGC 投入率** | 0%（停用）| >95% | 实现重新投运 |

*注：表中定性结论可靠，量化数值为基于文献的工程评估，精确运营统计数据以正式运营报告为准。*

### 14.8.3 典型调度工况示例：汛期大幅调峰响应

为使读者直观理解EDC在实际调度中的工作方式，以下给出一个汛期典型工况的完整决策推演，对应原始研究报告第3.3节中的典型案例逻辑。

**工况背景**（2019年汛期某一日，7月）：四川电网在下午用电高峰（15:00-18:00）前后，要求瀑深枕梯级总出力在15:00时从2500 MW增加至3200 MW，即增量700 MW，持续3小时后于18:00减回2500 MW。当日瀑布沟水库水位在845 m（正常蓄水位850 m，绿区），深溪沟水位640 m（绿区），枕头坝水位620.5 m（绿区，但接近高水位区阈值）。

**EDC分析过程**（15:00时刻）：
1. 电网指令下达梯级总出力增至3200 MW（增量700 MW）。
2. EDC系统当前工况：深枕水位均处绿区，但枕头坝水位较高，选用"最大蓄能模式"（优先保存瀑布沟库容）。
3. 查"经济运行总表"，当前瀑布沟水头约53 m，深溪沟水头约3 m，枕头坝水头约5 m，机组效率数据实时读取。
4. 分配方案：增量700 MW主要分配给深溪沟（+250 MW）和枕头坝（+200 MW），瀑布沟增量最小（+250 MW）——此分配将增大深溪沟和枕头坝的发电流量，即加快消耗已在途的来水，避免水位进一步升高。
5. 时序补偿：瀑布沟立即执行增量指令（$t_0$ = 15:00）；预测瀑布沟增出力导致出库流量增加约500 $\text{m}^3$/s，按汛期瀑-深水力时延95分钟推算，新的来流将在16:35到达深溪沟。因此深溪沟在16:35追加出力增量（而非15:00立即全部增加），避免来水尚未到达时水位提前被拉低。

**15:00-16:35期间（深溪沟来水尚未到达前）**：深溪沟按初始分配方案增出力+250 MW，但来水尚未增加，水位开始短暂下降（约0.2-0.3 m）；枕头坝因自身来水增加（深-枕时延约75分钟，深溪沟15:00增出力的水流约在16:15到达枕头坝），在16:15后水位趋于平稳。EDC的OSEM实时监测水位变化趋势，若深溪沟水位下降超过黄区阈值，自动触发"水位异常控制模式"临时减少深溪沟出力。

**16:35-18:00期间（梯级全线流量平衡建立后）**：瀑布沟增出力的流量波传递到深溪沟，三站入出库流量重新趋于平衡，水位稳定在绿区。梯级总出力维持3200 MW直至18:00。

**18:00撤回指令**：对称操作，瀑布沟先减、深枕延迟跟随，水位扰动约0.1-0.2 m，远低于各站0.5 m黄区阈值。

此工况充分体现了EDC相对于独立AGC的本质优势：独立AGC下，深溪沟15:00立即增出力+增负荷，来水尚未到达，水位快速下降触发越限报警，AGC被迫退出，调度员手动干预；EDC框架下，精确的时序补偿使三站协调行动，水位扰动完全可控，全程无需人工介入。

## 14.9 瀑深枕梯级 WNAL 等级评估

### 14.9.1 改造前等级：L1~L2（各站独立AGC阶段）

在2018年EDC投运之前，瀑深枕梯级的自主等级可评估为**WNAL L1~L2**（远程监控~辅助决策）。

**改造前五维度评分**：

| 维度 | 改造前状态描述 | 评分 |
|-----|------------|-----|
| **感知层** | 各站SCADA独立采集，无站间实时信息共享；瀑布沟入库流量有预报，但深枕入库完全依赖上游出库推算 | L1.5 |
| **决策层** | 各站AGC独立运行，无站间协调优化；深溪沟AGC因水位越限被迫停用 | L1（深溪沟实际退化为人工） |
| **执行层** | 瀑布沟AGC闭环运行，深溪沟人工操作，枕头坝AGC部分投运 | L1.5 |
| **监督层** | 各站独立报警，无梯级级别的系统性安全逻辑；水位越限全靠人工监视 | L1 |
| **验证层** | 无系统性在环验证，各站AGC仅完成出厂测试 | L1 |
| **综合评估** | **五维度均值1.2分** | **L1~L2** |

改造前的核心问题不在于各站装备落后（瀑布沟AGC本身是先进的），而在于**系统架构层面缺少全局协调能力**。各站分别处于L1~L2水平，但由于缺少协调层，梯级整体的自主能力甚至低于各站能力的简单叠加——深溪沟AGC因全局协调缺失而被迫退回人工模式，就是"系统架构缺陷导致个体能力退化"的典型例证。

### 14.9.2 改造后等级：稳定 L3（正式投运后）

瀑深枕梯级EDC系统于2018年底正式投运，运行数据表明三站联合调度已稳定达到 WNAL L3（条件自主）水平 [14-2]，且在常规输电工况下具备向L4过渡的部分能力。

**改造后五维度评分结果**：

| 维度 | 改造后状态描述 | 评分 |
|-----|------------|-----|
| **感知层** | 三站SCADA实时采集（5分钟），互相关函数确认水力时延，OSEM估计来流不确定性；区间入流感知能力L2 | L2.5 |
| **决策层** | BDPSA经济运行总表毫秒级查表；五种主动+三种应急策略全覆盖；EDC投入率>95%；ODD内自主决策不需人工确认 | L3 |
| **执行层** | 三站AGC均投运，EDC指令直接下达执行，形成完整闭环 | L3 |
| **监督层** | 22条退出规则+47条报警规则，异常自动退出；调度员监督状态而非监督具体决策 | L3 |
| **验证层** | MIL（300+场景）/SIL/HIL三层完整；汛末消落期和冬季高峰期专项ODD覆盖不足 | L2.5 |
| **综合评估** | **五维度均值2.8分** | **稳定L3** |

**从L1到L3的跃迁分析**：改造前后的WNAL等级提升了近两个完整级别，这在水利行业中是一个显著的跃迁。值得注意的是，这一跃迁并非通过"升级各站AGC硬件"实现（各站的底层控制器和SCADA系统基本未变），而是通过"增加全局协调层（EDC）"这一纯架构性改造实现的。这一实践经验有力地支撑了雷晓辉等 [14-3] 提出的基于无人驾驶理念的自主运行智慧水网架构中关于"架构先行"的核心主张。这再次印证了CHS理论的一个基本判断：**水利系统自主等级的提升，架构改进往往比算法升级更具杠杆效应**。

**五种调度策略模式与WNAL等级的对应关系**值得单独说明。EDC的五种主动策略模式并非简单的功能选择，而是隐含了不同的自主化程度：水位平稳模式和最大蓄能模式是高度自主的纯算法决策（ODD内L3行为）；少调负荷模式体现了对特定运行约束（深溪沟AGC投入率低）的自适应，具备初步的L4能力（感知自身系统能力局限并调整策略）；深枕负荷平衡模式和枕站负荷平衡模式则是人机协同模式（L3的P7原理体现），允许人工干预同时维持EDC框架完整性。这种"策略库按需选用"的设计，使EDC在不同工况下自动选择与之匹配的自主化层次，是梯级案例超越单站案例的一个重要设计维度。

### 14.9.3 制约因素分析：ODD边界未完全验证

虽然EDC已稳定达到L3，但向L4（高度自主）跃迁仍受多项因素制约，核心瓶颈是**ODD边界未完全验证**。

**制约因素一：极端工况ODD覆盖不足**。当前ODD的薄弱环节是汛末消落期（瀑布沟大幅降水位期间三站耦合关系发生显著变化，瀑布沟从"蓄水调节器"变为"泄水先行者"，水位-出力-流量三者之间的平衡关系完全不同于常规运行）和冬季电热高峰期（三站均需维持高出力导致调节裕度极小，EDC的约束软化机制面临更严苛的考验）。这两类工况在MIL阶段的场景覆盖不足，尚未进行专项HIL验证。

**制约因素二：跨梯级感知能力不足**。当前EDC以三站间水力电力模型为基础，对上游大渡河流域径流的中长期预报依赖外部气象水文系统，尚未实现"EDC感知流域降雨→主动调整蓄能策略"的闭环。升级至L4需要将流域水文预报直接纳入EDC决策层。

**制约因素三：梯级-末级联合未实现**。沙坪水电站作为枕头坝的下游反调节电站，当前沙坪MPC与梯级EDC是两套独立系统，两者之间仅有人工协调。将沙坪纳入梯级EDC统一框架，将显著提升全链条的协调自主能力。

**制约因素四：参数更新依赖人工**。经济运行总表每年汛前汛后各更新一次，更新频率取决于机组检修和参数漂移的速度，而非系统自适应能力。这意味着EDC停留在"静态ODD + 静态参数"的L3阶段，要达到L4需要引入在线参数学习和动态ODD边界更新。

### 14.9.4 跃迁路径：需补充的 xIL 测试项

基于上述制约因素分析，从L3向L4跃迁需要完成以下专项在环测试：

| **测试项** | 测试类型 | 验证目标 | 预计工作量 |
|-------|---------|---------|-----------|
| **汛末消落期联合调度** | MIL+HIL | 瀑布沟大幅降水位期间EDC策略有效性 | 6个月 |
| **冬季高峰全站满发** | MIL+HIL | 调节裕度极小条件下约束软化可靠性 | 3个月 |
| **沙坪联合前馈测试** | SIL | EDC→沙坪MPC数据链路和前馈效果 | 3个月 |
| **流域洪水预报接入** | MIL | 水文预报融入EDC决策的精度和时效 | 6个月 |
| **机组特性在线学习** | MIL+SIL | N-H-Q曲面自动校正的收敛性和安全性 | 6个月 |
| **四站联合EDC原型** | MIL | 瀑深枕沙四站统一优化的可行性 | 12个月 |

完成上述测试后，EDC可望从当前的"ODD内L3、ODD外人工"状态跃迁至"扩展ODD内L4、极端工况L3"的混合状态，进一步向全天候L4迈进。

## 14.10 与 CHS 理论框架的映射分析

CHS理论的学术根源可追溯至Wiener [14-26] 的控制论和钱学森 [14-25] 的工程控制论，两者奠定了反馈控制和系统工程的基本范式。梯级EDC案例为这一理论传承提供了鲜活的工程注脚。

### 14.10.1 CHS 八原理在梯级 EDC 中的体现

| CHS 原理 | 在梯级 EDC 的具体体现 | 若缺失的后果（2012年历史） |
|---------|---------------------|--------------------------|
| **P1 传递函数化** | 三站水力-电力耦合的统一传递函数模型；N-H-Q特性曲面为执行器传递函数 | 无法预测上游动作对下游水位的影响 |
| **P2 可控可观性** | 19台机组的出力可调区间分析；水位传感器布局确保可观性 | 振动区分配异常、水位盲区 |
| **P3 分层分布式** | 集控层 EDC + 电站层独立 AGC 的两级架构 | 三站各自为政，水位失控（深溪沟停用） |
| **P4 安全包络** | EDC 投入/退出逻辑 + 约束优先级 + 22条退出规则 + 47条报警规则 | 水位越限无系统性保护 |
| **P5 在环验证** | MIL→SIL→HIL 三阶段验证 300+ 场景 | 潜在时序冲突在现场暴露，修复代价高 |
| **P6 认知增强** | 经济运行总表将专家经验固化为查表逻辑（初步认知固化） | 每次优化重新计算，效率低 |
| **P7 人机共融** | 先辅助决策后自主控制的渐进式上线；M4/M5模式保留人工干预通道 | 调度员不信任系统，拒绝自动控制 |
| **P8 自主演进** | 年度特性曲线更新、策略库扩展（从最初3种模式到8种模式） | 参数漂移导致优化效果逐年衰减 |

### 14.10.2 梯级案例对 CHS 理论的新贡献

相比第十三章沙坪的单站案例，梯级案例揭示了 CHS 理论的**三个新维度**：

**维度一：多智能体协调的必要性**。深溪沟 2012 年 AGC 停用，是因为没有"全局协调层"——这是 CHS 原理三（P3 分层分布式）在实践中"缺失导致失败"的典型案例。EDC 系统的成功，直接验证了 P3 四层架构中"全局协调层"对于多站联合控制的不可或缺性。更深刻的是，这一失败不是因为任何一站的控制器出了问题，而是因为**系统架构层面缺少整合机制**。这对水利行业的启示是：面向多站联合运行的自动化改造，应当首先设计整体架构，再在架构框架内升级各站控制器，而非相反。

**维度二：时间维度的跨站协调**。单站控制的时间约束是"本站的入库流量在T分钟后到达安全极限"，而梯级控制的时间约束是"上游电站的决策在τ小时后才会影响下游电站"，这正是雷晓辉等 [14-21] 指出的从静态平衡到动态控制转变中时间维度复杂性急剧增加的体现。这要求控制器具有跨站、跨时间步长的预见能力，这是P1（传递函数化，包括空间传播的传递函数）核心价值的梯级体现。沙坪的P1是单段（枕头坝→沙坪），梯级的P1是三段串联（瀑布沟→深溪沟→枕头坝），相应地，梯级控制的预见期要求也更长（240分钟 vs 80分钟），对传递函数精度的要求更高。

**维度三：策略库 vs 单一策略的设计哲学**。沙坪的控制策略是单一的MPC优化框架，而梯级EDC设计了"五种主动模式+三种应急模式"的策略库。这一差异折射了系统复杂度增加后控制设计的变化：系统越复杂，运行工况越多样，单一策略的通用性越差，需要针对不同工况预先设计针对性策略，并配备状态机来管理策略切换。这一设计哲学与HydroOS中"四态机"（正常/降级/紧急/维护态）的运行时治理机制是一脉相承的（见第十一章），梯级EDC可视为HydroOS治理机制在水电站群场景的工程先行实践。

**梯级案例揭示的CHS理论局限**：梯级EDC在P3（分层分布式）上取得了重大突破，但在P8（全生命周期自主演进）方面仍依赖人工年度更新。经济运行总表每年汛前汛后各更新一次，更新频率取决于机组检修和参数漂移的速度，而非系统自适应能力。这意味着沙坪和梯级案例都停留在"静态ODD + 静态参数"的L3阶段，要达到L4（高度自主）需要引入在线参数学习和动态ODD边界更新，即P8原理的完整落地——这正是第十一章HydroOS和第十五章物理AI/认知AI所要系统性解决的问题。

## 14.11 实施反思与推广建议

### 14.11.1 深溪沟 AGC 重新投运的关键

2012 年的停用和 2018 年的重新投运，核心差别只有一个：**是否有全局协调层**。深溪沟 AGC 本身的算法并未根本改变，改变的是它工作的上下文——从"孤立运行"到"在 EDC 协调框架下运行"。

这个教训的推广价值：**在梯级或多站联合控制场景下，不要试图"升级单站控制器"来解决系统性问题——系统性问题需要系统性解决方案**。

### 14.11.2 推广建议

Oliveira和Loucks [14-13] 早已指出多水库系统的操作规则比单水库复杂得多，本案例模式适合推广的场景：
- 具有反调节关系的梯级电站群（上级电站调节能力强、下级容量小）
- 三站以上的串联梯级，且都接入同一电网调度系统
- 已具备集控层数据集成能力（各站实时数据汇聚到集控中心）

纪昌明等 [14-23] 的综述指出，梯级水电站联合优化调度问题的核心挑战在于维数灾和多约束耦合，而本案例的BDPSA+总表策略提供了一种工程实用的解决路径。推广时应特别注意以下两个容易被忽视的条件：第一，**机组N-H-Q特性曲面的准确性是EDC经济效益的前提**。若直接沿用出厂模型曲线而不进行真机试验校核，EDC的"经济最优"可能在实际中变为"经济次优"甚至产生分配失衡。推广时应将"现场真机试验+曲线精细化率定"列为必要前置工作。第二，**梯级一体化运营体制是EDC落地的制度前提**。EDC将分属不同管理主体的电站纳入统一决策框架，需要明确集控层的指令权限边界、各站的本地保护优先级以及事故责任划分。纯技术推广而忽视体制匹配，往往是梯级协调控制项目失败的深层原因。

### 14.11.3 EDC 的计算效率与未来演进

**计算效率的工程意义**：BDPSA算法通过"经济运行总表"将在线优化转化为离线预计算+在线查表，将原本需要若干秒的混合整数非线性规划求解压缩到毫秒级，解决了蒋传文等 [14-19] 指出的大规模梯级水电站短期优化调度中计算实时性与全局最优性之间的矛盾。这一设计的工程价值不仅在于满足AGC 5分钟采集步长的实时性要求，更在于为系统预留了应对指令加密（分钟级响应）甚至秒级响应的扩展空间。目前AGC指令频率为每5分钟一次，随着电网调频要求提高（如新能源大规模并网后调峰需求上升），指令可能加密至每分钟甚至每30秒一次，届时查表法的毫秒级响应将成为关键优势。

**计算复杂度的量化比较**：以3站、19台机组的瀑深枕梯级为例，若不预计算总表而每次在线求解，BDPSA的单次求解时间约为1-3秒，接近但仍略超AGC实时性要求；若扩展至6站、30台机组的更大梯级（如Mo等 [14-24] 研究的三峡-葛洲坝梯级），在线求解时间将超过10秒，完全不满足实时要求。有了"经济运行总表"，6站梯级只需将总表规模扩大，查表响应仍为毫秒级。这说明，总表预计算策略的计算效率优势随系统规模增大而愈发显著，是大规模梯级自动化调度的关键使能技术。

**未来深度学习替代可能性**：学术界已有研究探索用深度强化学习（DRL）替代传统优化算法进行梯级负荷分配，Feng等 [14-16] 采用并行多目标遗传算法平衡发电效益与生态需求的工作代表了这一方向的早期探索。DRL的潜在优势在于可以处理非结构化工况（如机组组合频繁变化、特大洪水极端工况），并隐式学习历史优化经验。然而，当前DRL在安全关键系统中面临的主要挑战是"可解释性不足"——调度员无法理解每次分配决策的原因，给事故追溯和合规性证明带来困难。CHS原理七（人机共融，P7）明确要求决策过程具备可解释性；因此，在可解释性问题根本解决之前，BDPSA等基于精确数学模型的方法仍将是工程实践的主流选择，DRL更适合作为策略发现和边界扩展的研究工具而非生产系统核心。

## 14.12 沙坪（点）与梯级（链）：两章案例的结构性对比

第十三章和本章共同构成"工程实践篇"的点-链两级，在阅读完两章后，有必要从整体视角梳理两案例的结构性异同，为第十五章（网状系统）做铺垫，也帮助读者将CHS理论框架与工程实践建立更清晰的映射关系。

**表14-2　沙坪（点）与瀑深枕梯级（链）的结构性对比**

| 维度 | 第十三章（沙坪，点） | 第十四章（瀑深枕，链） |
|-----|-------------------|---------------------|
| **系统边界** | 单站：1座坝+6台机组 | 三站串联：4620 MW、60 km河段 |
| **核心物理约束** | 时间约束（调节库容极小→响应必须快）| 空间耦合（水力传播时延→协调必须跨站）|
| **核心控制挑战** | 预见期内来流预测精度 | 多站异步时序协调 |
| 核心CHS原理 | P1（传递函数）→P4（安全包络）→P5（在环验证）| P3（分层分布式）→P1（跨站传递函数）→P4（系统级安全包络）|
| **核心算法** | MPC滚动优化 + 前馈负荷申报 | BDPSA经济运行总表 + 时序补偿 |
| **人机界面** | 调度员保留负荷申报最终审批权 | 调度员监督EDC运行，无需逐条确认 |
| **WNAL等级** | L2→L3过渡（研究阶段），L3（工程上线后）| L1~L2→L3跃迁（EDC投运后） |
| **失败教训** | 缺P1（模型精度不足）→P4预警失效 | 缺P3（无全局协调层）→深溪沟AGC停用 |
| **主要成果** | 闸门动作减少82%，水位越限消除 | 深溪沟AGC重新投运，水位越限>92%减少 |

两案例的最深刻差异不在于技术手段，而在于"失败的根本原因不同"：沙坪的困境是**物理时间约束**（库容太小、响应必须快），梯级的困境是**系统架构缺陷**（各自为政、缺少协调层）。这一对比印证了CHS理论的一个基本判断：水利系统的控制问题是异质的——不同类型的系统，主导性约束不同，需要不同的理论工具和工程路径。

在走向第十五章（胶东调水工程，网状系统）之前，已经建立的沙坪单站精控和梯级协调控制还面临一个共同的局限：两个系统都运行在相对稳定的边界条件下（来流范围有限，电力负荷指令较为规律）。当系统扩展到覆盖600公里的调水线路、涵盖13级泵站和95个闸站、同时受地形、节令、多目标供水需求约束时，问题的维度将发生质变，需要引入HydroOS三层架构和物理AI/认知AI双引擎才能应对。

## 本章小结

大渡河瀑深枕梯级 EDC 案例的核心教训：**链状系统的控制问题不能用多个点控制的叠加来解决**——局部最优不等于全局最优，缺少全局协调层的梯级各自为政必然导致水位失控和弃水。

EDC 系统通过"集控层统一优化 + 电站层独立执行"的分层分布式架构，将三个孤立的局部控制器整合为一个协调的整体。深溪沟 AGC 从 2012 年停用到 2018 年重新投运，技术本身没变，变的是系统架构——这是 CHS 原理三（分层分布式控制）最有力的工程佐证。

BDPSA算法与经济运行总表的组合，解决了"实时性"与"最优性"之间的矛盾：离线预计算保证了全局最优性，在线查表保证了毫秒级响应。八种控制模式构成的策略库，使EDC在复杂多变的工况下始终有适合的策略可选——这一"策略库+状态机"的设计范式，为更大规模水利系统的自主控制提供了可复制的工程模板。

梯级EDC将瀑深枕三站的WNAL等级从L1~L2跃迁至稳定L3，证明了**架构改进比算法升级更具杠杆效应**。但向L4的跃迁仍面临ODD边界未完全验证、跨梯级感知不足、参数更新依赖人工等制约，需要通过系统性的xIL测试和沙坪联合扩展来逐步突破。

第十五章将进一步扩展这一逻辑：当控制对象从"三站梯级"扩展到"600 公里长、涵盖 13 级泵站和 95 个闸站的水网"时，分层分布式控制将遭遇新的维数灾难，需要引入物理 AI 双引擎和认知 AI 协同来应对。

## 本章术语表

| **术语** | 英文 | 定义 |
|------|------|------|
| **EDC** | Economic Dispatch Control | 经济负荷分配控制，在梯级水电站间实时分配出力以最小化耗水率 |
| **AGC** | Automatic Generation Control | 自动发电控制，电站级的出力自动调节系统 |
| BDPSA | Bidirectional Dynamic Programming Successive Approximation | 改进双向逐次逼近算法，利用最优化原理正逆向同时求解以提升计算效率 |
| **反调节电站** | Re-regulating Station | 位于大容量调节电站下游、用于平滑其出库流量波动的小容量电站 |
| **纳什均衡陷阱** | Nash Equilibrium Trap | 各子系统局部最优导致全局次优的博弈困境 |
| **时序补偿** | Temporal Compensation | 在分布式控制中，将上游决策对下游的延迟影响显式纳入控制决策 |
| **软约束** | Soft Constraint | 允许轻微违反（以惩罚项代替）的约束，用于在多约束间取得平衡 |
| **经济运行总表** | Economic Operation Lookup Table | 将不同水头和负荷组合的最优分配预先离线计算，实时运行时查表获取分配方案 |
| **降阶模型** | Reduced-Order Model | 将复杂水动力方程简化为低阶传递函数，在保持精度的前提下降低计算复杂度 |
| **安全包络** | Safety Envelope | 系统运行的安全包络集合，超出包络自动触发保护动作 |
| **分层分布式控制** | Hierarchical Distributed Control (HDC) | 全局协调层与本地优化层的两级或多级控制架构 |
| **在环测试** | In-the-Loop Testing (xIL) | 包括MIL/SIL/HIL/PIL的系统性验证流程，逐步从仿真走向实物 |
