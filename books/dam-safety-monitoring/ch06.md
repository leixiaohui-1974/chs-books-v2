# 第6章 案例：混凝土坝与土石坝

## 本章导读

水工建筑物的安全运行直接关系到上下游人民生命财产安全与国民经济的稳定发展。不同坝型在结构特征、受力机制及破坏机理上存在显著差异，导致其安全监测的侧重点与预警逻辑各有不同。混凝土重力坝及拱坝依靠材料的刚度与坝体自重或拱端反力维持稳定，其主要失效模式包括整体滑动、坝体开裂以及基础失稳，因此监测的核心物理量通常为结构变形、应力应变与温度场。土石坝作为散粒体结构，依靠土石料的压实密度与分区防渗体系挡水，其典型破坏形式为渗透破坏（如管涌、流土）、边坡滑动及过大不均匀沉降，故而渗流场演变与孔隙水压力分布构成了其安全监控的重中之重。

本章系统梳理前期章节讲述的传感器技术、数据处理算法与系统架构理论，将其投射至实际工程环境。选取三峡大坝（混凝土重力坝）与小浪底水利枢纽（土石坝）作为典型工程案例，深度剖析不同坝型在安全监测体系构建、物理过程数学描述及智能预警方面的共性与差异。通过对这两类代表性工程的深入研究，旨在建立涵盖“感、传、知、用”全链条的工程实践认知，使读者能够将抽象的数学工具与具体的物理过程有机结合，掌握大型水利枢纽工程安全态势评估的系统方法论。

## 6.1 基本概念与理论框架

### 6.1.1 混凝土坝监测基本概念

混凝土大坝由于自身体积庞大且处于复杂的库水、环境温度及地质构造耦合作用下，其安全状态的外在表现首先集中于坝体的宏观位移。变形监测不仅能够直观反映坝体及坝基的整体性态，更是反演结构内部荷载传递规律的直接依据。以三峡大坝为例，该工程采用“垂线系统+引张线系统+静力水准仪”构建了空间三维高精度变形监测网。其中，正垂线与倒垂线联合组网，实现了坝体各高程水平位移相对于深部基岩基准点的绝对测量；多点位移计与基岩变形计则用于探明基础内部的岩体蠕变与断层错动。

理论框架层面，混凝土坝的变形行为被视为环境激励（水位、温度）与材料时变特性（徐变、老化）共同作用的系统响应。根据弹性力学与材料力学基本原理，坝体某测点的实测位移可分解为水压分量、温度分量与时效分量。这一分解原则构成了大坝变形统计模型的基础，将复杂的物理问题转化为多输入单输出的系统辨识问题。

### 6.1.2 土石坝监测基本概念

相对于刚性的混凝土结构，土石坝因其材料的多孔介质属性，呈现出明显的非线性、弹塑性与水力耦合特征。防渗体系（如心墙、面板）的完整性决定了土石坝的寿命。小浪底水利枢纽作为典型的黏土斜心墙堆石坝，其安全监控体系围绕渗流状态评估展开。渗流监测网络主要包括：埋设于防渗体及下游坝壳的渗压计（用于捕获孔隙水压力时空分布）、布置于不同高程的测压管（用于描绘浸润线动态变化）以及量水堰（用于汇测坝体及坝基的综合渗漏量）。

渗流理论框架建立在多孔介质流体力学之上。达西定律（Darcy's Law）与太沙基（Terzaghi）有效应力原理是描述土石坝内部固液两相相互作用的核心。当库水位发生骤降时，坝壳内的孔隙水若无法及时排出，将导致坝坡内部残留较高的超静孔隙水压力，降低土体的抗剪强度，进而引发滑坡风险。因此，监测孔水压力的消散速率并结合渗流量进行联合诊断，是土石坝安全评价的基础。

### 6.1.3 智能预警系统架构

针对上述两类大坝的监测需求，现代大坝安全智能预警系统普遍采用分层分布式架构，具体涵盖以下五个核心层级：
1. **感知层**：部署光纤光栅、振弦式传感器及自动化量测机器人，完成物理信号向数字信号的高保真转换。
2. **传输层**：融合工业以太网、LoRa与5G通信技术，构建低延迟、抗干扰的冗余数据链路。
3. **数据层**：依托时序数据库（如InfluxDB）与关系型数据库，实现海量异构监测数据（测值、巡视图像、运行日志）的结构化存储与清洗。
4. **分析层**：集成机理模型（有限元）与数据驱动模型（深度学习），实现在线状态估计、参数反演与异常识别。
5. **应用层**：提供三维可视化数字孪生界面，输出分级报警信息（如预警、报警、行动等级），并生成辅助决策报告。

## 6.2 数学建模与求解方法

### 6.2.1 混凝土坝变形统计模型推导

在建立混凝土坝变形数学模型时，通常采用静水压力-季节温度-时间（Hydrostatic-Season-Time, HST）统计模型。设坝体某测点的水平位移观测值为 $\delta(t)$，其数学表达式推导如下：

首先，将总位移分解为三个相互独立的加和项：
$$ \delta(t) = \delta_H(t) + \delta_T(t) + \delta_\theta(t) + \varepsilon(t) $$
其中，$\varepsilon(t)$ 为观测误差及未建模动态引起的随机噪声，假设其服从均值为零的正态分布。

**水压分量 $\delta_H(t)$**：根据材料力学梁理论与弹性半空间理论，坝体在静水压力作用下的水平位移与水深呈非线性关系。取库水位为 $H(t)$，基准水位为 $H_0$，展开为三次多项式：
$$ \delta_H(t) = \sum_{i=1}^3 a_i (H^i(t) - H_0^i) $$
其中，$a_1, a_2, a_3$ 为待估的水压回归系数。

**温度分量 $\delta_T(t)$**：气温与水温呈现显著的年度周期性变化。热传导方程的解析解表明，半无限大物体在边界受周期热激励时，内部温度响应也呈周期变化，但存在幅值衰减与相位滞后。利用傅里叶级数展开并截取前两阶谐波作为基函数：
$$ \delta_T(t) = \sum_{i=1}^2 \left[ b_{1i} \left( \sin \frac{2\pi i t}{365} - \sin \frac{2\pi i t_0}{365} \right) + b_{2i} \left( \cos \frac{2\pi i t}{365} - \cos \frac{2\pi i t_0}{365} \right) \right] $$
其中，$t$ 为观测日距起始日的累积天数，$t_0$ 为基准观测日，$b_{1i}, b_{2i}$ 为温度响应系数。

**时效分量 $\delta_\theta(t)$**：主要由混凝土徐变及基岩塑性变形引起，表现为变形速率随时间逐渐衰减并最终趋于稳定。采用对数与线性组合形式描述：
$$ \delta_\theta(t) = c_1 (\theta - \theta_0) + c_2 (\ln(1+\theta) - \ln(1+\theta_0)) $$
此处定义归一化时间变量 $\theta = t/100$。

在模型求解阶段，将上述方程写成矩阵形式 $\mathbf{Y} = \mathbf{X}\boldsymbol{\beta} + \boldsymbol{\varepsilon}$。由于库水位与季节温度等因素可能存在一定的共线性，直接使用普通最小二乘估计（OLS） $\hat{\boldsymbol{\beta}} = (\mathbf{X}^T \mathbf{X})^{-1} \mathbf{X}^T \mathbf{Y}$ 可能面临信息矩阵病态的问题，导致参数估计方差过大。工程数值计算中常引入岭回归（Ridge Regression），通过在对角线上增加非负惩罚项 $\lambda \mathbf{I}$ 改善数值稳定性：
$$ \hat{\boldsymbol{\beta}}_{ridge} = (\mathbf{X}^T \mathbf{X} + \lambda \mathbf{I})^{-1} \mathbf{X}^T \mathbf{Y} $$

### 6.2.2 土石坝渗流场控制方程与数值离散

土石坝内部地下水流动可视作多孔介质中的非稳定渗流。基于流体质量守恒定律与扩展的达西定律，三维各向异性介质内的非稳定渗流偏微分方程（Richards Equation）推导如下：

建立基本连续性方程：
$$ \frac{\partial (\rho_w n)}{\partial t} + \nabla \cdot (\rho_w \mathbf{q}) = 0 $$
其中，$\rho_w$ 为流体密度，$n$ 为孔隙率，$\mathbf{q}$ 为达西流速矢量。考虑到流体的可压缩性及土骨架的弹性变形，引入给水度（或贮水率）$S_s$，并代入 $\mathbf{q} = -\mathbf{K} \nabla H$ （$H$ 为总水头，$\mathbf{K}$ 为渗透系数张量），可得控制方程：
$$ \nabla \cdot (\mathbf{K} \nabla H) + Q_w = S_s \frac{\partial H}{\partial t} $$
展开为笛卡尔坐标系下的形式：
$$ \frac{\partial}{\partial x} \left( K_x \frac{\partial H}{\partial x} \right) + \frac{\partial}{\partial y} \left( K_y \frac{\partial H}{\partial y} \right) + \frac{\partial}{\partial z} \left( K_z \frac{\partial H}{\partial z} \right) + Q_w = S_s \frac{\partial H}{\partial t} $$

针对内部结构与材料分区复杂的土石坝，获得解析解存在困难，实际应用中常采用有限单元法（FEM）进行空间域离散。利用伽辽金（Galerkin）加权余量法，设定形函数 $N_i$，对求解域 $\Omega$ 内的残差方程进行积分：
$$ \int_\Omega N_i \left[ \nabla \cdot (\mathbf{K} \nabla \tilde{H}) - S_s \frac{\partial \tilde{H}}{\partial t} \right] d\Omega = 0 $$
对扩散项应用格林公式进行分部积分，不仅降低了对试探水头函数 $\tilde{H}$ 导数的连续性要求，也自然引入了边界条件。最终积分域上的方程组可化为矩阵描述的一阶常微分方程：
$$ \mathbf{C} \{\dot{H}\} + \mathbf{K_g} \{H\} = \mathbf{F} $$
式中，$\mathbf{C}$ 为系统的储水矩阵，$\mathbf{K_g}$ 为总体渗透传导矩阵，$\mathbf{F}$ 为包含源汇项与已知流量边界条件的荷载向量。对于时间维度的步进求解，工业界通用向后差分格式（Backward Euler），以保障无条件数值稳定性，并由此实现对各时间步内大型稀疏线性方程组的迭代推进。

## 6.4 工程启示与应用建议

基于本章理论推导与典型案例的数值仿真分析，在推进大坝安全智能监控工作向纵深发展时，提出如下工程实践建议：

第一，**构建多维动态的安全报警阈值体系**。传统的单参数极值报警规则（如简单设定绝对位移超限即报警）极易受季节更替引发的常规周期性波动干扰而产生大量的虚假警报。工程中应广泛引入基于物理力学逻辑的统计辨识模型，建立以“残差统计量”为核心的动态警戒管道。只有当实测数据偏离模型预测区间的置信下限或上限（如超出 $\pm 3\sigma$），且这一异常增量在相邻空间的测点序列中呈现出同向聚集性演变时，方可被确认为需采取人工干预的高级预警事件。

第二，**深化对土石坝渗流异常模式的早期识别**。量水堰的汇漏总量与环境边界条件（如库水位、降雨量）之间呈现复杂的非线性相伴规律。建议在关键渗漏通道及排水廊道部位融合分布式光纤传感技术（如DTS温度传感），利用水流示踪与坝体介质的热交换差异，将单一的一维流量数字升格为涵盖三维空间定位的流场分布特征图。若测压管测值呈现非预期的阶跃突变（突降或突升），且同步伴随渗流量的急剧放大或水质浊度的显著增加，则存在极高的内部管涌侵蚀风险，必须立即启动应急探测预案。

第三，**建立模型在线迭代与参数实时同化机制**。伴随水工建筑物服役年限的持续增长，材料固有的老化行为（例如大体积混凝土内部碱骨料反应碳化、防渗土体发生微观颗粒淤堵或流失）将引发初始构建的物理力学模型或统计模型回归参数逐渐失真。预警系统软件架构应具备自动化在线数据同化功能，利用动态滑动窗口提取最新时序数据，周期性地反演并校准模型状态参数。确保数字空间中的虚拟大坝能真实表征物理实体的现时状态，这是促成大坝安全管理从经验判别向数字孪生智能决策跃迁的核心前提。

## 本章小结

本章围绕混凝土重力坝及土石坝两类典型枢纽工程，系统阐述了其安全监测体系的基础理论架构、核心物理参量及演变过程的数学描述。重点解析了用于分离变形组分的HST多变量统计模型推导步骤，并系统建立了描述土石坝非稳定渗流场的三维有限元偏微分控制方程；通过导入三峡大坝垂线位移实测数据与小浪底水利枢纽土石坝渗流数值仿真案例，论证了参数估计算法效能及多孔介质中流场时滞演化的物理规律。本章的研究将抽象的数据处理算法赋能于具象的工程场景，为设计和开发涵盖状态自感知、演变可计算、风险可预判的现代水工建筑物智能诊断平台夯实了基础。

"""
书名：《大坝安全监测与预警》
章节：第6章 案例：混凝土坝/土石坝（6.1 基本概念与理论框架）
功能：构建“监测数据 -> 机理回归建模 -> 阈值预警 -> KPI评估 -> 图形展示”的完整仿真流程
"""

import numpy as np
from scipy.optimize import least_squares
import matplotlib.pyplot as plt

# =========================
# 1) 关键参数（可直接调参）
# =========================
N_DAYS = 365 * 2                  # 仿真天数
TRAIN_RATIO = 0.70                # 训练集比例
RANDOM_SEED = 2026                # 随机种子，保证可复现
CONCRETE_NOISE_STD = 0.60         # 混凝土坝观测噪声标准差
EARTH_NOISE_STD = 0.80            # 土石坝观测噪声标准差
N_ANOMALY = 12                    # 每类坝注入异常点个数
WARNING_K1 = 2.0                  # 一级预警阈值系数（k*sigma）
WARNING_K2 = 3.0                  # 二级预警阈值系数（k*sigma）

# 混凝土坝“真实”参数（用于生成仿真真值）
TRUE_CONCRETE = np.array([2.0, 0.070, 0.050, 1.20, -0.80, 0.030])
# 土石坝“真实”参数（用于生成仿真真值）
TRUE_EARTH = np.array([1.0, 0.050, 0.080, 0.180, 6.00, 0.015])

rng = np.random.default_rng(RANDOM_SEED)

# 中文显示设置（若本机无中文字体，可删除这两行）
plt.rcParams["font.sans-serif"] = ["SimHei", "Microsoft YaHei", "Arial Unicode MS", "DejaVu Sans"]
plt.rcParams["axes.unicode_minus"] = False


# =========================
# 2) 数据生成与机理模型
# =========================
def generate_drivers(n_days: int, rng_obj):
    """生成外部驱动因子：库水位、气温、降雨。"""
    t = np.arange(n_days, dtype=float)

    # 库水位：年周期 + 随机扰动
    h = 100 + 10 * np.sin(2 * np.pi * t / 365 - 0.6) + rng_obj.normal(0, 1.2, n_days)
    # 气温：年周期 + 随机扰动
    temp = 15 + 12 * np.sin(2 * np.pi * t / 365 - np.pi / 2) + rng_obj.normal(0, 1.0, n_days)
    # 降雨：Gamma随机 + 季节性调制
    rain = rng_obj.gamma(1.5, 5.0, n_days) * (0.3 + 0.7 * (np.sin(2 * np.pi * t / 365) + 1) / 2)

    return t, h, temp, rain


def concrete_model(p, t, h, temp):
    """
    混凝土坝响应模型（示例）：
    位移/效应 = 水位项 + 温度项 + 周期项 + 时效项
    """
    a0, a1, a2, a3, a4, a5 = p
    return (
        a0
        + a1 * h
        + a2 * temp
        + a3 * np.sin(2 * np.pi * t / 365)
        + a4 * np.cos(2 * np.pi * t / 365)
        + a5 * np.log1p(t)
    )


def earth_model(p, t, h, temp, rain):
    """
    土石坝响应模型（示例）：
    位移/渗流效应 = 水位项 + 降雨项 + 固结时变项 + 衰减项 + 温度项
    """
    b0, b1, b2, b3, b4, b5 = p
    return b0 + b1 * h + b2 * rain + b3 * np.sqrt(t + 1) + b4 * np.exp(-t / 500) + b5 * temp


def inject_observation_noise_and_anomaly(y_true, noise_std, n_anomaly, rng_obj):
    """叠加观测噪声并注入异常点，返回观测值与真实异常标签。"""
    y_obs = y_true + rng_obj.normal(0, noise_std, len(y_true))
    labels = np.zeros(len(y_true), dtype=int)

    anomaly_idx = rng_obj.choice(len(y_true), size=n_anomaly, replace=False)
    spike = rng_obj.normal(0, 4 * noise_std, n_anomaly) + rng_obj.choice([-1, 1], n_anomaly) * (5 * noise_std)
    y_obs[anomaly_idx] += spike
    labels[anomaly_idx] = 1
    return y_obs, labels


# =========================
# 3) 模型拟合、预警、KPI
# =========================
def fit_concrete_params(t, h, temp, y, train_end):
    """用最小二乘拟合混凝土坝模型参数。"""
    def residuals(p):
        return concrete_model(p, t[:train_end], h[:train_end], temp[:train_end]) - y[:train_end]

    p0 = np.array([0.0, 0.05, 0.02, 1.0, 1.0, 0.01])
    return least_squares(residuals, p0).x


def fit_earth_params(t, h, temp, rain, y, train_end):
    """用最小二乘拟合土石坝模型参数。"""
    def residuals(p):
        return earth_model(p, t[:train_end], h[:train_end], temp[:train_end], rain[:train_end]) - y[:train_end]

    p0 = np.array([0.0, 0.04, 0.03, 0.2, 5.0, 0.01])
    return least_squares(residuals, p0).x


def warning_levels(residual, train_end, k1=2.0, k2=3.0):
    """
    按训练期残差标准差建立预警阈值：
    |残差| > k1*sigma -> 一级预警
    |残差| > k2*sigma -> 二级预警
    """
    sigma = np.std(residual[:train_end], ddof=1)
    th1, th2 = k1 * sigma, k2 * sigma

    level = np.zeros_like(residual, dtype=int)
    abs_r = np.abs(residual)
    level[abs_r > th1] = 1
    level[abs_r > th2] = 2
    return level, th1, th2


def calc_kpi(y, yhat, true_anomaly, alarm2):
    """计算拟合与预警性能指标。"""
    err = y - yhat
    rmse = np.sqrt(np.mean(err ** 2))
    mae = np.mean(np.abs(err))
    r2 = 1 - np.sum(err ** 2) / np.sum((y - np.mean(y)) ** 2)

    tp = np.sum((alarm2 == 1) & (true_anomaly == 1))
    fp = np.sum((alarm2 == 1) & (true_anomaly == 0))
    fn = np.sum((alarm2 == 0) & (true_anomaly == 1))
    tn = np.sum((alarm2 == 0) & (true_anomaly == 0))

    precision = tp / (tp + fp + 1e-12)
    recall = tp / (tp + fn + 1e-12)
    false_alarm_rate = fp / (fp + tn + 1e-12)

    return {
        "RMSE": rmse,
        "MAE": mae,
        "R2": r2,
        "Precision": precision,
        "Recall": recall,
        "FalseAlarm": false_alarm_rate,
    }


def print_kpi_table(kpi_map):
    """打印KPI结果表格。"""
    print("\n================ KPI结果表 ================")
    print(f"{'坝型':<10}{'RMSE':>10}{'MAE':>10}{'R2':>10}{'Precision':>12}{'Recall':>10}{'FalseAlarm':>12}")
    print("-" * 74)
    for dam_name, k in kpi_map.items():
        print(
            f"{dam_name:<10}"
            f"{k['RMSE']:>10.3f}"
            f"{k['MAE']:>10.3f}"
            f"{k['R2']:>10.3f}"
            f"{k['Precision']:>12.3f}"
            f"{k['Recall']:>10.3f}"
            f"{k['FalseAlarm']:>12.3f}"
        )
    print("=" * 74)


def normalize(x):
    """用于多驱动同图展示的标准化。"""
    return (x - np.mean(x)) / (np.std(x) + 1e-12)


# =========================
# 4) 主流程
# =========================
if __name__ == "__main__":
    # 生成驱动因子
    t, h, temp, rain = generate_drivers(N_DAYS, rng)
    train_end = int(N_DAYS * TRAIN_RATIO)

    # 生成“真实响应”
    y_true_c = concrete_model(TRUE_CONCRETE, t, h, temp)
    y_true_e = earth_model(TRUE_EARTH, t, h, temp, rain)

    # 生成观测值并注入异常
    y_obs_c, label_c = inject_observation_noise_and_anomaly(y_true_c, CONCRETE_NOISE_STD, N_ANOMALY, rng)
    y_obs_e, label_e = inject_observation_noise_and_anomaly(y_true_e, EARTH_NOISE_STD, N_ANOMALY, rng)

    # 参数反演（拟合）
    p_hat_c = fit_concrete_params(t, h, temp, y_obs_c, train_end)
    p_hat_e = fit_earth_params(t, h, temp, rain, y_obs_e, train_end)

    # 全时段预测
    y_hat_c = concrete_model(p_hat_c, t, h, temp)
    y_hat_e = earth_model(p_hat_e, t, h, temp, rain)

    # 残差与预警
    res_c = y_obs_c - y_hat_c
    res_e = y_obs_e - y_hat_e
    lv_c, th1_c, th2_c = warning_levels(res_c, train_end, WARNING_K1, WARNING_K2)
    lv_e, th1_e, th2_e = warning_levels(res_e, train_end, WARNING_K1, WARNING_K2)

    # 二级预警作为“报警事件”用于KPI统计
    alarm2_c = (lv_c == 2).astype(int)
    alarm2_e = (lv_e == 2).astype(int)

    kpi_c = calc_kpi(y_obs_c, y_hat_c, label_c, alarm2_c)
    kpi_e = calc_kpi(y_obs_e, y_hat_e, label_e, alarm2_e)

    # 打印结果
    print("混凝土坝拟合参数:", np.round(p_hat_c, 4))
    print("土石坝拟合参数  :", np.round(p_hat_e, 4))
    print(f"混凝土坝阈值: 一级={th1_c:.3f}, 二级={th2_c:.3f}")
    print(f"土石坝阈值  : 一级={th1_e:.3f}, 二级={th2_e:.3f}")

    print_kpi_table({
        "混凝土坝": kpi_c,
        "土石坝": kpi_e
    })

    # 画图
    fig, axes = plt.subplots(3, 1, figsize=(13, 10), sharex=True)

    # 图1：驱动因子（标准化后同图）
    axes[0].plot(t, normalize(h), label="库水位(标准化)", linewidth=1.2)

    axes[0].plot(t, normalize(rain), label="降雨(标准化)", linewidth=1.2)
    axes[0].axvline(train_end, color="gray", linestyle="--", linewidth=1.0, label="训练/验证分界")
    axes[0].set_title("驱动因子时序")
    axes[0].legend(loc="upper right")
    axes[0].grid(alpha=0.25)

    # 图2：混凝土坝
    axes[1].plot(t, y_obs_c, label="观测值", color="#1f77b4", linewidth=1.0)
    axes[1].plot(t, y_hat_c, label="模型预测", color="#ff7f0e", linewidth=1.5)
    axes[1].scatter(t[lv_c == 1], y_obs_c[lv_c == 1], s=20, c="goldenrod", label="一级预警")
    axes[1].scatter(t[lv_c == 2], y_obs_c[lv_c == 2], s=28, c="red", label="二级预警")
    axes[1].axvline(train_end, color="gray", linestyle="--", linewidth=1.0)
    axes[1].set_title("混凝土坝：监测-预测-预警")
    axes[1].legend(loc="upper right")
    axes[1].grid(alpha=0.25)

    # 图3：土石坝
    axes[2].plot(t, y_obs_e, label="观测值", color="#2ca02c", linewidth=1.0)
    axes[2].plot(t, y_hat_e, label="模型预测", color="#d62728", linewidth=1.5)
    axes[2].scatter(t[lv_e == 1], y_obs_e[lv_e == 1], s=20, c="goldenrod", label="一级预警")
    axes[2].scatter(t[lv_e == 2], y_obs_e[lv_e == 2], s=28, c="red", label="二级预警")
