**位置**: 3.1.3节

**问题**: 仅文字描述"评估聚集程度",未给出计算公式

**建议补充**:
```markdown
**拥挤度距离计算**:

对于同一非支配层级F_i中的个体p,其拥挤度距离定义为:

$$d_p = \sum_{j=1}^{m} \frac{f_j(p_{next}) - f_j(p_{prev})}{f_j^{max} - f_j^{min}}$$

其中:
- m为目标函数个数
- p_{next}和p_{prev}为在第j个目标上相邻的个体
- f_j^{max}和f_j^{min}为第j个目标的最大最小值
- 边界个体(最大/最小)的拥挤度距离设为∞

**物理意义**: d_p越大,表明个体p周围解的分布越稀疏,应优先保留以维持Pareto前沿的多样性。

### 3.1.4.1 启发式修正法详细步骤

**场景**: 决策变量R_t导致水位越限

**修正流程**:
1. 计算时段末水位: Z_{t+1} = f_Z(V_{t+1})
2. 判断越限:
   - If Z_{t+1} > Z_max: 水位上越限
   - If Z_{t+1} < Z_min: 水位下越限
3. 修正水位: Z_{t+1}' = clip(Z_{t+1}, Z_min, Z_max)
4. 反算库容: V_{t+1}' = f_V(Z_{t+1}')
5. 反算下泄流量: R_t' = (V_t - V_{t+1}' + I_t·Δt - E_t·Δt) / Δt
6. 更新决策变量: R_t ← R_t'

**Python实现**:
```python
def repair_constraint(V_t, R_t, I_t, E_t, dt, Z_min, Z_max, Z_V_curve, V_Z_curve):
    # 计算时段末库容
    V_next = V_t + (I_t - R_t - E_t) * dt

    # 计算水位
    Z_next = Z_V_curve(V_next)

    # 修正越限
    if Z_next > Z_max:
        Z_next = Z_max
    elif Z_next < Z_min:
        Z_next = Z_min

    # 反算库容和流量
    V_next_repaired = V_Z_curve(Z_next)
    R_t_repaired = (V_t - V_next_repaired + I_t * dt - E_t * dt) / dt

    return R_t_repaired, V_next_repaired

**复杂度分析**:

传统NSGA: 每次排序需要O(MN²)比较,共N层,总复杂度O(MN³)

NSGA-II改进:
1. 预计算每个个体的被支配数n_p和支配集合S_p: O(MN²)
2. 构建第一层F_1: O(N)
3. 后续层级通过更新n_p递减构建: O(N)
4. 总复杂度: O(MN²)

其中M为目标函数个数,N为种群规模。
