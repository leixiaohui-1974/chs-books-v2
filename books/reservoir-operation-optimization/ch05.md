```markdown
## 本章导读

水库系统作为流域水资源时空尺度再分配与防洪调度的核心枢纽设施，其运行环境长期处于高度的动态演变之中。在多变的气候背景与复杂下垫面条件下，流域降雨径流过程呈现出显著的非平稳时间序列特征。传统水库调度通常依赖基于历史长系列实测径流资料编制的静态调度图，这种范式受限于设计之初的稳态假设与统计均值思维，在面对极端突发性水文事件或频繁的短期水情波动时，往往暴露其在响应速度和决策灵活性上的不足。现代水资源综合管理对防洪、发电、航运及生态环境供水等多维调度目标的协同提出了更苛刻的量化要求，单纯沿用常规固化规则已难以挖掘水利枢纽工程的综合效益潜能。

## 5.1 基本概念与理论框架

实时调度中的滚动MPC理论，从控制论视角剖析，是将一个在无限时域内运行的复杂最优控制难题，等效转换为在每一个推进步长内反复求解有限预测时域内的约束优化问题。其系统架构坚固地建立在三大核心机制模块之上：预测模型、滚动优化与反馈校正。

首先是预测模型机制。预测模型构成了MPC算法洞悉物理系统未来状态演变趋势的数学底座。在水库及流域管理的物理语境下，该模型矩阵通常涵盖降雨径流产汇流模型、河道一维或二维水动力学演进模型以及水库节点的水量平衡方程。假设当前系统处于调度时刻 $t$，预测模型以水库当前的实际水位、流域土壤含水率及前期降水指数等状态变量为基准，融合未来一段时间内（即预测时域）的气象降水预报或入库流量预报，推演系统未来的时空状态轨迹。预测模型的构建需要在物理机制的描述保真度与在线数学求解的计算开销之间寻求均衡点。

其次为滚动优化机制。在获取预测模型输出的未来状态演进空间后，MPC算法在设定的前瞻窗口（预测时域 $H_p$）内，求解使预设多目标成本函数达到最小的控制指令序列（如未来多时段的闸门启闭度或下泄流量规划）。有别于经典动态规划追求全生命周期全局最优的特征，滚动优化的逻辑精髓在于“前瞻规划，单步执行”。具体而言，优化求解器计算出未来 $H_p$ 个步长的最优下泄流量序列 $\mathbf{u}^* = [u^*_{t|t}, u^*_{t+1|t}, \dots, u^*_{t+H_p-1|t}]$，但在物理执行层面，调度中心仅向执行机构下发该序列的第一个控制变量 $u^*_{t|t}$。当时间推演至下一调度步长 $t+1$ 时，整个优化窗口在时间轴上向右平移，算法重新采集最新边界条件，再次启动全流程的优化解算。

最后是反馈校正闭环机制。受限于气象科学的发展阶段，短临及中长期水文预报普遍存在不同程度的随机误差，且该误差往往随预见期的延伸而呈指数级放大。若单纯依托开环的控制框架，模型预报误差与系统扰动将在时域内发生级联累积，最终驱动水库运行状态触碰工程安全红线。反馈校正机制依托广泛部署的遥测系统（SCADA），在每个新步长初采集大坝上游水位、尾水位及入库流量的真实观测值，强行修正系统内部状态变量，并以此作为新一轮优化的刚性初始条件。这种引入实测数据的闭环校正策略，阻断了误差的长期传播链条，大幅提升了控制系统在面对不确定扰动时的鲁棒性能。

## 5.2 数学建模与求解方法

本节聚焦于将水库调度的工程物理过程抽象为严格的非线性离散时间数学模型，并探讨满足在线求解要求的数值近似与优化技术。

### 5.2.1 综合目标函数的构建

水库实时调度往往需要在防洪保安、水力发电最大化与下游生态及生活供水保障等多个存在竞争冲突的目标间进行妥协与折中。设当前决策步长为 $t$，优化预测时域为 $H_p$（步长长度为 $\Delta t$）。在滚动框架内，每一时刻所需求解的目标函数 $J$ 可表达为多元成本的加权聚合：

$$ \min_{\mathbf{u}_{t}} J = \sum_{k=0}^{H_p-1} \left[ \omega_1 L_{\text{flood}}(V_{t+k|t}, q_{t+k|t}) - \omega_2 P_{\text{power}}(V_{t+k|t}, q_{t+k|t}) + \omega_3 L_{\text{eco}}(q_{t+k|t}) \right] + \Phi(V_{t+H_p|t}) $$

式中变量与算子的物理映射关系如下：
- 决策变量矩阵 $\mathbf{u}_{t} = [q_{t|t}, q_{t+1|t}, \dots, q_{t+H_p-1|t}]^T$ 定义了预测时域内的下泄流量序列。
- $\omega_1, \omega_2, \omega_3$ 为无量纲的惩罚权重系数，用于表达决策者在特定调度周期内的主观偏好（例如主汛期将赋予 $\omega_1$ 绝对的主导权重）。
- $L_{\text{flood}}$ 为防洪风险惩罚项，采用针对警戒界限的二次非对称罚函数：
  $$ L_{\text{flood}} = \alpha_1 \max(0, Z(V_{t+k|t}) - Z_{\text{fl}})^2 + \alpha_2 \max(0, q_{t+k|t} - q_{\text{safe}})^2 $$
  其中 $Z(V_{t+k|t})$ 为当前蓄水量对应的上游水位，$Z_{\text{fl}}$ 为防洪汛限水位，$q_{\text{safe}}$ 标定了下游河道堤防的安全泄量上限。
- $P_{\text{power}}$ 表征水电站发电效益，基于流体动力学能量转换方程推导：
  $$ P_{\text{power}} = A \cdot \eta \cdot q_{t+k|t} \cdot \left[ Z(V_{t+k|t}) - Z_{\text{tail}}(q_{t+k|t}) - \Delta h_{\text{loss}} \right] $$
  $A$ 为机组综合出力系数，$\eta$ 为水轮发电机组的能量转换效率，$Z_{\text{tail}}$ 为随下泄流量动态变化的尾水高程，$\Delta h_{\text{loss}}$ 包含沿程与局部水头损失。
- $\Phi(V_{t+H_p|t})$ 定义为终端状态约束函数（Terminal Cost），其存在是为了评估优化末端时刻水库蓄水状态的长远期经济价值，强制算法储备必要水量以避免“短视（Myopic）”放空行为。

### 5.2.2 状态转移与空间物理约束

构建与真实物理环境同构的状态空间模型是MPC成功运行的前提。

1. **离散状态转移方程**
预测核心机理依据质量守恒定律建立水库的水量连续性方程。采用前向差分离散，状态演进方程写作：
$$ V_{t+k+1|t} = V_{t+k|t} + \left[ I_{t+k|t} - q_{t+k|t} - q_{\text{spill}, t+k|t} - E_v(V_{t+k|t}, T_{\text{air}}) \right] \cdot \Delta t $$
其中：$I_{t+k|t}$ 为由上游水文预报模型注入的流量序列；$q_{\text{spill}}$ 为溢洪道弃水流量；$E_v$ 表述了随水库表面积与气象气温 $T_{\text{air}}$ 变化的蒸散发速率。此方程组构成了系统演化的刚性等式约束。

2. **多维运行能力边界约束**
水工建筑物的结构安全与机械设备的工况极限，投射在数学模型中成为严格的不等式约束集：
- 库容上下界限：$$ V_{\text{dead}} \le V_{t+k|t} \le V_{\text{max}} $$ 
  （对应死库容与校核洪水位库容，防止吸水真空与漫坝失事）
- 泄洪过流能力约束：$$ q_{\text{min}} \le q_{t+k|t} + q_{\text{spill}, t+k|t} \le Q_{\text{capacity}}(Z_{t+k|t}) $$ 
  （最大泄流量受控于当前库水位提供的高程水头及泄水孔口的几何水力特性）
- 流量变幅（Ramping Rate）约束：$$ \Delta q_{\text{down}} \le q_{t+k|t} - q_{t+k-1|t} \le \Delta q_{\text{up}} $$ 
  （抑制相邻时段下泄流量的剧烈阶跃，以维系下游河床地质稳定与航区通航条件）

### 5.2.3 算法求解与非线性处理技术

目标函数中的发电效益表达式包含状态变量与控制变量的非线性乘积项，且水位-库容转换关系常呈不规则曲线形态。此模型本质上属于非凸的非线性规划（NLP）问题。在包含多节点梯级水库的高维状态空间中，传统动态规划（DP）因面临指数级暴涨的“维数灾难”而失效。
为确保工业级应用中的求解时效，业内常采用**混合整数线性规划（MILP）**配合**SOS2（Special Ordered Sets of type 2）**约束的线性化逼近策略。具体而言，针对非线性水位-库容曲线 $Z = f(V)$，选取 $N$ 个离散断点组合 $(V_i, Z_i)$。引入连续权重变量 $\lambda_i \in [0, 1]$ 且 $\sum_{i=1}^N \lambda_i = 1$。
则水位和库容可近似表达为：
$$ Z \approx \sum_{i=1}^N \lambda_i Z_i, \quad V \approx \sum_{i=1}^N \lambda_i V_i $$
SOS2约束强制规定在向量 $\mathbf{\lambda}$ 中，最多仅允许有两个相邻的元素非零。此类约束可通过引入额外的二进制0-1变量实现。经由分段线性化处理后，原复杂的NLP被映射为标准的MILP问题，借助分支定界（Branch and Bound）算法即可在确定时间内获取具备全局次优数学下界的稳定解。

## 5.4 工程启示与应用建议

将高度抽象的滚动MPC理论顺利部署至复杂真实的流域梯级管理体系，面临着架构衔接与安全冗余的严峻考验，以下工程范式转化路径建议具备实施价值。

1. **构建“预报-调度”深度耦合的数字孪生体**
实施高精度水文预测与调度控制模型的无缝焊接。建议扬弃单一确定性预报体制，全面引入集合预报（Ensemble Forecasting）输出群作为优化引擎的扰动输入矩阵，进而升级为随机MPC（Stochastic MPC）或鲁棒MPC。利用数字孪生技术底座，在虚拟空间内对控制策略集进行预演与压力测试，过滤高危决策，再将安全指令流映射至物理实体。

2. **部署算力剥离与“云-边”协同架构**
鉴于非线性优化的时空复杂度，工业化系统应采取算力分层策略。在厂站边缘端控制层，部署轻量级、完全线性化的局部降维MPC模型或显式查表MPC（Explicit MPC），以毫秒级响应速度执行基层水位维持控制；在远端云中心，部署搭载全量水动力学与长周期的复杂非线性调度运算，并以小时级频次将状态锚点或目标函数引导参数下发至边缘层进行轨迹纠偏。

3. **构筑无条件硬件防线与柔性降级机制**
数学优化的可行解域存在理论盲区，传感器失真或通讯信道阻断皆可导致控制矩阵呈现发散崩溃。在可编程逻辑控制器（PLC）等底层硬件的底层执行逻辑中，必须强制硬编码大坝结构的物理保安红线。一旦诊断识别出算法陷入不可行状态（Infeasibility）或指令长期缺失，系统需在中断层面触发软隔离，平稳降维切换至基于实测水位的保坝调度图规则，确立全天候的工程安全底线。
```

```python
"""
书名：《水库调度优化与决策》
章节：第5章 实时调度（滚动MPC）
功能：构建含SOS2分段线性化的水库调度MILP仿真，输出KPI结果表并绘制调度过程图。
"""

import numpy as np
import matplotlib.pyplot as plt
from scipy.optimize import milp, Bounds, LinearConstraint
from scipy.sparse import lil_matrix


def main():
    # =========================
    # 1) 关键参数定义
    # =========================
    T = 24                      # 调度时段数（例如24个6小时）
    dt_h = 6.0                  # 每时段小时数
    dt_v = dt_h * 3600.0 / 1e6  # 流量(m3/s)转体积(百万m3)的系数

    # 合成入库流量过程（可替换为实测/预报序列）
    t_idx = np.arange(T)
    inflow = (
        700
        + 650 * np.exp(-((t_idx - 7) / 2.8) ** 2)
        + 900 * np.exp(-((t_idx - 16) / 2.2) ** 2)
    )
    inflow = np.clip(inflow, 250, None)

    # 库容-水位分段点（SOS2）
    V_bp = np.array([180, 220, 260, 300, 340, 380, 420], dtype=float)      # 百万m3
    Z_up_bp = np.array([148.0, 151.0, 154.0, 157.0, 159.5, 161.5, 163.0])   # m

    # 总泄量-尾水位分段点（SOS2）
    Q_bp = np.array([100, 300, 600, 900, 1300, 1800], dtype=float)          # m3/s
    Z_tail_bp = np.array([120.0, 120.8, 121.9, 123.0, 124.2, 125.8], dtype=float)

    # 工程运行边界
    s0 = 300.0                 # 初始库容（百万m3）
    s_target = 310.0           # 末期目标库容（百万m3）
    s_flood = 360.0            # 防洪控制库容（百万m3）
    s_min, s_max = V_bp[0], V_bp[-1]

    q_min, q_max = 120.0, 1500.0     # 机组下泄约束
    spill_max = 1800.0               # 弃水上限
    q_eco = 220.0                    # 生态流量下限
    q0 = 350.0                       # 初始下泄
    ramp_up, ramp_dn = 400.0, 400.0  # 下泄爬坡约束

    # 目标函数权重（可做灵敏度分析）
    release_value = 1.0      # 下泄收益（线性近似）
    spill_penalty = 7.0      # 弃水惩罚
    flood_penalty = 40.0     # 超汛限惩罚
    eco_penalty = 80.0       # 生态缺额惩罚
    terminal_penalty = 8.0   # 末库容偏差惩罚

    # 发电计算参数（用于KPI评估，不直接入MILP目标）
    eta = 0.90
    head_loss = 2.0

    Jv = len(V_bp)
    Jq = len(Q_bp)

    # =========================
    # 2) 变量索引
    # =========================
    cur = 0
    idx_s = np.arange(cur, cur + T); cur += T
    idx_r = np.arange(cur, cur + T); cur += T
    idx_sp = np.arange(cur, cur + T); cur += T
    idx_z = np.arange(cur, cur + T); cur += T
    idx_e = np.arange(cur, cur + T); cur += T

    idx_dev_p = cur; cur += 1
    idx_dev_n = cur; cur += 1

    idx_lam_v = np.arange(cur, cur + T * Jv).reshape(T, Jv); cur += T * Jv
    idx_y_v = np.arange(cur, cur + T * (Jv - 1)).reshape(T, Jv - 1); cur += T * (Jv - 1)

    idx_lam_q = np.arange(cur, cur + T * Jq).reshape(T, Jq); cur += T * Jq
    idx_y_q = np.arange(cur, cur + T * (Jq - 1)).reshape(T, Jq - 1); cur += T * (Jq - 1)

    n_var = cur

    # =========================
    # 3) 目标函数
    # =========================
    c = np.zeros(n_var)
    c[idx_r] = -release_value
    c[idx_sp] = spill_penalty
    c[idx_z] = flood_penalty
    c[idx_e] = eco_penalty
    c[idx_dev_p] = terminal_penalty
    c[idx_dev_n] = terminal_penalty

    # 二进制变量：SOS2段选择变量
    integrality = np.zeros(n_var, dtype=int)
    integrality[idx_y_v.ravel()] = 1
    integrality[idx_y_q.ravel()] = 1

    # =========================
    # 4) 变量上下界
    # =========================
    lb = np.full(n_var, -np.inf)
    ub = np.full(n_var, np.inf)

    lb[idx_s], ub[idx_s] = s_min, s_max
    lb[idx_r], ub[idx_r] = q_min, q_max
    lb[idx_sp], ub[idx_sp] = 0.0, spill_max
    lb[idx_z], ub[idx_z] = 0.0, np.inf
    lb[idx_e], ub[idx_e] = 0.0, np.inf
    lb[idx_dev_p], ub[idx_dev_p] = 0.0, np.inf
    lb[idx_dev_n], ub[idx_dev_n] = 0.0, np.inf

    lb[idx_lam_v.ravel()], ub[idx_lam_v.ravel()] = 0.0, 1.0
    lb[idx_lam_q.ravel()], ub[idx_lam_q.ravel()] = 0.0, 1.0
    lb[idx_y_v.ravel()], ub[idx_y_v.ravel()] = 0.0, 1.0
    lb[idx_y_q.ravel()], ub[idx_y_q.ravel()] = 0.0, 1.0

    # =========================
    # 5) 线性约束构建
    # =========================
    rows = []

    def add_row(cols, vals, low, up):
        rows.append((np.asarray(cols, dtype=int), np.asarray(vals, dtype=float), float(low), float(up)))

    # 5.1 质量平衡
    for t in range(T):
        cols = [idx_s[t], idx_r[t], idx_sp[t]]
        vals = [1.0, dt_v, dt_v]
        rhs = dt_v * inflow[t]
        if t == 0:
            rhs += s0
        else:
            cols.append(idx_s[t - 1]); vals.append(-1.0)
        add_row(cols, vals, rhs, rhs)

    # 5.2 库容-SOS2关系
    for t in range(T):
        add_row(idx_lam_v[t], np.ones(Jv), 1.0, 1.0)
        add_row([idx_s[t], *idx_lam_v[t]], [1.0, *(-V_bp)], 0.0, 0.0)
        add_row(idx_y_v[t], np.ones(Jv - 1), 1.0, 1.0)

        # λ0 <= y0
        add_row([idx_lam_v[t, 0], idx_y_v[t, 0]], [1.0, -1.0], -np.inf, 0.0)
        # λj <= yj-1 + yj
        for j in range(1, Jv - 1):
            add_row([idx_lam_v[t, j], idx_y_v[t, j - 1], idx_y_v[t, j]], [1.0, -1.0, -1.0], -np.inf, 0.0)
        # λ_last <= y_last
        add_row([idx_lam_v[t, Jv - 1], idx_y_v[t, Jv - 2]], [1.0, -1.0], -np.inf, 0.0)

    # 5.3 总泄量-SOS2关系
    for t in range(T):
        add_row(idx_lam_q[t], np.ones(Jq), 1.0, 1.0)
        add_row([idx_r[t], idx_sp[t], *idx_lam_q[t]], [1.0, 1.0, *(-Q_bp)], 0.0, 0.0)
        add_row(idx_y_q[t], np.ones(Jq - 1), 1.0, 1.0)

        add_row([idx_lam_q[t, 0], idx_y_q[t, 0]], [1.0, -1.0], -np.inf, 0.0)
        for j in range(1, Jq - 1):
            add_row([idx_lam_q[t, j], idx_y_q[t, j - 1], idx_y_q[t, j]], [1.0, -1.0, -1.0], -np.inf, 0.0)
        add_row([idx_lam_q[t, Jq - 1], idx_y_q[t, Jq - 2]], [1.0, -1.0], -np.inf, 0.0)

    # 5.4 防洪软约束：S_t - z_t <= s_flood
    for t in range(T):
        add_row([idx_s[t], idx_z[t]], [1.0, -1.0], -np.inf, s_flood)

    # 5.5 生态约束：R_t + Spill_t + e_t >= q_eco
    for t in range(T):
        add_row([idx_r[t], idx_sp[t], idx_e[t]], [1.0, 1.0, 1.0], q_eco, np.inf)

    # 5.6 下泄爬坡约束
    add_row([idx_r[0]], [1.0], q0 - ramp_dn, q0 + ramp_up)
    for t in range(1, T):
        add_row([idx_r[t], idx_r[t - 1]], [1.0, -1.0], -np.inf, ramp_up)
        add_row([idx_r[t - 1], idx_r[t]], [1.0, -1.0], -np.inf, ramp_dn)

    # 5.7 末库容偏差绝对值线性化：S_T - d+ + d- = s_target
    add_row([idx_s[T - 1], idx_dev_p, idx_dev_n], [1.0, -1.0, 1.0], s_target, s_target)

    # 稀疏矩阵组装
    m = len(rows)
    A = lil_matrix((m, n_var), dtype=float)
    lc = np.zeros(m)
    uc = np.zeros(m)
    for i, (cols, vals, lo, up) in enumerate(rows):
        A[i, cols] = vals
        lc[i] = lo
        uc[i] = up

    constraints = [LinearConstraint(A.tocsr(), lc, uc)]
    bounds = Bounds(lb, ub)

    # =========================
    # 6) 求解MILP
    # =========================
    res = milp(
        c=c,
        integrality=integrality,
        bounds=bounds,
        constraints=constraints,
        options={"disp": False}
    )

    if (not res.success) or (res.x is None):
        raise RuntimeError(f"MILP未收敛，状态码={res.status}, 信息={res.message}")

    x = res.x
    s = x[idx_s]
    r = x[idx_r]
    sp = x[idx_sp]
    z = x[idx_z]
    eco = x[idx_e]
    lam_v = x[idx_lam_v]
    lam_q = x[idx_lam_q]

    # =========================
    # 7) KPI计算与打印
    # =========================
    z_up = lam_v @ Z_up_bp
    z_tail = lam_q @ Z_tail_bp
    head = np.maximum(z_up - z_tail - head_loss, 0.0)

    power_mw = 9.81 * eta * r * head / 1000.0
    energy_gwh = np.sum(power_mw * dt_h) / 1000.0

    kpi = [
        ("优化目标函数值", res.fun),
        ("总入库水量(百万m3)", np.sum(inflow) * dt_v),
        ("总机组下泄水量(百万m3)", np.sum(r) * dt_v),
        ("总弃水量(百万m3)", np.sum(sp) * dt_v),
        ("生态缺额水量(百万m3)", np.sum(eco) * dt_v),
        ("峰值库容(百万m3)", np.max(s)),
        ("超汛限持续时间(小时)", np.count_nonzero(s > s_flood) * dt_h),
        ("累计超汛限库容(百万m3)", np.sum(np.maximum(s - s_flood, 0.0))),
        ("平均净水头(m)", np.mean(head)),
        ("总发电量(GWh)", energy_gwh),
    ]

    print("\n=== KPI结果表 ===")
    print(f"{'指标':<28}{'数值':>16}")
    print("-" * 46)
    for name, value in kpi:
        print(f"{name:<28}{value:>16.3f}")

    print("\n=== 分时段结果(前10段) ===")
    print(f"{'时段':>4} {'入流':>10} {'下泄':>10} {'弃水':>10} {'库容':>10} {'水头':>10}")
    for i in range(min(10, T)):
        print(f"{i+1:>4d} {inflow[i]:>10.1f} {r[i]:>10.1f} {sp[i]:>10.1f} {s[i]:>10.1f} {head[i]:>10.2f}")

    # =========================
    # 8) 绘图
    # =========================
    plt.rcParams["font.sans-serif"] = ["Microsoft YaHei", "SimHei", "Arial Unicode MS", "DejaVu Sans"]
    plt.rcParams["axes.unicode_minus"] = False

    x_plot = np.arange(1, T + 1)
    fig, axes = plt.subplots(3, 1, figsize=(11, 10), sharex=True)

    # 图1：流量过程
    axes[0].plot(x_plot, inflow, "b-", lw=2, label="入库流量")
    axes[0].plot(x_plot, r, "g-", lw=2, label="机组下泄")
    axes[0].bar(x_plot, sp, color="orange", alpha=0.4, label="弃水流量")
    axes[0].set_ylabel("流量 (m3/s)")
    axes[0].set_title("水库调度流量过程")
    axes[0].grid(alpha=0.25)
    axes[0].legend()

    # 图2：库容过程
    axes[1].plot(x_plot, s, "k-", lw=2, label="库容")
    axes[1].axhline(s_flood, color="r", ls="--", lw=1.8, label="防洪控制库容")
    axes[1].axhline(s_target, color="purple", ls=":", lw=1.8, label="末期目标库容")
    axes[1].fill_between(x_plot, s_flood, s, where=(s > s_flood), color="red", alpha=0.25, label="超汛限区")
    axes[1].set_ylabel("库容 (百万m3)")
    axes[1].set_title("库容演化与防洪控制")
    axes[1].grid(alpha=0.25)
    axes[1].legend()

    # 图3：净水头与出力
    ax3 = axes[2]
    ax3.plot(x_plot, head, "c-", lw=2, label="净水头")
    ax3.set_ylabel("净水头 (m)")
    ax3.set_xlabel("调度时段")
    ax3.grid(alpha=0.25)
    ax3.set_title("水头与出力过程")

    ax3b = ax3.twinx()
    ax3b.plot(x_plot, power_mw, "m-", lw=2, label="出力")
    ax3b.set_ylabel("出力 (MW)")

    lines1, labels1 = ax3.get_legend_handles_labels()
    lines2, labels2 = ax3b.get_legend_handles_labels()
    ax3b.legend(lines1 + lines2, labels1 + labels2, loc="upper right")

    plt.tight_layout()
    plt.show()


if __name__ == "__main__":
    main()
```

代码解读（约800字）  
这段脚本把“水库实时调度”组织成一个标准的混合整数线性规划（MILP）问题，核心难点是把水库中的非线性关系转换为可求解的线性形式。代码首先在参数区集中定义时段长度、入流过程、库容-水位曲线、总泄量-尾水位曲线以及防洪、生态、爬坡等运行边界。这样做的目的，是把所有工程假设都显式化，后续做灵敏度分析时只改参数，不改模型结构。  
模型变量分为五类：连续状态变量（库容）、连续控制变量（机组下泄与弃水）、软约束松弛变量（超汛限量与生态缺额）、终端偏差变量（绝对值线性化），以及SOS2分段线性化所需的λ权重与0-1段选择变量y。代码通过“索引切片”管理变量位置，避免手工拼接时出错，这是大规模优化脚本里非常关键的工程化写法。  
约束部分按物理逻辑分层构建。第一层是质量平衡方程，确保每时段的蓄泄关系符合守恒。第二层是两组SOS2近似：一组用于库容-水位，一组用于总泄量-尾水位。脚本采用“λ和y邻接约束”实现SOS2等价形式：所有λ非负且和为1，且最多只有相邻两段可以同时起作用。这样既保持曲线分段特性，又不引入非线性。第三层是运行规则约束，包括防洪上限软约束、生态下限软约束、机组爬坡约束与末库容约束。软约束通过惩罚系数进入目标函数，体现“可行优先、代价最小”的工程思想。  
目标函数本质是多目标加权：鼓励有效下泄（近似发电收益），惩罚弃水、超汛限、生态缺额和末库容偏差。因为直接把“流量×水头”写进MILP会形成双线性项，脚本将发电收益在优化阶段做线性近似，求解后再依据上游水位、尾水位和损失水头计算真实出力与电量作为KPI评估。这个“优化内近似、优化后校核”的流程在工程调度中很常见，能够平衡求解速度与物理解释性。  
输出部分分两层：先打印KPI表格，再画三幅图。KPI表覆盖水量平衡、防洪安全、生态保障和发电绩效，便于论文或教材直接引用。图形分别展示流量过程、库容控制与水头-出力响应，能直观看到策略在洪峰来临前后的调度动作。整体上，这份代码具备“参数集中、模型清晰、结果可解释”的教学与工程双重属性，适合作为《水库调度优化与决策》章节配套仿真模板。
