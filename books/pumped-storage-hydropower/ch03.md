# 第3章 调频/调峰/旋转备用策略

## 导读

在深入探讨变速抽水蓄能机组的具体控制技术之后，我们必须将视线从单一设备层面提升至整个电力系统的运行维度。第2章详细解析了变速机组在变频器调节下的全工况灵活控制原理，这种底层硬件和控制算法的突破，赋予了机组卓越的动态响应能力。然而，任何灵活性资源的价值都无法脱离其所服务的系统而独立存在。机组层面的控制能力必须与宏观电力系统的实际需求高度契合，才能真正发挥抽水蓄能电站作为“超级充电宝”和“电网稳定器”的枢纽作用。特别是在以风电、光伏为主导的新型电力系统中，电源侧的随机性、波动性与负荷侧的复杂变化交织，系统对调节资源的需求呈现出多时间尺度、多维度的特征。

电力系统对抽水蓄能电站的需求，可以精确地归结为三项核心辅助服务，它们分别对应着不同的时间尺度与物理过程：第一是调频（Frequency Regulation），主要在秒至分钟的微观时间尺度上发挥作用，旨在平抑由于发用电功率瞬时供需不平衡引起的系统频率偏差，维持交流电网的同步稳定；第二是调峰（Peak Shaving），作用于小时至日的时间尺度，目标是平衡由于人类作息规律和气象条件变化导致的日负荷曲线巨大峰谷差，实现电能的跨时段转移；第三是旋转备用（Spinning Reserve），这是一种应急状态下的防御机制，要求机组在电网发生发电机组跳闸、特高压直流闭锁等突发严重故障时，能够快速释放预留的功率容量，提供坚实的应急功率支撑，防止系统崩溃。

本章将系统且深入地剖析这三项核心辅助服务。我们将从物理机制出发，分别建立调频、调峰与旋转备用策略的严密数学模型，并推导其控制律。随后，我们将详细分析各类策略在实施过程中对水轮泵水力特性、电机电气特性以及电站整体运行边界的约束影响。在此基础上，本章将构建含高比例新能源的区域电力系统全景仿真平台，通过多场景的数据对比，定量评估抽水蓄能电站在多重辅助服务联合优化下的综合性能与经济效益。

## 3.1 调频服务策略

电力系统的频率是全网统一的全局性变量，它直接反映了系统内所有发电机机械输入功率与所有负荷电气消耗功率之间的实时平衡关系。当供需出现毫秒级或秒级的失配时，系统频率将偏离其额定值（如我国电网的50 Hz）。抽水蓄能电站凭借其快速的响应特性，在多级调频体系中扮演着不可替代的角色。

### 3.1.1 一次调频与下垂控制

一次调频是电力系统抵御频率波动的首道防线。当电网负荷突然增加或某台发电机组非计划停运时，系统总发电功率小于总负荷功率，所有并网的同步发电机转子会自发地释放旋转动能以弥补功率缺额，导致转子转速下降，宏观表现即为系统频率的降低。为阻止频率持续跌落，抽水蓄能机组的调速器必须根据频率偏差自动增加导叶开度，提升水轮机的机械功率输出。

这一自动响应过程遵循下垂（Droop）控制策略，即机组的有功功率调节量 $\Delta P$ 与系统频率偏差 $\Delta f$ 之间保持严格的负比例关系。其数学表达式为：

$$\Delta P = -\frac{1}{R} \Delta f$$

其中，$R$ 为调差系数（Droop coefficient）。在工程应用中，为了便于不同容量机组之间的参数整定与比较，$R$ 通常以标幺值或百分比形式表示。其物理意义为：当系统频率发生 1 pu（即100%）的偏差时，机组有功功率按比例变化的标幺值。计算公式可以展开为：

$$R\% = \frac{\Delta f / f_N}{\Delta P / P_N} \times 100\%$$

抽水蓄能电站的典型调差系数设定范围为 $R = 0.04 \sim 0.05$（即4%至5%）。以 $R = 0.04$ 为例，这意味着当系统频率下降1%（即0.5 Hz）时，机组将增加等于其额定容量25%的出力。下垂控制的核心逻辑在于通过引入稳态误差来实现多台并联运行机组间的负荷自动分配。根据控制理论，如果所有机组均采用无差调节（PI控制中的积分环节），它们将为了争夺对统一频率的主导权而发生严重的功率振荡。下垂特性确保了各机组按照自身容量的比例分担系统的功率缺额。

假设某抽水蓄能电站单机额定容量 $P_N = 300$ MW，额定频率 $f_0 = 50$ Hz，调差系数 $R = 0.04$。当系统负荷突增导致频率从 50.00 Hz 跌落至 49.90 Hz 时，频率偏差为 $\Delta f = -0.1$ Hz。此时该机组的一次调频功率调节量理论值为：

$$\Delta P = \frac{P_N}{R \cdot f_0} \cdot (-\Delta f) = \frac{300}{0.04 \times 50} \times 0.1 = 15 \text{ MW}$$

在实际控制回路中，为了避免系统微小频率波动引起导叶频繁动作从而加剧机械磨损，控制器内部会设置一个频率死区（Deadband），通常取值为 $\pm 0.033$ Hz 或 $\pm 0.05$ Hz。只有当频率偏差突破这一死区边界时，调速器的电液转换器和主配压阀才会动作，驱动接力器改变导叶开度。此外，受限于水轮机引水钢管的“水锤效应”（水流惯性导致的水压反向变化），水力系统呈现出非最小相位特性，这就要求PID调速器参数必须进行精细化整定，以防止一次调频响应过程出现反调（功率短暂朝着相反方向变化）甚至失稳。

### 3.1.2 二次调频（AGC）

一次调频虽然能够迅速遏制频率的恶化，但由于下垂特性的固有属性，系统最终将在一个偏离额定值（如49.85 Hz）的新稳态平衡点运行，这被称为有差调节。为了彻底消除这一稳态频率偏差，并确保相邻区域电网之间的联络线交换功率维持在日前交易的计划值，必须引入二次调频。二次调频是一个集中式的闭环控制过程，主要由调度中心的自动发电控制（Automatic Generation Control, AGC）系统统筹执行。

AGC系统的核心在于计算并处理区域控制偏差（Area Control Error, ACE）。对于互联电网中的任意一个控制区域，其ACE的物理模型定义为该区域对整个互联系统造成的功率不平衡总量，公式推导如下：

$$\text{ACE} = \Delta P_{\text{tie}} + B \cdot \Delta f$$

其中，$\Delta P_{\text{tie}} = P_{\text{tie, actual}} - P_{\text{tie, scheduled}}$ 代表该区域所有跨区联络线实际交换功率与计划交换功率的差值（规定流出为正）。$\Delta f = f_{\text{actual}} - f_0$ 为系统频率偏差。参数 $B$ 被称为频率偏差系数或自然频率响应系数（单位通常为 MW/Hz，且 $B > 0$）。$B$ 值的精确设定直接关系到AGC系统的稳定性。若 $B$ 设定等于该区域的实际综合一次调频能力与负荷频率调节效应之和，ACE就能准确反映本区域内部发生的功率扰动量。如果某区域内部发生发电机跳闸，该区域的 $\Delta f < 0$，同时由于相邻区域通过联络线向其输送事故支援功率，导致 $\Delta P_{\text{tie}} < 0$。此时ACE为较大的负值，AGC系统将向该区域内的机组发送增发功率的指令。

AGC分配给抽蓄电站的调节指令 $P_{\text{AGC}}$ 是通过优化算法下发的。现代经济调度不仅考虑机组的发电成本，更看重调节品质。机组响应AGC指令的过程并非瞬间完成，其功率变化速率严格受到物理约束。对于抽蓄机组而言，发电工况下的最大功率变化率通常受限于导叶的开启速度与水压的脉动极限。定速机组的响应速率大约为额定功率的3%至5%/s，而采用全功率变频器或双馈感应电机（DFIG）的变速抽蓄机组，由于通过电力电子装置实现了电磁功率与机械功率的解耦，其电气响应极快，整体响应速率可提升至5%至8%/s。

电力市场中衡量机组响应AGC质量的核心指标是调频性能指标（Regulation Performance Score, RPS），有时也称为综合性能指标 $K$ 值。该指标综合考量了机组响应的调节速率（$K_1$）、响应精度（$K_2$）和响应延迟时间（$K_3$）。抽蓄电站由于水电机组启停快、爬坡率高，能够精准跟踪调度中心下发的动态测试信号（如PJM市场的RegD快速调节信号），其RPS评分往往远高于传统的燃煤火电机组，从而在辅助服务市场中获取更高的里程补偿收益。

### 3.1.3 综合惯量响应

随着风力发电与光伏发电在系统中渗透率的不断攀升，电网中大量通过电力电子逆变器并网的新能源机组取代了传统的同步发电机。这些逆变器设备大多工作在最大功率点跟踪（MPPT）模式或定功率输出模式，其转子转速（对于风机而言）或内部电压频率与电网的系统频率完全解耦，因此无法像同步机那样自然地释放旋转动能，导致系统的整体物理惯量急剧下降。低惯量系统在面临功率缺额扰动时，频率变化率（Rate of Change of Frequency, ROCOF）会大幅增加，频率天底（Frequency Nadir）更低，极易触发低频减载保护（UFLS），引发大面积停电。

变速抽水蓄能机组的引入为解决这一难题提供了优异的方案。尽管变速机组同样通过变频器与电网相连，但利用现代控制理论，可以在变频器的有功功率控制环中增加附加的频率微分负反馈环节，人为地模拟同步发电机的惯量响应特性，这一技术被称为虚拟惯量控制（Virtual Inertia Control）。

根据经典同步发电机的转子运动方程（摇摆方程）：

$$2H \frac{d\omega}{dt} = P_m - P_e$$

其中 $H$ 为发电机转子固有的惯量时间常数，$P_m$ 为机械输入功率，$P_e$ 为电磁输出功率。为了使变速机组表现出类似的动态特性，控制系统在检测到系统频率发生瞬变时，强制变频器吸收或释放有功功率，其附加指令 $\Delta P_{\text{inertia}}$ 设计为：

$$\Delta P_{\text{inertia}} = -2H_{\text{virtual}} \frac{d(\Delta f)}{dt}$$

公式中的 $H_{\text{virtual}}$ 为虚拟惯量时间常数。这并非一个固定的物理参数，而是可以通过修改控制软件实时调整的增益系数。工程实践中，$H_{\text{virtual}}$ 的设定值通常可达到该机组物理结构惯量的1.5倍至3倍。当系统频率突降时（$d(\Delta f)/dt < 0$），$\Delta P_{\text{inertia}}$ 为正，变频器瞬间增加向电网注入的有功功率。这部分多发出的电能来源于机组巨大的转子旋转动能（$E_k = \frac{1}{2} J \omega_r^2$），在此过程中机组转速会短暂下降。

虚拟惯量响应的优势在于其纯电气控制特性，响应延迟极短。在系统扰动发生后的 0.1至0.5 秒内，变频器即可调整电流输出完成功率支撑。这远远快于依赖机械液压系统动作的常规调速器一次调频响应（后者通常需要1至2秒的延迟才能看到明显的功率爬升）。大量实测数据表明，在这关键的最初两秒钟内，虚拟惯量注入的能量起到了“踩刹车”的作用，极大地平抑了频率跌落的速率，对抬高频率天底发挥了决定性的作用。需要强调的是，虚拟惯量控制释放的是转子有限的动能，它不能持续提供稳态功率支持，必须与随后赶到的一次调频和AGC控制实现无缝、平滑的交接与配合，否则在转速恢复阶段会引起频率的二次跌落。

## 3.2 调峰服务策略

如果说调频是维持系统秒级平稳的“微操”，那么调峰则是应对电力系统宏观能量失衡的战略调度。受工业生产和居民生活规律的影响，电力负荷在一天内存在明显的早高峰、晚高峰和深夜低谷。而光伏在正午出力最大，风电则常在夜间负荷低谷时迎来大发（逆负荷特性）。这种源荷严重错配不仅降低了系统运行效率，更导致了严重的“弃风弃光”现象。抽水蓄能电站通过在低谷时段充当巨型用电负荷（抽水上抬水位），在高峰时段充当主力发电源（放水发电），实现了电能在时间轴上的大规模平移。

### 3.2.1 日运行模式优化

抽水蓄能电站的调峰运行本质上是一个带有多重硬性物理约束的多时段日前调度优化问题（Day-ahead Scheduling Optimization）。为建立数学模型，通常将一天的调度周期划分为 $T$ 个离散的时间段（按照1小时分辨率则 $T = 24$，按照15分钟分辨率则 $T = 96$）。设定决策变量为电站在每个时段 $t$ 的发电功率 $P_g(t)$ 和抽水功率 $P_p(t)$。

在成熟的电力现货市场或峰谷分时电价机制下，电站作为独立的市场主体，其首要经济目标是最大化全天的电能量套利收益，即峰谷电价差收益。目标函数可严谨地表述为：

$$\max \sum_{t=1}^{T} \left[ \lambda(t) \cdot P_g(t) - \lambda_p(t) \cdot P_p(t) \right] \cdot \Delta t$$

式中，$\lambda(t)$ 为时段 $t$ 的电网结算电价，$\lambda_p(t)$ 为抽水时段的购电成本（在某些市场结构中这两者可能统一为节点边际电价 LMP），$\Delta t$ 为单个调度时段的时间长度。

为了确保给出的调度指令在物理上是完全可行且安全的，模型必须囊括一系列严格的约束条件：

首先是**库容与水位约束**。上下水库的物理容积限制了电站储存电能的上限和下限。令 $E(t)$ 表示时段 $t$ 初始时刻上水库蕴含的等效蓄能量（单位为 MWh）：
$$E_{\min} \leq E(t) \leq E_{\max}$$
为了防止机组发生汽蚀或水库漫坝，$E_{\min}$ 对应于规定的死水位，$E_{\max}$ 对应于正常蓄水位。

其次是**能量平衡状态方程**，它描述了水库蓄能量随调度决策的动态演化过程：
$$E(t+1) = E(t) + \eta_p P_p(t) \Delta t - \frac{P_g(t)}{\eta_g} \Delta t + E_{\text{inflow}}(t) - E_{\text{spill}}(t)$$
此处，$\eta_p$ 为抽水过程的综合效率（电能转化为水势能的比例），$\eta_g$ 为发电过程的综合效率（水势能转化为电能的比例）。典型的抽水蓄能电站往返综合效率 $\eta_{\text{cycle}} = \eta_p \cdot \eta_g$ 通常在 75% 至 82% 之间。这表明每储存并释放 1 kWh 的电能，系统将发生约 0.25 kWh 的能量损耗。$E_{\text{inflow}}(t)$ 代表可能的天然径流入库量（对于纯抽水蓄能电站该项为零），$E_{\text{spill}}(t)$ 为弃水量。同时，为了保证第二天调度的连续性，通常还需设定始末状态约束：$E(T+1) = E(1)$。

第三是**机组出力边界约束**。任何机组的运行工况都有明确的上下限限制：
$$0 \leq P_g(t) \leq P_g^{\max}(H), \quad 0 \leq P_p(t) \leq P_p^{\max}(H)$$
此处特别标注了上限值是随水头 $H$ 变化的函数。随着上水库水位下降，实际工作水头降低，机组能发出的最大有效功率会随之缩减，这一水力学非线性特性在精细化调度中必须予以考虑。

最后是**工况互斥约束**。由于同一台机组在同一时段不可能既作为水泵又作为水轮机运行，因此必须施加互斥条件：
$$P_g(t) \cdot P_p(t) = 0$$
在实际求解时，这一非线性约束常常通过引入二进制辅助变量转换为线性形式，从而方便利用标准优化求解器处理。

### 3.2.2 机组组合决策

在现代大型抽水蓄能电站中，通常配置多台机组（如4台或6台300 MW机组）。此时，上述简单的总量调度问题必须拓展为更为复杂的机组组合（Unit Commitment, UC）问题。调度系统不仅要确定电站总的出力曲线，还要精确决策每个具体时段中各台机组的具体状态（开机还是停机、发电还是抽水）以及它们之间的负荷分配。

引入二维二进制状态变量 $u_g(i,t)$ 和 $u_p(i,t)$，分别表示第 $i$ 台机组在第 $t$ 时段是否处于发电或抽水状态（1为是，0为否）。在此细化的模型下，必须额外增加与设备健康和运维成本密切相关的约束：

1. **最小运行时间与最小停机时间约束**：机组一旦启动或停止，必须保持该状态一定的最小时间，以防止金属部件因温度剧烈交变而产生疲劳裂纹。
2. **启停次数约束**：定频机组在水泵工况启动时，采用SFC（静止变频器）拖动或背靠背启动，这一过程对转子冲击极大。因此每天的启停总次数被严格限定。
3. **启停成本**：每次启动机组都会产生一定的燃料（若是抽蓄则为额外的损耗和折旧费）成本，这被计入总的目标函数之中。

机组组合问题在数学上属于混合整数线性规划（Mixed Integer Linear Programming, MILP）问题。在包含海量节点和机组的省级电网层面求解此问题往往面临“维度灾难”。工业界普遍采用分支定界法（Branch and Bound）结合割平面法的商用求解器（如Gurobi或CPLEX）进行求解；而在学术研究中，为了处理抽蓄机组独有的非线性水头损失与效率曲线，有时也会采用改进的动态规划（Dynamic Programming）或基于拉格朗日松弛（Lagrangian Relaxation）的启发式算法。

## 3.3 旋转备用策略

即使在调度计划做得再完美无缺的情况下，电力系统依然面临着诸多不可预见的随机风险，例如特高压直流输电线路突然发生双极闭锁导致数千兆瓦功率输送中断，或者主干网内大型核电机组发生紧急停堆。为了应对这种小概率但高风险的N-1甚至N-2严重故障，系统必须在日常运行中预留足够的旋转备用（Spinning Reserve）。

旋转备用是指那些已经与电网保持同步并网运行，但当前输出功率尚未达到额定满载的机组。当调度中心下达紧急指令时，这些机组能够在极短的时间内（通常在10分钟以内，快速备用则要求在1分钟内）迅速增加其实际有功出力。抽水蓄能电站由于其水轮机转轮响应速度极快、不涉及复杂的热力学循环过程，被公认为系统中质量最高、动作最可靠的旋转备用资源。

抽水蓄能电站在实际运行中，可以通过以下三种截然不同的状态模式来提供旋转备用服务：

**（1）发电部分负荷运行模式**
当机组处于发电状态但并未满载时（如出力200 MW，额定300 MW），其能够提供的上调备用容量为 $R_{\text{spin}} = P_g^{\max} - P_g(t)$。在此状态下，机组转速已与电网同步，只需伺服机构驱动导叶进一步开启，增加水流流量即可增加功率。由于水流不可压缩且具有较大惯性，导叶开启速度受到水压上升率的严格限制，通常从部分负荷攀升至满载需要 10 至 30 秒。这属于典型的秒级快速备用。

**（2）抽水满负荷运行模式（专属于变速机组）**
传统的定速抽蓄机组在抽水工况下只能以满功率运行，无法调节。但变速机组由于采用了交流励磁技术，可以改变转子转速，从而在抽水工况下实现一定范围（通常为额定功率的70%至100%）的连续功率调节。更关键的是，当电网发生严重功率缺额时，处于抽水状态的变速机组可以通过逆变器控制迅速降低抽水负荷（例如从 300 MW 降至 210 MW），这相当于向系统间接注入了 90 MW 的有功功率。电气功率的响应时间极短，仅需 2 到 5 秒即可完成，这为阻止频率深度跌落赢得了极其宝贵的喘息时间。在此过程中，导叶随之缓慢关闭以匹配新的流量工况。

**（3）旋转凝相运行模式（Synchronous Condenser Mode）**
这是抽水蓄能电站独有且极其重要的一种运行方式。在此模式下，机组转子与电网保持同步旋转，但水轮机转轮室内的水被高压压缩空气压低至转轮下方。转轮在空气中空转，极大程度地消除了水流摩擦阻力。此时机组不发出有功功率，仅从电网吸收极少量（约额定容量的1%至2%）的有功功率以维持轴承摩擦和通风损耗，同时向电网提供大量无功功率以支撑电压。一旦接收到备用调用指令，系统迅速开启排气阀释放压缩空气，上游高压水流瞬间充满转轮室，导叶随即迅速打开。从空转状态加载至满负荷发电状态，通常在 60 至 90 秒内即可完成。虽然时间略长于第一种模式，但凝相运行能够释放出机组 100% 的容量作为备用，具有无与伦比的容量优势。

在现代电力市场中，上述三种状态对应的备用容量 $R_{\text{spin}}$ 与其响应时间 $t_{\text{resp}}$ 共同构成了备用产品质量的两个维度。响应速度越快、确定性越高的备用资源，在市场清算中获得的容量补偿价格就越高。这也促使电站在制定策略时，必须将旋转备用与日常调峰进行联合优化。

## 3.4 仿真设置

为了定量评估抽水蓄能电站在多重辅助服务联合优化策略下的实际效能，本节利用时域仿真工具，构建了一个包含高比例新能源的区域级互联电力系统综合评价模型。

**系统基础参数设置：**
- **常规电源**：配置4台参数一致的燃煤火电机组，总装机容量 2000 MW（单机 500 MW）。火电机组的爬坡率限制为额定容量的 1.5%/min，最小技术出力设定为额定容量的 40%（即单机 200 MW，低于此值需投油助燃，成本急剧增加）。其燃料成本曲线采用典型的二次多项式拟合。
- **新能源电源**：风电场总装机容量为 800 MW，占系统总装机容量的 25% 左右，属于典型的高比例新能源渗透场景。风电出力曲线采用西北某大型风电场春季典型日的实测历史数据，时间分辨率为 15 分钟。该曲线具有显著的反调峰特性（夜间22:00至凌晨4:00大发，正午期间出力较低）。
- **负荷特性**：系统总日负荷曲线取自华东电网春季典型工作日数据。日最大负荷（峰值）达到 3000 MW，出现在傍晚 19:00；日最小负荷（谷值）为 1875 MW，出现在凌晨 3:00。日峰谷差比达到1.6，系统调峰压力巨大。
- **抽水蓄能电站**：模拟一座配备4台全功率变速机组的大型抽水蓄能电站。单机额定发电功率 300 MW，额定抽水功率 320 MW。电站总发电装机 1200 MW。上水库正常蓄水位与死水位的有效水位落差为 50 米，有效运行库容为 200 万立方米，对应最大可连续发电时间约为 2.25 小时，总等效蓄能量约为 2700 MWh。机组综合往返效率设定为 75%。

**仿真场景与时段划分：**
整个调度仿真周期设定为 24 小时（0:00 - 24:00）。对于调峰与机组组合优化求解，时间步长设定为 15 分钟，全天共 96 个调度时段；对于涉及系统动态频率响应的AGC与一次调频仿真，在重点扰动时段截取样本，采用 1 秒甚至 0.1 秒的微观步长进行电磁暂态与机电暂态混合仿真。

**价格机制设定：**
峰谷电价机制采用三段式：高峰时段（9:00-12:00, 17:00-21:00）电价为 1.20 元/kWh；平段电价为 0.60 元/kWh；低谷时段（23:00-次日7:00）电价为 0.30 元/kWh。辅助服务市场中，调频容量价格设定为 15 元/MW·h，旋转备用容量价格设定为 10 元/MW·h。

**测试用例规划：**
仿真将严格对比三种不同策略配置下的系统表现：
场景（a）：单一调峰模式（仅以峰谷套利为目标）。
场景（b）：调峰+调频联合模式（响应AGC指令并提供一次调频死区设置）。
场景（c）：调峰+调频+旋转备用全方位综合优化模式（加入机组凝相运行调度）。

## 3.5 结果分析

针对上述仿真设置进行全周期演算后，得到了一系列具有深度工程参考价值的数据指标与时序曲线。

**第一，调峰效果的宏观数据解读**。
在场景（a）单纯追逐峰谷电价差的调度策略驱动下，抽水蓄能电站完美展现了其“削峰填谷”的搬运工角色。在0:00至7:00的低谷时段，电网负荷降至最低且风电处于大发期。若无抽蓄电站，负荷扣除风电后的净负荷仅为 1100 MW左右，远低于4台火电机组全部保持最小出力的总和（800 MW，假设不考虑深度调峰改造）。这将不可避免地导致大规模“弃风”。在模型中，抽水蓄能电站在此时段启动了全部4台机组进入满载抽水工况，吸收了约 1200 MW 的冗余电力。整个低谷期累计消耗抽水电量高达 5400 MWh。
随后，在9:00至12:00及17:00至21:00的两个用电高峰期，电网净负荷飙升。抽蓄电站转换为发电工况，4台机组并网放水，累计贡献发电量约 4050 MWh。通过能量的跨时段转移，系统等效峰负荷被大幅削减，日等效峰谷差从原始的 1200 MW 剧降至 780 MW，削峰率高达 35%。从经济账来看，电站在低谷期支付购电费 162万元（$5400 \times 0.3$），在高峰期实现售电收入 486万元（$4050 \times 1.2$），仅凭能量套利便可实现单日毛利润 324 万元。

**第二，调频性能的微观动态解析**。
在场景（b）中，为验证机组控制策略的有效性，我们在11:00时刻人为注入了一个严重的功率扰动：风电场受突发阵风影响，输出功率在 15 秒内直线下降了 120 MW。在仅依靠常规火电机组提供调频响应的对照组中，由于汽轮机锅炉热力系统的固有迟延，系统频率出现了深度的“V”型跌落，最低点（频率天底）探底至 49.72 Hz，并经历了长达 38 秒的振荡才缓慢回升至稳态允许范围内。
而当变速抽水蓄能电站介入后，由于采用了虚拟惯量控制策略，在风电出力下降后的仅仅 0.3 秒，变频器便迅速响应频率的变化率，释放转子动能，向电网强劲注入了最高达 45 MW 的短时功率支撑。这使得系统频率跌落的斜率显著放缓，最终频率天底被有效抬高至 49.85 Hz，避免了触碰低频减载定值。随后，通过AGC的二次调频接力，系统频率仅用 18 秒即恢复至 50 Hz 附近。评估数据显示，全天域内抽蓄电站的综合调频性能指标（CPS1）高达 98.2%，全面超越火电机组均值 92.5%，展现了极为优越的动态调节品质。

**第三，旋转备用价值的综合量化与经济性测算**。
在场景（c）的综合优化模式下，模型展现出更为复杂的博弈决策。优化算法主动选择在10:00至15:00的日间发电时段内，不让所有机组满载运行，而是保留了一台机组以 150 MW 的部分负荷状态运行，专门留出 150 MW 的容量作为旋转备用。显然，这种策略牺牲了一部分在高峰电价时段发电卖钱的机会，导致单纯的调峰收益下降了约 3%（约损失 10 万元）。然而，这 150 MW 的高品质旋转备用容量在辅助服务市场中获得了可观的备用容量费补偿（约合 15 万元）。综合核算下来，提供旋转备用服务不仅弥补了发电量减少的损失，反而使得全天的总综合收益较单一调峰模式提升了约 8%。
在仿真中，我们模拟了14:00时刻一台 500 MW 火电机组突然跳闸的极端 N-1 故障。由于留有备用预案，抽蓄电站内3台处于部分负荷的机组迅速增加导叶开度，同时一台处于凝相空转模式的备用机组紧急排气充水加载。整个电站在短短 12 秒内向系统增发了 450 MW 的紧急功率，将原本可能跌破 49.00 Hz 导致电网解列的严重事故，强行遏制在 49.65 Hz 的安全阈值之上。这种化险为夷的价值，是单纯的经济账所无法完全衡量的。

## 3.6 工程启示

本章阐述的控制策略与数学模型并非束之高阁的理论推导，它们正在现代大型电网的工程实践中发挥着不可替代的作用。从理论模型向工程应用转化时，有几个深层次的实践认识必须引起规划与调度人员的高度重视。

第一，调峰、调频和旋转备用这三项辅助服务之间，在物理容量上存在着天然的强耦合与严酷竞争关系。这种竞争在有限的库容量面前尤为突出。分配给旋转备用的可调容量，注定无法同时用于最大功率的发电调峰；而频繁地响应AGC调频指令不仅会消耗额外的水库蓄水量，更会成倍增加水泵水轮机组的开停机及工况转换次数，加剧止水密封和转轮叶片的疲劳损伤。例如，美国PJM市场的 Bath County 抽水蓄能电站（总装机3000 MW），其日常调度就是在能源套利与响应高度频繁的 RegD 调频信号之间进行极度复杂的概率优化平衡。多服务联合优化（Co-optimization），利用运筹学方法在动态边界下寻找效益最大化的帕累托最优解，是电网实际调度控制系统中每天必须实时计算的核心难题。

第二，变速抽蓄机组的虚拟惯量响应技术虽然在物理机制上非常先进且响应极快，但工程师在应用时必须对其局限性保持清醒的认识。虚拟惯量释放的能量本质上来源于机组巨大转子系统的旋转动能。当转速因释放能量而下降到安全边界限制（例如额定转速的90%）时，变频器必须强制退出惯量支撑并进入转速恢复阶段。这就意味着惯量支撑的持续时间非常短暂（通常只有 3 到 5 秒钟）。如果在虚拟惯量退出的关口，机组的水力导叶动作尚未跟上，或是系统内其他机组的一次调频尚未大规模发力接续，系统频率将不可避免地遭遇二次跌落（Secondary Frequency Dip），这种二次跌落有时甚至比初次跌落更具破坏性。因此，电气控制策略与机械水力特性的平滑协同控制，是该技术落地的核心技术壁垒。

第三，在宏观层面的工程经济性评价上，必须正视抽水蓄能电站固有的能量损耗特性。如前文所述，其综合效率通常在 75% 至 82% 之间。这意味着电站每向电网送出 1 kWh 的平稳电能，在抽水环节必须消耗掉 1.22 至 1.33 kWh 的低谷电能。这种约四分之一的能量“蒸发”，决定了抽水蓄能的调峰经济性高度甚至绝对依赖于电网的峰谷电价差空间。以我国河北丰宁抽水蓄能电站（装机规模3600 MW，世界第一）为例，该电站全面接入张北柔性直流电网，其主要任务是平抑张北地区海量风电和光伏的剧烈波动并为首都电网提供备用。在缺乏完善的现货市场电价波动的背景下，完全依靠能量套利很难收回巨额的建设投资。因此，在电力市场化改革不断深化的今天，建立健全包括两部制电价（容量电价+电量电价）和辅助服务独立补偿机制在内的合理价格体系，才是确保此类大型基础设施得以持续运营、健康发展的根本保障。

## 本章小结

本章自底向上、系统性地构建了抽水蓄能电站参与电力系统调频、调峰和旋转备用三项核心辅助服务的数学模型与工程控制策略。在调频层面，深入解析了一次调频下垂控制、二次AGC闭环调频的过程，并重点推导了变速机组应对低惯量挑战的虚拟惯量响应机制；在调峰层面，将电站的日运行抽象为一个涵盖多重水力与电气约束、以综合效益最大化为目标的日前混合整数规划问题；在旋转备用层面，则剖析了部分负荷、抽水降功率及凝相运行等不同模式在响应速度与容量提供上的差异。通过构建多源互联系统的全景仿真测试，用翔实的数据证明了多服务联合优化策略在提升削峰填谷效果、改善频率动态品质并增加电站总体收益方面的显著优势，充分论证了其在保证系统安全与提升经济性中的核心价值。下一章将改变单一储能视域，进一步探讨抽水蓄能电站如何与风电、光伏等间歇性新能源机组组成虚拟电厂，开展更为复杂的联合多目标调度控制模型研究。

# -*- coding: utf-8 -*-
"""
第3章：抽水蓄能调频 / 调峰 / 旋转备用一体化仿真
依赖：numpy, scipy, matplotlib

import numpy as np
import matplotlib.pyplot as plt
from scipy.optimize import minimize, LinearConstraint, Bounds


def build_profiles(n, dt_h, seed=42):
    rng = np.random.default_rng(seed)
    h = np.arange(n) * dt_h  # hour

    # 负荷与新能源出力（示例日曲线）
    load = (
        780
        + 110 * np.sin(2 * np.pi * (h - 7) / 24)
        + 80 * np.sin(4 * np.pi * (h - 18) / 24)
        + 45 * np.exp(-0.5 * ((h - 10) / 2.2) ** 2)
        + 120 * np.exp(-0.5 * ((h - 20) / 2.0) ** 2)
    )
    wind = 160 + 45 * np.sin(2 * np.pi * (h + 2) / 24) + rng.normal(0, 10, n)
    wind = np.clip(wind, 70, 260)
    solar = 230 * np.maximum(0.0, np.sin(np.pi * (h - 6) / 12)) ** 1.7

    net0 = load - wind - solar  # 抽蓄动作前净负荷

    # 备用需求：与净负荷爬坡和新能源出力相关
    ramp = np.abs(np.gradient(net0, dt_h))
    r_up = 70 + 0.18 * ramp + 0.08 * (wind + solar)
    r_dn = 50 + 0.12 * ramp
    r_up = np.clip(r_up, 60, 180)
    r_dn = np.clip(r_dn, 40, 140)

    return h, load, wind, solar, net0, r_up, r_dn


def optimize_schedule(net0, r_up, r_dn, cfg):
    n = len(net0)
    dt_h = cfg["dt_h"]
    p_gen_max = cfg["p_gen_max"]
    p_pump_max = cfg["p_pump_max"]
    eta_d = cfg["eta_d"]
    eta_c = cfg["eta_c"]
    e0 = cfg["e0"]
    e_min = cfg["e_min"]
    e_max = cfg["e_max"]
    lam = cfg["lambda_throughput"]
    ramp_limit = cfg["ramp_limit_per_step"]

    # x = [p_gen(0..n-1), p_pump(0..n-1)]
    target = np.mean(net0)

    def objective(x):
        p_gen = x[:n]
        p_pump = x[n:]
        net = net0 - p_gen + p_pump
        track = np.sum((net - target) ** 2)
        throughput = np.sum((p_gen + p_pump) ** 2)
        return track + lam * throughput

    def grad(x):
        p_gen = x[:n]
        p_pump = x[n:]
        net = net0 - p_gen + p_pump
        g_gen = -2 * (net - target) + 2 * lam * (p_gen + p_pump)
        g_pump = 2 * (net - target) + 2 * lam * (p_gen + p_pump)
        return np.concatenate([g_gen, g_pump])

    # SOC 轨迹约束
    L = np.tril(np.ones((n, n))) * dt_h
    A_soc = np.hstack(((-1.0 / eta_d) * L, eta_c * L))
    lb_soc = np.full(n, e_min - e0)
    ub_soc = np.full(n, e_max - e0)

    # 日终回到初始SOC
    A_end = np.hstack(
        ((-dt_h / eta_d) * np.ones(n), (eta_c * dt_h) * np.ones(n))
    ).reshape(1, -1)

    # 上/下旋转备用约束
    # R_up = p_gen_max - p_gen + p_pump >= r_up
    # R_dn = p_pump_max - p_pump + p_gen >= r_dn
    I = np.eye(n)
    A_up = np.hstack((-I, I))
    A_dn = np.hstack((I, -I))

    # 抑制同时充放电（线性近似约束）
    A_overlap = np.hstack((I, I))
    overlap_cap = max(p_gen_max, p_pump_max)

    # 功率爬坡约束（对签约功率 p = p_gen - p_pump）
    D = np.zeros((n - 1, n))
    for i in range(n - 1):
        D[i, i + 1] = 1.0
        D[i, i] = -1.0
    A_ramp = np.hstack((D, -D))

    constraints = [
        LinearConstraint(A_soc, lb_soc, ub_soc),
        LinearConstraint(A_end, [0.0], [0.0]),
        LinearConstraint(A_up, r_up - p_gen_max, np.full(n, np.inf)),
        LinearConstraint(A_dn, r_dn - p_pump_max, np.full(n, np.inf)),
        LinearConstraint(A_overlap, np.zeros(n), np.full(n, overlap_cap)),
        LinearConstraint(
            A_ramp,
            np.full(n - 1, -ramp_limit),
            np.full(n - 1, ramp_limit),
        ),
    ]

    lb = np.zeros(2 * n)
    ub = np.concatenate([np.full(n, p_gen_max), np.full(n, p_pump_max)])
    bounds = Bounds(lb, ub)

    x0 = np.zeros(2 * n)
    res = minimize(
        objective,
        x0,
        jac=grad,
        method="SLSQP",
        bounds=bounds,
        constraints=constraints,
        options={"maxiter": 600, "ftol": 1e-7, "disp": False},
    )
    if not res.success:
        raise RuntimeError(f"优化失败: {res.message}")

    p_gen = res.x[:n]
    p_pump = res.x[n:]
    p_sched = p_gen - p_pump
    soc = e0 + L @ (eta_c * p_pump - p_gen / eta_d)

    return p_gen, p_pump, p_sched, soc


def simulate_frequency(p_sched, cfg, seed=7):
    rng = np.random.default_rng(seed)
    n_slot = len(p_sched)
    slot_sec = int(cfg["dt_h"] * 3600)
    n_rt = n_slot * slot_sec

    p_gen_max = cfg["p_gen_max"]
    p_pump_max = cfg["p_pump_max"]

    # 简化系统频率模型参数
    M = cfg["M"]      # 惯量参数
    D = cfg["D"]      # 负荷阻尼
    Kp = cfg["Kp"]    # 一次调频增益
    Ki = cfg["Ki"]    # 二次调频增益

    f = np.zeros(n_rt)       # Hz偏差（相对50Hz）
    p_reg = np.zeros(n_rt)   # 调频附加功率
    p_act = np.zeros(n_rt)   # 实际抽蓄功率(计划+调频)
    integ = 0.0

    # 每5分钟一个慢变预测误差 + 秒级快扰动
    slot_err = rng.normal(0, 18, n_slot)

    for i in range(n_rt - 1):
        s = min(i // slot_sec, n_slot - 1)
        t = float(i)

        fast_noise = (
            8.0 * np.sin(2 * np.pi * t / 90.0)
            + 5.0 * np.sin(2 * np.pi * t / 23.0)
            + rng.normal(0, 4.0)
        )
        delta_p = slot_err[s] + fast_noise  # 系统功率失衡(MW)

        integ = np.clip(integ + f[i], -0.08, 0.08)
        cmd = -Kp * f[i] - Ki * integ

        up_headroom = p_gen_max - p_sched[s]
        down_headroom = p_pump_max + p_sched[s]
        p_reg_i = np.clip(cmd, -down_headroom, up_headroom)

        p_reg[i] = p_reg_i
        p_act[i] = p_sched[s] + p_reg_i

        f[i + 1] = f[i] + (delta_p - D * f[i] - p_reg_i) / M

    p_reg[-1] = p_reg[-2]
    p_act[-1] = p_act[-2]
    return f, p_reg, p_act


def compute_kpis(net0, p_sched, r_up, r_dn, soc, f, p_reg, cfg):
    net1 = net0 - p_sched
    p_gen_max = cfg["p_gen_max"]
    p_pump_max = cfg["p_pump_max"]
    dt_h = cfg["dt_h"]

    pv0 = np.max(net0) - np.min(net0)
    pv1 = np.max(net1) - np.min(net1)
    std0 = np.std(net0)
    std1 = np.std(net1)

    up_avail = p_gen_max - p_sched
    dn_avail = p_pump_max + p_sched
    reserve_ok = np.mean((up_avail >= r_up) & (dn_avail >= r_dn)) * 100

    throughput = np.sum(np.abs(p_sched)) * dt_h
    mileage = np.sum(np.abs(p_reg)) / 3600.0

    f_rms_mhz = np.sqrt(np.mean(f ** 2)) * 1000
    f_max_mhz = np.max(np.abs(f)) * 1000
    soc_end_dev = soc[-1] - cfg["e0"]

    rows = [
        ("峰谷差削减率", (pv0 - pv1) / pv0 * 100, "%"),
        ("净负荷标准差削减率", (std0 - std1) / std0 * 100, "%"),
        ("备用满足率", reserve_ok, "%"),
        ("抽蓄日内能量吞吐", throughput, "MWh"),
        ("AGC调节里程", mileage, "MW*h"),
        ("频差RMS", f_rms_mhz, "mHz"),
        ("最大频差", f_max_mhz, "mHz"),
        ("日终SOC偏差", soc_end_dev, "MWh"),
    ]
    return rows, net1, up_avail, dn_avail


def print_kpi_table(rows):

    print("KPI 指标汇总")
    print("=" * 56)

    print("-" * 56)
    for name, val, unit in rows:
        print(f"{name:<20}{val:>16.2f}{unit:>12}")
    print("=" * 56)
