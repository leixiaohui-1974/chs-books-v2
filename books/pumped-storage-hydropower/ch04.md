# 第4章 与新能源联合调度模型

## 导读

在构建以新能源为主体的新型电力系统进程中，风能和太阳能的开发与利用规模正呈现出指数级的增长态势。然而，风电和光伏发电具有强烈的随机性、波动性和间歇性特征。上一章从电力系统辅助服务的宏观角度，深入探讨了抽水蓄能电站的调频、调峰和旋转备用策略。在那些研究范式中，策略的制定通常将新能源出力的变化视为一种外部给定的、不可控的扰动输入，系统调度主要处于被动响应的状态。

然而，随着电网拓扑结构的演进和源网荷储一体化概念的落地，当抽水蓄能电站与大型风电场或光伏电站在同一地理区域集中布局时（例如中国三北地区的风光储综合能源基地），二者在物理与信息层面上可以深度耦合，作为一个统一的“虚拟电厂”（Virtual Power Plant, VPP）或综合能源系统进行联合优化调度。这种联合运行模式能够将风光的“不可控”转化为整体的“可控”，从根本上削峰填谷，降低新能源出力波动对主干电网的安全冲击，提升跨区输电通道的利用率。

本章致力于建立风-光-蓄联合系统的随机优化调度模型。首先，从气象学与能量转换的物理基础出发，引入新能源出力不确定性的严密数学描述方法；其次，推导基于场景的确定性等价模型（随机规划）以及基于不确定集的鲁棒优化模型的构建与求解机制；最后，通过详尽的全时间尺度仿真，全方位评估联合调度策略相对于独立运行模式在经济效益与运行可靠性上的显著提升。

## 4.1 新能源出力不确定性建模

### 4.1.1 风电出力的概率模型及其物理解释

风电出力的不确定性从根本上来源于大尺度大气环流与微地形气象学相互作用导致的局部风速预测误差。从流体力学和能量守恒定律出发，穿过风力发电机叶片扫掠面积 $A$ 的风的动能转化为机械能的理论功率为 $P_{\text{kinetic}} = \frac{1}{2} \rho A v^3$，其中 $\rho$ 为空气密度，$v$ 为来流风速。根据贝茨极限（Betz Limit），风力机最多只能提取风流中59.3%的动能。考虑到机械传动损耗与发电机效率，实际风速通过风力机的非线性功率曲线转换为电功率输出：

$$P_w(v) = \begin{cases} 0, & v < v_{\text{ci}} \text{ 或 } v > v_{\text{co}} \\ P_w^{\max} \cdot \frac{v^3 - v_{\text{ci}}^3}{v_R^3 - v_{\text{ci}}^3}, & v_{\text{ci}} \leq v \leq v_R \\ P_w^{\max}, & v_R < v \leq v_{\text{co}} \end{cases}$$

其中，$v_{\text{ci}}$、$v_{\text{co}}$ 和 $v_R$ 分别为切入风速（风机开始并网发电的最小风速）、切出风速（为保护机组结构安全而停机的最大风速）和额定风速（达到额定功率的风速）。

在日前调度中，给定数值天气预报（NWP）提供的风速预测值 $\hat{v}(t)$，真实风速可表示为 $v(t) = \hat{v}(t) + \epsilon(t)$。预测误差 $\epsilon(t)$ 通常近似服从均值为零的正态分布 $\mathcal{N}(0, \sigma_v^2)$ 或有界的Beta分布。由于上述功率曲线存在强烈的非线性分段特征，风电出力预测误差的概率密度函数（PDF）展现出高度的非对称性。具体而言，根据随机变量函数的分布变换定理，风电出力的概率密度 $f_P(P)$ 为：

$$f_P(P) = f_v(v) \left| \frac{dv}{dP} \right|$$

当预报风速接近额定风速 $v_R$ 时，由于功率曲线存在上限平移限幅（即达到 $P_w^{\max}$），正向的风速误差无法转化为更高的功率，导致风电出力在 $P_w^{\max}$ 处形成概率质量的冲激函数（Dirac Delta Function），使得整体误差分布呈现严重的左偏态。这种物理层面的非线性限幅效应要求调度模型必须具备处理非对称风险的能力。

### 4.1.2 光伏出力的概率模型与热力学推导

光伏发电的物理机制是半导体材料的PN结在光子激发下产生电子-空穴对的本征光电效应。光伏出力的波动受地表接收到的太阳短波辐照度与环境温度的双重影响，其预测误差的主要来源是高空云层的快速移动与消长对直射光的遮挡。

基于等效电路模型与半导体热力学特性，光伏阵列的输出功率可精确表示为：

$$P_{pv}(t) = P_{pv}^{\text{STC}} \cdot \frac{G(t)}{G_{\text{STC}}} \cdot \left[1 + \alpha_T (T_c(t) - T_{\text{STC}})\right]$$

公式中，$P_{pv}^{\text{STC}}$ 为标准测试条件（Standard Test Conditions, STC）下的铭牌额定功率；$G(t)$ 为实际接收到的倾斜面总辐照度（W/m²）；$G_{\text{STC}} = 1000$ W/m² 为参考辐照度。$\alpha_T$ 为组件的功率温度系数，通常为负值（晶硅组件约为 $-0.004$ /°C），其物理本质在于温度升高会导致半导体禁带宽度变窄，进而引发开路电压的急剧下降，造成输出功率衰减。$T_c(t)$ 为太阳能电池内部的工作温度，由环境温度 $T_a(t)$、辐照度及风速共同决定，通常采用稳态热平衡方程进行估算：$T_c(t) = T_a(t) + G(t) \cdot \frac{\text{NOCT} - 20}{800}$，其中 NOCT 为电池额定工作温度。

在不确定性建模中，实际辐照度 $G(t)$ 通常表示为晴空辐照度 $G_{\text{clear}}(t)$ 与晴空指数 $k_c(t)$ 的乘积。晴空辐照度可通过天文与地理学公式精确计算，而云层遮挡带来的随机性全部集中在 $k_c(t)$ 上。由于 $k_c(t)$ 受限于 $[0,1]$ 区间，其概率分布通常采用双峰或单峰的 Beta 分布进行拟合。

### 4.1.3 场景生成与削减的数学机理

为了将连续的概率密度模型转化为可嵌入混合整数线性规划（MILP）框架的离散形式，学术界与工程界广泛采用场景分析法（Scenario Analysis）。

**场景生成**：首先采用蒙特卡洛（Monte Carlo）模拟或拉丁超立方采样（Latin Hypercube Sampling, LHS）生成海量初始场景集 $\{P_w^{(s)}(t), P_{pv}^{(s)}(t)\}_{s=1}^{S}$（例如 $S=1000$）。相比于传统蒙特卡洛的完全随机性，拉丁超立方采样通过将概率累积分布函数（CDF）划分为等概率的多个区间并在每个区间内强制采样，确保了样本在概率空间中的均匀覆盖，有效避免了维度灾难。

**场景削减**：海量场景会导致优化模型规模极其庞大、无法求解。因此，引入基于概率距离测度的场景削减算法。当前主流采用 Kantorovich 距离（亦称 Wasserstein 距离）作为不同概率分布间差异的度量标准。Kantorovich 距离的物理意义是将一个概率分布转化为另一个概率分布所需的最小“搬运成本”。给定初始庞大场景集，场景削减算法（如前向选择法或K-medoids聚类）迭代地寻找最具代表性的子集，逐步将相似场景合并，最终将场景集削减至计算资源可承受的规模（通常 $S' = 10 \sim 20$ 个代表性典型场景）。削减后的每个场景被赋予新的概率权重 $\pi_s$，且严格满足全概率公式 $\sum_{s=1}^{S'} \pi_s = 1$。

## 4.2 联合调度优化模型

### 4.2.1 随机规划模型的推导与演化

基于上述离散场景，风-光-蓄联合系统调度的核心架构采用两阶段随机规划（Two-stage Stochastic Programming）模型。该模型的数学哲学在于将决策过程严格映射为物理时间轴上的先后顺序，平衡“预先承诺”与“事后纠偏”。

**第一阶段决策（日前计划）**：在真实的新能源出力揭晓之前（Day-Ahead），调度中心必须基于全局预测信息，提前敲定响应迟缓设备的运行状态。具体而言，包括常规火电机组的启停计划 $u_j(t)$ 以及抽水蓄能电站的基准运行模式安排。这些决策被称为“Here-and-Now”变量，它们不依赖于具体的随机场景。

**第二阶段决策（实时调整）**：在系统进入实时运行（Real-Time）阶段后，各个场景 $s$ 下的真实新能源出力偏差随之暴露。此时系统利用具有快速响应能力的资源（如抽水蓄能的出力深度调节、火电机组的自动发电控制 AGC 响应、乃至弃风弃光操作）进行功率补偿。这些决策被称为“Wait-and-See”变量，依附于具体的场景 $s$。

综合两阶段，系统级目标函数被构建为最小化总运行成本的数学期望：

$$\min \left\{ C_{\text{DA}} + \sum_{s=1}^{S'} \pi_s \cdot C_{\text{RT}}^{(s)} \right\}$$

日前确定的硬性成本（包含常规机组燃料成本与所有机组的启停成本）推导为：

$$C_{\text{DA}} = \sum_{t=1}^{T} \left[ \sum_{i \in \mathcal{G}} c_i P_i(t) + \sum_{j \in \mathcal{S}} c_j^{\text{SU}} \cdot u_j(t) \right] \Delta t$$

其中，$\mathcal{G}$ 为常规机组集合，$\mathcal{S}$ 为抽蓄机组集合，$c_i$ 为基于煤耗特性曲线线性化后的发电边际成本，$c_j^{\text{SU}}$ 为包含热态与冷态启动的高昂启停成本，$u_j(t) \in \{0,1\}$ 为启停二元逻辑变量。

第二阶段的实时期望调整成本为：

$$C_{\text{RT}}^{(s)} = \sum_{t=1}^{T} \left[ c^+ \cdot r^{+(s)}(t) + c^- \cdot r^{-(s)}(t) + c^{\text{curt}} \cdot P_{\text{curt}}^{(s)}(t) \right] \Delta t$$

其中，$r^+$ 和 $r^-$ 为系统调用正向和负向备用容量的实际功率，$P_{\text{curt}}$ 为因网络阻塞或调节能力枯竭而被迫舍弃的风光功率总量。系数 $c^+$、$c^-$ 和 $c^{\text{curt}}$ 为相应的市场惩罚价格信号。通常 $c^{\text{curt}}$ 设定为极高值，以从经济上驱使系统最大限度消纳绿电。

### 4.2.2 系统与物理设备约束条件

模型的成立需受到严格的网络与设备物理边界约束限制。

**系统功率绝对平衡约束**（独立作用于每个场景 $s$）：
确保在任何时间断面，系统的电源注入总和等于负荷与网络损耗总和。

$$\sum_{i \in \mathcal{G}} P_i(t) + P_g^{(s)}(t) + P_w^{(s)}(t) + P_{pv}^{(s)}(t) - P_p^{(s)}(t) - P_{\text{curt}}^{(s)}(t) = P_L(t) + r^{+(s)}(t) - r^{-(s)}(t)$$

**抽蓄电站水动力学与蓄能约束**：
抽水蓄能电站的核心在于上水库势能的跨时段转移。基于能量守恒，水库蓄能量 $E^{(s)}(t)$ 满足一阶差分方程：

$$E^{(s)}(t+1) = E^{(s)}(t) + \eta_p P_p^{(s)}(t) \Delta t - \frac{P_g^{(s)}(t)}{\eta_g} \Delta t$$

其中 $\eta_p$ 和 $\eta_g$ 分别为抽水工况和发电工况的综合水机电效率。
此外，受限于大坝物理结构，水库蓄能量存在上下限边界，且为了保证次日具备同等的调节能力，必须强加日循环一致性约束：

$$E_{\min} \leq E^{(s)}(t) \leq E_{\max}, \quad E^{(s)}(T+1) = E^{(s)}(1)$$

**常规火电机组物理约束**：
除了基本的出力上下限，火电机组受到严格的热应力限制，其出力变化率（爬坡率）被死区所约束：

$$-\Delta P_i^{\text{dn}} \leq P_i(t+1) - P_i(t) \leq \Delta P_i^{\text{up}}$$
同时还包含最小连续运行时间与最小连续停机时间的逻辑时间约束，以防频繁启停导致转子疲劳损伤。

### 4.2.3 鲁棒优化（Robust Optimization）替代方案

随机规划强烈依赖于场景概率 $\pi_s$ 的精确估计。在极端天气频发、历史气象数据难以准确拟合真实概率分布的情况下，分布鲁棒或纯鲁棒优化方法成为一种更安全的替代数学工具。鲁棒优化摒弃了精确的概率分布需求，转而将新能源出力的不确定性封装在一个确定性的区间集合（Box Uncertainty Set）或多面体集合（Budget Uncertainty Set）中。

多面体不确定集通常定义为：

$$\mathcal{U} = \left\{ \boldsymbol{\xi} : |\xi_t| \leq \hat{\xi}_t, \, \sum_{t=1}^{T} \frac{|\xi_t|}{\hat{\xi}_t} \leq \Gamma \right\}$$

其中，$\xi_t$ 为实际出力与预报值的偏差变量，$\hat{\xi}_t$ 为物理可能的最大偏差极限（如由置信区间推导）。核心参数 $\Gamma \in [0, T]$ 称为不确定性预算（Budget of Uncertainty）。其物理涵义在于：系统不可能在全天 $T$ 个时段同时遭遇极端的预测误差。$\Gamma$ 允许决策者根据风险偏好灵活调节模型的保守度。

鲁棒优化的架构转化为一个 Min-Max-Min 的三层博弈模型：第一层最小化日前成本；第二层（自然界的恶意）寻找不确定集中导致系统成本最高的恶劣出力轨迹；第三层（调度员的抗争）在给定恶劣轨迹下寻找最优的再调度措施以最小化实时成本。该复杂模型在算法工程上无法直接求解，必须依赖列与约束生成算法（Column-and-Constraint Generation, C&CG）或 Benders 分解法，将其解耦为迭代求解的混合整数主问题（Master Problem）和双层子问题（Subproblem），直至上下界收敛。

## 4.3 仿真设置与系统参数配置

为验证上述理论模型的有效性，本节基于典型的区域电网结构开展全景仿真。系统物理拓扑与参数配置如下：风电场总装机容量达 600 MW（由 100 台 6 MW 的直驱永磁风力发电机组构成）；光伏电站装机容量为 400 MW；与之配套的抽水蓄能电站配置为 4 台 300 MW 的可逆式水泵水轮机组（发电工况总容量 1200 MW，抽水工况额定输入总功率为 1280 MW，设计参数与第3章保持一致）；系统内包含 3 台 500 MW 的主力燃煤火电机组，负责提供基荷支撑。该区域电网夏季典型日的峰值负荷达到 2500 MW，呈现明显的双峰特征。

气象与出力数据采用我国华北某大型风光储一体化基地的实测历史典型日数据序列。为模拟真实预报误差水平，设定日前预报误差标准差为风电总装机的 10%、光伏装机的 8%。在数值实验中，首先通过基于时空相关性协方差矩阵的拉丁超立方采样技术生成了 1000 个高维初始场景集，随后利用 Kantorovich 距离测度与快速前向选择算法，将场景集削减收敛至 15 个代表性极强的典型场景，以输入随机规划模型。

调度优化时域 $T$ 设为 24 小时，时间步长 $\Delta t$ 划分为 15 分钟（共 96 个连续时段），以充分捕捉新能源的秒级至分钟级高频波动特征。MILP 优化模型在 Python 环境中通过 YALMIP 建模语言搭建，底层调用 Gurobi 商业数学优化求解器进行分支定界计算，为保证求解速度与质量的平衡，混合整数规划的相对收敛间隙（MIPGap）严格设为 0.5%。

本研究设立并详细对比了三种递进的调度方案体系：
（a）**独立运行模式**：风电场、光伏电站和抽蓄电站各自为战，风光按预测曲线被动并网，抽蓄仅配合火电进行常规的负荷削峰填谷。
（b）**确定性联合调度模式**：建立联合调度中心，但优化过程仅采信新能源日前预测曲线的单一期望值，不考虑任何误差波动场景。
（c）**两阶段随机规划联合调度模式**：全面考量经削减生成的 15 个概率场景，以系统期望运行成本最优为目标进行全局协同优化。

## 4.4 结果分析与数据深度解读

仿真结果揭示了多能互补系统在不同调度机制下的显著性能差异。以下从四个核心维度进行深入剖析。

**弃风弃光率的大幅改善**：
在基准的独立运行方案中，系统的全天综合弃风弃光率高达 11.3%。通过对出力曲线序列的追踪发现，严重的“弃能”现象主要集中发生在凌晨 02:00 至 05:00 的夜间负荷极小值时段。此时，由于风力资源丰富导致风电大发（局部时段出力超过 550 MW），而火电机组受限于最小技术出力（通常为额定容量的 40%~50%）已无法进一步下调，系统下调峰能力彻底枯竭。引入确定性联合调度后，弃能率降至 5.8%，联合系统开始显现初步的调节优势。
而在采用随机规划联合调度时，弃风弃光率进一步断崖式下降至仅 3.2%。深挖其内在机制发现，随机规划方案的独特优势在于其具备“前瞻性裕度”——模型在日前阶段识别出夜间高风速场景的概率风险，主动在傍晚时段适当增加抽水蓄能的发电流，提前腾空上水库库容。当凌晨风电实际出力远超预期时，抽水蓄能电站能够瞬间全开多台机组进入高强度抽水工况，以近乎零成本的方式将冗余的风电转化为上水库的重力势能，实现了新能源的物理时空平移。

**系统整体运行成本的压缩**：
以独立运行方案的全天总成本（计为基准值 1.00）为参照系，确定性联合调度成本降至 0.91，而随机规划联合调度的成本最低，达到 0.88（即总成本显著降低了 12%）。详细的成本分解数据表明，经济效益的跃升源于两方面：一是弃风弃光惩罚成本的大幅削减；二是火电启停次数的断崖式降低。在独立模式下，为了应对风光的剧烈波动，火电机组被迫频繁启停以提供备用，全天启停总次数达 12 次；而在随机规划联合模式中，抽蓄电站接管了大部分的高频调节任务，使得火电启停次数锐减至 7 次，不仅节约了海量的点火燃油与机械损耗成本，更大幅提升了火电机组的运行寿命与安全性。

**抽蓄电站潜能的深度激活与利用率评估**：
观测抽水蓄能电站的运行曲线可知，在随机规划方案指引下，抽蓄电站的日等效利用小时数攀升至 6.8 小时（其中发电工况 3.2 小时，抽水工况 3.6 小时），远超独立运行状态下的 5.2 小时。更具工程指导意义的数据是上水库蓄能量 $E(t)$ 的日内波动幅度：在随机规划中，水位轨迹从最低库容的 20% 剧烈波动至最高库容的 85%，日变幅达 65%（独立运行仅为 45%）。这一数据直接证明，基于场景的联合调度机制犹如一条纽带，将风光的随机波动深度绑定至抽蓄的物理水库中，更加彻底地榨取了抽蓄电站作为巨型储能单元的调节容量价值。

**蒙特卡洛回测与鲁棒性验证**：
为评估模型在真实极端环境下的生存能力，研究引入 200 个包含了极端恶劣气象条件的实际日数据序列进行事前盲测（Out-of-sample Test）。测试表明，确定性联合调度方案由于过度依赖单一预测值，在面临预报严重失效时，有高达 15% 的测试日次中出现了严重的功率平衡违约现象（即系统必须紧急呼叫昂贵的外部应急备用或实施切负荷拉闸）。相反，随机规划方案得益于其场景覆盖能力，违约率被压制在 2.3%。
为进一步探索安全性边界，研究测试了不确定性预算 $\Gamma = 6$ 的鲁棒优化方案。该方案在全样本回测中实现了惊人的 0% 违约率，系统在任何恶劣天气下均坚如磐石。然而，这种绝对的安全性伴随着经济代价：其日常运行总成本比随机规划方案高出 4.5%。这组数据深刻揭示了工程系统调度中“安全性与经济性不可兼得”的权衡博弈（Trade-off），调度中心必须根据电网实际的安全等级要求，在二者之间寻找最佳平衡点。

## 4.5 工程启示与实践案例

风-光-蓄联合优化调度理论不仅停留在算法层面，它已成为支撑新型电力系统实质性建设的核心技术基座。从国家重大工程实施与电力市场演进的视角，以下几点工程启示具有极高的指导价值。

第一，**联合调度的经济闭环与预测精度强耦合**。联合调度的核心经济价值在于有效减少了弃风弃光造成的能源浪费，并大幅削减了火电机组的启停折旧与燃料成本。然而，这一切红利的兑现高度依赖于气象预测算法的前端精度与数学规划后端的求解效率。工程经验表明，气象数值模型预报精度每实现 1 个百分点的微小提升，配合先进的联合调度算法，即可在系统端减少约 0.5 个百分点的弃能。这启示业界应加大对人工智能气象大模型（如基于深度学习的风光预测）的研发投入。

第二，**尾部风险的量化与条件风险价值控制**。传统的随机规划模型在削减场景数量以换取求解速度时，往往会抹杀低概率但破坏力极大的极端场景（如风机大面积脱网或日全食对光伏的冲击），从而低估尾部风险。在实际调度控制中心的软件系统中，强烈建议引入条件风险价值（Conditional Value-at-Risk, CVaR）指标。CVaR 能够专门针对成本分布的尾端最恶劣的 5% 或 10% 场景进行惩罚建模，为调度员提供更加稳健的风险抵御防线。

第三，**实际工程案例的成功印证**。以国家重点工程——河北丰宁抽水蓄能电站（总装机容量高达 3.6 GW，目前世界规模最大）为例。该电站直接服务于张北风光储输示范基地与北京柔性直流电网。在实际运行中，丰宁抽蓄不仅被动响应电网指令，更主动与张北地区数千兆瓦的风光基地进行实时虚拟电厂级联调度。通过平抑张北基地的宽频波动，丰宁电站成功确保了输送至北京的绿电是一条平滑、稳定的基荷带，完美体现了本章探讨的联合调度内核。

第四，**规划阶段的库容前置考量与市场机制建设**。抽蓄电站的物理库容限制是制约联合调度调节深度的硬约束边界。研究表明，水库库容越大，系统在时间轴上挪移能量的灵活性呈非线性增长。因此，在风光大基地的选址与配套抽蓄规划阶段，应打破“就发供电算容量”的传统思路，将风光不确定性吸收需求前置，充分论证扩建库容的经济性。同时，随着电力现货市场的日益成熟，风-光-蓄作为联合主体参与现货市场与辅助服务市场，能够利用抽蓄的灵活吞吐，对冲新能源在日前预报偏差导致的现货偏差考核巨额罚款，实现虚拟电厂商业模式的盈利闭环。

## 本章小结

本章系统性地建立了风-光-蓄多能互补系统在面对强不确定性时的随机优化调度理论框架。章节内容严密涵盖了基于物理机制的新能源出力概率分布建模、消除维度灾难的场景生成与 Kantorovich 距离削减技术、捕捉时间耦合特性的两阶段随机规划架构，以及防御极端风险的鲁棒优化替代算法等诸多关键核心技术。长时段高颗粒度的数值仿真结果明确证实：相较于各主体独立松散的运行模式，以随机规划为内核的联合协同调度策略能够将系统的综合弃风弃光率从 11.3% 压降至 3.2%，并使系统整体运行经济成本降低约 12%。整个模型设计与验证流程传递出一个关键的核心理念：即通过智能算法将新能源的强随机波动性，巧妙地传导并由抽水蓄能电站卓越的物理灵活调节能力所内部“吸收平滑”，最终使这一复杂的联合能源系统在主干电网侧呈现为如同常规火电机组般高度可控、可靠且平滑的优质电源。本章侧重于物理与经济层面的协同调度，在下一章节中，我们将视角转向市场博弈，深入探讨在复杂的双边电力现货市场与辅助服务市场环境下，上述优化调度动作如何转化为使联合主体利润最大化的最优动态竞价（Optimal Bidding）策略体系。

## 拓展视野

风-光-蓄联合调度模型中推导的上水库蓄能动态微分方程 $\dot{E} = \eta_p P_p - P_g/\eta_g$ ，与宏观跨流域调水工程体系中庞大水库群联合调度所遵循的水量平衡方程 $\dot{V}_i = Q_{\text{in},i} - Q_{\text{out},i}$ ，在系统的数学底层结构上呈现出完美的同构特征。这两类看似迥异的工程系统，在本质上均面临着同一类型的挑战：即边界输入条件的强不确定性（前者是风力光照自然资源的随机波动，后者是降水径流等水文序列的随机演化），且均需要在多阶段时间序列中围绕“能量/水量的蓄积与释放”做出最优的非线性决策。Lei（2025a）在其提出的水系统控制论（Cybernetics of Water Systems, CHS）宏大理论框架下，将这类涵盖储能介质的跨期优化问题，统一抽象并归纳为“积分型受控对象的随机最优控制”理论范畴。该理论指出，CHS 在流域水文调度中积累的成熟算法（特别是预测-校正滚动调度机制与随机动态规划逼近方法），能够为当前复杂多变的虚拟电厂风-光-蓄联合高频次微观调度系统，提供强有力的底层跨学科方法论借鉴与理论升华。

def build_profiles(T=24, seed=7):
    rng = np.random.default_rng(seed)
    t = np.arange(T)

    # 系统负荷曲线
    load = 1500 + 220 * np.sin(2 * np.pi * (t - 7) / 24) + 120 * np.sin(4 * np.pi * (t - 18) / 24)
    load = np.clip(load, 1150, 1900)

    # 风电预测
    wind_fc = 420 + 150 * np.sin(2 * np.pi * (t + 2) / 24) + rng.normal(0, 25, T)
    wind_fc = np.clip(wind_fc, 100, 720)

    # 光伏预测
    solar_shape = np.sin(np.pi * (t - 6) / 12)
    pv_fc = 420 * np.clip(solar_shape, 0, None) + rng.normal(0, 10, T)
    pv_fc = np.clip(pv_fc, 0, 450)

    renew_fc = wind_fc + pv_fc
    return t, load, wind_fc, pv_fc, renew_fc


def generate_and_reduce_scenarios(renew_fc, n_mc=100, n_rep=6, sigma=0.12, rho=0.65, seed=11):
    rng = np.random.default_rng(seed)
    T = renew_fc.size

    # Monte Carlo 场景生成（含时间相关误差）
    mc = []
    for _ in range(n_mc):
        eps = np.zeros(T)
        eps[0] = rng.normal(0, sigma)
        for k in range(1, T):
            eps[k] = rho * eps[k - 1] + np.sqrt(1 - rho ** 2) * rng.normal(0, sigma)
        s = np.clip(renew_fc * (1.0 + eps), 0.0, None)
        mc.append(s)
    mc = np.asarray(mc)

    # 简化场景削减：按全天电量排序后等分抽代表场景
    order = np.argsort(mc.sum(axis=1))
    sorted_mc = mc[order]
    idx = np.linspace(0, n_mc - 1, n_rep).round().astype(int)
    reps = sorted_mc[idx]

    bins = np.linspace(0, n_mc, n_rep + 1).astype(int)
    weights = np.array([(bins[i + 1] - bins[i]) / n_mc for i in range(n_rep)], dtype=float)
    weights /= weights.sum()
    return reps, weights


def solve_two_stage_stochastic(load, scen, w):
    # scen: [S, T]
    S, T = scen.shape
    dt = 1.0

    # 抽蓄参数
    Pp_max, Pg_max = 520.0, 520.0
    eta_p, eta_g = 0.90, 0.90
    E_min, E_max, E0 = 200.0, 2800.0, 1500.0

    # 火电参数
    Pth_min, Pth_max = 450.0, 2200.0
    ramp = 230.0

    # 成本系数
    a, b = 0.0022, 8.0
    c_curt = 260.0
    c_ramp = 0.06
    c_psh = 2.0
    c_simul = 20.0  # 抑制同一时段“边抽边发”

    n1 = 2 * T
    n_th = S * T

    def unpack(x):
        pump = x[:T]
        gen = x[T:2 * T]
        th = x[n1:n1 + n_th].reshape(S, T)
        cur = x[n1 + n_th:].reshape(S, T)
        return pump, gen, th, cur

    def soc_traj(pump, gen):
        e = np.zeros(T)
        ee = E0
        for k in range(T):
            ee = ee + eta_p * pump[k] * dt - gen[k] / eta_g * dt
            e[k] = ee
        return e

    def objective(x):
        pump, gen, th, cur = unpack(x)
        fuel = np.sum(w[:, None] * (a * th * th + b * th)) * dt
        ramp_cost = c_ramp * np.sum(w[:, None] * (np.diff(th, axis=1) ** 2))
        curt_cost = c_curt * np.sum(w[:, None] * cur) * dt
        psh_cost = c_psh * np.sum(pump + gen) * dt + c_simul * np.sum(pump * gen) * dt
        return fuel + ramp_cost + curt_cost + psh_cost

    def c_balance(x):
        pump, gen, th, cur = unpack(x)
        # 功率平衡：火电 + 新能源 - 弃电 + 抽蓄发电 - 抽水 = 负荷
        return (th + scen - cur + gen[None, :] - pump[None, :] - load[None, :]).ravel()

    def c_ramp_up(x):
        _, _, th, _ = unpack(x)
        return (ramp - np.diff(th, axis=1)).ravel()

    def c_ramp_dn(x):
        _, _, th, _ = unpack(x)
        return (ramp + np.diff(th, axis=1)).ravel()

    def c_soc_min(x):
        pump, gen, _, _ = unpack(x)
        return soc_traj(pump, gen) - E_min

    def c_soc_max(x):
        pump, gen, _, _ = unpack(x)
        return E_max - soc_traj(pump, gen)

    def c_soc_terminal(x):
        pump, gen, _, _ = unpack(x)
        return np.array([soc_traj(pump, gen)[-1] - E0])

    # 初值：不启用抽蓄
    pump0 = np.zeros(T)
    gen0 = np.zeros(T)
    cur0 = np.clip(Pth_min - (load[None, :] - scen), 0, scen)
    th0 = load[None, :] - scen + cur0
    x0 = np.concatenate([pump0, gen0, th0.ravel(), cur0.ravel()])

    bounds = []
    bounds += [(0.0, Pp_max)] * T
    bounds += [(0.0, Pg_max)] * T
    bounds += [(Pth_min, Pth_max)] * (S * T)
    for s in range(S):
        for t in range(T):
            bounds.append((0.0, float(scen[s, t])))

    constraints = [
        {"type": "eq", "fun": c_balance},
        {"type": "ineq", "fun": c_ramp_up},
        {"type": "ineq", "fun": c_ramp_dn},
        {"type": "ineq", "fun": c_soc_min},
        {"type": "ineq", "fun": c_soc_max},
        {"type": "eq", "fun": c_soc_terminal},
    ]

    res = minimize(
        objective,
        x0,
        method="SLSQP",
        bounds=bounds,
        constraints=constraints,
        options={"maxiter": 450, "ftol": 1e-7, "disp": False},
    )
    if not res.success:
        raise RuntimeError(f"优化失败: {res.message}")

    pump, gen, th, cur = unpack(res.x)
    return {
        "pump": pump,
        "gen": gen,
        "th": th,
        "cur": cur,
        "obj": res.fun,
        "msg": res.message,
        "Pth_min": Pth_min,
    }

def calc_cost(th, cur, pump, gen, w, a=0.0022, b=8.0, c_curt=260.0, c_ramp=0.06, c_psh=2.0, c_simul=20.0):
    fuel = np.sum(w[:, None] * (a * th * th + b * th))
    ramp_cost = c_ramp * np.sum(w[:, None] * (np.diff(th, axis=1) ** 2))
    curt_cost = c_curt * np.sum(w[:, None] * cur)
    psh_cost = c_psh * np.sum(pump + gen) + c_simul * np.sum(pump * gen)
    return fuel + ramp_cost + curt_cost + psh_cost


def calc_kpis(scen, w, th, cur, pump, gen, cost_value):
    total_renew = np.sum(w[:, None] * scen)
    curtailed = np.sum(w[:, None] * cur)
    acc = 100.0 * (1.0 - curtailed / (total_renew + 1e-12))

    ramp_rms = np.sqrt(np.sum(w * np.mean(np.diff(th, axis=1) ** 2, axis=1)))
    th_exp = np.sum(w[:, None] * th, axis=0)
    peak_valley = np.max(th_exp) - np.min(th_exp)
    shifted = np.sum(gen) - np.sum(pump)

    return {
        "新能源消纳率(%)": acc,
        "弃风弃光电量(MWh)": curtailed,
        "火电爬坡RMS(MW/h)": ramp_rms,
        "火电峰谷差(MW)": peak_valley,
        "抽蓄净搬移电量(MWh)": shifted,
        "综合成本指标": cost_value,
    }

    pump0, gen0, th0, cur0 = baseline_without_psh(load, scen, sol["Pth_min"])

    cost0 = calc_cost(th0, cur0, pump0, gen0, w)
    cost1 = calc_cost(sol["th"], sol["cur"], sol["pump"], sol["gen"], w)

    kpi0 = calc_kpis(scen, w, th0, cur0, pump0, gen0, cost0)
    kpi1 = calc_kpis(scen, w, sol["th"], sol["cur"], sol["pump"], sol["gen"], cost1)

    print("优化状态:", sol["msg"])
    print("代表场景数:", scen.shape[0])
    print_kpi_table(kpi0, kpi1)

    # 仅生成1张图
    w2 = w[:, None]
    renew_exp = np.sum(w2 * scen, axis=0)
    renew_acc_exp = np.sum(w2 * (scen - sol["cur"]), axis=0)
    th0_exp = np.sum(w2 * th0, axis=0)
    th1_exp = np.sum(w2 * sol["th"], axis=0)
    psh_net = sol["gen"] - sol["pump"]

    plt.figure(figsize=(11, 5))
    plt.plot(t, load, lw=2.3, label="负荷")

    plt.plot(t, th0_exp, "--", lw=2.0, label="火电(独立运行)")
    plt.plot(t, th1_exp, lw=2.3, label="火电(联合调度)")
    plt.plot(t, psh_net, lw=2.0, label="抽蓄净出力(+发电/-抽水)")
    plt.xlabel("时段(h)")
    plt.ylabel("功率(MW)")
