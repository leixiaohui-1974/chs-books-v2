# 第 1 章：水系统建模与控制论基础

## 1. 学习目标
本章从基础水力学原理出发，建立水务系统的第一性原理数学模型。
读者需掌握以下内容：
1. 连续性方程（质量守恒）与动量守恒在储水和输水设备中的数学表达。
2. 线性化系统分析方法，以及拉普拉斯变换与传递函数的求解。
3. 系统纯滞后（Dead Time）与执行器动态特性在闭环反馈控制中产生的不利影响及其机制。

## 2. 教材理论：水务系统采用控制理论的必要性
水务系统属于动态、强不确定性且受严格物理约束的公共基础设施。河流行洪受气象条件影响呈现持续波动，城市用水需求在小时尺度上存在显著的时变特性。泵站、闸门、阀门和水处理单元在广阔空间范围与多时间尺度上耦合运行。因此，水网运行优化不仅是水力学问题，更是典型的**动态反馈控制问题**。

控制理论提供了系统化的方法论，用于处理水务工程中的核心任务：在降雨不确定性、用水量突变等强扰动条件下，对有限物理执行器（如变频水泵）进行可验证、可约束的调度，以保障大型水网的安全性与运行效率。

在实际运行控制中：
- **开环控制（Open-Loop）**：基于历史经验和固定时间表进行调度（例如定时启停泵）。在面对未建模的突发扰动时缺乏鲁棒性，极易失效。
- **闭环反馈控制（Closed-Loop Feedback）**：利用实时传感器测量值（如液位、压力）连续修正控制动作。在水务系统中，鉴于扰动频繁且后果严重，闭环反馈控制是保障系统稳定运行的必要配置。

## 3. 数学基础与推导：动态系统建模与线性化
控制器设计需以物理模型为基础。对于水系统，系统模型通常源于质量守恒定律。

### 3.1 单容水箱的非线性质量守恒

**物理概化图（Physical Schematic）：**
![单容水箱控制系统物理概化图](assets/ch01/problem_nano.png)

考虑一个横截面积为 $A$ 的开口水箱，进水流量为 $q_{in}$，出水流量为 $q_{out}$。系统状态为水位 $h$。
根据质量守恒定律：
$$ A \frac{dh(t)}{dt} = q_{in}(t) - q_{out}(t) $$

若出水过程依赖于底部孔口的重力自流，根据托里拆利定律，出水流量与水深的平方根成正比：
$$ q_{out}(t) = k \sqrt{h(t)} $$
其中 $k = C_d a \sqrt{2g}$，$C_d$ 为流量系数，$a$ 为孔口面积，$g$ 为重力加速度。
将上式代入质量守恒方程，可得一阶**非线性微分方程**：
$$ A \frac{dh(t)}{dt} = q_{in}(t) - k \sqrt{h(t)} $$

### 3.2 工作点上的泰勒展开与线性化
在工业控制领域，为应用经典的 PID 控制或频域分析方法，通常需在某一稳态工作点 $(h_0, q_{in,0})$ 附近对系统进行线性化处理。
当系统处于平衡状态时，状态量的导数为零：
$$ q_{in,0} = k \sqrt{h_0} $$
定义偏差变量：$\tilde{h} = h - h_0$，$\tilde{q}_{in} = q_{in} - q_{in,0}$。
对非线性项在 $h_0$ 处进行一阶泰勒展开，忽略高阶极小项，可得：
$$ \frac{d\tilde{h}}{dt} = -\frac{k}{2A\sqrt{h_0}}\tilde{h} + \frac{1}{A}\tilde{q}_{in} $$
该等式可重写为标准的一阶惯性系统形式：
$$ \frac{d\tilde{h}}{dt} = -\frac{1}{\tau}\tilde{h} + K_u \tilde{q}_{in} $$
其中时间常数 $\tau = \frac{2A\sqrt{h_0}}{k}$（其物理量纲为秒 $s$），系统增益 $K_u = \frac{1}{A}$（物理量纲为 $m^{-2}$）。
在零初始条件假设下，通过拉普拉斯变换可得该系统的传递函数：
$$ G(s) = \frac{\tilde{H}(s)}{\tilde{Q}_{in}(s)} = \frac{K_u}{\tau s + 1} $$

## 4. 执行器动态与低通滤波在真实工业中的影响
在基础控制理论中，常假设控制器输出 $u(t)$ 可瞬时且无偏差地作用于被控对象。然而，在实际工业系统中，控制器与被控对象之间通常存在非理想环节：执行器本体动态特性、机械/液压传动迟延，以及信号滤波与采样延迟。

针对水务系统，典型的执行器包括电动阀门、液压驱动闸门以及变频水泵。由于其物理响应速率受限，通常可等效为一阶惯性加纯滞后（Dead Time）环节：
$$ G_a(s)=\frac{K_a}{T_a s+1}e^{-Ls} $$
其中 $L$ 为纯滞后时间（例如长距离明渠输水的水波传播延迟、SCADA通信延迟）。

**工程实现中的稳定性风险说明**：
纯滞后项 $e^{-Ls}$ 是限制系统稳定性的关键因素。它不改变系统的幅频特性，但会引入严重的相位滞后（在角频率 $\omega$ 下，相角将额外滞后 $-\omega L$ 弧度），从而直接削减系统的相角裕度（Phase Margin）。当调度中心下达开闸指令时，水流需经历较长传输时间方可抵达下游（即纯滞后时间 $L$ 较大），此时若控制器采用较高的 PID 比例增益，系统极易引发等幅振荡或发散振荡（如“水锤”或“涌波”现象）。

## 5. 案例分析：理论与实践的桥梁（单容水箱水动力学仿真）

### 5.1 案例背景 (Context)
本案例旨在将抽象的控制理论转化为可计算的数学与物理模型。研究对象为城市生命线工程中的高位水塔注水系统。该系统在早晚用水高峰期面临显著的负荷波动（可类比为洪水冲击或水质突发污染场景）。分析任务为：在水泵恒定功率注水条件下，预判水塔是否会发生溢流，并定量评估水位上升速率的动态变化。
此案例提炼自工程核心代码库，反映了水务工程师在实际工程中面临的典型问题：在存在物理边界约束和非线性泄流阻力的复杂工况下，分析系统状态并寻求动态平衡。

### 5.2 问题描述 (Problem)
当系统面临极端的边界输入条件时，传统的经验法则可能导致误判。典型的认知偏差是将恒定进水条件下的水位演化过程简化为持续匀速上升直至漫溢。
**核心难点**：随着水位的升高，底部静水压力逐渐增大，进而导致出水孔流速提升。该过程具有典型的非线性特征。若在控制系统设计中采用基础线性化假设构建单回路 PID 控制器而忽略此非线性特性，将导致系统在低水位区间响应迟缓，而在高水位区间响应过度激进，严重时可能引发管网结构的物理损伤风险。因此，必须构建能够准确预判系统物理状态演变规律的算法框架。

### 5.3 解题思路 (Solution Approach)
本案例采用白盒稳健数值迭代方法进行分析。
1. **连续性建模**：基于不可压缩流体连续性方程和托里拆利定律，建立描述物理状态的非线性微分方程（即前文推导的 $A \frac{dh}{dt} = q_{in} - k \sqrt{h}$）。
2. **数值稳定性保护**：在工业级代码实现中不可完全依赖求解器本身的稳定性。若求解器计算步长过大导致出现负水深，平方根函数将产生非法数值（NaN）。因此，必须在算法中引入 `h = max(h, 0.0)` 的物理截断边界以保障计算稳定。
3. **数值积分孪生**：运用 `scipy.integrate.odeint` 进行时间步进的数值模拟，构建真实水箱动态响应的数字孪生模型。

### 5.4 代码执行与图表 (Code & Charts)
> **学习提示**：理论推导最终需通过代码实现验证。以下核心逻辑模拟了物理引擎的自动计算过程。本案例运行了底层微分方程求解器（`odeint`），生成了随时间演进的数据表格与高精度动态响应曲线。此代码框架可作为工业级部署原型的验证基础。

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy.integrate import odeint

# 系统参数
A = 2.0       # 水箱截面积 m^2
C_d = 0.6     # 流量系数
a = 0.05      # 出水孔面积 m^2
g = 9.81      # 重力加速度 m/s^2

def tank_dynamics(h, t, Q_in):
    # 【数值稳定性保护】：物理系统中水深非负，防止 sqrt 函数出现 NaN
    h = max(h, 0.0)
    Q_out = C_d * a * np.sqrt(2 * g * h)
    # 质量守恒定律的数值表达
    dhdt = (Q_in - Q_out) / A
    return dhdt

# 模拟 200 秒的物理时间演进过程
t = np.linspace(0, 200, 500)
Q_in_step = 0.1 # 设定上游水泵恒定注入流量为 0.1 m^3/s

# 采用高精度数值积分求解状态轨迹
h_sim = odeint(tank_dynamics, 0.0, t, args=(Q_in_step,))
```

### 5.5 结果解析 (Result Interpretation)
仿真结果表明，系统的水位演变轨迹呈现为非线性逐渐收敛的曲面，而非简单的线性上升，这符合一阶惯性系统的动态响应特征。
在仿真初期（$t \in [0, 40]$ 秒），由于水箱初始液位较低，底部静水压力不足以产生较大的出流，水位呈现快速上升趋势。若在实际泵站运行中仅依据此短时趋势采取过度关阀操作，极易导致系统产生超调等误动作。
随着水位的持续上升，底部静水压力相应增加，进而驱动出水流速提升（图中红色虚线代表恒定进水流量 $0.1\text{ m}^3/\text{s}$，红色实线代表随时间增加的出水流量）。水深曲线的变化率逐渐趋缓，并于约 $100$ 秒时稳定在 **0.566 米** 的稳态液位；同时，出水流量逐渐收敛并等同于进水流量（$0.1\text{ m}^3/\text{s}$）。此结果证实：在出水孔畅通且水塔容量满足边界条件的前提下，系统能够依赖物理法则中的非线性反馈机制实现动态**自平衡 (Self-Regulating)**。

### 5.6 专家建议 (Recommendations)
1. **针对工程实现的抗失稳建议**：在将此类控制算法部署至 PLC（可编程逻辑控制器）或边缘计算网关时，需着重处理现场传感信号（如超声波液位计因水面波动产生的高频噪声）。若在 $0.566\text{ m}$ 稳态工作点附近频繁受到噪声干扰，微分方程的离散求解过程极易产生剧烈的控制指令抖动，进而可能损伤变频器等电气设备。因此，在物理传感信号接入算法运算单元前，必须引入**卡尔曼滤波（Kalman Filter）**或一阶低通滤波算法进行平滑去噪处理。
2. **系统安全边界声明**：本教材所提供的算法模型内置了数值截断保护机制（`h = max(h, 0.0)`）。当研发人员将该模块接入大语言模型智能体（LLM Agent）或强化学习（Reinforcement Learning）控制器进行策略寻优探索时，严禁移除底层的物理约束边界；否则，在探索未知状态空间时，可能导致数值积分器出现计算溢出错误（NaN），并最终引发系统级控制失效或仿真程序崩溃。
