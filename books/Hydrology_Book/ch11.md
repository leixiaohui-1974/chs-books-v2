# 第11章 城市内涝防洪预警与联合调度

## 11.1 从气象接入、产流计算到街道级内涝告警的全链路

这一章我们将前面的所有概念和组件串联起来，应对城市水务中最棘手的问题：内涝。
一场暴雨从天上降落到演变成街道积水，在我们的系统里体现为一个自动化执行链：
雷达数据接入 -> 降雨网格化 -> 触发 `rainfall_runoff.py` 产流计算 -> 触发 1D/2D 水动力演进 -> 生成网格水位。
在 `core/hydrology/use_cases/urban_flood.py` 中，我们展示了这一业务流程。系统会根据计算出的降雨强度和汇水区容量，自动将内涝风险评级为 `High` 或 `Low`。

## 11.2 “库-河-网”联合调度降低溢流风险

单独靠加粗排水管网并不能根治内涝。现代城市的应对策略是“库（水库调蓄）、河（河道行洪）、网（管网抽排）”的联合调度。
当系统预判到内涝风险（`risk: High`）时，Agent 将自动检索水文拓扑图，计算是否可以提前 12 小时降低上游水库的水位，或者腾空城区的景观湖以形成调蓄容积。
`tests/hydrology/test_ch11_end_to_end_urban_flood.py` 验证了从降雨强度输入到高风险状态判断的端到端逻辑通路。

## 本章小结与证据锚点

- **核心结论**：城市防洪排涝不再是单一水利设施的单打独斗，而是数字孪生体系下的全局联合博弈。
