# 第5章 堤防安全评估

## 本章导读

堤防工程是流域防洪工程体系的基础屏障，其物理结构的完整性与功能状态的稳定性直接关系到沿江河湖泊广大地区的人民生命财产安全与社会经济发展。随着全球气候变化加剧，极端气象水文事件呈现频发态势，传统基于单一静态工况和纯经验校核的堤防设计与评估方法，已难以满足现代流域多尺度复杂防洪调度的实际需求。现代洪水预报与防洪调度要求将堤防视为动态响应的物理实体，将其瞬态安全裕度作为调度决策的关键边界条件。

本章在《洪水预报与防洪调度》的宏观框架下，聚焦堤防工程微观尺度的安全演化机制，系统探讨堤防安全评估的理论基础、数学模型、仿真计算与工程应用。本章涵盖非稳定渗流场演变、土体多场耦合变形破坏机制、极限平衡与数值计算方法，并延伸至堤防溃决动力学及洪水演进分析。通过解析各类失效模式的物理边界与诱发机理，建立从实时状态感知、数据同化到风险定量评估的完整闭环，从而为流域级防洪系统的智能调度提供坚实的工程学约束与理论支撑。

## 5.1 基本概念与理论框架

堤防安全评估的理论框架建立在土力学、水力学与工程地质学的交叉基础上，主要包括渗流场分析、结构稳定性验算、破坏模式识别与监测预警等核心板块。

### 5.1.1 非饱和渗流与渗透稳定
在汛期洪水位抬升或退水过程中，水流在水头梯度驱动下穿过堤身及地基孔隙的运动构成渗流场。堤防通常由压实土料筑成，浸润线以上的土体处于非饱和状态。非饱和土的渗透系数是体积含水率或基质吸力的非线性函数，必须引入土-水特征曲线（Soil-Water Characteristic Curve, SWCC）来描述土骨架与孔隙水的相互作用。

渗透变形是土石堤防失效的主要前导因素，可分为管涌（Piping）与流土（Boiling）两类基本形式。管涌多发生于内部细颗粒在粗颗粒孔隙中被渗流挟带流失的无黏性土层中；流土则多见于粉细砂或粉土层，在局部水力坡降超过土体临界有效浮重度时，土体结构发生整体悬浮与破坏。临界水力坡降的判定是渗透稳定分析的核心。

### 5.1.2 堤防溃决模式与机制演化
堤防破坏往往是多种失效机制叠加与演化的结果，典型的宏观溃决模式包括漫顶溃决、管涌诱发溃决与边坡失稳溃决。
- **漫顶溃决**：当洪水位超过堤顶高程，水流越顶形成跌水，在背水坡面产生强烈的水流剪切应力。当该剪切应力超过坡面土体的抗冲刷起动切应力时，将引发溯源冲刷，进而导致堤身断面迅速削薄并最终整体垮塌。
- **内部管涌溃决**：伴随渗透破坏的发展，堤防内部形成贯通的优势渗流通道（管涌通道）。随着孔径扩大，水流由层流转变为紊流，挟沙能力成倍增长，导致堤顶塌陷并最终撕裂堤身。
- **边坡失稳**：高水位持续期，堤身大范围浸水导致土体基质吸力丧失、孔隙水压力急剧升高，有效应力降低；而在洪水骤降期，迎水坡土体内的高孔隙水压力来不及消散，产生向外的动水压力，极易诱发迎水坡的大规模圆弧滑动。

### 5.1.3 溃坝（堤）洪水演进与危害评估
堤防一旦溃决，积蓄在河道内的巨大水体将通过溃口以突发涌波的形式向防洪保护区倾泻。溃决洪水的演进是一个强烈的三维非恒定水流过程。对其进行评估需要首先确定溃口的发展过程（溃口流量过程线），随后利用浅水动力学方程对洪水在淹没区内的爬坡、绕流及滞留现象进行二维或三维动力学路由分析，以输出最大淹没水深、流速及到达时间分布图，为应急疏散提供依据。

### 5.1.4 物理感知与堤防安全监测
现代堤防工程广泛部署了渗压计、测斜仪、沉降磁环及分布式光纤等传感设备，形成全天候的状态监测网络。通过获取孔隙水压力、内部变形及表面位移的时序数据，结合反演分析理论，可动态修正岩土力学参数，从而克服地质勘探数据的空间局限性，实现模型驱动与数据驱动相融合的数字孪生评估。

## 5.2 数学建模与求解方法

本节从偏微分方程与计算力学角度，建立堤防状态响应与破坏演化的严格数学模型。

### 5.2.1 非饱和-饱和渗流场控制方程
基于质量守恒定律与扩展的达西定律（Darcy's Law），三维非恒定非饱和渗流场的主控方程（Richards方程）可表示为：

$$ \frac{\partial}{\partial x} \left( k_x(h_w) \frac{\partial H}{\partial x} \right) + \frac{\partial}{\partial y} \left( k_y(h_w) \frac{\partial H}{\partial y} \right) + \frac{\partial}{\partial z} \left( k_z(h_w) \frac{\partial H}{\partial z} \right) + W = m_w \gamma_w \frac{\partial H}{\partial t} $$

式中：$H$ 为总水头；$h_w$ 为压力水头；$k_x(h_w), k_y(h_w), k_z(h_w)$ 分别为各向异性的非饱和渗透系数函数；$W$ 为源汇项；$m_w$ 为比水容积（即单位体积含水率对压力水头的导数）；$\gamma_w$ 为水的重度；$t$ 为时间。

方程求解需要定义初始条件 $H(x,y,z,0) = H_0(x,y,z)$ 以及两类典型边界条件：
1. 第一类边界（Dirichlet边界条件）：水库或河道水位边界，$H(x,y,z,t) = H_b(t)$。
2. 第二类边界（Neumann边界条件）：不透水地基或已知流量边界，$-\mathbf{k} \nabla H \cdot \mathbf{n} = q(t)$。

在有限元方法（FEM）框架下，采用伽辽金（Galerkin）加权残数法对上述方程进行空间离散，并结合隐式向后欧拉法进行时间积分，可将原偏微分方程转化为大型非线性代数方程组，采用Newton-Raphson方法进行迭代求解。

### 5.2.2 边坡稳定性的极限平衡模型
极限平衡法假定破坏面上土体处于屈服状态，通过刚体静力平衡条件求解抗滑稳定安全系数（Factor of Safety, $F_s$）。针对复杂多地层堤防结构，广泛采用Morgenstern-Price法或Spencer法，其严格满足所有条块的力和力矩平衡。

设滑动体被离散为 $n$ 个垂直条块，条块底部受到的抗剪强度 $\tau_f$ 遵循莫尔-库仑（Mohr-Coulomb）破坏准则：

$$ \tau_f = c' + (\sigma_n - u_w) \tan \phi' $$

式中：$c'$ 为有效黏聚力；$\phi'$ 为有效内摩擦角；$\sigma_n$ 为底面法向总应力；$u_w$ 为孔隙水压力（由渗流方程求解获得）。

全局安全系数 $F_s$ 定义为抗剪强度与实际发挥的平均剪应力 $\tau_m$ 之比。Spencer法的积分表达形式可简化求解如下超越方程组：

$$ \sum_{i=1}^{n} (W_i \sin \alpha_i - \frac{c'_i l_i + (W_i \cos \alpha_i - u_{wi} l_i) \tan \phi'_i}{F_s}) = 0 $$

式中：$W_i$ 为条块自重；$\alpha_i$ 为条块底面倾角；$l_i$ 为条块底面长度。方程需通过迭代算法求得使得整个滑动面 $F_s$ 取最小值的临界滑动面位置。

### 5.2.3 漫顶溃口演变动力学模型
堤防溃口的展宽与下切是一个流固强耦合过程。设溃口为梯形断面，底宽为 $b(t)$，底高程为 $Z_b(t)$。进入溃口的水流近似采用宽顶堰流公式计算：

$$ Q_{out} = C_d b(t) \sqrt{2g} (H_{up} - Z_b(t))^{3/2} $$

水流对堤防土体的垂向下切率 $\frac{dZ_b}{dt}$ 和侧向扩宽率 $\frac{db}{dt}$ 由土体侵蚀定律决定，通常假设冲刷速率与水流超额剪切应力成正比：

$$ \frac{dZ_b}{dt} = -k_d (\tau_b - \tau_c)^a $$

式中：$k_d$ 为冲刷系数；$\tau_b = \gamma_w n^2 v^2 / R^{1/3}$ 为床面实际水流剪切应力，由曼宁公式反演流速 $v$ 及水力半径 $R$ 得到；$\tau_c$ 为土体临界起动切应力；$a$ 为经验指数，对于黏性土常取1.0。侧向扩宽受静力失稳与水流冲刷共同控制，其演化方程通过类似动量传递推导获得。

### 5.3.2 瞬态渗流与浸润线演化计算
设定外部扰动边界条件为一典型双峰型洪水过程：初始水位处于死水位 25.0m，随后在7天内以匀速涨至设计洪水位 32.5m，维持该高水位运行5天后，遭遇上游水库控泄，水位以 1.2m/d 的速度骤降至 26.0m。

通过瞬态渗流场解算发现：涨水初期，浸润线主要在迎水坡表层土体缓慢推进；高水位维持期间，由于下部砂卵石层的高透水性，地下水压迅速向背水侧传递，导致背水侧坡脚处的承压水头显著上升。仿真计算提取出背水侧地表出流点处的最大水力坡降达到 0.72，十分接近粉质粉土层的临界水力坡降 0.85。若未设置减压井或导渗沟，该区域存在极高的流土破坏风险。

## 5.4 工程启示与应用建议

上述模型推导与仿真论证为实际防洪工程的设计与运行提供了科学理性的约束依据：
1. **防渗与导排的协同优化**：针对二元结构地基，一味加宽堤身并不能有效缓解坡脚管涌风险。工程上必须坚持“前堵后排”原则——在迎水侧构筑垂直防渗墙或铺设防渗土工膜切断主要渗径，同时在背水侧增设减压井、反滤盖重或贴坡排水体系，两者协同方能将出逸坡降严格控制在许用范围内。
2. **退水期的水库联合控调**：在防洪系统调度实践中，上游水库的泄洪不仅需考虑下游河道的行洪能力，更必须将下游重点堤防的“最大允许水位骤降速率”作为硬性刚性约束。通过梯级水库的平缓调蓄，可人为拉长退水历时，避免土体因内外部水压严重失衡而诱发工程灾难。
3. **数字孪生下的动态预警**：静态的阈值预警往往滞后于物理变形。建议将本文所述的瞬态渗流偏微分方程解算器嵌入流域防洪管理平台，引入基于粒子滤波（Particle Filter）的数据同化算法，利用实时传感器读数反演动态渗透边界，实现对未来 72 小时内 $F_s$ 演化轨迹的超前滚动预测。

## 本章小结

本章深入解剖了堤防安全评估的多物理场耦合机制，建立了涵盖 Richards 渗流偏微分控制方程、极限平衡非线性积分系统及溃口侵蚀动力学方程的核心数学体系。通过典型二元结构堤防的瞬态仿真，定量论证了非稳定渗流对孔隙水压力场与有效剪切强度的时空调制作用。相关理论模型不仅为揭示管涌、滑坡及漫顶溃决的内在逻辑提供了科学度量，更为流域防洪工程提供了从被动防御向主动控制转变的坚实算力基础。

## 拓展视野：水系统控制论中的堤防状态方程

若将目光从固体力学延伸至系统科学，“水系统控制论（Water System Cybernetics）”为重新审视堤防安全提供了全新范式。在这一视角下，整个流域防洪体系被抽象为一个大规模的分布式参数系统控制网络（Distributed Parameter Control System）。

堤防沿线的结构完整度（如安全系数 $F_s(x, t)$、渗透水头 $H(x, y, z, t)$）构成了系统的状态变量（State Variables）；上游水库的放水流量及自然降雨是驱动系统状态演化的控制输入（Control Inputs）与随机扰动（Stochastic Disturbances）。前文推导的偏微分方程及极限平衡模型，本质上即为该非线性系统的状态转移方程。通过遍布堤防的物联网传感器获取的观测向量，利用卡尔曼滤波（Kalman Filter）及变分伴随算子技术，能够连续不断地进行状态观测与时空同化。基于此，流域防洪调度就不再是单一追求水量平衡的水文调度，而升华为了一个带有复杂偏微分时空约束的非线性最优控制问题：在满足系统状态不进入危险域（即保证全线 $F_s(x,t) \ge [F_s]$）的前提下，实现流域洪灾损失的全局最小化配置。这种数学结构上的同构性，使得航空航天领域的自适应鲁棒控制理论得以跨界移植，成为未来智慧水利的核心引擎。

# -*- coding: utf-8 -*-
"""
教材：《洪水预报与防洪调度》
章节：第5章 堤防安全评估
功能：仿真“非稳定渗流-稳定安全系数-溃决演化-洪水淹没”闭环，输出KPI并绘图
"""

import numpy as np
import matplotlib.pyplot as plt
from scipy.sparse import diags, csc_matrix
from scipy.sparse.linalg import spsolve

# ------------------------------
# 0) 全局绘图设置（中文显示）
# ------------------------------
plt.rcParams["font.sans-serif"] = ["SimHei", "Microsoft YaHei", "Arial Unicode MS", "DejaVu Sans"]
plt.rcParams["axes.unicode_minus"] = False

# ------------------------------
# 1) 关键参数（可按工程项目调整）
# ------------------------------
np.random.seed(42)  # 固定随机种子，便于教学复现

T_hour = 240.0       # 总仿真时长（h）
dt_hour = 1.0        # 时间步长（h）
dt_sec = dt_hour * 3600.0
t = np.arange(0.0, T_hour + dt_hour, dt_hour)
nt = len(t)

# 空间离散（堤基一维渗流剖面）
L = 60.0             # 计算域长度（m）
Nx = 121             # 网格点
x = np.linspace(0.0, L, Nx)
dx = x[1] - x[0]

# 水位与几何
H_land_m = 21.0      # 背水侧基准水位（m）
levee_crest_m = 33.0 # 堤顶高程（m）

# 水力扩散系数（真值用于构造“观测”，模型值用于预报）
D_true = 0.080       # m2/s
D_model = 0.060      # m2/s（带偏差，体现模型不确定性）

# 边坡稳定参数（简化无限边坡法）
slope_angle_deg = 20.0
phi_deg = 30.0
c_kpa = 18.0
gamma_sat = 19.0e3   # N/m3
slip_depth = 6.0     # m
i_crit = 0.85        # 临界出逸坡降

# 损伤与溃决参数
damage_crit = 0.65
k_damage_i = 0.080
k_damage_fs = 0.150
k_breach = 0.45      # m/h
B_max = 16.0         # m
C_weir = 0.90        # 宽顶堰综合系数

# 保护区淹没参数
polder_area_m2 = 2.5e7
drain_coeff = 25.0
risk_depth_ref = 1.5

# 数据同化参数
sensor_x = np.array([8.0, 18.0, 35.0, 52.0])
sensor_idx = np.array([int(round(v / dx)) for v in sensor_x], dtype=int)
obs_every = 3
obs_sigma = 0.08
assim_gain = 0.55


def build_river_level(t_hour):
    """构造双峰外水位过程（绝对高程，m）"""
    base = 24.0
    peak1 = 6.0 * np.exp(-0.5 * ((t_hour - 72.0) / 18.0) ** 2)
    peak2 = 4.5 * np.exp(-0.5 * ((t_hour - 132.0) / 16.0) ** 2)
    tail = 1.2 * np.exp(-np.maximum(t_hour - 150.0, 0.0) / 45.0)
    H = base + peak1 + peak2 + tail
    return np.clip(H, 22.0, 34.2)


def build_matrix(n, r):
    """组装隐式差分三对角矩阵"""
    main = (1.0 + 2.0 * r) * np.ones(n - 2)
    off = -r * np.ones(n - 3)
    A = diags([off, main, off], offsets=[-1, 0, 1], format="csc")
    return csc_matrix(A)


def run_simulation(diffusivity, H_river, params, truth_h=None, obs_dict=None, use_assim=False):
    """
    运行一次堤防安全评估仿真
    diffusivity: 水力扩散系数
    truth_h: 真值水头场（用于RMSE评估）
    obs_dict: 观测字典（k -> 传感器观测向量）
    use_assim: 是否启用同化
    """
    r = diffusivity * dt_sec / (dx ** 2)
    A = build_matrix(Nx, r)

    # 状态变量初始化
    h = np.zeros(Nx)                 # 当前步水头（相对背水侧）
    h_fields = np.zeros((nt, Nx))    # 全时段水头场

    exit_grad = np.zeros(nt)         # 背水侧出逸坡降
    FS = np.zeros(nt)                # 稳定安全系数
    damage = np.zeros(nt)            # 累计损伤（0~1）
    breach_width = np.zeros(nt)      # 溃口宽度
    Q_breach = np.zeros(nt)          # 溃口流量
    V_inland = np.zeros(nt)          # 保护区蓄水体积
    H_inland = np.zeros(nt)          # 保护区平均淹深
    risk = np.zeros(nt)              # 综合风险指数
    rmse = np.full(nt, np.nan)       # 相对真值误差

    breach_start_idx = None

    theta = np.deg2rad(params["slope_angle_deg"])
    phi = np.deg2rad(params["phi_deg"])
    c_pa = params["c_kpa"] * 1000.0
    slip_zone = (x >= 0.45 * L) & (x <= 0.90 * L)

    for k in range(nt):
        # 边界条件：迎水侧定水头，背水侧定水头
        left_bc = max(H_river[k] - params["H_land_m"], 0.0)
        right_bc = 0.0

        # 非稳定渗流隐式推进
        if k == 0:
            h[:] = np.linspace(left_bc, right_bc, Nx)
        else:
            b = h[1:-1].copy()
            b[0] += r * left_bc
            b[-1] += r * right_bc
            h[1:-1] = spsolve(A, b)
            h[0] = left_bc
            h[-1] = right_bc

        # 数据同化（Nudging）
        if use_assim and (obs_dict is not None) and (k in obs_dict):
            y_obs = obs_dict[k]
            h[sensor_idx] = h[sensor_idx] + params["assim_gain"] * (y_obs - h[sensor_idx])

        h_fields[k, :] = h

        # 出逸坡降
        exit_grad[k] = max((h[-2] - h[-1]) / dx, 0.0)

        # 极限平衡简化安全系数
        ru = np.clip(np.mean(h[slip_zone]) / (params["slip_depth"] + 1e-6), 0.0, 0.99)
        term_c = c_pa / (params["gamma_sat"] * params["slip_depth"] * np.sin(theta) * np.cos(theta))
        term_phi = (np.tan(phi) / np.tan(theta)) * (1.0 - ru)
        FS[k] = max(term_c + term_phi, 0.05)

        # 损伤累计
        if k > 0:
            over_i = max(exit_grad[k] / params["i_crit"] - 1.0, 0.0)
            over_fs = max(1.0 - FS[k], 0.0)
            dD = (params["k_damage_i"] * over_i + params["k_damage_fs"] * over_fs) * dt_hour
            damage[k] = np.clip(damage[k - 1] + dD, 0.0, 1.0)

        # 溃决触发判定
        if breach_start_idx is None:
            if (damage[k] >= params["damage_crit"]) and (exit_grad[k] > 0.6 * params["i_crit"]):
                breach_start_idx = k

        # 溃口展宽
        if k > 0:
            breach_width[k] = breach_width[k - 1]
        if breach_start_idx is not None and k >= breach_start_idx:
            h_over = max(H_river[k] - params["levee_crest_m"], 0.0)
            i_over = max(exit_grad[k] / params["i_crit"] - 1.0, 0.0)
            dB = params["k_breach"] * (h_over + 0.8 * i_over) ** 1.25 * dt_hour
            breach_width[k] = np.clip(breach_width[k] + dB, 0.0, params["B_max"])

        # 溃口流量
        if breach_width[k] > 0.0:
            delta_h = max(H_river[k] - params["H_land_m"], 0.0)
            Q_breach[k] = params["C_weir"] * breach_width[k] * delta_h ** 1.5

        # 洪水演进（保护区蓄排）
        if k > 0:
            Q_drain = params["drain_coeff"] * np.sqrt(max(H_inland[k - 1], 0.0))
            V_inland[k] = max(V_inland[k - 1] + (Q_breach[k] - Q_drain) * dt_sec, 0.0)
            H_inland[k] = V_inland[k] / params["polder_area_m2"]

        # 综合风险指标
        risk[k] = np.clip(
            0.30 * (exit_grad[k] / params["i_crit"])
            + 0.35 * (1.0 / FS[k] / 2.0)
            + 0.20 * damage[k]
            + 0.15 * np.clip(H_inland[k] / params["risk_depth_ref"], 0.0, 2.0),
            0.0, 1.5
        )

        # 与真值误差
        if truth_h is not None:
            rmse[k] = np.sqrt(np.mean((h - truth_h[k]) ** 2))

    return {
        "h_fields": h_fields,
        "exit_grad": exit_grad,
        "FS": FS,
        "damage": damage,
        "breach_width": breach_width,
        "Q_breach": Q_breach,
        "V_inland": V_inland,
        "H_inland": H_inland,
        "risk": risk,
        "rmse": rmse,
        "breach_start_idx": breach_start_idx,
    }


def print_kpi_table(rows):
    """打印ASCII KPI表格"""
    w1 = max(len(r[0]) for r in rows)
    w2 = max(len(r[1]) for r in rows)
    line = f"+{'-' * (w1 + 2)}+{'-' * (w2 + 2)}+"
    print("\n=== KPI结果表（第5章 堤防安全评估）===")
    print(line)
    print(f"| {'指标'.ljust(w1)} | {'数值'.ljust(w2)} |")
    print(line)
    for k, v in rows:
        print(f"| {k.ljust(w1)} | {v.ljust(w2)} |")
    print(line)


if __name__ == "__main__":
    # 2) 边界过程与观测构造
    H_river = build_river_level(t)
    params = {
        "H_land_m": H_land_m,
        "levee_crest_m": levee_crest_m,
        "slope_angle_deg": slope_angle_deg,
        "phi_deg": phi_deg,
        "c_kpa": c_kpa,
        "gamma_sat": gamma_sat,
        "slip_depth": slip_depth,
        "i_crit": i_crit,
        "damage_crit": damage_crit,
        "k_damage_i": k_damage_i,
        "k_damage_fs": k_damage_fs,
        "k_breach": k_breach,
        "B_max": B_max,
        "C_weir": C_weir,
        "polder_area_m2": polder_area_m2,
        "drain_coeff": drain_coeff,
        "risk_depth_ref": risk_depth_ref,
        "assim_gain": assim_gain,
    }

    # 真值模拟
    truth = run_simulation(D_true, H_river, params)

    # 构造传感器观测
    obs_dict = {}
    for k in range(0, nt, obs_every):
        obs_dict[k] = truth["h_fields"][k, sensor_idx] + np.random.normal(0.0, obs_sigma, len(sensor_idx))

    # 开环与同化模拟
    open_loop = run_simulation(D_model, H_river, params, truth_h=truth["h_fields"], use_assim=False)
    assim = run_simulation(D_model, H_river, params, truth_h=truth["h_fields"], obs_dict=obs_dict, use_assim=True)

    # 3) KPI统计
    peak_level = np.max(H_river)
    max_exit = np.max(assim["exit_grad"])
    min_fs = np.min(assim["FS"])
    max_damage = np.max(assim["damage"])
    breach_time_txt = "未触发" if assim["breach_start_idx"] is None else f"{t[assim['breach_start_idx']]:.0f} h"
    max_breach_width = np.max(assim["breach_width"])
    max_q_breach = np.max(assim["Q_breach"])
    vol_breach_mcm = np.sum(assim["Q_breach"]) * dt_sec / 1.0e6
    max_depth = np.max(assim["H_inland"])
    rmse_open = np.nanmean(open_loop["rmse"])
    rmse_assim = np.nanmean(assim["rmse"])
    rmse_gain = (rmse_open - rmse_assim) / (rmse_open + 1e-12) * 100.0
    high_risk_hours = np.sum(assim["risk"] >= 1.0) * dt_hour

    kpis = [
        ("洪峰外水位", f"{peak_level:.2f} m"),
        ("最大出逸坡降", f"{max_exit:.3f}"),
        ("最小安全系数FS", f"{min_fs:.3f}"),
        ("最大累计损伤", f"{max_damage:.3f}"),
        ("溃决触发时刻", breach_time_txt),
        ("最大溃口宽度", f"{max_breach_width:.2f} m"),
        ("最大溃口流量", f"{max_q_breach:.2f} m3/s"),
        ("溃决总水量", f"{vol_breach_mcm:.3f} 百万m3"),
        ("最大淹没深度", f"{max_depth:.3f} m"),
        ("开环RMSE", f"{rmse_open:.4f} m"),
        ("同化RMSE", f"{rmse_assim:.4f} m"),
        ("同化误差降幅", f"{rmse_gain:.2f} %"),
        ("高风险历时", f"{high_risk_hours:.0f} h"),
    ]
    print_kpi_table(kpis)

    # 4) 绘图
    fig, axes = plt.subplots(2, 2, figsize=(14, 10), sharex=True)

    ax = axes[0, 0]
    ax.plot(t, H_river, color="#1f77b4", lw=2.2, label="河道外水位")
    ax.axhline(levee_crest_m, color="black", ls="--", lw=1.4, label="堤顶高程")
    ax.axhline(H_land_m, color="gray", ls=":", lw=1.2, label="背水侧基准水位")
    if assim["breach_start_idx"] is not None:
        ax.axvline(t[assim["breach_start_idx"]], color="red", ls="--", lw=1.2, label="溃决触发")
    ax.set_ylabel("水位 (m)")
    ax.set_title("(a) 外部水位边界")
    ax.grid(alpha=0.25)
    ax.legend(loc="upper right", fontsize=9)

    ax = axes[0, 1]
    ax.plot(t, open_loop["exit_grad"], color="#ff7f0e", lw=1.6, alpha=0.75, label="开环出逸坡降")
    ax.plot(t, assim["exit_grad"], color="#2ca02c", lw=2.0, label="同化出逸坡降")
    ax.axhline(i_crit, color="red", ls="--", lw=1.3, label="临界坡降")
    ax.set_ylabel("坡降 (-)")
    ax.set_title("(b) 背水侧出逸坡降")
    ax.grid(alpha=0.25)
    ax.legend(loc="upper right", fontsize=9)

    ax = axes[1, 0]
    ax.plot(t, open_loop["FS"], color="#9467bd", lw=1.5, alpha=0.75, label="开环FS")
    ax.plot(t, assim["FS"], color="#8c564b", lw=2.0, label="同化FS")
    ax.axhline(1.0, color="red", ls="--", lw=1.2, label="FS=1.0")
    ax.set_ylabel("安全系数 FS")
    ax.grid(alpha=0.25)
    ax.set_title("(c) 稳定与损伤演化")
    ax2 = ax.twinx()
    ax2.plot(t, assim["damage"], color="#17becf", lw=1.8, label="累计损伤D")
    ax2.axhline(damage_crit, color="#17becf", ls=":", lw=1.2, label="损伤阈值")
    ax2.set_ylabel("损伤 D")
    l1, lb1 = ax.get_legend_handles_labels()
    l2, lb2 = ax2.get_legend_handles_labels()
    ax2.legend(l1 + l2, lb1 + lb2, loc="upper right", fontsize=8)

    ax = axes[1, 1]
    ax.plot(t, assim["Q_breach"], color="#d62728", lw=2.0, label="溃口流量")
    ax.plot(t, assim["breach_width"], color="#bcbd22", lw=1.8, label="溃口宽度")
    ax.set_ylabel("流量/宽度")
    ax.grid(alpha=0.25)
    ax.set_title("(d) 溃口动力学与淹没演进")
    ax2 = ax.twinx()
    ax2.plot(t, assim["H_inland"], color="#1f77b4", lw=2.0, label="保护区淹深")

    l1, lb1 = ax.get_legend_handles_labels()

    for a in axes[1, :]:

    fig.suptitle("第5章 堤防安全评估：多过程耦合仿真", fontsize=14)
    plt.tight_layout()
    plt.show()

代码解读（约800字）  
这份脚本把第5章的核心思路组织为一个“可计算闭环”：先用非稳定渗流方程更新堤体内部水头，再把孔压效应映射到边坡稳定安全系数，随后用超限坡降与稳定不足共同驱动损伤累计，当损伤达到阈值后触发溃口展宽，最后把溃口流量输入保护区蓄排方程得到淹没过程，并汇总成KPI供调度决策使用。  
在渗流层面，脚本采用一维扩散型控制方程并用隐式差分离散，核心优势是数值稳定性好，适合长历时洪水过程。迎水侧和背水侧分别作为定水头边界，迎水侧水位过程由双峰洪水位函数构造，能够体现“先涨后落、二次抬升”的典型工况。隐式离散后形成三对角线性方程组，利用 `scipy.sparse` 与 `spsolve` 高效求解，既符合教学可读性，也保留工程计算的规模扩展空间。  
稳定分析部分采用简化极限平衡表达式，把滑动带平均水头换算为孔压比 `ru`，再进入黏聚力项与摩擦项组合得到 `FS`。这一步对应教材中“渗流场-力学响应耦合”的思想：水头上升会抬高孔隙水压力、削弱有效应力，从而降低抗剪强度。脚本设置 `FS=1.0` 作为失稳警戒线，便于与图形和KPI联动解释。  
损伤与溃决模块体现了失效演化机制。`damage` 不是瞬时判别，而是随时间累积：若出逸坡降超过临界值或 `FS` 低于1，则按权重累加。该处理反映“慢变劣化+阈值突变”的工程现实。达到 `damage_crit` 后进入溃口动力学，溃口宽度按超限水力条件增长，并通过宽顶堰公式计算溃口流量 `Q_breach`。  
洪水演进层面，脚本把溃口流量输入保护区体积平衡方程，同时考虑排涝项，得到平均淹深 `H_inland`。这样就把堤防局部失效与区域洪灾后果打通，形成从工程状态到灾害影响的因果链。风险指标 `risk` 则把坡降、`FS`、损伤和淹深统一量化，为防洪调度系统提供可直接调用的综合安全信号。  
“状态感知-同化”通过传感器点实现：先用真值扩散系数生成带噪观测，再用偏差模型进行开环模拟与同化模拟对比。同化采用 Nudging 形式，代码简单但效果直观，可显著降低水头场RMSE，体现了教材强调的数据同化价值。  
最后输出的KPI表包括最大坡降、最小 `FS`、溃决时刻、最大溃口宽度、溃决总水量、最大淹深与同化误差降幅等指标；配套4幅图对应“外部致灾边界、渗流响应、结构安全演化、灾害后果扩散”四个层次，适合作为第5章课堂演示与作业扩展的统一脚手架。
