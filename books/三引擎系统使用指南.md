## 二、工具使用

### 2.1 三引擎评审系统

**文件**: `three_engine_simple.py`

**功能**: 对单个章节进行Codex和Gemini双引擎评审

**使用方法**:
```bash
cd D:\cowork\教材\chs-books-v2\books
python three_engine_simple.py <book_name> <chapter_num>
```

**示例**:
```bash
# 评审reservoir-operation-optimization的第2章
python three_engine_simple.py reservoir-operation-optimization 02

# 评审flood-forecasting-control的第1章
python three_engine_simple.py flood-forecasting-control 01
```

**输出文件**:
```
<book_name>/reviews/
├── ch<num>_codex_review.txt    # Codex技术评审
├── ch<num>_gemini_review.txt   # Gemini工程评审
└── ch<num>_summary.md          # Claude评审汇总
```

### 2.2 三引擎修改系统

**文件**: `three_engine_fix.py`

**功能**: 根据评审意见执行修改

**使用方法**:
```bash
# 第一步: 生成修改方案
python three_engine_fix.py --book <book_name> --chapter <num>

# 第二步: 执行修改
python three_engine_fix.py --book <book_name> --chapter <num> \
    --engine codex --instruction修改指令.txt
```

**示例**:
```bash
# 生成修改方案
python three_engine_fix.py --book reservoir-operation-optimization --chapter 02

# 使用Codex执行修改
python three_engine_fix.py --book reservoir-operation-optimization --chapter 02 \
    --engine codex --instruction reviews/ch02_fix_instruction.txt
```

## 三、典型工作流程

### 3.1 单章评审与修改

**步骤1: 运行三引擎评审**
```bash
python three_engine_simple.py reservoir-operation-optimization 02
```

**步骤2: Claude分析评审意见**

打开生成的评审汇总文件:
```
reservoir-operation-optimization/reviews/ch02_summary.md
```

阅读Codex和Gemini的评审意见,然后在Claude对话中请求:
```
请分析reservoir-operation-optimization/ch02的Codex和Gemini评审意见,
生成详细的修改方案,包括:
1. 问题优先级排序
2. 具体修改建议
3. 修改执行计划
```

**步骤3: 生成修改指令**

Claude会生成修改方案,保存为:
```
reservoir-operation-optimization/reviews/ch02_fix_instruction.txt
```

**步骤4: 执行修改**
```bash
python three_engine_fix.py --book reservoir-operation-optimization --chapter 02 \
    --engine codex --instruction reviews/ch02_fix_instruction.txt
```

**步骤5: 验证修改**

Claude会自动验证修改结果,检查:
- 文件完整性
- 参考文献
- 代码块
- CHS关联

### 3.2 批量评审

**方法1: 使用Shell循环**
```bash
cd D:\cowork\教材\chs-books-v2\books

# 评审reservoir-operation-optimization的所有章节
for i in 01 02 03 04 05 06 07 08; do
    python three_engine_simple.py reservoir-operation-optimization $i
    sleep 60  # 避免API限流
done
```

**方法2: 使用Python脚本**
```python
# batch_review.py
books = ["reservoir-operation-optimization", "flood-forecasting-control"]
for book in books:
    for chapter in range(1, 9):
        os.system(f"python three_engine_simple.py {book} {chapter:02d}")
        time.sleep(60)
```

## 四、评审标准

### 4.1 Codex评审重点

**技术准确性** (P1):
- 公式推导是否严谨
- 符号使用是否一致
- 单位体系是否闭合
- 算法描述是否完整

**数学严谨性** (P1):
- 定理证明是否完整
- 假设条件是否明确
- 边界条件是否考虑
- 收敛性是否分析

**工程可行性** (P2):
- 算法复杂度是否可接受
- 实时性是否满足要求
- 可扩展性是否良好

### 4.2 Gemini评审重点

**工程实践** (P1):
- 案例数据是否真实
- 工程参数是否合理
- 操作流程是否可行
- 安全约束是否完备

**行业规范** (P2):
- 术语使用是否规范
- 标准引用是否准确
- 设计规范是否符合
- 工程经验是否合理

**内容质量** (P2):
- 逻辑是否清晰
- 结构是否合理
- 案例是否充分
- 图表是否完整

### 4.3 Claude综合评审

**学术质量**:
- 创新性
- 完整性
- 一致性
- 可读性

**参考文献**:
- 覆盖率
- 质量
- 自引率
- 格式规范

**CHS关联**:
- 理论映射
- 术语统一
- 概念一致

## 五、常见问题

### 5.1 Codex CLI问题

**问题**: `FileNotFoundError: codex`

**解决**:
```bash
# 检查Codex是否安装
which codex

# 如果未安装,使用npm安装
npm install -g @anthropic-ai/codex-cli

# 配置Codex
codex config set api_key YOUR_API_KEY
```

### 5.2 Gemini CLI问题

**问题**: `Authentication required`

**解决**:
```bash
# 重新认证Gemini
gemini auth login

# 检查认证状态
gemini auth status
```

### 5.3 编码问题

**问题**: `UnicodeEncodeError: 'gbk' codec`

**解决**: 使用`three_engine_simple.py`,已内置UTF-8处理

### 5.4 API限流

**问题**: `Rate limit exceeded`

**解决**:
- 在批量评审时增加sleep时间
- 使用`--delay`参数控制请求间隔
- 分批次处理,避免短时间大量请求
