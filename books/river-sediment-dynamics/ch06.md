# 第6章 案例：黄河/长江泥沙

## 本章导读

河流泥沙动力学不仅是流体力学与地貌学的交叉学科，更是指导大型水利枢纽工程建设与运行的基础理论。本章作为《河流泥沙动力学与河床演变》的第6章，聚焦于全球最具代表性的两条河流——黄河与长江。黄河以其极高的含沙量和复杂的河床演变规律闻名于世，其水沙关系的不协调是导致下游“悬河”发育的根本原因；而长江作为我国第一大河，三峡水库的建成蓄水深刻改变了中下游的水沙通量，引发了长距离、长时期的河道冲刷与形态调整。

本章围绕黄河小浪底调水调沙与长江三峡清水下泄两大典型案例展开，系统阐述高含沙水流与清水冲刷的基本概念、数学建模方法、数值仿真技术及工程应用启示。通过剖析这两大复杂系统工程中的泥沙问题，读者能够将前序章节所述的泥沙起动、推移质与悬移质运动、河床演变等基础理论融会贯通，掌握解决实际宏观水利工程泥沙问题的方法论。

## 6.1 基本概念与理论框架

黄河与长江的泥沙问题在物理机制上呈现出截然不同的特征，构成了泥沙动力学研究的两个极端面：一端是超高浓度的泥沙输运，另一端是含沙量骤减后的河床剧烈冲刷。

### 6.1.1 黄河小浪底调水调沙机制

黄河泥沙的核心矛盾在于“水少沙多，水沙异源”。小浪底水利枢纽是黄河干流三门峡以下唯一能够进行较大库容调节的控制性工程。调水调沙的基本概念是通过工程手段，在短时间内塑造出人工洪峰，利用强大的水流动力将水库内的淤沙以及下游河床的沉积物输送入海。

这一过程涉及高含沙水流的复杂动力学行为。当含沙量达到一定阈值（通常体积浓度超过10%）时，水流的流变性质发生根本改变，由牛顿流体转变为非牛顿流体（主要表现为宾汉流体特征）。此时，细颗粒泥沙在水中形成絮凝网状结构，具备了屈服应力。在调水调沙期间，异重流（Density Current）的排沙机制发挥着核心作用。高含沙洪水进入水库后，由于密度较大，潜入清水底部沿库底向坝前运动。合理调度排沙底孔的开启时机，能够将异重流直接排出库外，从而维持水库的有效库容。

### 6.1.2 长江三峡清水下泄效应

三峡水库采用“蓄清排浑”的运行方式，汛期降低水位排泄携带大量泥沙的洪水，枯水期蓄水发电。水库蓄水后，大量泥沙拦截在库内，导致坝下泄放的水流含沙量急剧降低，远低于水流的挟沙能力，这种现象称为“清水下泄”。

清水下泄对下游河道产生深远的物理生态影响。水流为恢复输沙平衡，必然大量冲刷河床。这一过程伴随着“床沙粗化”（Bed Armoring）现象：水流优先带走床面上的细颗粒泥沙，留下抗冲能力较强的粗颗粒泥沙。粗颗粒在床面形成一层保护层，增加床面阻力，限制了冲刷的进一步下切。长江中下游的冲刷不仅表现为纵向的河底高程降低，还引发了横向的崩岸以及河型的局部转化，对防洪堤防、航道通航条件以及取水设施产生了深远影响。

### 6.1.3 一维泥沙模型验证理论

针对长河段的泥沙输移与河床演变，三维或二维模型计算代价过高且受限于边界数据的精度，一维非平衡泥沙数学模型是目前工程界最基础且成熟的理论框架。一维模型基于断面平均的水动力学参数，结合泥沙非平衡输移方程（即泥沙浓度调整存在空间滞后效应）进行求解。验证过程要求模型计算的水位、流量、含沙量以及断面冲淤变化量与实测数据高度吻合，从而确认模型参数（如床面糙率、恢复饱和系数、泥沙沉速）的合理性。

## 6.2 数学建模与求解方法

本节从数学角度建立描述黄河/长江泥沙动力学演变的核心模型，推导关键公式，并分析相关参数的物理与工程意义。

### 6.2.1 水动力学控制方程

河流一维非恒定流动的基本控制方程为圣维南方程组（Saint-Venant Equations），由质量守恒与动量守恒推导而来。
连续方程：
$$ \frac{\partial A}{\partial t} + \frac{\partial Q}{\partial x} = q_l $$
动量方程：
$$ \frac{\partial Q}{\partial t} + \frac{\partial}{\partial x}\left(\frac{Q^2}{A}\right) + g A \frac{\partial Z}{\partial x} + g A S_f = 0 $$
式中：$A$为过水断面面积；$Q$为流量；$t$为时间；$x$为流程坐标；$q_l$为侧向入流；$g$为重力加速度；$Z$为水位；$S_f$为摩阻斜坡，通常采用曼宁公式估算 $S_f = \frac{n^2 Q |Q|}{A^2 R^{4/3}}$，其中$n$为糙率，$R$为水力半径。

### 6.2.2 悬移质不平衡输沙方程

自然界中的泥沙输移极少处于绝对的平衡状态。一维非平衡输沙方程描述了含沙量向水流挟沙能力调整的过程：
$$ \frac{\partial (A S)}{\partial t} + \frac{\partial (Q S)}{\partial x} = -\alpha \omega B (S - S_*) $$
式中：$S$为断面平均含沙量；$S_*$为水流挟沙能力（可通过张瑞瑾公式或Engelund-Hansen公式计算）；$B$为河宽；$\omega$为泥沙沉速；$\alpha$为恢复饱和系数，表征含沙量恢复到挟沙能力的快慢程度。

针对长江三峡下游清水冲刷，水流挟沙能力$S_*$远大于实际含沙量$S$，方程右侧为正，水流从河床中挟带泥沙，引发冲刷；针对黄河调水调沙期，$S > S_*$，发生大规模淤积或通过异重流输送。

### 6.2.3 高含沙流变模型与变形方程

对于黄河的高含沙水流，流体表现出宾汉流体特性，其本构方程为：
$$ \tau = \tau_y + \eta_B \frac{du}{dy} \quad (\tau > \tau_y) $$
式中：$\tau$为剪切应力；$\tau_y$为屈服应力，与泥沙浓度呈指数关系；$\eta_B$为宾汉极限粘滞系数；$\frac{du}{dy}$为速度梯度。这种流变特性使得高含沙水流的流速分布趋于平缓，阻力特性发生改变。

河床变形由泥沙连续方程（Exner方程）控制：
$$ \rho_s (1 - p) \frac{\partial A_b}{\partial t} = \alpha \omega B (S - S_*) + \frac{\partial G_b}{\partial x} $$
式中：$\rho_s$为泥沙密度；$p$为床面孔隙率；$A_b$为断面冲淤面积；$G_b$为推移质输沙率。通过耦合求解上述水动力、泥沙输移与河床变形方程，即可实现对复杂河段河床演变的定量描述。

### 6.2.4 数值求解方法

上述偏微分方程组具有高度非线性特征，无法求得解析解。工程应用中多采用有限差分法（Finite Difference Method）或有限体积法（Finite Volume Method）。以Preissmann隐式四点偏心差分格式为例，将时间导数与空间导数离散化：
$$ \frac{\partial f}{\partial t} \approx \frac{f_{i}^{n+1} + f_{i+1}^{n+1} - f_{i}^{n} - f_{i+1}^{n}}{2 \Delta t} $$
$$ \frac{\partial f}{\partial x} \approx \theta \frac{f_{i+1}^{n+1} - f_{i}^{n+1}}{\Delta x} + (1-\theta) \frac{f_{i+1}^{n} - f_{i}^{n}}{\Delta x} $$
其中 $\theta$ 为时间加权系数（$0.5 < \theta \le 1$保证无条件稳定）。离散后的非线性代数方程组通过牛顿-拉夫逊（Newton-Raphson）迭代法联合求解。求解过程中，水流与泥沙往往采用解耦（Decoupled）或半耦合方式计算，即先求解水流场更新水动力参数，再代入泥沙方程计算输沙率和床面变形，最后更新断面地形进入下一时间步。

## 6.4 工程启示与应用建议

数学建模与数值仿真的最终目的是指导大型水利工程的科学规划与安全运行。基于对黄河与长江典型案例的深入分析，提出以下工程应用建议：

第一，**构建流域梯级水库联合调控体系**。对于黄河泥沙管理，单一依赖小浪底水库的调水调沙难以长久维持。必须结合上游万家寨水库、中游三门峡水库实施水沙联合调度，利用上游清水冲刷中游淤积，再由小浪底接力输沙入海。在数学模型中，这种联合调度体现为空间多节点输入条件的非线性优化组合问题。

第二，**高度关注清水下泄引发的航道与堤防次生灾害**。三峡下游长距离河道下切导致枯水期水位大幅下降，对部分取水泵站和航道浅滩演变产生了不利影响。同时，河床深槽冲刷加剧了岸坡失稳的风险。工程实践中必须结合二维或三维水沙动力学模型对荆江弯曲河段等关键节点进行精细化评估，提前部署沉排护底、抛石护岸等工程措施，锁定微观河势。

第三，**建立泥沙测报与模型反馈修正闭环机制**。自然河流的地形与水文条件时刻处于演变之中。理论模型中的床沙级配、临界起动切应力等参数需依赖汛期水文站的实时监测数据进行动态标定（Data Assimilation）。利用现代声学多普勒流速剖面仪（ADCP）等技术获取的高精度场数据，定期对一维及二维泥沙模型进行冷启动更新，能够有效提高中长期演变预测的可靠性。

## 拓展视野：水沙演变与水系统控制论的同构性

宏观水系泥沙输运的过程，在系统科学视角下不再仅仅是经典的流体力学问题，它与控制科学在数学结构上展现出高度的同构性（Isomorphism）。水网控制论将复杂的河流网络抽象为有向图与状态空间模型。在这一框架中，河段是具有延迟和非线性阻尼特性的动态传递函数，水库大坝是受控的节点阀门，而水沙通量则是系统的控制输入与状态变量。

在黄河多水库联合调水调沙优化调度中，研究人员已经开始应用现代控制理论（如模型预测控制，Model Predictive Control, MPC）。通过求解含有约束条件的哈密顿-雅可比-贝尔曼（HJB）方程，将最大化排沙比、最小化沿程漫滩风险设为多目标泛函，系统能够在庞大的状态空间中自动寻优，输出坝群的最优开启时序与流量组合。类似的优化控制框架在大型跨流域调水工程（如南水北调工程）的明渠水力学控制中同样得到证实。泥沙动力学与控制论的深度交叉，正在推动传统水利工程向智慧水利、数字孪生流域加速演进。

def process_terms(t, y, p):
    """计算某时刻的水沙过程量。"""
    M_raw, eta = y
    M = max(M_raw, 1.0e5)  # 避免出现非物理负库存

    # 年内日期（用于季节循环）
    day = t % DAYS_PER_YEAR

    # 流量：季节项 + 洪峰项
    season_q = 1.0 + p["q_amp"] * np.sin(2.0 * np.pi * (day - p["q_phase"]) / DAYS_PER_YEAR)
    flood = 1.0 + p["flood_amp"] * np.exp(-0.5 * ((day - p["flood_center"]) / p["flood_width"]) ** 2)
    Q = p["Q_mean"] * max(0.2, season_q) * flood

    # 水深随流量变化（简化幂律）
    h = p["h_ref"] * max(0.2, (Q / p["Q_mean"]) ** 0.40)

    # 控制体积与平均含沙浓度
    volume = p["B"] * p["L"] * h
    C = M / max(volume, 1.0)  # kg/m3

    # 来沙通量（考虑上游拦沙）
    season_in = 1.0 + p["qin_amp"] * np.sin(2.0 * np.pi * (day - p["qin_phase"]) / DAYS_PER_YEAR)
    q_in = p["qin0"] * max(0.1, season_in) * flood * (1.0 - p["trap"])  # kg/s

    # 输沙能力（经验幂律）
    q_cap = p["k_cap"] * (Q / 1000.0) ** p["m"] * (p["S"] ** p["n"])  # kg/s

    # 可供输沙受浓度约束
    q_available = p["wash_coeff"] * C * Q
    q_out = min(q_cap, q_available + p["wash_base"])  # kg/s

    # 河床剪切与再悬浮/淤积
    tau = RHO_W * GRAVITY * h * p["S"]        # Pa
    entr = p["k_entr"] * max(tau - p["tau_c"], 0.0)  # kg/day
    dep = p["k_dep"] * M                       # kg/day

    return {
        "Q": Q,
        "h": h,
        "C": C,
        "q_in": q_in,
        "q_out": q_out,
        "q_cap": q_cap,
        "entr": entr,
        "dep": dep,
    }


def rhs(t, y, p):
    """状态方程：
    dM/dt = (来沙-出沙)*86400 + 再悬浮 - 淤积
    deta/dt = (淤积-再悬浮)/(rho_s*(1-n)*B*L)
    """
    terms = process_terms(t, y, p)
    M_raw = y[0]

    dMdt = (terms["q_in"] - terms["q_out"]) * SEC_PER_DAY + terms["entr"] - terms["dep"]

    # 防止库存已极低时继续向负方向漂移
    if M_raw <= 1.0e5 and dMdt < 0.0:
        dMdt = 0.0

    bed_den = RHO_S * (1.0 - POROSITY) * p["B"] * p["L"]  # kg/m
    detadt = (terms["dep"] - terms["entr"]) / bed_den      # m/day

    return [dMdt, detadt]


def simulate_one_river(name, p, t_eval):
    """对单条河流执行数值积分，并回算过程量。"""
    y0 = [p["M0"], p["eta0"]]
    sol = solve_ivp(
        fun=lambda t, y: rhs(t, y, p),
        t_span=(t_eval[0], t_eval[-1]),
        y0=y0,
        t_eval=t_eval,
        method="RK45",
        max_step=2.0,
        rtol=1e-6,
        atol=1e-9,
    )
    if not sol.success:
        raise RuntimeError(f"{name} 数值积分失败: {sol.message}")

    n = len(sol.t)
    Q = np.zeros(n)
    C = np.zeros(n)
    q_in = np.zeros(n)
    q_out = np.zeros(n)
    q_cap = np.zeros(n)
    dep = np.zeros(n)
    entr = np.zeros(n)

    for i in range(n):
        terms = process_terms(sol.t[i], [sol.y[0, i], sol.y[1, i]], p)
        Q[i] = terms["Q"]
        C[i] = terms["C"]
        q_in[i] = terms["q_in"]
        q_out[i] = terms["q_out"]
        q_cap[i] = terms["q_cap"]
        dep[i] = terms["dep"]
        entr[i] = terms["entr"]

    return {
        "name": name,
        "t": sol.t,
        "M": sol.y[0],
        "eta": sol.y[1],
        "Q": Q,
        "C": C,
        "q_in": q_in,
        "q_out": q_out,
        "q_cap": q_cap,
        "dep": dep,
        "entr": entr,
    }


def compute_kpi(result):
    """计算KPI指标。"""
    t = result["t"]
    years = t[-1] / DAYS_PER_YEAR

    mean_qout = np.mean(result["q_out"])  # kg/s
    export_total = np.trapezoid(result["q_out"] * SEC_PER_DAY, t)  # kg
    export_annual_yt = export_total / years / 1e8                  # 亿吨/年

    mean_C = np.mean(result["C"])  # kg/m3
    delta_eta = result["eta"][-1] - result["eta"][0]  # m

    cap_util = 100.0 * np.mean(result["q_out"] / np.maximum(result["q_cap"], 1e-8))  # %
    net_dep_annual = np.trapezoid(result["dep"] - result["entr"], t) / years / 1e8    # 亿吨/年

    return {
        "mean_qout": mean_qout,
        "export_annual_yt": export_annual_yt,
        "mean_C": mean_C,
        "delta_eta": delta_eta,
        "cap_util": cap_util,
        "net_dep_annual": net_dep_annual,
    }

**800字代码解读**  
这段脚本对应第6.1“基本概念与理论框架”，核心思想是把河流泥沙系统拆成三个可计算环节：来沙输入、输沙输出、河床冲淤反馈。状态变量只有两个，`M` 表示控制河段内悬沙库存，`eta` 表示河床高程相对变化，这样既保留了水沙耦合主线，又不会把教学案例做得过于复杂。`process_terms` 函数负责在任意时刻计算水沙过程量：先用季节正弦项和洪峰高斯项合成流量 `Q`，再由 `Q` 推算水深 `h` 与浓度 `C`。来沙通量 `q_in` 用基准值乘季节波动，并引入 `trap` 表示库坝拦沙影响；出沙通量 `q_out` 取“输沙能力上限”和“可供输沙量”两者较小值，体现“有能力不一定有沙、有沙也受能力约束”的物理逻辑。输沙能力 `q_cap` 采用经验幂律（流量指数 `m`、坡降指数 `n`），与教材里“水动力决定搬运能力”的观点一致。  
`rhs` 是动力学方程。第一式是泥沙质量守恒：`dM/dt = (q_in-q_out)*86400 + entr - dep`，其中 `entr` 为再悬浮、`dep` 为淤积；第二式是 Exner 型床沙连续关系：`deta/dt` 与“淤积-再悬浮”的净值成正比，并除以床沙有效体积系数 `rho_s*(1-n)*B*L`。这一步把“悬沙变化”与“河床演变”连成闭环，是本章案例最关键的理论落地。`solve_ivp` 以日步长积分20年，得到连续时间解。  
KPI 部分不是简单打印曲线，而是提取可对比指标：平均输沙量、年均出境泥沙、平均含沙浓度、20年河床变化、输沙能力利用率、净淤积量。这样可以直接比较黄河和长江在同一框架下的响应差异。参数上，黄河设置了更高坡降和来沙振幅，反映“高含沙、强冲淤”；长江设置更大流量和更高拦沙系数，体现“径流大但上游工程显著改变来沙”。最后四幅图分别看短期过程（前3年通量波动）和长期趋势（20年冲淤、浓度、累计出沙），兼顾事件尺度与演变尺度。整体上，这是一套可教学、可扩展的最小模型；后续可继续引入分粒径级配、分段河道、库区调度边界等内容，逐步从“概念仿真”升级到“实测校准”。
