# 第6章 案例：A²/O工艺智能控制

## 6.1 导读

前五章分别建立了活性污泥模型（第1章）、多变量控制框架（第2章）、MBR膜过滤控制（第3章）、能耗优化方法（第4章）和软测量技术（第5章），形成了从机理建模、状态感知到动态控制、全局优化的完整方法体系。本章将以我国水处理领域应用最广泛的A$^2$/O（Anaerobic-Anoxic-Oxic，厌氧-缺氧-好氧）工艺为核心对象，将前述五章的离散方法综合集成，构建一个面向真实生产场景的全流程智能控制系统。

A$^2$/O工艺自20世纪70年代问世以来，凭借其结构简单、水力停留时间短、除磷脱氮效果好等优势，成为大中型城市污水处理厂的主流选择。该工艺的核心在于通过厌氧、缺氧、好氧三个空间物理隔离但物质流紧密串联的反应区，实现碳（COD/BOD）、氮（N）、磷（P）的同步去除。然而，这种同步去除机制带来了高度的控制复杂性：三段之间不仅通过外回流污泥（二沉池至厌氧段）和内回流混合液（好氧段至缺氧段）实现物质循环，且各生化过程对环境条件（如溶解氧DO、碳源分配）的需求存在本质矛盾。例如，厌氧释磷要求绝对的无氧环境，若外回流携带过多硝酸盐，则反硝化菌会与聚磷菌竞争碳源导致释磷失败；好氧硝化需要充足的溶解氧，但过高的溶解氧又会通过内回流破坏缺氧区的反硝化环境。

本章将给出一个10万吨/日规模的A$^2$/O污水厂智能控制系统完整设计方案。我们将从工艺机理的数学推导出发，剖析控制变量间的耦合关系，设计包含单回路反馈、多变量协调与全厂级优化的三层控制架构，并结合长周期仿真数据，深入解读系统在正常波动与极端冲击下的动态响应特征，最终展示从仪表选型、算法部署到运行维护的完整工程实践路径。

## 6.2 理论基础

### 6.2.1 A²/O工艺的过程描述与物理解释

A$^2$/O工艺的物理拓扑结构由进水分配网络、三个串联的生物反应区和固液分离单元组成。物质在系统中的转化遵循质量守恒定律和生化反应动力学。

1. **厌氧段（Anaerobic Zone）物理生化过程**：
进水与二沉池回流的含磷污泥在此混合。该区域没有游离态氧和化合态氧（硝酸盐）。聚磷菌（PAOs）在这种压迫环境下，分解细胞内积聚的聚磷酸盐，释放出能量和磷酸盐（$PO_4^{3-}$），同时利用释放的能量将进水中的快速可生物降解有机物（如挥发性脂肪酸VFA）吸收入细胞，合成聚-β-羟基丁酸（PHB）储存于胞内。其反应速率受到单体底物浓度的Monod限制：
$$r_{\mathrm{release}} = \mu_{\mathrm{PAO,max}} \left( \frac{S_{\mathrm{VFA}}}{K_{\mathrm{VFA}} + S_{\mathrm{VFA}}} \right) X_{\mathrm{PAO}}$$
这一过程的物理意义在于完成了碳源的“圈占”，为后续的过量吸磷储备了能量和碳骨架。

2. **缺氧段（Anoxic Zone）物理生化过程**：
厌氧出水与好氧段通过内回流泵送来的富含硝态氮（$S_{\mathrm{NO}}$）混合液在此交汇。反硝化异养菌利用前段未消耗完的有机物或内源呼吸产生的碳源，将硝酸盐还原为氮气逸出水面。反硝化速率的推导需同时考虑碳源和电子受体的双重限制：
$$r_{\mathrm{denit}} = \mu_{\mathrm{H,max}} \left( \frac{S_{\mathrm{S}}}{K_{\mathrm{S}} + S_{\mathrm{S}}} \right) \left( \frac{S_{\mathrm{NO}}}{K_{\mathrm{NO}} + S_{\mathrm{NO}}} \right) \left( \frac{K_{\mathrm{O,H}}}{K_{\mathrm{O,H}} + S_{\mathrm{O}}} \right) X_{\mathrm{H}}$$
式中最后项为氧抑制函数，物理上表示只要存在微量溶解氧（$S_O > K_{O,H}$），反硝化酶的活性就会被显著抑制。

3. **好氧段（Oxic Zone）物理生化过程**：
通过底部曝气系统强制传质溶解氧，发生三个核心反应：自养型硝化菌将氨氮（$S_{\mathrm{NH}}$）氧化为硝态氮；异养菌彻底降解残余有机物；聚磷菌利用胞内PHB氧化分解产生的能量，超量吸收环境中的磷酸盐合成聚磷酸盐。好氧段的氧平衡方程可以推导为：
$$\frac{dS_O}{dt} = K_{L}a(S_{O}^* - S_O) - r_{\mathrm{O2,cons}} + \frac{Q}{V}(S_{O,\mathrm{in}} - S_O)$$
其中，$K_{L}a$ 为体积氧传质系数，物理上受气泡大小、混合强度和流体黏度影响；$r_{\mathrm{O2,cons}}$ 为微生物耗氧速率总和；$S_{O}^*$ 为饱和溶解氧浓度。

将A$^2$/O系统扩展至活性污泥模型（ASM2d），状态变量（如各类微生物浓度、基质浓度）增至约20个，反应过程增至约15个。宏观操纵变量向量定义为：
$$\mathbf{u} = [K_{La}, Q_a, Q_r, Q_{\mathrm{carbon}}, Q_w]^T$$
分别代表好氧段曝气强度、内回流量、外回流量、外碳源投加量和排泥量。系统的被控变量（输出）映射为：
$$\mathbf{y} = [S_O, S_{\mathrm{NO,anox}}, S_{\mathrm{NH,e}}, TN_e, TP_e]^T$$

### 6.2.2 控制系统分层架构与数学模型

污水处理过程具有大惯性、非线性和多时间尺度的物理特性，智能控制系统必须采用严格的分层架构，以解耦不同时间尺度上的动态矛盾。

**1. 基础控制层（秒级响应）**：
主要处理物理设备的快速动态过程。包含DO-PI控制回路和污泥界面高度控制回路。DO控制通过调节鼓风机频率和阀门开度改变曝气量，其控制律为：
$$u_{\mathrm{blower}}(t) = K_p \left[ e(t) + \frac{1}{T_i} \int_0^t e(\tau) d\tau \right]$$
其中偏差 $e(t) = S_{O}^{\mathrm{ref}} - S_O(t)$。基础控制层的任务是抵抗流体压力波动等高频扰动，确保执行机构准确跟踪上位指令。

**2. 协调控制层（分钟级响应）**：
处理生化反应过程的物质平衡与交互耦合。本系统核心为前馈-反馈串级多变量控制，典型如DO-$S_{\mathrm{NO}}$双变量控制与外碳源投加控制。
外碳源投加的物理基础在于反硝化过程的电子供体/受体化学计量比。理论上还原1克硝酸盐氮需2.86克COD，考虑到微生物合成等因素，实际需求更高。碳源投加控制律设计为前馈与反馈结合：
$$Q_{\mathrm{carbon}} = K_c \cdot \max\left(0, \frac{S_{\mathrm{NO,anox}}}{C_{\mathrm{BOD,anox}}} - r_{\mathrm{CN,ref}} \right) \cdot V_{\mathrm{anox}} + G_{\mathrm{PI}}(TN_{\mathrm{e}}^{\mathrm{ref}} - TN_{\mathrm{e}})$$
式中，$r_{\mathrm{CN,ref}}$ 为保障彻底反硝化的临界碳氮比；前馈项用于快速补偿进水水质波动引发的瞬时碳源缺口，反馈项 $G_{\mathrm{PI}}$ 用于消除模型误差带来的稳态余差。

**3. 优化决策层（小时级响应）**：
处理全厂级的经济性与合规性博弈。通过求解一个带约束的非线性规划问题，每小时向协调层下发一次最优设定值 $\mathbf{u}_{\mathrm{sp}} = [S_O^{\mathrm{ref}}, S_{\mathrm{NO}}^{\mathrm{ref}}, r_{\mathrm{CN,ref}}]^T$。目标函数融合了出水水质指数（EQI，衡量污染物排放总量）和运行成本指数（OCI，衡量能耗与物耗）：
$$\min_{\mathbf{u}_{\mathrm{sp}}} \; J = w_1 \cdot \hat{f}_{\mathrm{EQI}}(\mathbf{u}_{\mathrm{sp}}) + w_2 \cdot \hat{f}_{\mathrm{OCI}}(\mathbf{u}_{\mathrm{sp}})$$
约束条件为出水水质的严格国标限值：
$$\text{s.t.} \quad S_{\mathrm{NH,e}} \leq 5 \; \mathrm{mg/L}, \quad TN_e \leq 15 \; \mathrm{mg/L}, \quad TP_e \leq 0.5 \; \mathrm{mg/L}$$
OCI的数学表达综合了曝气能耗（$AE$）、泵送能耗（$PE$）、碳源消耗（$CE$）及污泥处理成本（$SP$）：
$$OCI = \int_0^T \left( \alpha \cdot AE(t) + \beta \cdot PE(t) + \gamma \cdot CE(t) + \delta \cdot SP(t) \right) dt$$
该优化模型利用近似梯度法或群智能算法进行在线求解。

### 6.2.3 软测量子系统的集成推导

由于在线氨氮、总氮等水质仪表存在滞后长、易受污染漂移的问题，系统引入软测量技术进行数据校准与预测替代。推导基于数据驱动模型的隐层映射机制：

1. **出水氨氮软测量（混合驱动）**：
利用物料守恒建立机理基准，用神经网络补偿未建模动态。物理量输入包含好氧段DO、pH、温度及进水流量。
$$\hat{S}_{\mathrm{NH,e}} = f_{\mathrm{Mechanistic}}(\mathbf{x}) + W \cdot \sigma(V \mathbf{x} + \mathbf{b}_1) + \mathbf{b}_2$$
温度和pH改变了氨氧化细菌（AOB）的最大比生长速率，DO则限制了电子受体，这些物理关系隐含在网络的权重矩阵中。

2. **出水总氮软测量（PLS模型）**：
提取缺氧段ORP、好氧段氨氮和内回流比等易测变量。偏最小二乘法（PLS）通过最大化输入变量得分矩阵 $\mathbf{T}$ 与输出变量 $\mathbf{Y}$ 的协方差来降维：
$$\max_{\|\mathbf{w}\|=1, \|\mathbf{c}\|=1} \mathrm{cov}(\mathbf{Xw}, \mathbf{Yc})$$
通过提取隐含潜变量，剥离了测量噪声，提高了对出水总氮预测的鲁棒性。

3. **模型与仪表的交叉验证体系**：
设在线仪表读数为 $y_m$，软测量估计值为 $\hat{y}$。定义动态残差 $e_d = |y_m - \hat{y}|$。利用滑动窗口计算残差均值与方差。当 $e_d$ 超过置信上限（如 $3\sigma$）且持续 $N$ 个采样周期时，系统判断为测量异常，将反馈回路的输入切换至软测量值，同时触发仪表声光报警，并启动软测量模型的增量在线学习过程。

### 6.2.4 异常工况识别的统计学推导

污水处理厂常面临暴雨、有毒物质泄漏等物理或化学冲击，必须具备早期预警能力。基于主成分分析（PCA）的多元统计异常检测，其核心在于将高维相关变量空间正交变换为低维独立的主元空间。

给定标准化后的历史正常数据集 $\mathbf{X} \in \mathbb{R}^{n \times m}$，通过特征值分解其协方差矩阵得到负荷矩阵 $\mathbf{P}$。对于新样本 $\mathbf{x}$，其主元得分 $\mathbf{t} = \mathbf{P}^T \mathbf{x}$。
提取两个监控统计量。Hotelling $T^2$ 衡量样本在主成分模型子空间内的变异程度：
$$T^2 = \mathbf{x}^T \mathbf{P} \mathbf{\Lambda}^{-1} \mathbf{P}^T \mathbf{x} \sim \frac{k(n^2-1)}{n(n-k)} F_{\alpha}(k, n-k)$$
其中 $\mathbf{\Lambda}$ 为保留特征值构成的对角阵。
SPE（Squared Prediction Error）衡量样本偏离主成分模型的残差大小：
$$\mathrm{SPE} = \|\mathbf{x} - \hat{\mathbf{x}}\|^2 = \|\mathbf{x} - \mathbf{P}\mathbf{P}^T\mathbf{x}\|^2$$
SPE的控制限 $Q_{\alpha}$ 基于多元正态假设计算得出。
从物理意义上讲，$T^2$ 越限通常对应于正常操作范围内的设定值漂移或负荷变化（如流量增大）；而 SPE 越限则意味着变量间的固有物理或生化相关性被打破（如毒性物质导致某种菌群死亡，切断了反应链条）。

## 6.3 仿真设置

为了全面验证上述理论与控制架构，基于Benchmark Simulation Model (BSM) 框架建立一个模拟10万吨/日规模A$^2$/O污水厂的严谨仿真平台。

物理尺寸与水力参数设定：系统包含进水泵房、反应池与二沉池。厌氧段分为2个完全混合反应器（CSTR）（各1500 $\mathrm{m^3}$，水力停留时间HRT=0.7h），缺氧段2个反应池（各3000 $\mathrm{m^3}$，HRT=1.4h），好氧段3个反应池（各5000 $\mathrm{m^3}$，HRT=3.6h）。二沉池采用10层一维沉淀模型（Takács模型），共2座，总表面积2500 $\mathrm{m^2}$。系统维持污泥浓度MLSS在3500 mg/L左右，通过排泥控制将污泥龄（SRT）稳定在12天。

进水水质与动态模式配置：依托中国南方某典型城市管网实测数据生成动态进水文件。基准干旱天气下，日均进水流量为 $10^5 \mathrm{m^3/d}$。污染物浓度设计值：COD 350 mg/L、BOD 180 mg/L、$S_{\mathrm{NH}}$ 35 mg/L、TN 45 mg/L、TP 5 mg/L。进水呈现双峰分布的日周期规律（早8点和晚20点为水量水质高峰），日变化系数介于1.0至1.5之间。

设计三个逐步递进的对比控制方案：
- **场景A（传统PID控制）**：各段溶解氧由单独的定值PI控制器维持，好氧段DO设定值为恒定2.0 mg/L。内回流比固定为200%，外回流比固定为100%，无碳源投加。
- **场景B（协调控制）**：激活协调控制层，包含DO-$S_{\mathrm{NO}}$双变量前馈-反馈控制，并根据前述碳氮比公式自动调配外碳源（乙酸钠）。
- **场景C（全流程智能控制）**：开启三层控制架构，引入软测量模块替代高频次水质仪表数据，并启用PCA异常检测模块。优化决策层每小时更新一次底层设定值。

仿真周期设定为90天。为考察系统的抗冲击韧性，在第30天叠加了一场持续6小时的短历时强降雨（暴雨冲击），进水流量激增80%，但污染物浓度因管网雨水混入被稀释50%；在第60天模拟了一次工业非法排污（COD冲击），进水COD浓度在4小时内陡增100%，而流量不变。系统评价指标覆盖达标稳定性、全生命周期能耗物耗以及对极端工况的镇定时间。

## 6.4 结果分析与数据解读

历经90天的动态仿真，输出的数百万条水质与能耗数据清晰地刻画了控制系统的内在运行机理与干预效果。

**出水水质的动态演变与达标率解析**
传统PID（场景A）在平稳工况下出水基本达标，整体达标率为89%。但深入剖析其时间序列数据发现，在早晚负荷高峰到来时，由于固定的好氧DO设定值无法匹配激增的氨氮负荷，曝气池末端实际DO经常被迅速消耗至1.2 mg/L以下，导致硝化反应速率 $r_{\mathrm{nit}}$ 受到严重的氧限制，出水氨氮出现显著的周期性超标峰值（最高达5.2 mg/L）。
协调控制（场景B）通过动态前馈补偿，将达标率推升至96%。当进水高负荷抵近时，双变量控制器提前预判需氧量并增大曝气指令，成功压制了氨氮的峰值波动。同时，缺氧区的碳源前馈投加使得反硝化动力学不再受限于进水低C/N比，出水总氮浓度从平均22 mg/L大幅平抑至14 mg/L，且数据方差从4.2缩小至1.5，表现出极强的稳定性。
全流程智能控制（场景C）实现了99.2%的极致达标率。通过优化层的全局寻优，各段状态参数被严格限制在最优包络线内。

**能耗优化背后的物理力学与生化机制**
从经济性维度考量，场景B的运行成本指数（OCI）较场景A下降12%。这部分节约并非凭空产生，其物理实质在于消除了非峰值时段的“过曝气”与“过循环”。传统方案在夜间低负荷时，仍维持2.0 mg/L的DO和200%的内回流，大量电能白白消耗在克服流体阻力与产生无效气泡上。
场景C的节能效果更为显著，OCI较场景A整体降低19%。数据图表显示，优化决策层在每日凌晨2点至5点期间，不仅将好氧段DO设定值下调至1.0 mg/L，更将内回流比智能调降至150%。从动力学角度看，1.0 mg/L的DO已足以满足夜间极低氨氮负荷下的硝化需求，同时有效避免了高溶氧回流液对缺氧段反硝化环境的毒害破坏。更精妙的是，场景C的外碳源总投加量比场景B进一步减少了15%，原因是全局优化算法在缺氧前段精准计算了剩余水解底物量，杜绝了单一PI控制器容易产生的超调投药现象。

**异常冲击下的系统响应轨迹分析**
面对第30天的暴雨冲击（流量骤增80%，浓度稀释），流体力学效应占据主导。高流速大幅缩短了水力停留时间，传统方案A未作响应，大量活性污泥被加速冲刷至二沉池，生化区污泥浓度（MLSS）在两小时内从3500 mg/L跌至2800 mg/L，系统微生物量严重流失，导致此后数天内氨氮飙升至7.5 mg/L，恢复期长达4天。
而在场景C中，PCA异常检测算法在进水流量突变的第8分钟，其 Hotelling $T^2$ 统计量即突破了95%的置信限。系统立即切入预设的“雨季应急模式”：自动关闭排泥阀（截留生物量），并根据流速前馈信号适度拉高DO设定值至3.0 mg/L，强化单体微生物的反应速率以弥补停留时间的缩短。数据曲线表明，场景C的MLSS最低仅降至3200 mg/L，出水氨氮峰值被死死按在3.2 mg/L，并在降雨结束后6小时内迅速恢复平稳。

第60天的COD冲击进一步展现了软测量的威力。高浓度有机物进入系统后，传统仪表存在约40分钟的检测延迟。而场景C的软测量模型凭借前端DO与ORP微小变化的敏锐捕捉，提前15分钟探测到生化耗氧速率（OUR）的异常飙升，据此推断出进水COD负荷的剧增。系统随即主动大幅拉升风机转速并同步增加碳源池备用容量，完成了一次预测-防御闭环，出水COD始终未超过47 mg/L。

## 6.5 工程启示与落地案例

@dataclass
class PlantParams:
    # 体积 (m3)
    V_an: float = 2000.0
    V_ax: float = 3500.0
    V_ox: float = 6000.0

    # 回流参数
    r_return: float = 0.7  # 外回流比 Qr/Qin

    # 反应参数（简化ASM）
    do_sat: float = 8.0
    K_do: float = 0.4
    K_do_nit: float = 0.6
    K_no3: float = 0.8

    k_cod_aer: float = 0.58

    k_do_decay_an: float = 0.8
    k_do_decay_ax: float = 1.0

    # 耗氧系数（简化）
    o2_per_cod: float = 0.25
    o2_per_nit: float = 1.2

    # 能耗系数
    alpha_air: float = 26.0    # kW/(u_air^2)
    alpha_pump: float = 0.004  # kW/(m3/h recycle)


def influent_profile(t):
    # 日变化 + 两次冲击
    q = 1000.0 * (1 + 0.15 * np.sin(2 * np.pi * t / 24 - 0.6) + 0.04 * np.sin(4 * np.pi * t / 24))
    cod = 330.0 + 65.0 * np.sin(2 * np.pi * (t - 3) / 24)
    nh4 = 34.0 + 7.0 * np.sin(2 * np.pi * (t - 1) / 24)
    no3 = 1.2 + 0.3 * np.sin(2 * np.pi * (t + 2) / 24)

    if 30 <= t <= 36:
        q += 120.0
        cod += 120.0
        nh4 += 8.0
    if 54 <= t <= 58:
        q += 80.0
        cod += 90.0
        nh4 += 5.0

    return {
        "Q": max(q, 650.0),
        "COD": max(cod, 120.0),
        "NH4": max(nh4, 8.0),
        "NO3": max(no3, 0.2),
        "DO": 0.2,
    }


class PIDController:
    def __init__(
        self, kp, ki, kd, u_min, u_max, bias=0.0, reverse=False,
        adaptive=False, e_scale=1.0, de_scale=1.0, aw=0.25
    ):
        self.kp0, self.ki0, self.kd0 = kp, ki, kd
        self.kp, self.ki, self.kd = kp, ki, kd
        self.u_min, self.u_max = u_min, u_max
        self.bias = bias
        self.reverse = reverse
        self.adaptive = adaptive
        self.e_scale = e_scale
        self.de_scale = de_scale
        self.aw = aw
        self.integral = 0.0
        self.last_e = 0.0

    def _fuzzy_gain_update(self, e, de):
        en = np.clip(abs(e) / (self.e_scale + 1e-12), 0.0, 1.0)
        den = np.clip(abs(de) / (self.de_scale + 1e-12), 0.0, 1.0)

        mu_small = np.clip((0.4 - en) / 0.4, 0.0, 1.0)
        mu_mid = np.clip(1.0 - abs(en - 0.5) / 0.3, 0.0, 1.0)
        mu_large = np.clip((en - 0.6) / 0.4, 0.0, 1.0)

        mu_de_small = np.clip((0.35 - den) / 0.35, 0.0, 1.0)
        mu_de_large = np.clip((den - 0.35) / 0.65, 0.0, 1.0)

        d_kp = 0.70 * mu_large + 0.30 * mu_mid - 0.10 * mu_small
        d_ki = 0.60 * (mu_small * mu_de_small) - 0.50 * mu_large
        d_kd = 0.70 * mu_de_large + 0.20 * mu_mid

        kp = self.kp0 * (1.0 + d_kp)
        ki = self.ki0 * np.clip(1.0 + d_ki, 0.2, 2.0)
        kd = self.kd0 * (1.0 + d_kd)
        return kp, ki, kd

    def update(self, sp, pv, dt, ff=0.0):
        e = (pv - sp) if self.reverse else (sp - pv)
        de = (e - self.last_e) / max(dt, 1e-9)

        if self.adaptive:
            self.kp, self.ki, self.kd = self._fuzzy_gain_update(e, de)
        else:
            self.kp, self.ki, self.kd = self.kp0, self.ki0, self.kd0

        i_candidate = self.integral + e * dt
        u_raw = self.bias + self.kp * e + self.ki * i_candidate + self.kd * de + ff
        u_sat = float(np.clip(u_raw, self.u_min, self.u_max))

        # back-calculation anti-windup
        if self.ki > 1e-12:
            self.integral = i_candidate + self.aw * (u_sat - u_raw) / self.ki
        else:
            self.integral = i_candidate

        self.last_e = e
        return u_sat


def process_rhs(t, x, u_air, r_int, infl, p: PlantParams):
    x = np.maximum(x, 0.0)
    (
        cod_an, nh4_an, no3_an, do_an,
        cod_ax, nh4_ax, no3_ax, do_ax,
        cod_ox, nh4_ox, no3_ox, do_ox
    ) = x

    q = infl["Q"]
    qr = p.r_return * q
    qi = r_int * q

    eps = 1e-9

    # 厌氧段
    r_den_an = p.k_denit * cod_an * (no3_an / (p.K_no3 + no3_an + eps)) * (p.K_i_do / (p.K_i_do + do_an + eps))
    r_hyd_an = p.k_cod_ana * cod_an
    reac_an = np.array([
        -r_hyd_an - 0.4 * r_den_an,
        0.0,
        -r_den_an / p.cod_per_no3,
        -p.k_do_decay_an * do_an
    ])

    # 缺氧段
    r_den_ax = p.k_denit * cod_ax * (no3_ax / (p.K_no3 + no3_ax + eps)) * (p.K_i_do / (p.K_i_do + do_ax + eps))
    reac_ax = np.array([
        -r_den_ax,
        0.0,
        -r_den_ax / p.cod_per_no3,
        -p.k_do_decay_ax * do_ax
    ])

    # 好氧段
    r_cod_ox = p.k_cod_aer * cod_ox * (do_ox / (p.K_do + do_ox + eps))
    r_nit = p.k_nit * nh4_ox * (do_ox / (p.K_do_nit + do_ox + eps))
    reac_ox = np.array([
        -r_cod_ox,
        -r_nit,
        +r_nit,
        u_air * (p.do_sat - do_ox) - p.o2_per_cod * r_cod_ox - p.o2_per_nit * r_nit
    ])

    cin = np.array([infl["COD"], infl["NH4"], infl["NO3"], infl["DO"]])
    can = np.array([cod_an, nh4_an, no3_an, do_an])
    cax = np.array([cod_ax, nh4_ax, no3_ax, do_ax])
    cox = np.array([cod_ox, nh4_ox, no3_ox, do_ox])

    d_an = (q * cin + qr * cox - (q + qr) * can) / p.V_an + reac_an
    d_ax = ((q + qr) * can + qi * cox - (q + qr + qi) * cax) / p.V_ax + reac_ax
    d_ox = ((q + qr + qi) * cax - (q + qr + qi) * cox) / p.V_ox + reac_ox

    return np.hstack([d_an, d_ax, d_ox])


def simulate_case(name, use_fuzzy, p: PlantParams, t_end=72.0, dt=1.0 / 12.0):
    t = np.arange(0.0, t_end + dt, dt)
    n = len(t)

    states = np.zeros((n, 12), dtype=float)
    states[0] = np.array([
        300.0, 30.0, 1.0, 0.10,
        220.0, 25.0, 4.0, 0.20,
        120.0, 12.0, 10.0, 1.50
    ])

    influent = {k: np.zeros(n, dtype=float) for k in ["Q", "COD", "NH4", "NO3", "DO"]}
    u_air_hist = np.zeros(n - 1, dtype=float)
    r_int_hist = np.zeros(n - 1, dtype=float)
    energy_kwh = 0.0

    if use_fuzzy:
        do_pid = PIDController(1.15, 0.55, 0.03, 0.2, 5.0, bias=1.2, adaptive=True, e_scale=1.2, de_scale=0.8)
        no3_pid = PIDController(0.30, 0.10, 0.015, 0.4, 3.0, bias=0.9, reverse=True, adaptive=True, e_scale=4.0, de_scale=2.0)
    else:
        do_pid = PIDController(1.60, 0.95, 0.04, 0.2, 5.0, bias=1.4, adaptive=False)
        no3_pid = PIDController(0.42, 0.16, 0.020, 0.4, 3.0, bias=1.0, reverse=True, adaptive=False)

    for k in range(n - 1):
        tk = t[k]
        xk = states[k]
        inf = influent_profile(tk)
        for key in influent:
            influent[key][k] = inf[key]

        # 多变量前馈：进水负荷和流量扰动
        ff_air = np.clip(0.018 * (inf["NH4"] - 34.0) + 0.0015 * (inf["Q"] - 1000.0), -0.3, 1.5)
        ff_rec = np.clip(0.030 * (inf["NH4"] - 34.0) + 0.0010 * (inf["Q"] - 1000.0), -0.2, 1.2)

        # DO控制（操纵量：曝气）
        u_air = do_pid.update(sp=2.0, pv=xk[11], dt=dt, ff=ff_air)
        # NO3控制（操纵量：内回流比）
        r_int = no3_pid.update(sp=8.0, pv=xk[10], dt=dt, ff=ff_rec)

        u_air_hist[k] = u_air
        r_int_hist[k] = r_int

        rhs = lambda tt, xx: process_rhs(tt, xx, u_air, r_int, inf, p)
        sol = solve_ivp(rhs, (tk, t[k + 1]), xk, method="RK23", max_step=dt / 3, rtol=1e-5, atol=1e-7)
        states[k + 1] = np.maximum(sol.y[:, -1], 0.0)

        p_air = p.alpha_air * (u_air ** 2)
        p_pump = p.alpha_pump * (r_int * inf["Q"])
        energy_kwh += (p_air + p_pump) * dt

    last_inf = influent_profile(t[-1])
    for key in influent:
        influent[key][-1] = last_inf[key]

    eff = states[:, [8, 9, 10, 11]]  # COD, NH4, NO3, DO (出水近似取好氧末端)
    mask = t >= 12.0  # 去掉前12h启动段

    cod_removal = 100.0 * (np.mean(influent["COD"][mask]) - np.mean(eff[:, 0][mask])) / np.mean(influent["COD"][mask])
    nh4_removal = 100.0 * (np.mean(influent["NH4"][mask]) - np.mean(eff[:, 1][mask])) / np.mean(influent["NH4"][mask])

    treated_volume = np.trapz(influent["Q"], t)  # m3
    specific_energy = energy_kwh / max(treated_volume, 1e-9)  # kWh/m3

    quality_ok = (eff[:, 0] <= 50.0) & (eff[:, 1] <= 5.0) & (eff[:, 2] <= 15.0)
    compliance = 100.0 * np.mean(quality_ok[mask])

    return {
        "name": name,
        "time": t,
        "states": states,
        "effluent": eff,
        "influent": influent,
        "u_air": u_air_hist,
        "r_int": r_int_hist,
        "kpi": {
            "COD去除率(%)": cod_removal,
            "NH4去除率(%)": nh4_removal,
            "单位能耗(kWh/m3)": specific_energy,
            "达标率(%)": compliance,
        },
    }

代码解读（约500字）  
这段程序把A²/O工艺抽象成三个串联CSTR，分别对应厌氧、缺氧、好氧区，每个区都保留四个关键水质状态：`COD/NH4/NO3/DO`。`process_rhs` 是核心机理方程：前半部分写生化反应，后半部分写水力输运。厌氧和缺氧段主要发生反硝化与有机物降解，好氧段发生有机物氧化和硝化，DO由“曝气传氧-生化耗氧”平衡决定。这样既保持了ASM思想（Monod型限制与氧抑制），又把模型复杂度控制在教学可用范围。控制层使用两回路：DO回路调曝气强度 `u_air`，NO3回路调内回流比 `r_int`。两者都叠加前馈项，前馈来源是进水`NH4`与流量扰动，作用是提前补偿负荷冲击，减少纯反馈滞后。固定PID与自适应模糊PID复用了同一控制框架，差别在 `adaptive` 开关：自适应版本在每个采样步根据误差和误差变化率做“模糊增益整定”，大偏差时提高`Kp/Kd`增强快速性，小偏差时提升`Ki`抑制稳态误差，并用反算抗饱和避免积分累积。仿真主循环按离散控制、连续过程的思路实现：控制量在采样间隔内保持常值，过程由 `solve_ivp` 积分推进。最后用三类KPI评估：COD去除率、NH4去除率、单位能耗（kWh/m³），并额外统计综合达标率（COD/NH4/NO3同时达标）。绘图部分给出两种策略的出水时序对比与限值线，便于直观看到冲击阶段谁恢复更快、谁振荡更小、谁能耗更低。
