# 第4章 能耗优化：曝气与回流系统

## 4.1 导读

在上一章中，我们深入讨论了MBR系统的膜污染建模与控制，其中为了减缓膜污染而引入的膜面曝气能耗，已经暴露了污水处理厂整体能耗问题的冰山一角。事实上，现代城市污水处理厂是市政基础设施中名副其实的用电大户。据统计，污水处理厂的总体电耗通常占到整个城市总用电量的1%-3%。在污水处理厂内部的能量流向中，曝气系统无疑是最大的能量消耗单元，其能耗通常占全厂总电耗的50%-60%；其次是提供物质循环动力和水力高程的回流与提升泵系统，占据了15%-25%的电耗份额。在全球应对气候变化以及国家“双碳”（碳达峰、碳中和）目标的强力驱动下，污水处理行业的节能降碳已经从一项单纯降低运营成本的经济诉求，上升为促进行业可持续发展的核心焦点。

从生物化学反应的本质来看，曝气系统的作用是向混合液中强制传递氧气，为好氧微生物氧化分解有机物（碳化过程）以及硝化细菌将氨氮氧化为硝态氮（硝化过程）提供必不可少的电子受体。充足的溶解氧（DO）是保障生化反应速率和出水水质达标的基础。然而，回流系统的高速运转和高强度的曝气不仅会带来高昂的电费账单，还会引发复杂的系统内部耦合效应。例如，过高的溶解氧不仅无益于好氧反应的进一步加速，反而会通过高达数倍的大比例内回流，将大量富含氧气的混合液带入缺氧区，直接破坏反硝化所需的严格缺氧环境，进而导致系统总氮去除效率显著下降。

本章将从宏观的系统工程视角出发，跳出单一设备的控制局限，建立涵盖物理过程与生化机制的曝气与回流系统能耗数学模型，并提出基于多目标优化的节能控制策略。全厂优化的核心矛盾极为鲜明：为了确保出水水质严格达标，系统倾向于进行充分的生化反应（即要求系统增大曝气量和内部回流量）；而为了实现节能降碳的绿色目标，系统则要求严格限制并减少一切不必要的能量输入。如何在这一相互牵制的矛盾中，寻找能够兼顾水质与能耗的帕累托（Pareto）最优解，实现能源的精准投放与按需分配，是贯穿本章的核心问题。

## 4.2 理论基础

为了对复杂污水处理系统进行科学的能耗优化，首要前提是建立具有明确物理意义的底层能耗模型，并将其与生化反应动力学模型进行有效对接。

### 4.2.1 曝气系统能耗模型及其物理解释

污水处理厂中最为广泛采用的鼓风曝气系统，其能耗主要源于鼓风机克服水头和管网阻力对抗空气做功。根据工程热力学第一定律与理想气体状态方程，鼓风机内部气体的压缩过程可近似视为绝热过程（无热量交换）。对体积为 $V$ 的气体施加压力 $p$ 进行压缩，所做的功通过对 $V dp$ 进行积分推导。考虑到理想气体可逆绝热过程满足 $pV^\gamma = \text{常数}$，绝热压缩过程的风机功率理论模型可表示为：

$$P_{\mathrm{blower}} = \frac{w R T_1}{\eta_b (\gamma - 1)} \left[\left(\frac{p_2}{p_1}\right)^{(\gamma-1)/\gamma} - 1 \right]$$

公式中的变量具有具体的物理内涵：$w$ 为空气的质量流量（kg/s），它直接决定了系统的供氧总量；$R$ 为空气特定气体常数（通常取287 J/(kg·K)）；$T_1$ 为风机进口处的绝对温度（K），表明夏季高温会增加气体压缩比功；$\eta_b$ 为风机的综合效率（通常在0.7-0.85之间，涵盖了机械效率与等熵效率）；$\gamma = 1.4$ 为双原子分子（如空气中的氧气和氮气）的绝热指数；$p_1$ 和 $p_2$ 分别为进出口的绝对压力。

特别地，出口压力 $p_2$ 必须克服反应池中的各项阻力才能将空气成功注入水体，其表达式为 $p_2 = p_1 + \rho g h + \Delta p_{\mathrm{pipe}}$。其中 $h$ 为曝气器在水下的浸没深度，$\rho g h$ 代表需克服的静水压强；$\Delta p_{\mathrm{pipe}}$ 为管路系统（含阀门）的沿程与局部阻力损失以及微孔曝气器的孔口射流阻力。随着设备运行时间的推移，曝气器表面容易发生生物污垢堵塞或无机结垢，导致 $\Delta p_{\mathrm{pipe}}$ 显著上升，从而推高整个系统的排气压力要求和运行能耗。

在生化反应侧，供气量转化为水中的溶解氧受双膜理论主导。曝气量与系统氧传质系数 $K_La$ 的关系并非简单的线性映射，而通常近似表示为以下经验关联式：

$$K_La = k_a \cdot Q_{\mathrm{air}}^{0.8}$$

其中 $k_a$ 为与曝气器类型、水流流态以及反应池几何深度密切相关的综合传质系数，$Q_{\mathrm{air}}$ 为单位池容的标准供气量。结合上述两个公式，如果忽略压力微小变化对密度的影响，则鼓风机的轴功率可以近似表达为氧传质系数 $K_La$ 的函数：

$$P_{\mathrm{blower}} \propto K_La^{1.25}$$

这一超线性比例关系揭示了一个重要的物理规律：曝气量的“边际能耗”是单调递增的。也就是说，在系统处于高供氧状态时，继续增加微量的氧气传递能力需要付出巨大的能量代价。经过数学计算可知，将 $K_La$ 从200 $\mathrm{d^{-1}}$ 提高到240 $\mathrm{d^{-1}}$（相对增加20%）所需的额外风机功率增幅将达到约24%。因此，从优化角度看，削减高负荷时期不必要的过量曝气，其节能收益远大于在低负荷时期削减同等绝对量的曝气。

### 4.2.2 回流泵系统能耗与管路流体力学模型

污泥外回流泵（用于维持好氧池生物量浓度）和混合液内回流泵（用于将硝态氮送回缺氧区反硝化）的运行能耗同样占据重要比例。基于流体力学中的伯努利（Bernoulli）方程，泵送功率模型推导如下：

$$P_{\mathrm{pump}} = \frac{\rho g Q_r H_r}{\eta_p}$$

其中，$Q_r$ 为回流体积流量（m³/s）；$H_r$ 为泵的总扬程（m）；$\eta_p$ 为泵及电机的联合效率。总扬程 $H_r$ 由实际提升的静水位高差（$\Delta z$）和管路系统的水头损失（$h_f$）两部分构成。对于内回流系统而言，由于缺氧池和好氧池通常处于同一构筑物内，静水位高差极为微小（大多仅需克服隔墙过流堰的水位降落），因此扬程 $H_r$ 主要被管路及阀门的动态摩擦阻力所占据，且阻力大小与流速的平方成正比。

根据水泵相似定律（Affinity Laws），在采用变频器（VFD）调节流量的工况下，泵的流量 $Q$ 与叶轮转速 $N$ 成正比（$Q \propto N$），扬程 $H$ 与转速的平方成正比（$H \propto N^2$），而轴功率 $P$ 与转速的三次方成正比（$P \propto N^3$）。这意味着，如果在低进水负荷时段能够将内回流流量降低20%，对应的泵送功率将大幅削减近49%。在国际水协提出的基准仿真模型（BSM1）框架内，污泥外回流比 $r = Q_r / Q_{\mathrm{in}}$ 通常设定在0.5-1.0之间，而内回流比 $a = Q_a / Q_{\mathrm{in}}$ 通常处于1.0-4.0的高位运行区间。

### 4.2.3 多目标优化问题的严格形式化表达

将污水处理厂的综合运行优化问题表述为严谨的多目标数学规划问题，是实施智能控制的理论前提：

$$\min_{\mathbf{u}} \; \mathbf{F}(\mathbf{u}) = \begin{bmatrix} f_{\mathrm{EQI}}(\mathbf{u}) \\ f_{\mathrm{OCI}}(\mathbf{u}) \end{bmatrix}$$

受到以下非线性过程约束和边界约束的限制：
$$\text{s.t.} \quad S_{\mathrm{NH,e}} \leq 4 \; \mathrm{mg/L}, \quad TN_e \leq 15 \; \mathrm{mg/L}, \quad TSS_e \leq 10 \; \mathrm{mg/L}$$

$$K_{La,\min} \leq K_{La} \leq K_{La,\max}, \quad Q_{a,\min} \leq Q_a \leq Q_{a,\max}$$

在上述公式中，$\mathbf{u} = [K_{La}, Q_a, r]^T$ 代表系统的核心决策变量向量。目标函数向量包括两个相互矛盾的指标：$f_{\mathrm{EQI}}$ 为出水水质指数（Effluent Quality Index），通过对出水悬浮物、化学需氧量、生化需氧量、氨氮及总氮等污染物通量进行加权积分计算得到，数值越小表示向受纳水体排放的污染越轻；$f_{\mathrm{OCI}}$ 为综合运行成本指数（Operational Cost Index），综合统计了曝气能耗（AE）、泵送能耗（PE）、污泥处置成本（SP）、外加碳源成本（EC）以及搅拌能耗（ME）。
由于这两个目标在绝大多数工况下存在深度耦合的相互冲突特性（例如提升水质必然增加能耗），因此不存在能够使两者同时达到绝对最小值的唯一解，而是形成一条代表最优化权衡关系的帕累托前沿（Pareto Front）。

### 4.2.4 加权目标函数法与标量化技术

在工程自动化控制层面，直接求解动态多目标优化不仅计算负荷庞大，且难以生成唯一的实时控制指令。因此，工程实践中常采用标量化技术中的加权目标函数法，将多目标协同问题降维转化为单目标最优化问题：

$$\min_{\mathbf{u}} \; J = w_1 \cdot \frac{f_{\mathrm{EQI}}}{f_{\mathrm{EQI}}^0} + w_2 \cdot \frac{f_{\mathrm{OCI}}}{f_{\mathrm{OCI}}^0}$$

引入 $f_{\mathrm{EQI}}^0$ 和 $f_{\mathrm{OCI}}^0$ 的目的在于消除量纲差异，使得目标函数处于同一数值标度之下进行归一化运算。权重参数 $w_1$ 和 $w_2$（满足 $w_1 + w_2 = 1$）则量化了决策管理者在应对特定季节或特定排污政策时，对水质安全裕度与节能降耗效益之间的优先级主观偏好。

### 4.2.5 基于BSM1的稳态系统寻优

为了宏观把握水质与能耗的边界关系，首先可在稳态进水条件下（忽略时间维度波动）执行决策变量的空间遍历扫描。具体算法设定为：将关键控制变量 $K_La$ 在 $100 \sim 400 \; \mathrm{d^{-1}}$ 的物理可行域内划定等间距网格，内回流流量 $Q_a$ 在 $10000 \sim 90000 \; \mathrm{m^3/d}$ 范围内进行离散采样，外回流比则依据工艺手册固定为0.5。通过运行稳态仿真求解器，计算空间内每一组离散参数所对应的EQI和OCI终值，剔除所有违反出水达标约束（如氨氮或总氮超标）的无效解，从而在目标空间中绘制出完整的帕累托前沿。

### 4.2.6 动态优化前馈-反馈控制：可变DO设定值策略

稳态优化的致命局限在于，它建立在进水流量和水质恒定不变的假设之上。现实中，污水管网汇集的流量和污染物负荷呈现出剧烈的日变化和季节性时变特性。传统采用固定DO设定值（例如恒定控制DO为2.0 mg/L）的反馈控制回路，虽然能够抵抗一定的扰动，但在系统低负荷时期极易发生供气量长期下探但仍过量曝气的现象。

动态前馈-反馈优化策略的核心机制在于：在低进水负荷时段（如凌晨），不仅依靠反馈闭环降低鼓风机出力，更从控制逻辑的源头上主动下调DO控制设定值，以切断过剩的能量输送并保护缺氧区环境；而在高负荷洪峰时段，则预先拉高DO设定值，利用充氧能力储备强力压制氨氮穿透的风险。通过引入进水氨氮负荷变量前馈信号 $L(t) = Q_{\mathrm{in}}(t) \cdot S_{\mathrm{NH,in}}(t)$，构建DO设定值的动态映射函数：

$$S_O^{\mathrm{ref}}(t) = S_{O,\min}^{\mathrm{ref}} + (S_{O,\max}^{\mathrm{ref}} - S_{O,\min}^{\mathrm{ref}}) \cdot \frac{L(t) - L_{\min}}{L_{\max} - L_{\min}}$$

其中，$S_{O,\min}^{\mathrm{ref}}$ 和 $S_{O,\max}^{\mathrm{ref}}$ 为基于微生物动力学特性确定的DO设定值安全下限与上限（一般分别取1.0 mg/L和3.0 mg/L）。该表达式实质上是一种带有边界饱和限制的线性前馈调度策略，它赋予了控制系统根据处理任务轻重“按需分配”能源的高级智能。

## 4.3 仿真设置与评价体系

本节仿真实验严格基于国际通用的基准仿真模型BSM1平台展开。BSM1包含5个串联的完全混合反应器（2个前端缺氧池和3个后端好氧池）以及1个二沉池模型。为了全面评估不同控制策略的效能，我们导入了标准14天动态干天气（Dry weather）进水水质和流量时间序列数据，该数据具有显著的日变周期特征。

仿真实验共设计四组对比方案以突显算法改进带来的效益：（1）基准开环方案——采用恒定的经验参数（第三、四、五好氧池 $K_La$ 统一固定为 $240\ \mathrm{d^{-1}}$，内回流 $Q_a$ 固定为 $55338\ \mathrm{m^3/d}$）；（2）常规反馈控制方案——在第五个好氧反应池部署位置式PI控制算法，通过调节三个好氧池的联合阀门开度，维持DO在固定值2.0 mg/L；（3）动态可变控制方案——实施前馈-反馈联合PI控制，DO设定值在1.0-3.0 mg/L之间依据进水氨氮瞬时负荷进行线性浮动调节；（4）单点稳态极值方案——直接从4.2.5节生成的帕累托前沿数据集中，提取出在严格满足出水标准前提下OCI数值最低的极限静止参数组并应用。

仿真数据统计分析区间设定为第7天至第14天，以消除模型初始状态带来的瞬态误差。能耗统计将详细拆解为曝气能耗积分值（$E_{\mathrm{aer}}$）与泵送能耗积分值（$E_{\mathrm{pump}}$）。水质安全评价指标则聚焦于抗扰动能力较差的出水氨氮小时峰值以及对内回流极其敏感的出水总氮14天均值。

## 4.4 仿真结果与深度数据分析

多维度能耗对比结果以高精度的数据颗粒度，清晰地勾勒出各级控制策略潜藏的巨大节能空间。
在无动态控制干预的（1）基准方案下，系统计算得到的总OCI高达364（采用BSM1标准能耗单位计算），其中曝气单元的运行能耗指数达到225.68，占据了总能耗的62%绝对主导地位。这一结果暴露出静态参数的粗放性：在夜间低流量阶段，恒定的大规模供风使得好氧池末端DO一路飙升至超过4.5 mg/L。这不仅白白浪费了大量压缩气体作功，其附带的高溶解氧更随着巨量内回流倾泻至缺氧池，导致原本用于反硝化的进水易降解有机碳源被好氧异养菌优先抢占耗竭，严重拖累了整体脱氮性能。

实施了（2）常规PI固定DO控制后，总OCI下降至335，较基准方案实现了约8%的节能减排。这8%的降幅确切表明，通过闭环反馈抑制夜间过度充氧是有效的。然而，在该策略下，当进水氨氮负荷出现极值飙升时（例如第10天下午14:00），固定的2.0 mg/L设定值限制了硝化速率的进一步拉升，导致系统对峰值冲击的抗性略显单薄。

当激活（3）可变DO设定值前馈-反馈策略时，系统表现出了卓越的自适应协同能力。总OCI进一步大幅滑落至308，节能比例扩大到惊人的15%。深度剖析运行日志可以发现，额外挖掘出的7%节能效益，主要归功于系统在凌晨时分果断将DO设定值下调至1.2 mg/L，此举触发了PI控制器指令风机大幅降低频率，彻底消灭了夜间无效能耗。同时，在应对白天的负荷洪峰时，系统临时将设定值拉升至2.8 mg/L，强化了传质驱动力，将出水氨氮峰值牢牢锁定在安全线以内。

反观（4）稳态优化方案提取的理论最低耗能点，其OCI虽然探底至295，但代价是EQI较基准方案大幅劣化并增加了约5%。这是由于该方案将全局 $K_La$ 强行压低至 $190 \mathrm{d^{-1}}$，同时将内回流截流至 $40000 \mathrm{m^3/d}$。双重削弱导致氧气和硝态氮的循环通量双双陷入瓶颈，生化系统的硝化和反硝化反应速率均出现明显滑坡。在此参数配置下，出水氨氮峰值攀升至3.6 mg/L（已危险逼近4.0 mg/L的排放红线），且出水总氮均值跃升至16.1 mg/L（已实质性越过了15 mg/L的法定标准）。这确凿表明，脱离了动态负荷实际、单纯依靠静态理论寻找极致节能点，必然引发灾难性的排放达标风险。

水质-能耗多维帕累托前沿的数据分布规律也印证了上述分析。可变DO设定值策略在复杂的权衡博弈中脱颖而出：其EQI较基准方案反而降低了18%（意味着出水水质获得实质性改善），同时OCI削减了15%。实现这一罕见“双赢”局面的核心物理实质在于——动态前馈控制打破了空间静态分配的局限，将全天有限的绝对曝气资源，从低需求时间段精准抽调、转移注入至高需求时间段，完成了能量维度在时间轴上的高效重构与再分配。它用事实告诉我们：机械固定的控制参数注定无法匹配强非线性的时变生化反应过程，动态控制的灵魂始终在于“精准感知，按需供给”。

此外，针对回流泵能耗的延伸分析显示，单纯依靠调节水量的优化空间相对收窄。仿真数据显示，将内回流比从粗放的300%阶跃降至200%，虽然能使内回流泵送能耗大幅跳水约33%（符合泵相似定律），但后果是出水总氮浓度硬性升高了约2 mg/L。实际上，内回流比的最优操作点高度依赖于原水的实际碳氮比（C/N）。当进水富含易降解有机物（高碳氮比）时，缺氧区具有旺盛的反硝化潜力，此时增大内回流比输送更多的硝态氮可获得极佳的脱氮收益；反之，若面对低碳氮比原水，缺氧区反硝化反应的速率限制步骤已从“缺电子受体”转变为“缺电子供体（碳源）”，此时盲目维持高回流比的边际效益将迅速衰减为零，纯粹成为电能的空转与浪费。

## 4.5 工程实践与启示案例

将上述学术级的能耗优化理论向一线工程转化落地时，必须深刻把握并恪守一系列关键的工程原则。

首要前提是坚定贯彻“先达标、后节能”的底线逻辑——任何形式的节能改造或算法优化，都绝对不能以削弱系统的抗冲击韧性或导致短暂性出水超标作为置换筹码。在具体的算法部署过程中，强烈建议在软件底层设置充足的水质安全裕度。例如，即便环保法规规定氨氮排放上限为4.0 mg/L，在编写优化目标函数时，也应将其内控约束阈值收紧至3.0 mg/L，以为大雨天气或工业偷排引发的极端毒性负荷冲击预留足够的生化缓冲空间。

其次，曝气系统的宏观节能并非只依赖于顶层控制算法的精雕细琢，底层硬件设备的选型和维护同样决定了能耗的基座高度。理论测算和工程实测均表明，采用先进硅胶膜片的微孔曝气系统，由于气泡直径微小、比表面积庞大，其氧利用率（OTE）通常可高达20%-35%，其效能远胜于氧利用率仅为5%-10%的传统穿孔管粗气泡曝气器。某位于南方、日处理能力达10万吨的A2/O工艺污水厂，在未更改任何自控逻辑的前提下，仅将老化板结的曝气盘全面更换并清洗底部管网，降低了管路摩擦阻力 $\Delta p_{\mathrm{pipe}}$，就使得单日风机电耗直降12%。可见，基础设备的物理升级往往能提供比纯控制优化更立竿见影的节能基础。

第三，驱动风机与水泵的变频硬件生态，是所有高级动态控制算法赖以生存的物理载体。根据流体力学与系统动力学原理，若厂区配备的风机仅支持传统的星三角启动、且只能维持固定转速或仅支持粗糙的开/关两档控制，则系统将完全丧失进行流量连续平滑调节的能力。在这种落后的硬件制约下，即便上位机搭载了再先进的非线性模型预测控制（NMPC）或前馈-反馈控制算法，都将面临“有脑无手”的执行死局。我国北方某大型污水处理厂通过引进磁悬浮离心鼓风机并加装全变频驱动阵列，成功将DO设定的跟踪误差缩小至 $\pm 0.15$ mg/L，使得吨水处理电耗从0.28千瓦时大幅压降至0.21千瓦时，每年为企业创造直接经济效益超150万元。

最后，运维管理人员应当敏锐地捕捉夜间低负荷时段这一极具挖掘价值的节能“黄金窗口”。城市污水管网的汇水特征决定了其日变化系数极大（通常在0.5-1.5之间剧烈波动）。在深夜至凌晨时分，管网入流流量往往锐减至白昼高峰期的三分之一甚至更低，生化池内的氨氮和有机负荷亦随之断崖式下跌。如果能够抓住这一窗口期，依托可靠的在线水质仪表进行负荷核算，联动下调系统曝气参数，将带来全年核算下最为丰沛的降耗红利。

本章沿着从物理机制推导到系统级寻优的严谨逻辑脉络，建立并求解了污水生化处理核心单元的能耗表征模型。通过构建多目标优化的数学形式，我们揭示了曝气量与回流比在保障出水水质和压降运行成本之间的深刻博弈。详尽的仿真论证确切地表明，摒弃传统静态思路、引入基于进水负荷前馈信号的动态可变DO设定值策略，能够以时间维度上的资源调配换取空间维度的综合效率，最终实现出水水质改善18%且运行总能耗削减15%的卓越绩效。通过对能耗-水质帕累托前沿的深度剖析，为污水处理厂的管理者提供了透明且量化的决策工具。而在下一章中，我们将进一步探索出水水质的软测量（Soft-sensing）技术，因为精准且低延迟的参数获取，才是支撑各类高级控制算法与能耗优化落地的真正信息基石。

# -*- coding: utf-8 -*-
"""
第4章仿真：能耗优化（曝气KLa + 内回流比R）
依赖：numpy, scipy, matplotlib
运行：python ch04_energy_opt_sim.py
"""

import numpy as np
from scipy.integrate import solve_ivp
from scipy.optimize import minimize
import matplotlib.pyplot as plt

# -----------------------------
# 1) 参数区（可按教材或项目校准）
# -----------------------------
P = {
    # 体积 (m3)
    "V_an": 6000.0,   # 缺氧池
    "V_ae": 12000.0,  # 好氧池
    # 反应动力学参数
    "k_nit": 4.0,     # 硝化最大速率 (mgN/L/h)
    "K_NH4": 1.0,
    "K_DO": 0.5,
    "k_den": 3.2,     # 反硝化最大速率 (mgN/L/h)
    "K_NO3": 0.5,
    "K_COD": 40.0,
    "DO_sat": 8.0,    # 饱和DO (mg/L)
    "alpha_O2": 4.2,  # 硝化耗氧系数 (mgO2/mgNH4-N)
    # 控制变量边界（对应教材思路）
    "KLa_min": 100.0,  # d^-1
    "KLa_max": 400.0,  # d^-1
    "R_min": 1.0,      # 内回流比 Qa/Qin
    "R_max": 4.0,
    # 排放约束
    "NH4_lim": 4.0,    # mg/L
    "TN_lim": 15.0,    # mg/L
    # 目标函数权重
    "w_energy": 1.0,
    "w_nh4": 260.0,
    "w_tn": 150.0,
    "w_do": 1.5,
    "DO_ref": 2.0,
    # 能耗模型系数
    # 鼓风机：P_air ∝ KLa^1.25（教材核心关系）
    "c_air": 0.13,     # kW / (d^-1)^1.25
    "n_air": 1.25,
    # 回流泵：P_pump ∝ R^3（相似定律简化）
    "c_pump": 6.0,     # kW
    "n_pump": 3.0,
    "Q_nom": 2500.0,   # m3/h（标称进水流量）
}

# -----------------------------
# 2) 扰动：进水日变化
# -----------------------------
def influent_profile(t_h):
    """
    t_h: 小时
    返回: Qin (m3/h), NH4_in (mg/L), NO3_in (mg/L), COD_in (mg/L)
    """
    Qin = 2500 + 450 * np.sin(2 * np.pi * (t_h - 6) / 24.0) + 180 * np.sin(4 * np.pi * (t_h - 2) / 24.0)
    NH4_in = 30 + 8 * np.sin(2 * np.pi * (t_h - 5) / 24.0)
    COD_in = 280 + 70 * np.sin(2 * np.pi * (t_h - 4) / 24.0)
    NO3_in = 0.5
    Qin = max(Qin, 1200.0)
    NH4_in = max(NH4_in, 8.0)
    COD_in = max(COD_in, 80.0)
    return Qin, NH4_in, NO3_in, COD_in


# -----------------------------
# 3) 过程模型（两区简化机理）
# 状态 x = [NH4_an, NO3_an, NH4_ae, NO3_ae, DO_ae]
# -----------------------------
def process_rhs(t, x, u, influent, p):
    NH4_an, NO3_an, NH4_ae, NO3_ae, DO_ae = x
    KLa_d, R = u
    Qin, NH4_in, NO3_in, COD_in = influent

    Qa = R * Qin
    Qmix = Qin + Qa
    KLa_h = KLa_d / 24.0  # d^-1 -> h^-1

    # 速率项
    r_nit = p["k_nit"] * (NH4_ae / (p["K_NH4"] + NH4_ae + 1e-9)) * (DO_ae / (p["K_DO"] + DO_ae + 1e-9))
    f_C = COD_in / (p["K_COD"] + COD_in + 1e-9)
    r_den = p["k_den"] * (NO3_an / (p["K_NO3"] + NO3_an + 1e-9)) * f_C

    # 缺氧池
    dNH4_an = (Qin * NH4_in + Qa * NH4_ae - Qmix * NH4_an) / p["V_an"]
    dNO3_an = (Qin * NO3_in + Qa * NO3_ae - Qmix * NO3_an) / p["V_an"] - r_den

    # 好氧池
    dNH4_ae = (Qmix * NH4_an - Qmix * NH4_ae) / p["V_ae"] - r_nit
    dNO3_ae = (Qmix * NO3_an - Qmix * NO3_ae) / p["V_ae"] + r_nit
    dDO_ae = KLa_h * (p["DO_sat"] - DO_ae) - p["alpha_O2"] * r_nit - (Qmix / p["V_ae"]) * DO_ae

    return np.array([dNH4_an, dNO3_an, dNH4_ae, dNO3_ae, dDO_ae], dtype=float)


def simulate_step(x0, u, influent, p, dt_h):
    sol = solve_ivp(
        fun=lambda t, y: process_rhs(t, y, u, influent, p),
        t_span=(0.0, dt_h),
        y0=x0,
        t_eval=[dt_h],
        method="RK45",
        rtol=1e-5,
        atol=1e-7,
        max_step=dt_h / 5.0,
    )
    if sol.success and sol.y.shape[1] > 0:
        x1 = sol.y[:, -1]
    else:
        # 失败时用一次欧拉步兜底
        x1 = x0 + dt_h * process_rhs(0.0, x0, u, influent, p)
    return np.maximum(x1, 0.0)


# -----------------------------
# 4) 能耗模型 + 单步优化目标
# -----------------------------
def power_model(u, influent, p):
    KLa_d, R = u
    Qin = influent[0]
    P_air = p["c_air"] * (KLa_d ** p["n_air"])
    P_pump = p["c_pump"] * (R ** p["n_pump"]) * (Qin / p["Q_nom"])
    return P_air, P_pump


def stage_cost(u, x, influent, p, dt_h):
    x1 = simulate_step(x, u, influent, p, dt_h)
    NH4_e = x1[2]
    TN_e = x1[2] + x1[3]
    DO_e = x1[4]

    P_air, P_pump = power_model(u, influent, p)
    E_step = (P_air + P_pump) * dt_h  # kWh

    # 软约束惩罚（达标优先）
    pen_nh4 = max(0.0, NH4_e - p["NH4_lim"]) ** 2
    pen_tn = max(0.0, TN_e - p["TN_lim"]) ** 2
    pen_do = (DO_e - p["DO_ref"]) ** 2

    J = (
        p["w_energy"] * E_step
        + p["w_nh4"] * pen_nh4
        + p["w_tn"] * pen_tn
        + p["w_do"] * pen_do
    )
    return J


def optimize_control(x, influent, p, dt_h, u_prev):
    bounds = [(p["KLa_min"], p["KLa_max"]), (p["R_min"], p["R_max"])]
    res = minimize(
        fun=lambda u: stage_cost(u, x, influent, p, dt_h),
        x0=np.array(u_prev, dtype=float),
        method="SLSQP",
        bounds=bounds,
        options={"maxiter": 35, "ftol": 1e-4, "disp": False},
    )
    if res.success:
        return res.x
    return np.array(u_prev, dtype=float)


# -----------------------------
# 5) 主仿真
# -----------------------------
def run_simulation():
    T_h = 24.0
    dt_h = 10.0 / 60.0  # 10分钟
    n_steps = int(T_h / dt_h)

    # 初值
    x = np.array([20.0, 6.0, 8.0, 10.0, 1.8], dtype=float)
    u = np.array([200.0, 2.5], dtype=float)  # [KLa(d^-1), R]

    hist = {
        "time": [],
        "Qin": [],
        "NH4_in": [],
        "NH4_e": [],
        "TN_e": [],
        "DO": [],
        "KLa": [],
        "R": [],
        "P_air": [],
        "P_pump": [],
    }

    for k in range(n_steps):
        t = k * dt_h
        inf = influent_profile(t)

        u = optimize_control(x, inf, P, dt_h, u)
        x = simulate_step(x, u, inf, P, dt_h)

        P_air, P_pump = power_model(u, inf, P)
        NH4_e = x[2]
        TN_e = x[2] + x[3]

        hist["time"].append(t + dt_h)
        hist["Qin"].append(inf[0])
        hist["NH4_in"].append(inf[1])
        hist["NH4_e"].append(NH4_e)
        hist["TN_e"].append(TN_e)
        hist["DO"].append(x[4])
        hist["KLa"].append(u[0])
        hist["R"].append(u[1])
        hist["P_air"].append(P_air)
        hist["P_pump"].append(P_pump)

    # 转 ndarray
    for k in hist:
        hist[k] = np.array(hist[k], dtype=float)
    return hist, dt_h

# 6) KPI输出

def print_kpi_table(hist, dt_h, p):

    TN = hist["TN_e"]
    KLa = hist["KLa"]
    R = hist["R"]
    Qin = hist["Qin"]

    P_pump = hist["P_pump"]

    E_air = np.sum(P_air * dt_h)      # kWh
    E_pump = np.sum(P_pump * dt_h)    # kWh
    E_total = E_air + E_pump
    V_total = np.sum(Qin * dt_h)      # m3
    sec_energy = E_total / max(V_total, 1e-9)

    kpi = [
