# 第5章 多系统耦合优化建模

## 本章导读

水、能源与粮食（Water-Energy-Food, WEF）是维系人类社会运转的基础资源。传统单一资源的管理模式往往忽略了各系统间的复杂交互作用，导致资源配置效率低下及跨部门冲突。本章围绕多系统耦合优化建模展开，深入探讨WEF纽带系统的基本概念、理论方法与工程应用。通过构建涵盖多目标的数学规划模型，并引入多主体仿真技术，本章旨在为复杂资源系统的协同规划与联合调度提供定量化的分析工具，实现水利工程综合效益的最大化与可持续发展。

## 5.1 基本概念与理论框架

### 5.1.1 WEF三系统耦合优化内涵

WEF纽带系统并非三个孤立子系统的简单物理叠加，而是通过物质流、能量流和信息流相互交织、动态交互的复杂适应系统。水系统为能源开采与粮食生产提供必需的水资源；能源系统驱动水资源的抽取、输配与净化，并支撑现代农业机械化与化肥生产；粮食系统则通过土地利用变化影响水文循环，其生物质能亦可转化为能源。

多系统耦合优化的核心在于打破部门壁垒，在给定资源环境承载力约束下，通过合理的控制策略与工程调度，寻求水、能、粮三个维度综合效益的全局最优解。这一过程要求处理资源的时空错位问题，例如在干旱区，农业灌溉的高耗水与能源基地的高耗水往往存在刚性竞争。

### 5.1.2 多目标规划理论

由于水、能、粮三个子系统的优化方向常常存在冲突，单一目标的极值优化无法满足工程实际需求。多目标规划（Multi-objective Programming, MOP）为解决此类问题提供了理论框架。在WEF系统中，常见的优化目标包括：区域经济系统总产出最大化、水土资源消耗量最小化、生态环境负面效应（如碳排放、面源污染）最低化等。由于这些目标之间无法同时达到绝对最优，系统求解转化为寻找非劣解集（Pareto最优解集）的过程，即在不恶化任意其他目标的前提下，无法进一步改善某一目标的解的集合。

### 5.1.3 Agent-Based模型在纽带系统中的应用

为刻画系统中多元利益主体的微观行为及其对宏观系统状态的涌现效应，基于智能体的模型（Agent-Based Modeling, ABM）被广泛引入。在WEF系统中，Agent可定义为农户、发电企业、水库调度机构及政府环保部门。每个Agent具有独立的决策规则与资源禀赋，依据有限理性在市场机制或政策约束下进行博弈。通过多主体环境下的自下而上建模，能够更真实地模拟水权交易、水价响应及农业种植结构调整等非线性动态过程。

### 5.1.4 可持续发展指标体系

评估耦合系统的优化效果，需建立多维度的定量指标体系。该体系深度对接联合国可持续发展目标（SDGs），尤其是SDG 2（零饥饿）、SDG 6（清洁饮水）与SDG 7（清洁能源）。表5-1列举了典型的WEF系统耦合评价指标。

## 5.2 数学建模与求解方法

本节从数学角度建立多系统耦合优化建模的核心模型，推导关键公式，并剖析模型参数的物理意义。建模工具主要涉及微积分、运筹学及现代启发式优化算法。

### 5.2.1 状态方程与控制变量

定义系统在时段 $t$ 的状态变量向量为 $\mathbf{S}(t) = [S_w(t), S_e(t), S_f(t)]^T$，分别代表水库蓄水量、煤炭/水能储备量与粮食库存量。定义控制变量向量为 $\mathbf{U}(t) = [\mathbf{Q}(t), \mathbf{P}(t), \mathbf{A}(t)]^T$，其中 $\mathbf{Q}(t)$ 为水资源分配矩阵，$\mathbf{P}(t)$ 为能源生产与调度向量，$\mathbf{A}(t)$ 为农作物种植面积与灌溉制度向量。

系统的动态演化由以下离散差分方程描述：
$$ \mathbf{S}(t+1) = \mathbf{S}(t) + f(\mathbf{S}(t), \mathbf{U}(t), \mathbf{\Omega}(t)) $$
式中，$\mathbf{\Omega}(t)$ 表示系统所受的外部随机强迫输入，如天然来水量、降水量及市场价格波动。

### 5.2.2 耦合约束条件推导

**（1）水资源供需平衡约束**
区域在时段 $t$ 的总可用水量 $W_{avail}(t)$ 必须大于等于能源、粮食及生活生态用水之和：
$$ \sum_{j \in J_e} \mu_j P_j(t) + \sum_{k \in K_f} m_k A_k(t) + W_{eco}(t) \leq W_{avail}(t) $$
式中，$J_e$ 为能源生产单元集合；$\mu_j$ 为第 $j$ 类能源生产的耗水定额（m³/MWh）；$K_f$ 为农作物种类集合；$m_k$ 为第 $k$ 类作物的毛灌溉定额（m³/hm²）；$P_j(t)$ 与 $A_k(t)$ 分别为能源产量与种植面积。

**（2）涉水系统能耗约束**
水系统的运行深度依赖能源驱动。泵站抽水、跨流域调水与污水处理的总能耗 $E_{water}(t)$ 需满足：
$$ E_{water}(t) = \sum_{i \in I_w} \left( \frac{\rho g Q_i(t) H_i}{\eta_i} \right) \Delta t + \sum_{n \in N_{treat}} e_n V_n(t) \leq E_{avail}(t) $$
式中，$I_w$ 为提水工程节点集合；$\rho$ 为水密度；$g$ 为重力加速度；$Q_i(t)$ 为抽水流量；$H_i$ 为净扬程；$\eta_i$ 为机组综合效率；$N_{treat}$ 为水处理节点集合；$e_n$ 为处理单位体积污水的能耗系数；$V_n(t)$ 为污水处理量。

**（3）粮食生产水分响应模型**
基于Jensen乘法模型推导作物产量与耗水量的非线性耦合关系：
$$ \frac{Y_k}{Y_{k,m}} = \prod_{s=1}^{N_s} \left( 1 - K_{y,s}^k \left( 1 - \frac{ET_{a,s}^k}{ET_{c,s}^k} \right) \right) $$
式中，$Y_k$ 与 $Y_{k,m}$ 分别为作物 $k$ 的实际产量与潜在最高产量；$s$ 为生育阶段；$K_{y,s}^k$ 为作物各生育期的缺水敏感指数；$ET_{a,s}^k$ 与 $ET_{c,s}^k$ 分别为实际蒸发蒸腾量与最大蒸发蒸腾量。该公式表明，水资源配置在时间分布上的微小偏差均会被敏感指数放大，从而显著影响最终粮食产量。

### 5.2.3 多目标优化函数

基于上述耦合机制，建立以经济性、环境性和可靠性为核心的三维目标函数空间：

**目标一：综合净效益最大化（经济性）**
$$ \max F_1 = \sum_{t=1}^T \left( \alpha \cdot \mathbf{R}_e^T \mathbf{P}(t) + \beta \cdot \mathbf{R}_f^T \mathbf{Y}(t) - \mathbf{C}^T \mathbf{U}(t) \right) $$
式中，$\mathbf{R}_e$ 与 $\mathbf{R}_f$ 为能源与粮食的市场价格向量；$\mathbf{C}$ 为各控制变量的操作成本系数向量；$\alpha$ 与 $\beta$ 为通货膨胀及折现因子。

**目标二：碳-水系统足迹最小化（环境性）**
$$ \min F_2 = \sum_{t=1}^T \left( \gamma_c E_{total}(t) + \gamma_w W_{extract}(t) \right) $$
式中，$\gamma_c$ 为区域平均碳排放因子（tCO₂/MWh）；$E_{total}$ 为系统总能耗；$\gamma_w$ 为水资源衰减惩罚系数。

### 5.2.4 求解算法分析

由于上述模型包含高度非线性的Jensen公式及离散的泵站启停决策变量，该问题属于混合整数非线性规划问题（MINLP）。常规的梯度下降法易陷入局部极小值。工程中多采用带精英保留策略的非支配排序遗传算法（NSGA-II）或多目标粒子群算法（MOPSO）进行求解。
求解过程利用罚函数法处理等式与不等式约束：
$$ Fitness(X) = \lambda_1 F_1(X) - \lambda_2 F_2(X) - \sum_{m=1}^M \phi_m \max(0, g_m(X))^2 $$
其中 $\phi_m$ 为巨大的惩罚因子，迫使种群在迭代中迅速逼近可行域。

### 5.3.2 仿真结果与动态特征演化

**（2）Pareto前沿与权衡机制**
观察绘制的经济性（$F_1$）与环境影响（$F_2$）的Pareto前沿曲线。曲线斜率的变化揭示了深刻的边际效应规律：在极度缺水时段，为保证农业基本产量（维持安全红线），需启动备用深层地下水水源，此时系统的抽水能耗呈指数级飙升，$F_1$ 的微小提升以 $F_2$ 的巨大牺牲为代价。这表明系统的物理约束极大地限制了经济指标的无限扩张。

**（3）参数敏感性分析**
对气候变异条件下的天然来水 $\mathbf{\Omega}(t)$ 进行扰动分析。当径流量下降20%时，S0情景下农业面临大面积减产，系统处于瘫痪边缘；而S2模型凭借储能系统与水库群的联合调节，自动执行“弃保”策略——优先削减非敏感生育期的作物灌水及低效益能矿企业用水，使粮食减产幅度控制在5%以内，展现出极强的系统韧性。

## 5.4 工程启示与应用建议

基于上述理论建模与数值仿真分析，对现代水利工程的规划、设计与运行提出以下具有现实指导意义的建议：

1. **重构工程规划评估维度**：传统水利工程的可行性研究多侧重于本行业的防洪、供水与发电效益计算。在WEF视角下，必须强制引入全生命周期的“水足迹”与“碳足迹”核算，评估水利设施全产业链（如大坝浇筑的隐含碳、调水工程的长期能耗）对能源与气候系统的反向冲击。
2. **推行水利能源化基础设施建设**：在硬件层面，推广“水-光-抽蓄”一体化工程。例如，在平原水库或干渠水面敷设光伏板，一方面提供抽水泵站所需的清洁电能，另一方面通过遮阴大幅降低库区无效水面蒸发，实现水能实体层面的直接耦合。
3. **建立多主体利益补偿机制**：优化的数学解落地需依托制度设计。由于全局优化往往要求某一局部（如上游农户）牺牲自身利益以成全整体（如保障下游能源基地用水），水行政主管部门应基于模型的影子价格（Shadow Price）计算结果，建立精准的跨部门生态与水权补偿机制。
4. **推进智能感知与数字孪生**：差分方程的高精度求解依赖于高频、多源的实时数据。工程上应加快部署覆盖水文气象、土壤墒情、电网负荷的物联网传感体系，构建流域级“数字孪生”平台，将离线的耦合优化算法转化为在线的滚动调度指令。

## 拓展视野：关联水系统控制论 (Cybernetics of Water Systems)

本章论述的WEF耦合系统在数学机理上与**水系统控制论**具有高度同构性。控制论的核心在于研究系统的状态观测、信息反馈与最优控制。在大型水网（如南水北调工程）及流域联合调度中，同样面临多维度复杂系统的解耦与控制问题。

将水网视为受控对象（Plant），天气预报与市场需求为干扰向量（Disturbance），各个节制闸、泵站与发电机组即为执行器（Actuator）。本章中求解出的Pareto非劣解，本质上构成了控制论中的最优反馈控制律（Optimal Feedback Control Law）。近年来，随着深度强化学习（DRL）与模型预测控制（MPC）技术的发展，控制论理论被深度融合进多系统耦合框架中。在具有长时滞性、强非线性及不确定性的水网调度工程中，通过前馈补偿与误差反馈调节，不仅能够在稳态下寻优，更能在突发极端干旱或电网断崖式负荷波动时，保证WEF全要素系统的动态渐近稳定性（Asymptotic Stability）。类似的多变量最优控制框架已在多项大型跨流域调水工程的自动化改造中得到验证并展现出卓越效能。

```python
# 《水-能源-粮食纽带关系》第5章：多系统耦合优化建模
# 功能：构建WEF（水-能源-粮食）耦合优化仿真，输出KPI结果表并生成可视化图形

import numpy as np
from scipy.optimize import minimize
import matplotlib.pyplot as plt

# =========================
# 1) 关键参数（可直接调参）
# =========================
PARAMS = {
    # 资源与需求
    "W_NATURAL": 120.0,      # 天然可用水量（百万m^3）
    "E_EXTERNAL": 230.0,     # 外部可供能源（GWh）
    "F_DEMAND": 95.0,        # 粮食需求（万吨）
    "E_DEMAND": 120.0,       # 社会能源服务需求（GWh）
    "W_ENV_MIN": 25.0,       # 生态最小需水（百万m^3）

    # 耦合生产系数
    "ALPHA_F": 1.80,         # 粮食生产系数
    "ALPHA_E": 0.90,         # 由水支撑的能源产出系数（如水电等）
    "ALPHA_R": 0.65,         # 能源投入带来的再生水系数

    # 非线性弹性指数（体现规模报酬递减）
    "FOOD_EXP_W": 0.58,
    "FOOD_EXP_E": 0.42,
    "ENERGY_EXP_W": 0.75,
    "REUSE_EXP_E": 0.85,

    # 成本与排放参数
    "COST_W": 0.22,          # 水资源调配成本（归一化）
    "COST_E": 0.38,          # 能源使用成本（归一化）
    "EF_EXT": 0.42,          # 外部供能排放系数
    "EF_OP": 0.18,           # 系统运行排放系数
    "EF_HYDRO_OFFSET": 0.12, # 清洁能源替代减排系数

    # 多目标加权系数（理论框架：加权求和法）
    "W_FOOD": 0.35,          # 粮食保障权重
    "W_ENERGY": 0.30,        # 能源保障权重
    "W_WATER": 0.20,         # 生态水保障权重
    "W_COST": 0.10,          # 成本惩罚权重
    "W_EMISSION": 0.05,      # 排放惩罚权重
}

# 决策变量顺序：x = [W_food, W_energy, E_agri, E_treat]
BOUNDS = [
    (10.0, 100.0),   # W_food：农业用水（百万m^3）
    (5.0, 80.0),     # W_energy：能源系统用水（百万m^3）
    (5.0, 140.0),    # E_agri：农业用能（GWh）
    (0.0, 120.0),    # E_treat：水处理/回用用能（GWh）
]

X0 = np.array([55.0, 28.0, 68.0, 25.0], dtype=float)  # 初值
SCENARIO_FACTORS = np.array([0.80, 0.90, 1.00, 1.10, 1.20])  # 外部供能情景


# =========================
# 2) 系统评价函数
# =========================
def evaluate_system(x, p):
    """根据决策变量计算耦合系统状态与KPI。"""
    W_food, W_energy, E_agri, E_treat = x

    # 防止零值在分数指数下数值不稳定
    W_food = max(W_food, 1e-9)
    W_energy = max(W_energy, 1e-9)
    E_agri = max(E_agri, 1e-9)
    E_treat = max(E_treat, 1e-9)

    # 粮食产出：受农业用水与农业用能共同影响
    food = p["ALPHA_F"] * (W_food ** p["FOOD_EXP_W"]) * (E_agri ** p["FOOD_EXP_E"])

    # 能源子系统产出：用水促进能源产出（可类比水电等）
    hydro_energy = p["ALPHA_E"] * (W_energy ** p["ENERGY_EXP_W"])

    # 再生水：水处理用能转化为回用水
    reused_water = p["ALPHA_R"] * (E_treat ** p["REUSE_EXP_E"])

    # 系统平衡量
    water_available = p["W_NATURAL"] + reused_water
    water_env = water_available - W_food - W_energy
    energy_service = p["E_EXTERNAL"] + hydro_energy - E_agri - E_treat

    # 成本与排放
    cost = p["COST_W"] * (W_food + W_energy) + p["COST_E"] * (E_agri + E_treat)
    emission = (
        p["EF_EXT"] * p["E_EXTERNAL"]
        + p["EF_OP"] * (E_agri + E_treat)
        - p["EF_HYDRO_OFFSET"] * hydro_energy
    )

    # 归一化保障指标（上限截断防止单指标主导）
    s_food = np.clip(food / p["F_DEMAND"], 0.0, 1.5)
    s_energy = np.clip(energy_service / p["E_DEMAND"], 0.0, 1.5)
    s_water = np.clip(water_env / p["W_ENV_MIN"], 0.0, 2.0)

    # 综合可持续指数（多目标加权）
    sustainability = (
        p["W_FOOD"] * s_food
        + p["W_ENERGY"] * s_energy
        + p["W_WATER"] * s_water
        - p["W_COST"] * (cost / 100.0)
        - p["W_EMISSION"] * (emission / 100.0)
    )

    # 耦合协调度：几何平均体现“短板效应”
    coordination = (max(s_food, 1e-6) * max(s_energy, 1e-6) * max(s_water, 1e-6)) ** (1 / 3)

    return {
        "food": food,
        "hydro_energy": hydro_energy,
        "reused_water": reused_water,
        "water_env": water_env,
        "energy_service": energy_service,
        "cost": cost,
        "emission": emission,
        "s_food": s_food,
        "s_energy": s_energy,
        "s_water": s_water,
        "sustainability": sustainability,
        "coordination": coordination,
    }


def objective(x, p):
    """SLSQP默认求最小值，这里对综合可持续指数取负号实现“最大化”目标。"""
    return -evaluate_system(x, p)["sustainability"]


def build_constraints(p):
    """构建约束：生态底线与能源服务底线。"""
    return [
        # 生态需水约束：水结余必须不低于生态阈值
        {
            "type": "ineq",
            "fun": lambda x, p=p: p["W_NATURAL"] + p["ALPHA_R"] * (max(x[3], 0.0) ** p["REUSE_EXP_E"])
                                  - x[0] - x[1] - p["W_ENV_MIN"]
        },
        # 能源保障约束：净能源服务不低于需求
        {
            "type": "ineq",
            "fun": lambda x, p=p: p["E_EXTERNAL"] + p["ALPHA_E"] * (max(x[1], 0.0) ** p["ENERGY_EXP_W"])
                                  - x[2] - x[3] - p["E_DEMAND"]
        },
    ]


def solve_wef(p, x0):
    """求解WEF耦合优化问题。"""
    result = minimize(
        objective,
        x0=x0,
        args=(p,),
        method="SLSQP",
        bounds=BOUNDS,
        constraints=build_constraints(p),
        options={"maxiter": 400, "ftol": 1e-9}
    )
    return result


def print_table(title, rows):
    """打印简洁KPI表格。"""
    print("\n" + title)
    print("-" * 66)
    print(f"{'指标':<24}{'数值':>14}{'单位':>12}")
    print("-" * 66)
    for name, value, unit in rows:
        print(f"{name:<24}{value:>14.3f}{unit:>12}")
    print("-" * 66)


if __name__ == "__main__":
    # 中文显示设置（系统缺少中文字体时会自动回退）
    plt.rcParams["font.sans-serif"] = ["SimHei", "Microsoft YaHei", "Arial Unicode MS", "DejaVu Sans"]
    plt.rcParams["axes.unicode_minus"] = False

    # =========================
    # 3) 基准情景优化
    # =========================
    res = solve_wef(PARAMS, X0)
    if not res.success:
        raise RuntimeError(f"优化失败：{res.message}")

    x_opt = res.x
    state = evaluate_system(x_opt, PARAMS)

    # 打印最优决策变量
    decision_rows = [
        ("农业用水 W_food", x_opt[0], "百万m^3"),
        ("能源系统用水 W_energy", x_opt[1], "百万m^3"),
        ("农业用能 E_agri", x_opt[2], "GWh"),
        ("水处理用能 E_treat", x_opt[3], "GWh"),
    ]
    print_table("最优决策变量", decision_rows)

    # 打印KPI结果表
    kpi_rows = [
        ("粮食产出", state["food"], "万吨"),
        ("能源净服务", state["energy_service"], "GWh"),
        ("生态结余水量", state["water_env"], "百万m^3"),
        ("再生水量", state["reused_water"], "百万m^3"),
        ("耦合协调度", state["coordination"], "-"),
        ("综合可持续指数", state["sustainability"], "-"),
        ("总成本", state["cost"], "归一化"),
        ("碳排放", state["emission"], "归一化"),
    ]
    print_table("KPI结果表", kpi_rows)

    # =========================
    # 4) 情景分析（外部供能变化）
    # =========================
    ext_energy_list, si_list, food_list = [], [], []
    warm_start = x_opt.copy()

    for factor in SCENARIO_FACTORS:
        p_sc = PARAMS.copy()
        p_sc["E_EXTERNAL"] = PARAMS["E_EXTERNAL"] * factor
        res_sc = solve_wef(p_sc, warm_start)

        if res_sc.success:
            st_sc = evaluate_system(res_sc.x, p_sc)
            ext_energy_list.append(p_sc["E_EXTERNAL"])
            si_list.append(st_sc["sustainability"])
            food_list.append(st_sc["food"])
            warm_start = res_sc.x
        else:
            ext_energy_list.append(p_sc["E_EXTERNAL"])
            si_list.append(np.nan)
            food_list.append(np.nan)

    ext_energy_arr = np.array(ext_energy_list)
    si_arr = np.array(si_list)
    food_arr = np.array(food_list)

    # =========================
    # 5) 可视化
    # =========================
    fig, axes = plt.subplots(1, 2, figsize=(13, 5))

    # 图1：基准情景下三系统保障与协调度
    labels = ["粮食保障", "能源保障", "生态水保障", "耦合协调度"]
    values = [state["s_food"], state["s_energy"], state["s_water"], state["coordination"]]
    colors = ["#2a9d8f", "#e9c46a", "#457b9d", "#e76f51"]

    axes[0].bar(labels, values, color=colors)
    axes[0].axhline(1.0, linestyle="--", color="gray", linewidth=1)
    axes[0].set_ylim(0, max(2.1, max(values) * 1.2))
    axes[0].set_title("基准情景：WEF保障指标")
    axes[0].set_ylabel("归一化指标值")

    # 图2：外部供能变化对可持续性与粮食产出的影响
    ax_left = axes[1]
    ax_right = ax_left.twinx()

    line1 = ax_left.plot(ext_energy_arr, si_arr, "o-", color="#2a9d8f", label="综合可持续指数")
    line2 = ax_right.plot(ext_energy_arr, food_arr, "s--", color="#e76f51", label="粮食产出")

    ax_left.set_title("情景分析：外部供能变化")
    ax_left.set_xlabel("外部供能规模 (GWh)")
    ax_left.set_ylabel("综合可持续指数", color="#2a9d8f")
    ax_right.set_ylabel("粮食产出 (万吨)", color="#e76f51")

    # 合并图例
    lines = line1 + line2
    labels = [l.get_label() for l in lines]
    ax_left.legend(lines, labels, loc="best")

    plt.tight_layout()
    plt.show()
```

代码解读（约800字）  
这段脚本对应第5章“多系统耦合优化建模”的核心思想：把水、能源、粮食三个子系统放在统一决策框架中，用一个优化器同时求解资源配置与可持续目标。脚本首先在 `PARAMS` 中集中定义参数，包括资源总量、需求阈值、生产系数、成本排放系数和目标权重，这样做的价值是“模型结构固定、参数可替换”，便于课堂演示和后续敏感性分析。决策变量为四个：农业用水、能源系统用水、农业用能、水处理用能。它们分别作用于粮食产出、能源产出与再生水产出，形成典型耦合链条：能源投入提升再生水，再生水缓解水约束；水投入支撑能源产出，能源产出反过来缓解能源约束。  
`evaluate_system()` 是模型主引擎。函数内部用幂函数表达非线性生产关系，体现边际收益递减的工程现实；然后计算三类关键状态量：粮食产量、净能源服务、生态结余水量。之后构造三个归一化保障指标 `s_food/s_energy/s_water`，并通过加权求和减去成本和排放惩罚，得到综合可持续指数 `sustainability`。这实际上就是多目标优化在工程中的常用折中法：把“多目标”转为“单目标”，再由优化算法搜索最优点。为避免单一高值掩盖短板，脚本又给出几何平均的 `coordination`（耦合协调度），它对低值更敏感，符合“木桶效应”。  
约束部分 `build_constraints()` 设置了两条硬约束：生态底线（水结余不得低于生态阈值）和能源服务底线（净能源不得低于需求）。这两条约束体现了第5.1节的理论框架要点：系统优化不能只追求总效益，还要满足安全与可持续边界。求解器采用 `scipy.optimize.minimize(..., method='SLSQP')`，因为SLSQP可同时处理非线性目标、边界和不等式约束，适合此类连续决策问题。  
输出层面，脚本先打印“最优决策变量表”，再打印“KPI结果表”，满足教材中“可解释、可复核”的仿真要求。KPI同时给出产出、成本、排放和协调度，便于课堂讨论“高产出是否伴随高代价”。可视化部分含两幅图：第一幅展示基准情景下三系统保障水平与耦合协调度，横向比较系统短板；第二幅做情景分析，改变外部供能规模，观察综合可持续指数与粮食产出的联动趋势，直观看到“能源约束变化如何传导到粮食与水系统”。  
整体上，这份脚本不是追求某个唯一“真实值”，而是提供一个可调参数、可扩展结构的教学型仿真骨架。你可在此基础上继续扩展为：加入随机来水、价格机制、碳约束、区域多主体（Agent）行为，从而自然过渡到第5.2和5.3节的更复杂模型。
