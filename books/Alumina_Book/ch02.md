# 第2章 蒸发工序的物理机理与数学模型

## 2.1 多效蒸发器的结构与能量流（热质平衡）

多效蒸发的工程价值是“蒸汽梯度利用”：前一效二次蒸汽成为后一效热源，从而降低单位蒸发水量的鲜蒸汽需求。
但这不是自动省汽，必须满足热质平衡闭合和工况匹配。

### 质量守恒框架

对单效可写为：

$$ % (Eq. Auto-Aligned)F_i = L_i + V_i$$

其中 $F_i$ 为入料，$L_i$ 为出液，$V_i$ 为二次蒸汽量。
若溶质不挥发：

$$F_i x_{F_i} = L_i x_{L_i}$$

### 能量守恒框架

$$F_i h_{F_i} + Q_i = L_i h_{L_i} + V_i h_{V_i} + Q_{loss,i}$$

多效系统通过热耦合把各效联立，形成求解闭环。

在 `core/alumina/thermo_balance.py` 中，核心逻辑应包含：

1. 分效质量平衡方程组。
2. 分效能量平衡方程组。
3. 多效耦合条件与边界条件。
4. 平衡残差计算与收敛判据。

对应服务接口是 `alumina_evap_server.solve_balance`，对应章节证据测试是 `tests/alumina/test_ch02_thermo_balance.py`。

## 2.2 气液两相流与闪蒸机理

蒸发过程并非理想稳态“单相换热”，而是伴随气液两相流动与局部闪蒸。
当物料进入较低压力区域，部分液体会瞬时汽化，影响温度、体积分率与流动阻力。

闪蒸可用焓差关系描述其汽化分率趋势：

$$\phi \approx \frac{h_{in} - h_f(P)}{h_g(P)-h_f(P)}$$

两相行为带来的控制挑战：

1. 有效传热温差在不同工况下变化明显。
2. 流量与压降关系呈强非线性。
3. 测量信号出现“看起来稳定、实际状态漂移”的现象。

因此，`thermo_balance.py` 在工程上不应仅做静态收支，还应为两相与闪蒸影响预留等效项或接口，使模型可在不同负荷下保持可解释性和可计算性。

## 2.3 物料浓度、温度与压力的强非线性耦合模型

蒸发系统的难点在于变量不是独立变化，而是强耦合闭环：

1. 压力变动改变沸点。
2. 沸点变化改变有效温差与蒸发速率。
3. 蒸发速率变化又反过来改变浓度和液位。
4. 浓度上升加剧沸点升高与粘度变化，进一步影响传热与流动。

典型关系可抽象为：

$$T_{boil} = T_{sat}(P) + \Delta T_{BPE}(x)$$

$$q = U(x, R_f)\,A\,\Delta T_{lm}$$

其中 $U$ 受浓度与结疤热阻 $R_f$ 影响。
这意味着“固定参数线性模型”只能在窄工况有效，跨工况会迅速失配。

`core/alumina/thermo_balance.py` 的核心价值，是把这些耦合关系组织成可求解、可校核的约束系统，而不是停留在单方程说明。
`test_ch02_thermo_balance.py` 的“通过”本质上验证了一个关键事实：即使在耦合条件下，质量与能量闭合仍可成立在可接受误差内。

## 2.4 数据驱动与机理融合（灰盒模型）的必然性

纯机理模型有可解释性，但参数漂移与设备老化会降低预测精度；纯数据模型有拟合能力，但外推与安全约束解释不足。
蒸发工序的现实答案是灰盒融合。

灰盒框架可写为：

$$x_{k+1}=f_{\text{phys}}(x_k,u_k,\theta_k)+w_k$$

$$y_k=g_{\text{phys}}(x_k,\theta_k)+\Delta g_{\text{data}}(z_k)+v_k$$

其中：

1. $f_{\text{phys}}, g_{\text{phys}}$ 由 `thermo_balance.py` 的守恒与热力学结构给出。
2. $\Delta g_{\text{data}}$ 用于补偿未建模扰动与慢变参数。
3. $\theta_k$ 可在线估计，形成“边运行边校准”的能力。

为什么是“必然性”：

1. 仅靠机理难覆盖全寿命工况。
2. 仅靠数据难满足高后果场景的可审计需求。
3. 双向迭代方法要求每个结论都能被测试复验，灰盒最易形成稳定证据链。
