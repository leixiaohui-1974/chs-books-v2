# 第3章 从 PID 到模型预测控制（MPC）

## 3.1 控制理论演进极简史

液位控制的工业实践经历了从经验到模型的发展：

1. **人工调节**：基于“看水位、拧阀门”的纯经验模式。
2. **PID 控制**：20世纪中叶起，PID 以其“无模型”的即插即用特性统治了单回路液位控制，解决了大部分线性和慢变系统的稳态问题。
3. **先进控制（APC/MPC）**：当系统出现强耦合、硬约束和多目标冲突时，PID 显得力不从心。模型预测控制（MPC）通过在每个控制步求解一个带有物理约束的优化问题，实现了多变量的全局协调。

## 3.2 为什么选择 MPC？（看得远、算得准、能应对约束）

在双容水箱这种存在耦合（两箱水位相互影响）和约束（阀门最大开度、水箱防溢流）的场景中，MPC 展示了结构性优势：

1. **多变量协同**：MPC 天生能处理多输入多输出（MIMO）系统，无需人工解耦。
2. **约束处理**：把“水不能溢出 ($h \le h_{max}$)”和“阀不能开过头 ($u \le 1.0$)”直接写入算法的边界条件中。
3. **滚动优化**：不仅看当下的误差，还能推演未来 $N$ 步的演化，实现“防患于未然”。

在代码证据链中，`core/coupled_tank/mpc.py` 的加入与 `core/coupled_tank/pid.py` 形成了直接的竞技场。

## 3.3 预测域与控制域：如何预知未来并当下决策

MPC 的核心概念是“预测域（Prediction Horizon）”和“控制域（Control Horizon）”：
- **预测域**：模型向未来推演的步数。
- **控制域**：允许控制动作发生变化的步数。

算法在每个采样周期：
1. 测量当前水位。
2. 预测未来演化，求解出一条最佳动作序列。
3. **仅执行该序列的第一个动作**。
4. 下个周期重复此过程（滚动）。

这种机制使系统能够持续抵抗未建模扰动。
对应测试 `tests/coupled_tank/test_ch03_pid_vs_mpc.py` 直接将两者的调控效果进行了代码级断言。

## 3.4 强化学习（RL）在液位控制中的探索与互补

随着 AI 发展，基于强化学习（RL）的无模型调控也逐渐兴起。RL 擅长处理难以建立精确数学模型的极端非线性场景，但在“安全可解释性”上仍不及 MPC。在本书构想中，MPC 提供安全基线与刚性约束，RL 则可用于上层的设定值寻优与调度，形成互补。
